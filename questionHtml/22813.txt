    <div class="post-text" itemprop="text">

<p>I have a file in the distributed cache. The driver class, based on the output of a job, updates this file and starts a new job.  The new job need these updates.</p>

<p>The way I currently do it is to replace the old Distributed Cache file with a new one (the updated one). </p>

<p>Is there a way of broadcasting the diffs (between the old file and the new one) to all the tasks trackers which need the file ? </p>

<p>Or is it the case that, after a job (the first one, in my case) is finished, all the directories/files specific to that job are deleted and consequently it doesn't even make sense to think in this direction ?</p>

<p>I think that distributed cache is not build with such scenario in mind. It simply put files locally. <br>
In Your case I would suggest to put file in HDFS and make all interested parties to take it from there <br>
As an optimization you can give this file high replication factor and it will be local to most of the tasks.</p>
