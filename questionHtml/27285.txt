    <div class="post-text" itemprop="text">

<p>Assuming I have lines of data like the following that show user names and their favorite fruits:</p>

<pre><code>Alice\tApple
Bob\tApple
Charlie\tGuava
Alice\tOrange
</code></pre>

<p>I'd like to create a pig query that shows the favorite fruit of each user. If a user appears multiple times, then I'd like to show "Multiple". For example, the result with the data above should be:</p>

<pre><code>Alice\tMultiple
Bob\tApple
Charlie\tGuava
</code></pre>

<p>In SQL, this could be done something like this (although it wouldn't necessarily perform very well):</p>

<pre><code>select user, case when count(fruit) &gt; 1 then 'Multiple' else max(fruit) end
from FruitPreferences
group by user
</code></pre>

<p>But I can't figure out the equivalent PigLatin. Any ideas?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Write a <a href="http://wiki.apache.org/pig/UDFManual" rel="nofollow">"Aggregate Function" Pig UDF</a> (scroll down to "Aggregate Functions"). This is a user-defined function that takes a bag and outputs a scalar. So basically, your UDF would take in the bag, determine if there is more than one item in it, and transform it accordingly with an if statement.</p>

<p>I can think of a way of doing this without a UDF, but it is definitely awkward. After your <code>GROUP</code>, use <code>SPLIT</code> to split your data set into two: one in which the count is 1 and one in which the count is more than one:</p>

<pre><code>SPLIT grouped INTO one IF COUNT(fruit) == 0, more IF COUNT(fruit) &gt; 0;
</code></pre>

<p>Then, separately use <code>FOREACH ... GENERATE</code> on each to transform it:</p>

<pre><code>one = FOREACH one GENERATE name, MAX(fruit); -- hack using MAX to get the item
more = FOREACH more GENERATE name, 'Multiple';
</code></pre>

<p>Finally, union them back:</p>

<pre><code>out = UNION one, more;
</code></pre>

<p>I haven't really found a better way of handing the same data set in two different ways based on some conditional, like you want. I typically do some sort of split/recombine like I did here. I believe Pig will be smart and make a plan that doesn't use more than 1 M/R job.</p>

<p>Disclaimer: I can't actually test this code at the moment, so it may have some mistakes.</p>

<hr>

<p>Update:</p>

<p>In looking harder, I was reminded of the <a href="http://pig.apache.org/docs/r0.9.0/basic.html#arithmetic" rel="nofollow">bicond operator</a> and I think that will work here.</p>

<pre><code>b = FOREACH a GENERATE name, (COUNT(fruit)==1 ? MAX(FRUIT) : 'Multiple');
</code></pre>
