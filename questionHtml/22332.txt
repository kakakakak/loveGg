<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The solution that worked for me was to run namenode and datanode one by one and not together using <code>bin/start-all.sh</code>. What happens using this approach is that the error is clearly visible if you are having some problem setting the datanodes on the network and also many posts on stackoverflow suggest that namenode requires some time to start-off, therefore, it should be given some time to start before starting the datanodes. Also, in this case I was having problem with different ids of namenode and datanodes for which I had to change the ids of the datanode with same id as the namenode.</p>

<p>The step by step procedure will be:</p>

<ol>
<li>Start the namenode <code>bin/hadoop namenode</code>. Check for errors, if any.</li>
<li>Start the datanodes <code>bin/hadoop datanode</code>. Check for errors, if any.</li>
<li>Now start the task-tracker, job tracker using 'bin/start-mapred.sh'</li>
</ol>

<p>Look at your namenode (probably <a href="http://localhost:50070" rel="nofollow">http://localhost:50070</a>) and see how many datanodes it says you have.  </p>

<p>If it is 0, then either your datanode isn't running or it isn't configured to connect to the namenode.  </p>

<p>If it is 1, check to see how much free space it says there is in the DFS.  It may be that the data node doesn't have anywhere it can write data to (data dir doesn't exist, or doesn't have write permissions).</p>

<td class="answercell">

<p>I had the same problem, I took a look at the datanode logs and there was a warning saying that the dfs.data.dir had incorrect permissions... so I just changed them and everything worked, which is kind of weird.</p>

<p>Specifically, my "dfs.data.dir" was set to "/home/hadoop/hd_tmp", and the error I got was:</p>

<pre><code>...
...
WARN org.apache.hadoop.hdfs.server.datanode.DataNode: Invalid directory in dfs.data.dir: Incorrect permission for /home/hadoop/hd_tmp/dfs/data, expected: rwxr-xr-x, while actual: rwxrwxr-x
ERROR org.apache.hadoop.hdfs.server.datanode.DataNode: All directories in dfs.data.dir are invalid.
...
...
</code></pre>

<p>So I simply executed these commands:</p>

<ul>
<li>I stopped all the demons with "bin/stop-all.sh"</li>
<li>Change the permissions of the directory with "chmod -R 755 /home/hadoop/hd_tmp"</li>
<li>I gave format again to the namenode with "bin/hadoop namenode -format".</li>
<li>I re-started the demons "bin/start-all.sh"</li>
<li>And voilà, the datanode was up and running! (I checked it with the command "jsp", where a process named DataNode was shown).</li>
</ul>

<p>And then everything worked fine.</p>

<p>I encountered the same problem. When I looked at <strong>localhost:50070</strong>, under the cluster summary, all properties were shown as 0 except "DFS Used% 100". <p>In my case, the cause is unable to resolve the hostname. I solved the problem simply by adding "IP_Address  hostname" to <strong>/etc/hosts</strong>.</p>
