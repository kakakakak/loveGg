<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The <code>Spring for Apache Hadoop</code> framework has a <code>Hive</code> integration, having a look at
the <a href="https://github.com/SpringSource/spring-hadoop/tree/master/src/main/java/org/springframework/data/hadoop/hive" rel="nofollow">source code</a> might give you some idea how to run hql scripts from code.</p>

<p>On the other hand you can also check the <code>Hive</code> sources as well (especially <a href="http://hive.apache.org/docs/r0.9.0/api/index.html?org/apache/hadoop/hive/cli/CliSessionState.html" rel="nofollow">CliSessionState</a> and <a href="http://hive.apache.org/docs/r0.9.0/api/index.html?org/apache/hadoop/hive/cli/CliDriver.html" rel="nofollow">CliDriver</a>)
to see how <em>Hive shell</em> picks up a hql file (i.e: <code>hive -f file.q</code>) . </p>

<p>Based on these such a raw implementation can do the job:</p>

<pre><code>import java.io.PrintStream;
import org.apache.hadoop.hive.cli.CliDriver;
import org.apache.hadoop.hive.cli.CliSessionState;
import org.apache.hadoop.hive.common.LogUtils;
import org.apache.hadoop.hive.conf.HiveConf;
import org.apache.hadoop.hive.ql.session.SessionState;

public class RunHQLScript {

    private static class MyCliSessionState extends CliSessionState {
        public MyCliSessionState(HiveConf conf, String host, int port) {
            super(conf);
            this.host = host;
            this.port = port;
        }
    }

    public static void main(String[] args) throws Exception {

        LogUtils.initHiveLog4j();
        CliSessionState ss = new MyCliSessionState(new HiveConf(SessionState.class),
                "localhost", 10000);

        ss.in = System.in;
        ss.out = new PrintStream(System.out, true, "UTF-8");
