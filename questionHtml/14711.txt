<p>I am using Cloudera 4.2.0 and Spark.</p>

<p>I just want to try out some examples given by Spark.</p>

<pre><code>// HdfsTest.scala
package spark.examples

import spark._

object HdfsTest {
  def main(args: Array[String]) {
    val sc = new SparkContext(args(0), "HdfsTest",
      System.getenv("SPARK_HOME"), Seq(System.getenv("SPARK_EXAMPLES_JAR")))

    val file = sc.textFile("hdfs://n1.example.com/user/cloudera/data/navi_test.csv")
    val mapped = file.map(s =&gt; s.length).cache()
    for (iter &lt;- 1 to 10) {
      val start = System.currentTimeMillis()
      for (x &lt;- mapped) { x + 2 }
      //  println("Processing: " + x)
      val end = System.currentTimeMillis()
      println("Iteration " + iter + " took " + (end-start) + " ms")
    }
    System.exit(0)
  }
}

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>(This question was also asked / answered on the <a href="https://groups.google.com/d/msg/spark-users/DO-HncHnXOs/rBlwXLeXpSIJ" rel="nofollow">spark-users mailing list</a>).</p>

<p>You need to compile Spark against the particular version of Hadoop/HDFS running on your cluster.  From the <a href="http://spark-project.org/docs/latest/#a-note-about-hadoop-versions" rel="nofollow">Spark documentation</a>:</p>

<blockquote>
  <p>Spark uses the Hadoop core library to talk to HDFS and other Hadoop-supported storage systems. Because the HDFS protocol has changed in different versions of Hadoop, you must build Spark against the same version that your cluster runs. You can change the version by setting the <code>HADOOP_VERSION</code> variable at the top of <code>project/SparkBuild.scala</code>, then rebuilding Spark (<code>sbt/sbt clean compile</code>).</p>
</blockquote>

<p>The <a href="http://groups.google.com/group/spark-users" rel="nofollow">spark-users</a> mailing list archives contain several questions about compiling against specific Hadoop versions, so I would search there if you run into any problems when building Spark.</p>

<p>You can set Coudera's Hadoop version with an environment variable when building Spark, look up your exact artifact version on Cloudera's maven repo, should be this:</p>

<pre><code>SPARK_HADOOP_VERSION=2.0.0-cdh4.2.0 sbt/sbt assembly publish-local
</code></pre>

<p>Make sure you run whatever you run with the same Java engine you use to build Spark. Also, there are pre-built Spark packages for different Cloudera Hadoop distributions, like <a href="http://spark-project.org/download/spark-0.8.0-incubating-bin-cdh4.tgz" rel="nofollow">http://spark-project.org/download/spark-0.8.0-incubating-bin-cdh4.tgz</a></p>
