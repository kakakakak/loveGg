    <div class="post-text" itemprop="text">

<p>I put some files into hdfs (<code>/path/to/directory/</code>) which contain data like following;</p>

<pre><code>63  EB44863EA74AA0C5D3ECF3D678A7DF59
62  FABBC9ED9719A5030B2F6A4591EDB180
59  6BF6D40AF15DE2D7E295EAFB9574BBF8
</code></pre>

<p>All of them named as <code>_user_hive_warehouse_file_name_000XYZ_A</code>. These files had downloaded from another hdfs.</p>

<p>I'm trying to create external table via Hive;</p>

<pre><code>CREATE EXTERNAL TABLE users(
id int,
user string
)
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t'
LOCATION '/path/to/directory/';
</code></pre>

<p>It says;</p>

<pre><code>OK
Time taken: 0.098 seconds
</code></pre>

<p><code>select * from users;</code> returns <strong>empty</strong>.</p>

<p><code>select count(1) from users;</code> returns <strong>0</strong>.</p>

<p>Hive creates the table successfully, but it's always empty. If I put another file like another.txt, that contains the sample data mentioned above, <code>select count(1) from users;</code> returns <strong>3</strong>.</p>

<p>What am I missing, why the table is empty?</p>

<p><strong>Environment</strong>:</p>

<ul>
<li>JDK 7</li>
<li>Hadoop 2.6.0</li>
<li>Hive 0.14.0</li>
<li>Ubuntu 14.04</li>
</ul>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>When you run any command on Hive, it is run internally as a MapReduce Job on the HDFS path that you stored the file. The job uses the FileInputFormat to read the HDFS files which has a hiddenFileFilter which ignores any files starting with underscore ("_") and ("."). You can actually set other files to ignore by setting the <a href="http://hadoop.apache.org/docs/current/api/org/apache/hadoop/mapred/FileInputFormat.html#setInputPathFilter(org.apache.hadoop.mapred.JobConf,%20java.lang.Class)" rel="nofollow">FileInputFormat.SetInputPathFilter</a> to a CustomPathFilter. Hadoop uses the files with underscores are "special" files to show job output and logs. This is probably why they are ignored.</p>
