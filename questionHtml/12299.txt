    <div class="post-text" itemprop="text">

<p>Its a continuation of this question: <a href="http://stackoverflow.com/questions/32276856/porting-a-multi-threaded-compute-intensive-job-to-spark">Porting a multi-threaded compute intensive job to spark</a></p>

<p>I am using <code>forEachPartition</code> as suggested <a href="http://stackoverflow.com/a/32277967/231917">here</a> to loop over a list of 10000 IDs, then I do a <code>repartition(20)</code> because every partition creates DB connection and if I create say 100 partitions, the job just dies because of 100 open connections to postgres and mongo. I use postgres connections not only to store data but to lookup some data from another table. 
I can get rid of storing the data to postgres directly from my task and do it as post processing from a sequence file.</p>

<p>But I ideally need to massively parallelize my spark job so that the task completed within a given time, currently it processed about 200 IDs in 20hrs, but I need to process 10000 IDs in 20hrs. So  <code>repartition(20)</code> is clearly not helping. I am bound by IO on db here. </p>

<p>So what are my options where I can efficiently share this data across all tasks? I want data in mongo and postgres to be treated as in memory lookup tables - total size is about 500gb.</p>

<p>My options are:</p>

<ol>
<li>RDD (I don't think RDD fits my usecase)  </li>
<li>Dataframe  </li>
<li>Broadcast variables (not sure of this will work as its creation needs 500gb available in the spark driver)  </li>
<li>Move data from mongo to s3 and tasks lookup from s3.</li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The techinique we follow for this kind of problem is to:</p>

<ol>
<li>Store the lookups in different collection of MongoDB.</li>
<li>Using Hadoop MongoDB connector get the data from MongoDB and store it in an RDD</li>
<li>Broadcast the variable so as it will be available to all the Node/Worker</li>
<li>Now if the data is in HDFS create an RDD for it or if the data is  in MongoDB using the Hadoop MongoDB connector.</li>
<li>Now perform the lookup matching part</li>
<li>Save the file as an Sequence file or you can also save it on S3 need to check on it as we store it back to MongoDB</li>
</ol>
