<p>I am running one node cassandra 2.0.3 and Apache Spark 2.0.3</p>

<p>I have created a scala program to create an RDD using Spark hadoop API to access Cassandra DB.</p>

<p>Also what environment variables should be set up for spaark in bashrc as I am using the below config in spark-env.sh</p>

<pre><code>export SPARK_MASTER_IP="10.0.3.15"

export SPARK_MASTER_PORT="7077"

export SCALA_HOME="/home/Desktop/CD/scala-2.9.3"

export SPARK_WORKER_MEMORY=1g

export SPARK_WORKER_INSTANCES=1

export SPARK_WORKER_DIR="/home/Desktop/CD/spark-0.8.0-incubating/sparkdata"
</code></pre>

<p>My sample scala code is as below</p>

<pre><code>            val job=new Job()
    job.setInputFormatClass(classOf[ColumnFamilyInputFormat])
    val host: String = "localhost"
            val port: String = "9160"
    ConfigHelper.setInputInitialAddress(job.getConfiguration(), host)
        ConfigHelper.setInputRpcPort(job.getConfiguration(), port)
    ConfigHelper.setInputColumnFamily(job.getConfiguration(), "demodb", "emp")
    ConfigHelper.setInputPartitioner(job.getConfiguration(), "Murmur3Partitioner")
    CqlConfigHelper.setInputColumns(job.getConfiguration(), "empid,deptid,first_name,last_name")
            CqlConfigHelper.setInputWhereClauses(job.getConfiguration(),"empid=104")

    // Make a new Hadoop RDD
    val casRdd = sc.newAPIHadoopRDD(job.getConfiguration(),
                            classOf[CqlPagingInputFormat],
                            classOf[Map[String, ByteBuffer]],
                            classOf[Map[String, ByteBuffer]])

    println(casRdd.count())
</code></pre>

<p>As you can see <a href="https://github.com/apache/cassandra/blob/cassandra-2.0.3/src/java/org/apache/cassandra/hadoop/cql3/CqlPagingRecordReader.java#L661" rel="nofollow">here</a> this is caused because a generic unhandled exception is being thrown in the Cassandra's CQL RecordReader. </p>

<p>Try to run Spark with Debug logging enabled and you should get the error that is causing this issue.</p>

<p>My <strong>guess</strong>  is the problem will be that you are trying to use setting ColumnFamilyInputFormat in jobConfiguration and passing classOf[CqlPagingInputFormat] to the newHadoopApiRdd method. But htat is just a guess.</p>

<p>Though <a href="http://tuplejump.com" rel="nofollow">we</a> have currently only released <a href="http://tuplejump.github.com/calliope/" rel="nofollow">Calliope</a> built only against Cassandra 1.2.12, our client have used it with Cassandra 2.x successfully without any change. Calliope gives you a simpler and higher level API to work with Cassandra from Spark. I will suggest you give it a try.</p>
