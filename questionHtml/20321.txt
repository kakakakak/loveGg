    <div class="post-text" itemprop="text">

<p>I have forked git repository of one project on github
and made my own changes to it.
I wanted to get diff between my repository and original
repository that I've forked. Can someone tell me git command
to get that diff ?
I need to submit the diff for review.</p>

<pre>
Original repository:
git://github.com/apache/hive.git
My repository:
git@github.com:prafullat/hive.git
</pre>

<p>Here are details from my .git/config</p>

<pre>
[remote "origin"]
    fetch = +refs/heads/*:refs/remotes/origin/*
    url = git@github.com:prafullat/hive.git
[remote "mirror"]
    fetch = +refs/heads/*:refs/remotes/origin/*
    url = git://github.com/apache/hive.git
</pre>

<p>I tried looking at already posted questions regarding the same topic and
could not get it to work.
Any help would be highly appreciated.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You need to fetch the latest of both remote repositories and compare the main branches to each other.  It looks like the main branch is the 'trunk' branch, so you can see what commits are unique to your project (and not in the trunk branch of the 'mirror' project) like this:</p>

<pre><code>$ git log --oneline origin/trunk ^mirror/trunk
1c4fa82 1. Modified the flag name for gb_to_idx rewrite to    hive.ql.rewrite.gb_to_idx    So

</code></pre>

<p>Or you can remove the <code>--oneline</code> to see the full commit messages.  It looks like they're all yours.  You can also add a <code>--no-merges</code> if you don't want to see those merge commits.</p>

<p>Next, you can get the actual diff by running this:</p>

<pre><code>$ git diff --stat mirror/trunk...origin/trunk
 README.txt                                         |    4 +-
 build.xml                                          |    1 +
 .../java/org/apache/hadoop/hive/conf/HiveConf.java |    1 +
 ivy/ivysettings.xml                                |    4 +-

 .../hadoop/hive/metastore/IMetaStoreClient.java    |   14 +

(bunch more lines)
 ql/src/test/queries/clientpositive/index_compact.q |   13 +

 .../clientpositive/ql_rewrite_gbtoidx.q.out        |  211 ++++++

<p>If you remove the <code>--stat</code>, you'll get the actual diff.  The <code>...</code> in between the <code>mirror/trunk</code> and <code>origin/trunk</code> is a shorthand saying you want the diff between the common ancestor, so it doesn't give you a diff removing everything added to the original project since you started, it just gives you the changes you've made on your branch.</p>

    <div class="post-text" itemprop="text">
<p>There is a flaw in this, in case anyone comes across this question again.</p>

<pre><code>[remote "origin"]
    fetch = +refs/heads/*:refs/remotes/origin/*
    url = git@github.com:prafullat/hive.git
[remote "mirror"]
    fetch = +refs/heads/*:refs/remotes/origin/*
    url = git://github.com/apache/hive.git
</code></pre>

<p>Whenever you do a git fetch on mirror it overwrites the same remote branch in your .git/refs/remotes. You should make sure to change the remote mirror fetch to reflect the new name. </p>

<pre><code>[remote "origin"]
    fetch = +refs/heads/*:refs/remotes/origin/*
    url = giturl
[remote "mirror"]
    fetch = +refs/heads/*:refs/remotes/mirror/*
    url = giturl
</code></pre>

<p>Then a simple</p>

<pre><code>git diff origin/master..mirror/master
</code></pre>

    <div class="post-text" itemprop="text">
<p>One downside of the approaches above is that they diff a specific <em>pair of branches</em>. If you want to find <em>all</em> commits where your local repo differs from the remote -- across all branches you may have created, even if some aren't merged back into master yet -- you can do the following:</p>

<pre><code>git log --branches --not --remotes
</code></pre>
