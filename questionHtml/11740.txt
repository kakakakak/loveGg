<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Definitively. Check out, for example, <a href="http://en.wikipedia.org/wiki/Bulk_synchronous_parallel" rel="nofollow">Bulk Synchronous Parallel</a>. Map/Reduce is in fact a very restricted way of reducing problems, however that restriction makes it manageable in a framework like Hadoop. The question is if it is less trouble to press your problem into a Map/Reduce setting, or if its easier to create a domain-specific parallelization scheme and having to take care of all the implementation details yourself. Pig, in fact, is only an abstraction layer on top of Hadoop which automates many standard problem transformations from not-Map-Reduce-y to Map-Reduce-compatible.</p>

<p>Edit 26.1.13: Found a <a href="http://www.dataminingblog.com/think-you-need-hadoop-think-again/" rel="nofollow">nice up-to-date overview here</a></p>

<p>Phil Colella  identified seven numerical methods for scientific computation based on the patterns of scattering and gathering of data between processing nodes, and called them 'dwarfs'. These have been added to by others, a list is available at the <a href="http://view.eecs.berkeley.edu/wiki/Dwarf_Mine">Dwarf Mine</a>:</p>

<ol>
<li>Dense Linear Algebra</li>
<li>Sparse Linear Algebra</li>
<li>Spectral Methods</li>
<li>N-Body Methods</li>
<li>Structured Grids</li>
<li>Unstructured Grids</li>
<li>MapReduce</li>
<li>Combinational Logic</li>
<li>Graph Traversal</li>
<li>Dynamic Programming</li>
<li>Backtrack and Branch-and-Bound</li>
<li>Graphical Models</li>
<li>Finite State Machines</li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>Update (August 2014):</strong> Stratosphere is now called <a href="http://flink.incubator.apache.org/" rel="nofollow">Apache Flink</a> (incubating).</p>

<p>Have a look at <a href="http://stratosphere.eu/" rel="nofollow">Stratosphere</a>. It is another Big Data runtime that offers more operators (map, reduce, join, union, cross, iterate, ...). It also allows to define advanced data flow graphs (with Hadoop MR, you would have to chain jobs).</p>

<p>Stratosphere also supports BSP with its graph processing abstraction (called <a href="http://stratosphere.eu/docs/0.4/programming_guides/spargel.html" rel="nofollow">Spargel</a>).</p>

<p>If you like to read scientific papers, have a look at <a href="http://stratosphere.eu/assets/papers/NephelePACTs_10.pdf" rel="nofollow">Nephele/PACTs: A Programming Model and Execution Framework for Web-Scale Analytical Processing</a>, it explains the theoretical backgrounds of the system.</p>

<p>Another system in the field is <a href="http://spark.incubator.apache.org/" rel="nofollow">Spark</a> which has its own model (RDDs). Since BSP has been mentioned here, also have a look at <a href="http://graphlab.org/" rel="nofollow">GraphLab</a>, the offer an alternative to BSP.</p>
