    <div class="post-text" itemprop="text">

<p>I'd like to control the Hadoop job retry policy using command line options. For example, I'd like to write the following Sqoop job and have it be fault tolerant to MySQL outages:</p>

<pre><code>sqoop
  GENERIC_SQOOP_OPTIONS
  -Dmap.retry.exponentialBackOff=TRUE
  -Dmap.retry.numRetries=10
</code></pre>

<p>For a general Hadoop job, it might look something like:</p>

<pre><code>hadoop job -submit
  GENERIC_JOB_OPTIONS
  -Dmap.retry.exponentialBackOff=TRUE
  -Dmap.retry.numRetries=10
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Yes, hadoop "-D" arguments are supported in Sqoop. However their position on command line is crucial. They must be present directly after "sqoop" and before any other Sqoop arguments:</p>

<pre><code>sqoop -Dmap.retry.exponentialBackOff=TRUE -Dmap.retry.numRetries=10 import --connect ...
</code></pre>

<p>Please note that different positioning will not work as expected.</p>
