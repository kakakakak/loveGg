<p>I have a very simply formatted XML document that I would like to translate into TSV suitable for an import into Hive. The formatting of this document is straightforward:</p>

<pre><code>&lt;root&gt;
   &lt;row&gt;
      &lt;ID&gt;0&lt;/ID&gt;
      &lt;ParentID&gt;0&lt;/ParentID&gt;
      &lt;Url&gt;&lt;/Url&gt;
      &lt;Title&gt;&lt;/Title&gt;
      &lt;Text&gt;&lt;/Text&gt;
      &lt;Username&gt;&lt;/Username&gt;
      &lt;Points&gt;0&lt;/Points&gt;
      &lt;Type&gt;0&lt;/Type&gt;
      &lt;Timestamp&gt;&lt;/Timestamp&gt;
      &lt;CommentCount&gt;0&lt;/CommentCount&gt;
   &lt;/row&gt;
&lt;/root&gt;
</code></pre>

<p>I have a working Ruby script that will translate a document formatted as above into TSVs properly. That's here:</p>

<pre><code>require "rubygems"
require "crack"

xml = Crack::XML.parse(File.read("sample.xml"))

xml['root']['row'].each{ |i|
  puts "#{i['ID']}      #{i['ParentID']}        #{i['Url']}     #{i['Title']}..." 
}
</code></pre>

<p>Unfortunately, the files I need to translate are substantially larger than this script can handle (> 1 GB). </p>

<p>Which is where Hadoop comes in. The simplest solution is probably to write a MapReduce job in Java, but that's not an option given that I lack Java skills. So I wanted to write the a mapper script in either Python or Ruby which I am far from expert in, but can at least navigate. </p>

<p>My plan then was to do the following:</p>

<ol>
<li>use StreamXmlRecordReader to parse the file record by record</li>
<li>map the deserialization using crack</li>
<li>reduce it with a simple regurgitation of the elements spaced by tabs</li>
</ol>

<p>This approach has failed consistently, however. I've used a variety of Ruby/Wukong scripts with no success. Here's one based off the article <a href="http://steveeichert.com/hadoop,/mapreduce/2010/04/22/using-hadoop-streaming-for-xml-processing.html" rel="nofollow">here</a>:</p>

<pre><code>#!/usr/bin/env ruby

require 'rubygems'
require 'crack'

xml = nil
STDIN.each_line do |line|
  puts |line|
  line.strip!

  if line.include?("&lt;row")
    xml = Crack::XML.parse(line)
    xml['root']['row'].each{ |i|
      puts "#{i['ID']}      #{i['ParentID']}        #{i['Url']}..."     
  else
    puts 'no line'
  end

  if line.include?("&lt;/root&gt;")
    puts 'EOF'
  end
end
</code></pre>

<p>This and other jobs fail as follows:</p>

<p>The second problem is that I'm told by a gracious and helpful expert that because StreamXmlRecordReader doesn't produce an additional record delimiter, this approach probably isn't going to work, and that I'll need to read in single lines, grep for row, stack up everything until you get /row, and then parse it.</p>

<p>Is this the simplest approach, and if so, how I might best accomplish that?</p>

<p>Performance isn't a huge issue, because these files are batch processed every few weeks or so, just in case that helps.</p>
