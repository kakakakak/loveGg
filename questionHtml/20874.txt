<p>The order of the mapWithIndex in a RDD isn't fixed if the parameter preservesPartitioning is not set.</p>

<p>So add after the function a ,true and it should work... Like:</p>

<pre><code>val csv = sc.textFile("./project/Casos_Notificados_Dengue_01_2010.csv")

val rdd = csv.mapPartitionsWithIndex(
    ((i,iterator) =&gt; if (i == 0 &amp;&amp; iterator.hasNext){
      iterator.next
      iterator
    }else iterator), true)
</code></pre>

<p>For the new info: I think it's already a whole new question but...
First you're taking a text file and putting it in a RDD it will be an RDD of strings, so you probably will not be able to use a GroupBy...</p>

<p>First let's make a schema:</p>

<pre><code>import org.apache.spark.sql._
import org.apache.spark.sql.types._
val schemaArray = csv.collect()(0).split(",")
val schema =
  StructType(
    schemaArray.map(fieldName =&gt; StructField(fieldName, StringType, true)))
</code></pre>

<p>Now that we have an schema... It's as simple as the tutorial goes, but i will write it here:</p>

<pre><code>val rowRDD = rdd.map(_.split(",")).map(p =&gt; Row.fromSeq(p))

// Apply the schema to the RDD.
val schemaRDD = sqlContext.applySchema(rowRDD, schema)

// Register the SchemaRDD as a table.
schemaRDD.registerTempTable("casos")

// SQL statements can be run by using the sql methods provided by sqlContext.
val results = sqlContext.sql("SELECT Column1,Column2,Column3,Column4,Column5 FROM casos")

// The results of SQL queries are SchemaRDDs and support all the normal RDD operations.
// The columns of a row in the result can be accessed by ordinal.
results.map(t =&gt; "Column1: " + t(0)).collect().foreach(println)
//You may also want to refine the new object somehow
results.registerTempTable("tempTable")
val results2 = sqlContext.sql("SELECT Column1,Column2,Column3,Column4,Column5 FROM casos WHERE Column1=1")
</code></pre>

<p>I may have forgotten some import,so i will leave here the link to the tutorial: <a href="https://spark.apache.org/docs/1.1.0/sql-programming-guide.html#rdds" rel="nofollow">https://spark.apache.org/docs/1.1.0/sql-programming-guide.html#rdds</a></p>

<p>So for your specific task I'd suggest looking at the spark-csv package on <a href="http://spark-packages.org" rel="nofollow">http://spark-packages.org</a> . If you end up having to do the parsing by hand, you might just want to call <code>first()</code> to get the first line.</p>

<p>The other option is you could do this similar to how CsvRleation in spark-csv works which is grabbing the first line and then filtering out that line if it shows up anywhere:</p>

<pre><code>  /**
   * Returns the first line of the first non-empty file in path
   */
  private lazy val firstLine = {
    // Using Spark to read the first line to be able to handle all Hadoop input (gz, bz, etc.)
    sqlContext.sparkContext.textFile(location).first()
  }
  rdd.filter(_ != firstLine)
</code></pre>
