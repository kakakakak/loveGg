<p>It is easy to join datasets by single key simply by sending join field as a reducer key.
But joining records by several keys where at least one shoud be the same is not that easy for me.</p>

<p>Example I have logs and I want to group them by user parameters, I want to join them by (ipAddress, sessionId,visitorCockies)</p>

<p>So log1 should be grouped with log2 if log1.ip == log2.ip OR log1.session = log2.session OR log1.cockie = log2.coockie. Perhaps it is possible to create composite key or some probabalistic approach like minHash...</p>

<p>Is it possible? </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The issue is that MapReduce joins are typically implemented by giving records that match on some field the same reduce key so that they get sent to the same reducer. So anything to get around this is going to be a bit of a hack, but it is possible...</p>

<p>Here's what I would recommend: for each input record, generate three copies, each with a new "key" field that is prefixed by the field it's coming from. So for example, say you had the following input:</p>

<pre><code>(ip=1.2.3.4, session=ABC, cookie=123)
(ip=3.4.5.6, session=DEF, cookie=456)
</code></pre>

<p>Then you would generate</p>

<pre><code>(ip=1.2.3.4, session=ABC, cookie=123, key=ip_1.2.3.4)
(ip=1.2.3.4, session=ABC, cookie=123, key=session_ABC)
(ip=1.2.3.4, session=ABC, cookie=123, key=cookie_123)
(ip=3.4.5.6, session=DEF, cookie=456, key=ip_3.4.5.6)
(ip=3.4.5.6, session=DEF, cookie=456, key=session_DEF)
(ip=3.4.5.6, session=DEF, cookie=456, key=cookie_456)
</code></pre>

<p>And then you could simply group on this new field.</p>

<p>I'm not too familiar with scalding/cascading (although I've been meaning to learn more about it) but this would definitely conform to how joins are generally done in Hadoop.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>After you created the separate joins as described by Joe above, you need to get rid of the duplicates. Two tuples in your data are duplicates iff they are equal in all fields you use in your "OR-join". So if you do a natural join on a key representing all relevant fields afterwards, you'll get all duplicates grouped together. Hence you can replace them by a single occurrence of the respective tuple.</p>

<p>Let's look at an example: Assume you've got tuples with fields (A,B,C,D) and the fields you're interested in are A, B, and C. You would first do equi-joins on A, B, and C separately. For each, you'd join the initial tuple stream with itself. Denote the first stream with (A0, B0, C0, D0) and the second with (A1, B1, C1, D1). The result will be tuples (A0, B0, C0, D0, A1, B1, C1, D1). For each of those tuples, you'd create a tuple (A0A1B0B1C0C1, A0, B0, C0, D0, A1, B1, C1, D1), so all duplicates will be grouped together in a subsequent reducer. For each group, return only one of the contained tuples.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>Tip</strong>: Use type aliases to make your Scalding code nice to read</p>

<p><strong>Note 0</strong>: This solution is particularly nice because it will always be just 1 mapred job, even when there are more keys to join on.</p>

<p><strong>Note 1</strong>: Assumes each pipe doesn't have duplicate keys, otherwise you would have to make 'key also have an index for which log it came from, and the mapTo would be a flatMapTo and bit more complicated.</p>

<p><strong>Note 2</strong>: For simplicity this will discard the joining fields, to keep them you'd need a big ugly tuple (ip1, ip2, session1, session2, ...etc). If you really want I can write out an example that keeps them.</p>

<p><strong>Note 3</strong>: If you really wanted to merge the duplicate values, you could follow this with a groupBy each of logEntry1 and logEntry2, produce a logEntryList, then cat (as mentioned in a comment this is not normal for a join). This will create 2 more mapred jobs.</p>

<pre><code>type String2 = (String, String)
type String3 = (String, String, String)

def addKey(log: Pipe): Pipe = log.flatMap[String3, String](('ip, 'session, 'cookie) -&gt; 'key)(
  _.productIterator.toList.zipWithIndex.map {
    case (key: String, index: Int) =&gt; index.toString + key
  }
)

(addKey(log1) ++ addKey(log2)).groupBy('key)(_.toList[String]('logEntry -&gt; 'group))
.mapTo[Iterable[String], String2]('group -&gt; ('logEntry1, 'logEntry2))(list =&gt; (list.head, list.last))
</code></pre>
