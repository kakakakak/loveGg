<p>In simple word count map reduce program the output we get is sorted by words. Sample output can be :<Br>
Apple 1<Br>
Boy 30<Br>
Cat 2<Br>
Frog 20<Br>
Zebra 1<Br>
If you want output to be sorted on the basis of number of occrance of words, i.e in below format<Br>
1 Apple<Br>
1 Zebra<Br>
2 Cat<Br>
20 Frog<Br>
30 Boy<Br>
You can create another MR program using below mapper and reducer where the input will be the output got from simple word count program.</p>

<pre><code>class Map1 extends MapReduceBase implements Mapper&lt;Object, Text, IntWritable, Text&gt;
{
    public void map(Object key, Text value, OutputCollector&lt;IntWritable, Text&gt; collector, Reporter arg3) throws IOException 
    {
        String line = value.toString();
        StringTokenizer stringTokenizer = new StringTokenizer(line);
        {
            int number = 999; 
            String word = "empty";

            if(stringTokenizer.hasMoreTokens())
            {
                String str0= stringTokenizer.nextToken();
                word = str0.trim();
            }

            if(stringTokenizer.hasMoreElements())
            {
                String str1 = stringTokenizer.nextToken();
                number = Integer.parseInt(str1.trim());
            }

            collector.collect(new IntWritable(number), new Text(word));
        }

    }

}


class Reduce1 extends MapReduceBase implements Reducer&lt;IntWritable, Text, IntWritable, Text&gt;
{
    public void reduce(IntWritable key, Iterator&lt;Text&gt; values, OutputCollector&lt;IntWritable, Text&gt; arg2, Reporter arg3) throws IOException
    {
        while((values.hasNext()))
        {
            arg2.collect(key, values.next());
        }

    }

}
</code></pre>

    <div class="post-text" itemprop="text">
<p>The output from the Hadoop MapReduce wordcount example is sorted by the key. So the output should be in alphabetical order.</p>

<p>With Hadoop you can create your own key objects that implement the <code>WritableComparable</code> interface allowing you to override the <code>compareTo</code> method. This allows you to control the sort order.</p>

<p>To create an output that is sorted by the number of occurances you would probably have to add another MapReduce job to process the output from the first as you have said. This second job would be very simple, maybe not even requiring a reduce phase. You would just need to implement your own <code>Writable</code> key object to wrap the word and its frequency. A custom writable looks something like this:</p>

<pre><code> public class MyWritableComparable implements WritableComparable {
       // Some data
       private int counter;
       private long timestamp;

       public void write(DataOutput out) throws IOException {
         out.writeInt(counter);
         out.writeLong(timestamp);
       }

       public void readFields(DataInput in) throws IOException {
         counter = in.readInt();
         timestamp = in.readLong();
       }

       public int compareTo(MyWritableComparable w) {
         int thisValue = this.value;
         int thatValue = ((IntWritable)o).value;
         return (thisValue &lt; thatValue ? -1 : (thisValue==thatValue ? 0 : 1));
       }
     }
</code></pre>

<p>I grabbed this example from <a href="http://hadoop.apache.org/common/docs/current/api/org/apache/hadoop/io/WritableComparable.html" rel="nofollow">here</a>.</p>

<p>You should probably override <code>hashCode</code>, <code>equals</code> and <code>toString</code> as well.</p>

<p>As you have said, one possibility is to write two jobs to do this.
First job:
Simple wordcount example</p>

<p>Second job:
Does the sorting part.</p>

<p>The pseudo code could be:</p>

<p><strong>Note : The output file generated by the first job will be the input for the second job</strong></p>

<pre><code>    Mapper2(String _key, Intwritable _value){
    //just reverse the position of _value and _key. This is useful because reducer will get the output in the sorted and shuffled manner.
    emit(_value,_key);
    }

    Reduce2(IntWritable valueofMapper2,Iterable&lt;String&gt; keysofMapper2){
//At the reducer side, all the keys that have the same count are merged together.
        for each K in keysofMapper2{
        emit(K,valueofMapper2); //This will sort in ascending order.
        }

    }
</code></pre>

<p>You can also sort in descending order for which it is feasible to write a separate comparator class which will do the trick.
Include comparator inside the job as:</p>

<pre><code>Job.setComparatorclass(Comparator.class);
</code></pre>

<p>This comparator will sort the values in descending order before sending to the reducer side. So on the reducer, you just emit the values.</p>

<p>In Hadoop sorting is done between the Map and the Reduce phases.  One approach to sort by word occurance would be to use a custom group comparator that doesn't group anything; therefore, every call to reduce is just the key and one value.  </p>

<pre><code>public class Program {
   public static void main( String[] args) {

      conf.setOutputKeyClass( IntWritable.class);
      conf.setOutputValueClass( Text.clss);
      conf.setMapperClass( Map.class);
      conf.setReducerClass( IdentityReducer.class);
      conf.setOutputValueGroupingComparator( GroupComparator.class);   
      conf.setNumReduceTasks( 1);
      JobClient.runJob( conf);
   }
}

public class Map extends MapReduceBase implements Mapper&lt;Text,IntWritable,IntWritable,Text&gt; {

   public void map( Text key, IntWritable value, OutputCollector&lt;IntWritable,Text&gt;, Reporter reporter) {
       output.collect( value, key);
   }
}

public class GroupComaprator extends WritableComparator {
    protected GroupComparator() {
        super( IntWritable.class, true);
    }

    public int compare( WritableComparable w1, WritableComparable w2) {
        return -1;
    }
}
</code></pre>
