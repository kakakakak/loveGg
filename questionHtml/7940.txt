<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The datanode on one of the slave machines in my cluster was throwing a similar exception with port binding:</p>

<pre><code>ERROR org.apache.hadoop.hdfs.server.datanode.DataNode: java.net.BindException: Address already in use
</code></pre>

<p>I noticed that the default web interface port of datanode i.e. 50075 was already bind to another application:</p>

<pre><code>[ap2]-&gt; netstat -an | grep -i 50075
tcp        0      0 10.0.1.1:45674              10.0.1.1:50075              ESTABLISHED
tcp        0      0 10.0.1.1:50075              10.0.1.1:45674              ESTABLISHED
[ap2]-&gt; 
</code></pre>

<p>I have changed the Datanode web interface in <code>conf/hdfs-site.xml</code>:</p>

<pre><code>&lt;property&gt;
&lt;name&gt;dfs.datanode.http.address&lt;/name&gt;
&lt;value&gt;10.0.1.1:50080&lt;/value&gt;
&lt;description&gt;Datanode http port&lt;/description&gt;
&lt;/property&gt;
</code></pre>

<p>This helped to resolve the issue, similarly you can change the default address and port where the web interface listens by setting <code>dfs.http.address</code> in <code>conf/hadoop-site.xml</code>, e.g. localhost:9090, but ensure that port is available.</p>
