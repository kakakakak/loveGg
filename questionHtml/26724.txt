<p>I have to use Hadoop for my research work and I am deciding for the best option to start with. So far I have end up to go with Cloudera. I've downloaded the <a href="http://www.cloudera.com/content/cloudera/en/downloads/quickstart_vms/cdh-5-3-x.html" rel="nofollow">quick start VM</a>
and started learning different <a href="http://www.cloudera.com/content/cloudera/en/training/roles/developers.html" rel="nofollow">turorials</a>.</p>

<p>The issue is that my system can't afford to run it and perform very slow and I think it might just stop working after I feed it with all the data and run other services.</p>

<p>I was advised to go for a cloud service with 4 cluster node. Can someone please help me by providing the best option and estimated pricing to consider? 1 year plan might be enough to complete my research.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you are a linux user, Just download the individual components(like hdfs, MR1, YARN, Hbase, Hive etc...) from this  <a href="http://archive.cloudera.com/cdh5/cdh/5/" rel="nofollow">Cloudera Archives</a> instead of loading Cloudera Quickstart VM.</p>

<p>If you want to try the 4 node cluster, easiest option is to use cloud. 
There are plenty of cloud providers. I have personally used AWS, Google Cloud, Microsoft Azure, IBM SmartCloud. Out of which, AWS is the best to start with.
It is like pay as you go(use).I can recommend you to use a decent EC2 Machine(4 X m3.large Machines)</p>

<p>Type: m3.large  CPU:2   RAM:7.5G    Storage: 1 x 32 SSD Price: $0.133 per Hour <a href="https://aws.amazon.com/ec2/pricing/" rel="nofollow">AWS Pricing</a></p>

<p>If you plan to do the research for one year, I recommend you to go for VPC. </p>

<p><strong><em>Cons of AWS EC2:</em></strong>
If you launch a machine in EC2, the moment you restart your machine, Your IP and the hostname will get changed. </p>

<p>In AWS VPC, your IP and hostname will remain the same. </p>

<p><strong>If you use 4 Machinesx24x7xone month,it costs you $389.44</strong>.
You can calculate the <a href="http://calculator.s3.amazonaws.com/index.html" rel="nofollow">AWS cost</a> by yourself </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As best as I can see you have two paths: </p>

<ol>
<li>Setup Hadoop in a cloud service provider (i.e. Amazon's EC2 or
Redhat's Openshift. </li>
<li>Use Hadoop-as-a-service (i.e. Amazon's EMR or Microsoft's HDInsight).</li>
</ol>

<p>The first path, setting up Hadoop in a cloud service provider will require you to become a semi-competent Hadoop administrator. If that's your goal, great! However you'll spend a great deal of time learning the necessary skills and mindset to become that. I don't suspect that that is your goal.</p>

<p>The second path is the one I'd recommend out of these two. Using Hadoop-as-a-service you will get up and running faster, but will cost more up front and on an ongoing (per hour basis). You'll still probably save money because you'll be spending less time troubleshooting your Hadoop cluster and more time doing the work you wanted to do in the first place.</p>

<p>I have to wonder, if you can even fit your dataset on your laptop, why are you using big data tools in the first place? True, they'll work. However Big Data is at least partially defined as data sets and computational problems that just don't fit on a single machine.</p>
