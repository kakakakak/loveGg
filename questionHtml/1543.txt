<p>I Installed Sqoop in my local machine. Following are the config information.</p>

<p>Bash.bashrc:</p>

<pre><code>export HADOOP_HOME=/home/hduser/hadoop
export HBASE_HOME=/home/hduser/hbase
export HIVE_HOME=/home/hduser/hive
export HCAT_HOME=/home/hduser/hive/hcatalog
export SQOOP_HOME=/home/hduser/sqoop

export PATH=$PATH:$HIVE_HOME/bin
export PATH=$PATH:$HADOOP_HOME/bin
export PATH=$PATH:$HBASE_HOME/bin
export PATH=$PATH:$SQOOP_HOME/bin
export PATH=$PATH:$HCAT_HOME/bin
</code></pre>

<p>Hadoop:</p>

<pre><code>Version: Hadoop 1.0.3
</code></pre>

<p>Hive:</p>

<pre><code>Version: hive 0.11.0 
</code></pre>

<p>Mysql Connector driver</p>

<pre><code>version: mysql-connector-java-5.1.29

"The driver is copied to the lib folder of sqoop"
</code></pre>

<p>Sqoop : </p>

<pre><code>version: sqoop 1.4.4
</code></pre>

<p>After making all the installation I create a table in mysql named practice_1, But when I run the load command to load data from mysql to hdfs the command throws an exception:</p>

<pre><code>ERROR sqoop.Sqoop: Got exception running Sqoop: java.lang.RuntimeException: Could not     load db driver class: com.mysql.jdbc.Driver
</code></pre>

<p>Coud anyone please guide me what can be the possible problem.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You need database driver in 'SQOOP' classpath check <a href="http://sqoop.apache.org/docs/1.4.0-incubating/SqoopUserGuide.html#id1763114">this</a>
It has wonderful explanation about the 'SQOOP'</p>

<p>SQOOP has other options like</p>

<blockquote>
  <p>Ex: --driver com.microsoft.jdbc.sqlserver.SQLServerDriver -libjars=".*jar"</p>
</blockquote>

<p>from <a href="http://sqoop.apache.org/docs/1.4.0-incubating/SqoopUserGuide.html#id1763114">here</a> </p>

<blockquote>
  <p>You can use Sqoop with any other JDBC-compliant database. First, download the appropriate JDBC driver for the type of database you want to import, and install the .jar file in the $SQOOP_HOME/lib directory on your client machine. (This will be /usr/lib/sqoop/lib if you installed from an RPM or Debian package.) Each driver .jar file also has a specific driver class which defines the entry-point to the driver. For example, MySQL's Connector/J library has a driver class of com.mysql.jdbc.Driver. Refer to your database vendor-specific documentation to determine the main driver class. This class must be provided as an argument to Sqoop with --driver.</p>
</blockquote>

<p>You may be interested in understanding the difference between connector and driver <a href="https://blogs.apache.org/sqoop/entry/connectors_and_drivers_in_the">here</a> is the article </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Another solution which avoids using a shared library is adding the driver jar to the classpath of sqoop by using HADOOP_CLASSPATH. I haven't got the -libjars option to work. This solution works also on a secure cluster using kerberos. </p>

<pre><code>HADOOP_CLASSPATH=/use.case/lib/postgresql-9.2-1003-jdbc4.jar
sqoop export --connect jdbc:postgresql://db:5432/user --driver org.postgresql.Driver --connection-manager org.apache.sqoop.manager.GenericJdbcManager --username user -P --export-dir /user/hive/warehouse/db1/table1 --table table2
</code></pre>

<p>This one works at least with sqoop 1.4.3-cdh4.4.0 </p>

    <div class="post-text" itemprop="text">
<p>You need to grant <code>priveleges</code> to the tables as below:</p>

<blockquote>
  <p>grant all privileges on marksheet.* to 'root'@'192.168.168.1'
  identified by 'root123';</p>
  
  <p>flush privileges;</p>
</blockquote>

<p>Here is sample command that I have successfully executed:</p>

<blockquote>
  <p>sqoop import --verbose --fields-terminated-by ',' --connect
  jdbc:mysql://192.168.168.1/test --username root --password root123
      --table student --hive-import --create-hive-table  --hive-home /home/training/hive --warehouse-dir /user/hive/warehouse
      --fields-terminated-by ',' --hive-table studentmysql</p>
</blockquote>
