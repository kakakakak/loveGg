<p>I've been having issues with ssh connections terminating, and thought it might be better to install mosh on the emr master - this way there would be some protection from loss of connectivity. I've created the following script to act as part of my bootstrap, but when I run it, I'm having a few issues.</p>

<pre><code>#!/bin/bash

#
# Make sure we've got git, autoconf, automake, protobuf-compilers
#

sudo yum install -y \
    git \
    autoconf-2.69-11.9.amzn1.noarch \
    automake-1.13.4-3.15.amzn1.noarch \
    protobuf-compiler-2.5.0-1.8.amzn1.x86_64 \
    protobuf-2.5.0-1.8.amzn1.x86_64

#
# Pull the latest code from github
#

cd /home/hadoop
git clone https://github.com/keithw/mosh

#
# Build and install
#

cd mosh
./autogen.sh &amp;&amp; ./configure &amp;&amp; make &amp;&amp; sudo make install
</code></pre>

<p>Currently this falls over on the configure step with the following:</p>

<pre><code>checking for protobuf... no
configure: error: Package requirements (protobuf) were not met:

No package 'protobuf' found

Consider adjusting the PKG_CONFIG_PATH environment variable if you
installed software in a non-standard prefix.
</code></pre>

<p>I've tried setting <code>PREFIX=/usr</code> and <code>PKG_CONFIG_PATH=${PREFIX}/lib64/pkgconfig</code>, then running ./configure with <code>--prefix=$PREFIX</code> - that doesn't help, I've also tried installing the 32 bit package, which also didn't help. This would all be easier if there were a source for <code>sudo yum install -y mosh</code>, however <code>yum whatprovides mosh</code> comes back empty.</p>

<p>Any pointers would be welcome!</p>

    <div class="post-text" itemprop="text">
<p>It depends on which version of AMI you make an installation.
There is a changes you need to make to build it on AMI 3.7.0:</p>

<ul>
<li><p>Do not specify particular version of protobuf package.</p>

<p><em>There is protobuf version 2.5.0-1.10 on new AMIs</em></p></li>
<li><p>Additionally install package <code>protobuf-devel</code> </p></li>
</ul>

<p>So your <code>yum install</code> command should look like this:</p>

<pre><code>sudo yum install -y \
  git \
  autoconf-2.69-11.9.amzn1.noarch \
  automake-1.13.4-3.15.amzn1.noarch \
  protobuf-compiler \
  protobuf \
  protobuf-devel
</code></pre>

<p><strong>UPD:</strong></p>

<p>Don't forget to modify your ElasticMapReduce-master security group to open UDP port 60001.</p>

<ul>
<li>Go to the AWS EC2 WebConsole / Network &amp; Security / Security Groups; </li>
<li>Pick an <code>ElasticMapReduce-master</code> security group from a list;</li>
<li>Go to the <strong>Inbound</strong> tab and and a new rule for UDP port 6001.</li>
</ul>
