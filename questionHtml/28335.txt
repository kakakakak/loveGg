<p><strong>In brief:</strong></p>

<ol>
<li>How to find which Java/Scala thread has locked a file?
I know that a class/thread in JVM has locked a concrete file (overlaps a region of file), but I don't know how. It's possible to find out what class/thread is doing this when I stop application in breakpoint? </li>
</ol>

<p>The following code throws <a href="https://docs.oracle.com/javase/7/docs/api/java/nio/channels/OverlappingFileLockException.html">OverlappingFileLockException</a>:</p>

<pre><code>FileChannel.open(Paths.get("thisfile"), StandardOpenOption.APPEND).tryLock().isValid();
FileChannel.open(Paths.get("thisfile"), StandardOpenOption.APPEND).tryLock()..isShared();
</code></pre>

<ol start="2">
<li>How can Java/Scala lock this file (<a href="https://github.com/apache/spark/">Spark</a>)? I know about how to lock files using java.nio.channels, but I didn't found appropriate invocation in <a href="https://github.com/apache/spark/">github repository of Spark</a>.</li>
</ol>

<hr>

<hr>

<p>I try to do search in the internet, but found just <a href="https://issues.apache.org/jira/browse/SPARK-8333">in progress</a> issues in Spark (one user try to do some patch, but it isn't working, if I undenstand comment to this pull request correctly) and some unanswered questions in SO. </p>

<p>It looks like the problem is in deleteRecursively() method of <a href="https://github.com/apache/spark/blob/3af53e61fd604fe8000e1fdf656d60b79c842d1c/core/src/main/scala/org/apache/spark/util/Utils.scala">Utils.scala</a> class. I set breakpoint to this method and rewrite it to Java:    </p>

<pre><code>public class Test {
    public static void deleteRecursively(File file) {
        if (file != null) {
            try {
                if (file.isDirectory()) {
                    for (File child : listFilesSafely(file)) {
                        deleteRecursively(child);
                    }
                    //ShutdownHookManager.removeShutdownDeleteDir(file)
                }
            } finally {
                if (!file.delete()) {
                    if (file.exists()) {
                        throw new RuntimeException("Failed to delete: " + file.getAbsolutePath());
                    }
                }
            }
        }
    }

    private static List&lt;File&gt; listFilesSafely(File file) {
        if (file.exists()) {
            File[] files = file.listFiles();
            if (files == null) {
                throw new RuntimeException("Failed to list files for dir: " + file);
            }
            return Arrays.asList(files);
        } else {
            return Collections.emptyList();
        }
    }

    public static void main(String [] arg) {
        deleteRecursively(new File("C:\\Users\\MyUser\\AppData\\Local\\Temp\\spark-9ba0bb0c-1e20-455d-bc1f-86c696661ba3")); 
    }
</code></pre>

<p>When <a href="https://github.com/apache/spark/">Spark</a> is stopped at breakpoint of this method, I found out that JVM of one thread of <a href="https://github.com/apache/spark/">Spark</a> locked "C:\Users\MyUser\AppData\Local\Temp\spark-9ba0bb0c-1e20-455d-bc1f-86c696661ba3\metastore\db.lck" file and Windows <a href="https://technet.microsoft.com/en-us/sysinternals/processexplorer.aspx">Process Explorer</a> also show that Java locks this file. Also FileChannel shows that file is locked in JVM.  </p>

<p>Now, I have to:  </p>

<ol>
<li><p>Find out what thread/class has locked this file</p></li>
<li><p>Find out what way to locking files <a href="https://github.com/apache/spark/">Spark</a> is using to lock "metastore\db.lck", what class does it and how to unlock it before shutdown</p></li>
<li><p>Do some pull request to <a href="https://github.com/apache/spark/">Spark</a> or <a href="https://hive.apache.org/">Hive</a> to unlock this file ("metastore\db.lck") before a call to deleteRecursively() method or at least leave a comment about the problem</p></li>
</ol>

<blockquote>
  <ol>
  <li>How to find which Java/Scala thread has locked a file?</li>
  </ol>
</blockquote>

<p>I have some problem and find out this solution: All locked object you can see at least in Thread.threadLocals field. </p>

<p>If file locked following code:  </p>

<pre><code>    File newFile = new File("newFile.lock");
    newFile.createNewFile();
    FileLock fileLock = FileChannel.open(Paths.get(newFile.getAbsolutePath()), StandardOpenOption.APPEND).tryLock();
</code></pre>

<p>In <code>Thread.threadLocals</code> you can see <code>sun.nio.fs.NativeBuffer</code> class with field <code>owner</code> = ".../newFile.lock".</p>

<p>So you can try following code, that returns all thread with all class in threadLocals, you need find what Threads have NativeBuffer class or Spark/Hive objects and so on (and after check this threadLocals of this threads in Eclipse or IDEA debug mode):   </p>

<pre><code>private static String getThreadsLockFile() {
    Set&lt;Thread&gt; threads = Thread.getAllStackTraces().keySet();
    StringBuilder builder = new StringBuilder();
    for (Thread thread : threads) {
        builder.append(getThreadsLockFile(thread));
    }
    return builder.toString();
}

private static String getThreadsLockFile(Thread thread) {
    StringBuffer stringBuffer = new StringBuffer();
    try {
        Field field = thread.getClass().getDeclaredField("threadLocals");
        field.setAccessible(true);
        Object map = field.get(thread);
        Field table = Class.forName("java.lang.ThreadLocal$ThreadLocalMap").getDeclaredField("table");
        table.setAccessible(true);
        Object tbl = table.get(map);
        int length = Array.getLength(tbl);
        for (int i = 0; i &lt; length; i++) {
            try {
                Object entry = Array.get(tbl, i);
                if (entry != null) {
                    Field valueField = Class.forName("java.lang.ThreadLocal$ThreadLocalMap$Entry").getDeclaredField("value");
                    valueField.setAccessible(true);
                    Object value = valueField.get(entry);
                    if (value != null) {
                        stringBuffer.append(thread.getName()).append(" : ").append(value.getClass()).
                                append(" ").append(value).append("\n");
                       }
                }
            } catch (Exception exp) {
                // skip, do nothing
            }
        }
    } catch (Exception exp) {
        // skip, do nothing
    }
    return stringBuffer.toString();
}
</code></pre>

<p>Or you can try to use following code, but this code find only <code>NativeBuffer</code> class with <code>owner</code> parameter (so it isn't work in all case): </p>

<pre><code>private static String getThreadsLockFile(String fileName) {
    Set&lt;Thread&gt; threads = Thread.getAllStackTraces().keySet();
    StringBuilder builder = new StringBuilder();
    for (Thread thread : threads) {
        builder.append(getThreadsLockFile(thread, fileName));
    }
    return builder.toString();
}

private static String getThreadsLockFile(Thread thread, String fileName) {
    StringBuffer stringBuffer = new StringBuffer();
    try {
        Field field = thread.getClass().getDeclaredField("threadLocals");
        field.setAccessible(true);
        Object map = field.get(thread);
        Field table = Class.forName("java.lang.ThreadLocal$ThreadLocalMap").getDeclaredField("table");
        table.setAccessible(true);
        Object tbl = table.get(map);
        int length = Array.getLength(tbl);
        for (int i = 0; i &lt; length; i++) {
            try {
                Object entry = Array.get(tbl, i);
                if (entry != null) {
                    Field valueField = Class.forName("java.lang.ThreadLocal$ThreadLocalMap$Entry").getDeclaredField("value");
                    valueField.setAccessible(true);
                    Object value = valueField.get(entry);
                    if (value != null) {
                        int length1 = Array.getLength(value);
                        for (int j = 0; j &lt; length1; j++) {
                            try {
                                Object entry1 = Array.get(value, j);
                                Field ownerField = Class.forName("sun.nio.fs.NativeBuffer").getDeclaredField("owner");
                                ownerField.setAccessible(true);
                                String owner = ownerField.get(entry1).toString();
                                if (owner.contains(fileName)) {
                                    stringBuffer.append(thread.getName());
                                }
                            } catch (Exception exp) {
                                // skip, do nothing
                            }
                        }
                    }
                }
            } catch (Exception exp) {
                // skip, do nothing
            }
        }
    } catch (Exception exp) {
        // skip, do nothing
    }
    return stringBuffer.toString();
}
</code></pre>

<p>See <a href="http://stackoverflow.com/questions/2177553/how-to-find-out-which-thread-is-locking-a-file-in-java">How to find out which thread is locking a file in java?</a></p>

<p>Files are locked by Windows process. Threads may open files for reading writing, but a class that holds a reference to file handle is responsible to close it. Therefore you should look for an object, not a thread.</p>

<p>See <a href="http://stackoverflow.com/questions/154309/how-can-i-figure-out-what-is-holding-on-to-unfreed-objects">How can I figure out what is holding on to unfreed objects?</a> to find out how.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I give you information what I find out about my own guestion unsing other answer (thank you <a href="http://stackoverflow.com/a/34259195/4318868">Basilevs</a>, <a href="http://stackoverflow.com/a/34293890/4318868">tploter</a> very much), may be it's help someone in the same case: </p>

<ol>
<li><p>Every time when JVM thread lock a <strong>file</strong> exclusively, also JVM lock
some <strong>Jave object</strong>, for example, I find in my case:</p>

<ul>
<li>sun.nio.fs.NativeBuffer</li>
<li>sun.nio.ch.Util$BufferCache</li>
</ul>

<p>So you need just find this locked Java object and analyzed them and you find what <strong>thread locked your file</strong>.</p></li>
</ol>

<p>I not sure that it work if file just open (without locked exclusively), but I'm sure that is work if file be locked exclusively by Thread (using <a href="https://docs.oracle.com/javase/7/docs/api/java/nio/channels/FileLock.html" rel="nofollow">java.nio.channels.FileLock</a>, <a href="https://docs.oracle.com/javase/7/docs/api/java/nio/channels/FileChannel.html" rel="nofollow">java.nio.channels.FileChannel</a> and so on)</p>

<ol start="2">
<li>Unfortunately, about Spark, I find a lot of other locked <code>Hive</code> object (<code>org.apache.hadoop.hive.ql.metadata.Hive</code>, <code>org.apache.hadoop.hive.metastore.ObjectStore</code>, <code>org.apache.hadoop.hive.ql.session.SessionState</code>, <code>org.apache.hadoop.hive.ql.metadata.Hive</code> and so on) when a <code>Spark</code> try to delete db.lck, and that means the <code>Spark</code> hadn't shutdown <code>Hive</code> correctly at all, before it tried to delete <code>Hive's</code> files. Fortunately, this problem absent in <code>Linux OS</code> (may be <code>Linux</code> allowed delete locked files).</li>
</ol>
