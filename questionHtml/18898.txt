    <div class="post-text" itemprop="text">

<p>I have several, similar map/reduce jobs with divergent counter names and different <code>getCounter()</code> conventions.  </p>

<ol>
<li><p>Is there an effective, idiomatic hadoop way for managing the
uniform use of counter logging in large map/reduce applications ?</p></li>
<li><p>I think it is a somewhat scary idea that different map/reduce jobs can make their own counters.  Is there a way you can disable this, so that all counters have to be created from a single resource?  I think this would improve the quality of output for some of my classes.</p></li>
<li><p>Any other techniques for analyzing or managing all counters in an application would be appreciated...  </p></li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The following code is in the <a href="http://svn.apache.org/repos/asf/hadoop/common/branches/branch-0.20-security-205/src/mapred/org/apache/hadoop/mapred/Counters.java" rel="nofollow">Counters.java</a>. Note that this code is in the 20.203, 20.204 and 20.205 (now called 1.0) releases. Also note that some of the parameters are configurable and some are not.</p>

<pre><code>/** limit on the size of the name of the group **/

</code></pre>
