<td class="answercell">
    <div class="post-text" itemprop="text">
<p>if you have a hive setup you can put the local dataset directly using Hive load command  in hdfs/s3. </p>

<p>You will need to use "Local" keyword when writing your load command.</p>

<p>Syntax for hiveload command</p>

<pre><code>LOAD DATA [LOCAL] INPATH 'filepath' [OVERWRITE] INTO TABLE tablename [PARTITION (partcol1=val1, partcol2=val2 ...)]
</code></pre>

<p>Refer below link for more detailed information.
<a href="https://cwiki.apache.org/confluence/display/Hive/LanguageManual%20DML#LanguageManualDML-Loadingfilesintotables">https://cwiki.apache.org/confluence/display/Hive/LanguageManual%20DML#LanguageManualDML-Loadingfilesintotables</a> </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There is another way of enabling this,</p>

<ol>
<li><p>use hadoop hdfs -copyFromLocal to copy the .csv data file from your local computer to somewhere in HDFS, say '/path/filename'</p></li>
<li><p>enter Hive console, run the following script to load from the file to make it as a Hive table. Note that '\054' is the ascii code of 'comma' in octal number, representing fields delimiter.</p></li>
</ol>

<p>CREATE EXTERNAL TABLE table name (foo INT, bar STRING)
 COMMENT 'from csv file'
 ROW FORMAT DELIMITED FIELDS TERMINATED BY '\054'
 STORED AS TEXTFILE
 LOCATION '/path/filename';</p>

<p>You may try this,
Following are few examples on how files are generated. 
Tool -- <a href="https://sourceforge.net/projects/csvtohive/?source=directory" rel="nofollow">https://sourceforge.net/projects/csvtohive/?source=directory</a></p>

<ol>
<li><p>Select a CSV file using Browse and set hadoop root directory ex: /user/bigdataproject/</p></li>
<li><p>Tool Generates Hadoop script with all csv files and following is a sample of 
generated Hadoop script to insert csv into Hadoop</p>

<p><pre><code>#!/bin/bash -v<br/>
hadoop fs -put ./AllstarFull.csv /user/bigdataproject/AllstarFull.csv
hive -f ./AllstarFull.hive<br/></p>

<p>hadoop fs -put ./Appearances.csv /user/bigdataproject/Appearances.csv
hive -f ./Appearances.hive<br/></p>

<p>hadoop fs -put ./AwardsManagers.csv /user/bigdataproject/AwardsManagers.csv
hive -f ./AwardsManagers.hive</pre></code></p></li>
<li><p>Sample of generated Hive scripts</p>

<p><pre><code>CREATE DATABASE IF NOT EXISTS lahman;<br/>
USE lahman;<br/>
CREATE TABLE AllstarFull (playerID string,yearID string,gameNum string,gameID string,teamID string,lgID string,GP string,startingPos string) row format delimited fields terminated by ',' stored as textfile;<br/>
LOAD DATA INPATH '/user/bigdataproject/AllstarFull.csv' OVERWRITE INTO TABLE AllstarFull;<br/>
SELECT * FROM AllstarFull;</pre></code></p></li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Let me work you through the following simple steps:</p>

<p>Steps
1. Create a table on hive using the field names in your csv file. Lets say for example, your csv file contains three fields (id, name, salary) and you want to create a table in hive called "staff". Use the below code to create the table in hive.</p>

<p>hive> CREATE TABLE Staff (id int, name string, salary double) row format delimited fields terminated by ‘,’;</p>

<ol start="2">
<li>Now that your table is created in hive, let us load the data in your csv file to the "staff" table on hive.</li>
</ol>

<p>hive>  LOAD DATA LOCAL INPATH '/home/yourcsvfile.csv' OVERWRITE INTO TABLE Staff;</p>

<ol start="3">
<li>Display the contents of your "Staff" table on hive to check if the data were successfully loaded</li>
</ol>

<p>hive> SELECT * FROM Staff;</p>

<p>Thanks.</p>
