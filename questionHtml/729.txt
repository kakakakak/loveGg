<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I recently got this same issue. But it turned out that the message from Spark was misleading; there were no missing jars. The problem for me was that the Java class <code>HiveContext</code>, which is called by PySpark, parses the <code>hive-site.xml</code> when it's constructed and there was an exception being raised during construction. (PySpark catches this exception and incorrectly suggests that it's due to a missing jar.) It ended up being an error with the property <code>hive.metastore.client.connect.retry.delay</code>, which was set to <code>2s</code>. The <code>HiveContext</code> class tries to parse this as an integer, which fails. Change it to <code>2</code> and remove the characters in <code>hive.metastore.client.socket.timeout</code> and <code>hive.metastore.client.socket.lifetime</code>.</p>

<p>Note that you can get a more descriptive error by calling <code>sqlContext._get_hive_ctx()</code> directly.</p>

<p>It also says:
'An error occurred while calling None.org.apache.spark.sql.hive.HiveContext.\n'</p>

<p>So, the problem seems to be that Hive part is not provided in the spark-submit command, and cluster fails to find Hive dependency.
Just do as it says, and:</p>

<p><code>Export 'SPARK_HIVE=true'</code></p>

<p>In theory, it should allow you to build your jar with Hive dependency included, so that spark would find the lib it misses.</p>
