<p>The exec command that this error is referring to is in <a href="https://github.com/Cascading/vagrant-cascading-hadoop-cluster/blob/2.5/modules/hadoop/manifests/init.pp" rel="nofollow">/modules/hadoop/manifests/init.pp</a></p>

<p>I Downloaded the grrr file myself and it worked. So there is no problem in downloading the file itself</p>

<p><strong>the grrr file contains :</strong></p>

<pre><code>#!/bin/bash

# author: André Kelpe &lt;efeshunderelf at googlemail.com&gt;
# licencse: Apache v2

GRRR_WGET_OPTIONS="--user-agent grrr/1.0"

# find out our region and yes, you can get this as csv file. How cool is that?
GEOIP_REGION=$(wget -qO- freegeoip.net/csv/ | tr '[A-Z]' '[a-z]' | tr -d '"'| awk -F, '{print $2}')

# classic confusion between geoip db and apache mirror list
if [ $GEOIP_REGION == "gb" ]; then 
    GEOIP_REGION=uk
fi

MIRRORLIST_FILE_NAME=$(mktemp)

# download the latest mirror list from apache. we ignore the last
# sync times and hope for the best...
wget -qO- http://www.apache.org/mirrors/mirrors.list | grep -v '^$' \
    | grep http | grep -v ' 0$' | grep -v '^#' &gt; $MIRRORLIST_FILE_NAME

# use US as the default region. apache does the same in their scripts...
REGION=us

# check if there is a mirror in our region
if grep -q " $GEOIP_REGION " $MIRRORLIST_FILE_NAME; then
    REGION=$GEOIP_REGION
fi

# finally download it all
wget $GRRR_WGET_OPTIONS $(grep " $REGION " $MIRRORLIST_FILE_NAME | shuf | head -1 | awk '{print $3}')/$*

retval=$?

# clean up after ourselves.
rm $MIRRORLIST_FILE_NAME

exit $retval
</code></pre>

<p>So because some of other exec commands require the download_grrr exec, they are skipped because of failed dependencies. How can I solve this error?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Normally a timeout implies the file took too long to download from the server. You'll need to add a <code>timeout =&gt; 0</code> to that command, or a high enough value. Puppet's default timeout for exec is 300 seconds. </p>

<p>However, since it's downloading a quite small shell script there might be a networking issue with that URL it wgets. It's possible you were rate-limited or Github took long to respond when it tried to run that command.</p>

<p>The easiest way would be to manually fix it by doing:</p>

<pre><code>vagrant ssh
wget --no-check-certificate http://raw.github.com/fs111/grrrr/master/grrr -O /tmp/grrr &amp;&amp; chmod +x /tmp/grrr
exit
vagrant --provision
</code></pre>

<p>I just cloned that repo and ran vagrant up and it worked fine for me. It might have been a temporary network blip.</p>
