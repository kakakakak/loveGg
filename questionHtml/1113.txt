<p>I am setting up a <code>Hadoop 2.6.0</code> Single Node Cluster. I follow the <a href="https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-common/SingleCluster.html" rel="nofollow">hadoop-common/SingleCluster documentation</a>. I work on <code>Ubuntu 14.04</code>. So far I have managed to run Standalone Operation successfully. </p>

<p>I face an error when trying to perform <strong>Pseudo-Distributed Operation</strong>. I managed to start NameNode daemon and DataNode daemon. <strong>jps</strong> oputut:</p>

<pre><code>martakarass@marta-komputer:/usr/local/hadoop$ jps
4963 SecondaryNameNode
4785 DataNode
8400 Jps
martakarass@marta-komputer:/usr/local/hadoop$ 
</code></pre>

<p>But when I try to make the HDFS directories required to execute MapReduce jobs, I receive the following error:</p>

<pre><code>martakarass@marta-komputer:/usr/local/hadoop$ bin/hdfs dfs -mkdir /user
15/05/01 20:36:00 WARN util.NativeCodeLoader: Unable to load native-hadoop library for your platform... using builtin-java classes where applicable
mkdir: Failed on local exception: com.google.protobuf.InvalidProtocolBufferException: Protocol message tag had invalid wire type.; Host Details : local host is: "marta-komputer/127.0.0.1"; destination host is: "localhost":9000; 
martakarass@marta-komputer:/usr/local/hadoop$ 
</code></pre>

<p>(I believe I can ignore the <code>WARN util.NativeCodeLoader: Unable to load native-hadoop library for your platform...</code> warning at this point.)</p>

<hr>

<p>When it comes to <code>Hadoop</code> config files, I changed only the files mentioned in the documentation. I have: </p>

<p><strong>etc/hadoop/core-site.xml</strong>:</p>

<pre><code>&lt;configuration&gt;
    &lt;property&gt;
        &lt;name&gt;fs.defaultFS&lt;/name&gt;
        &lt;value&gt;hdfs://localhost:9000&lt;/value&gt;
    &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>etc/hadoop/hdfs-site.xml</strong>:</p>

<pre><code>&lt;configuration&gt;
    &lt;property&gt;
        &lt;name&gt;dfs.replication&lt;/name&gt;
        &lt;value&gt;1&lt;/value&gt;
    &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>I managed to connect to localhost:</p>

<pre><code>martakarass@marta-komputer:~$ ssh localhost
martakarass@localhost's password: 
Welcome to Ubuntu 14.04.1 LTS (GNU/Linux 3.13.0-45-generic x86_64)

 * Documentation:  https://help.ubuntu.com/

Last login: Fri May  1 20:28:58 2015 from localhost
</code></pre>

<p>I formatted the filesystem:</p>

<pre><code>martakarass@marta-komputer:/usr/local/hadoop$  bin/hdfs namenode -format
15/05/01 20:30:21 INFO namenode.NameNode: STARTUP_MSG: 
/************************************************************
STARTUP_MSG: Starting NameNode
STARTUP_MSG:   host = marta-komputer/127.0.0.1
STARTUP_MSG:   args = [-format]
STARTUP_MSG:   version = 2.6.0
(...)
15/05/01 20:30:24 INFO namenode.NameNode: SHUTDOWN_MSG: 
/************************************************************
SHUTDOWN_MSG: Shutting down NameNode at marta-komputer/127.0.0.1
************************************************************/
</code></pre>

<p><strong>/etc/hosts</strong>:</p>

<pre><code>127.0.0.1       localhost
127.0.0.1       marta-komputer

# The following lines are desirable for IPv6 capable hosts
::1     ip6-localhost ip6-loopback
fe00::0 ip6-localnet

</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Do these changes in <strong>/etc/hosts</strong>:</p>

<p><strong>1. Change:</strong></p>

<pre><code>127.0.0.1    localhost
</code></pre>

<p>to</p>

<pre><code>127.0.0.1    localhost    marta-komputer
</code></pre>

<p><strong>2. Delete:</strong></p>

<pre><code>127.0.0.1    marta-komputer
</code></pre>

<p><strong>3. Add:</strong></p>

<pre><code>your-system-ip    marta-komputer
</code></pre>

<p>To find your system IP, type this in terminal </p>

<pre><code>ifconfig
</code></pre>

<p>(find your IP address here) or type this:</p>

<pre><code>ifdata -pa eth0
</code></pre>

<p>Your final <strong>/etc/hosts</strong> file should look like:</p>

<pre><code>127.0.0.1       localhost       marta-komputer
your-system-ip       marta-komputer

# The following lines are desirable for IPv6 capable hosts
::1     ip6-localhost ip6-loopback
fe00::0 ip6-localnet
ff00::0 ip6-mcastprefix
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters
</code></pre>

<p>Change <strong>hdfs-site.xml</strong>:</p>

<p><strong>1. Change:</strong></p>

<pre><code>hdfs://localhost:9000
</code></pre>

<p>to </p>

<pre><code>hdfs://marta-komputer:9000
</code></pre>

<p>Now, stop and start hadoop processes.</p>

<p>Your <strong>jps</strong> command should list these processes:</p>

<pre><code>Namenode
Datanode
TaskTracker
SecondaryNameNode
</code></pre>

<p>If it does not list all these processes, check respective logs for errors.</p>

<p><strong>UPDATE:</strong></p>

<ol>
<li><p>Follow this tutorial <a href="http://pingax.com/install-hadoop2-6-0-on-ubuntu/" rel="nofollow">here</a></p></li>
<li><p>If the problem persists, it might be due to permission issue.</p></li>
</ol>

<p><strong>UPDATE II:</strong></p>

<ol>
<li>Create a directory and change permissions for namenode and datanode:</li>
</ol>

<p><code>sudo mkdir -p /usr/local/hdfs/namenode</code></p>

<p><code>sudo mkdir -p /usr/local/hdfs/datanode</code></p>

<p><code>sudo chown -R hduser:hadoop /usr/local/hdfs/namenode</code></p>

<p><code>sudo chown -R hduser:hadoop /usr/local/hdfs/datanode</code></p>

<ol start="2">
<li>Add these properties in <code>hdfs-site.xml</code>:</li>
</ol>

<p><code>dfs.datanode.data.dir</code> with value <code>/usr/local/hdfs/datanode</code></p>

<p><code>dfs.namenode.data.dir</code> with value <code>/usr/local/hdfs/namenode</code></p>

<ol start="3">
<li>Stop and start hadoop processes.</li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>This is a set of steps I followed when facing exactly the same problem but with 2.7.1, the steps shouldn't differ much.</p>

<ol>
<li><p>Format of my <strong>/etc/hosts</strong> folder:</p>

<pre><code>127.0.0.1    localhost   &lt;computer-name&gt;
# 127.0.1.1    &lt;computer-name&gt;
&lt;ip-address&gt;    &lt;computer-name&gt;

# Rest of file with no changes
</code></pre></li>
<li><p>.xml configuration files (displaying contents inside <code>&lt;configuration&gt;</code> tag):</p>

<ul>
<li><p><strong>core-site.xml</strong>:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;fs.defaultFS&lt;/name&gt;
    &lt;value&gt;hdfs://localhost/&lt;/value&gt;
&lt;/property&gt;
&lt;!-- set value to a directory you want with an absolute path --&gt;
&lt;property&gt;
    &lt;name&gt;hadoop.tmp.dir&lt;/name&gt;
    &lt;value&gt;"set/a/directory/on/your/machine/"&lt;/value&gt;
    &lt;description&gt;A base for other temporary directories&lt;/description&gt;
&lt;/property&gt;
</code></pre></li>
<li><p><strong>hdfs-site.xml</strong>:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;dfs.replication&lt;/name&gt;
    &lt;value&gt;1&lt;/value&gt;
&lt;/property&gt;
</code></pre></li>
<li><p><strong>yarn-site.xml</strong>:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;yarn.recourcemanager.hostname&lt;/name&gt;
    &lt;value&gt;localhost&lt;/value&gt;
&lt;/property&gt;

&lt;property&gt;
    &lt;name&gt;yarn.nodemanager.aux-services&lt;/name&gt;
    &lt;value&gt;mapreduce_shuffle&lt;/value&gt;
&lt;/property&gt;
</code></pre></li>
<li><p><strong>mapred-site.xml</strong>:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;mapreduce.framework.name&lt;/name&gt;
    &lt;value&gt;yarn&lt;/value&gt;
&lt;/property&gt;
</code></pre></li>
</ul></li>
<li><p>This is a good opportunity to verify that you are indeed using this configuration. In the folder where your <strong>.xml</strong> files reside, view contents of script <strong>hadoop_env.sh</strong> and make sure <code>$HADOOP_CONF_DIR</code> is pointing at the right directory.</p></li>
<li><p>NameNode binds ports <strong>50070</strong> and <strong>8020</strong> on my standard distribution and DataNode binds ports <strong>50010</strong>, <strong>50020</strong>, <strong>50075</strong> and <strong>43758</strong>. Run <code>sudo lsof -i</code> to be certain no other services are using them for some reason.</p></li>
<li><p>At this point, if you have changed the value hadoop.tmp.dir you should reformat the NameNode by <code>hdfs namenode -format</code>. If not <strong>remove</strong> the temporary files already present in the tmp directory you are using (default /tmp/):</p></li>
<li><p>In <strong>/sbin/</strong> start the name and data node by using the <strong>start-dfs.sh</strong> script and yarn with <strong>start-yarn.sh</strong> and evaluate the output of jps:</p>

<pre><code>./start-dfs.sh   
./start-yarn.sh
</code></pre></li>
<li><p>At this point if NameNode, DataNode, NodeManager and ResourceManager are all running you should be set to go! </p></li>
</ol>

<p>If any of these hasn't started, share the log output for us to re-evaluate.</p>
