<p>I just setup Hadoop/Yarn 2.x (specifically, v0.23.3) in Psuedo-Distributed mode.</p>

<p>I followed the instructions of a few blogs &amp; websites which, more-or-less provide the
same prescription for setting it up. I also followed the 3rd-Edition of O'reilly's
Hadoop book (which ironically was the least helpful).</p>

<p>THE PROBLEM:</p>

<pre><code>After running "start-dfs.sh" and then "start-yarn.sh", while all of the daemons
do start (as indicated by jps(1)), the Resource Manager web portal
(Here: http://localhost:8088/cluster/nodes) indicates 0 (zero) job-nodes in the
cluster. So while submitting the example/test Hadoop job indeed does get
scheduled, it pends forever because, I assume, the configuration doesn't see a
node to run it on.

Below are the steps I performed, including resultant configuration files.
Hopefully the community help me out... (And thank you in advance).
</code></pre>

<p>THE CONFIGURATION:</p>

<p>The following environment variables are set in both my and hadoop's UNIX account profiles: ~/.profile:</p>

<pre><code>export HADOOP_HOME=/home/myself/APPS.d/APACHE_HADOOP.d/latest
  # Note: /home/myself/APPS.d/APACHE_HADOOP.d/latest -&gt; hadoop-0.23.3

export HADOOP_COMMON_HOME=${HADOOP_HOME}
export HADOOP_INSTALL=${HADOOP_HOME}
export HADOOP_CLASSPATH=${HADOOP_HOME}/lib
export HADOOP_CONF_DIR=${HADOOP_HOME}/etc/hadoop/conf
export HADOOP_MAPRED_HOME=${HADOOP_HOME}
export YARN_HOME=${HADOOP_HOME}
export YARN_CONF_DIR=${HADOOP_HOME}/etc/hadoop/conf
export JAVA_HOME=/usr/lib/jvm/jre
</code></pre>

<p>hadoop$ java -version<br></p>

<pre><code>java version "1.7.0_06-icedtea&lt;br&gt;
OpenJDK Runtime Environment (fedora-2.3.1.fc17.2-x86_64)&lt;br&gt;
OpenJDK 64-Bit Server VM (build 23.2-b09, mixed mode)&lt;br&gt;

# Although the above shows OpenJDK, the same problem happens with Sun's JRE/JDK.
</code></pre>

<p>The NAMENODE &amp; DATANODE directories, also specified in etc/hadoop/conf/hdfs-site.xml:</p>

<pre><code>/home/myself/APPS.d/APACHE_HADOOP.d/latest/YARN_DATA.d/HDFS.d/DATANODE.d/
/home/myself/APPS.d/APACHE_HADOOP.d/latest/YARN_DATA.d/HDFS.d/NAMENODE.d/
</code></pre>

<p>Next, the various XML configuration files (again, YARN/MRv2/v0.23.3 here):</p>

<pre><code>hadoop$ pwd; ls -l
/home/myself/APPS.d/APACHE_HADOOP.d/latest/etc/hadoop/conf
lrwxrwxrwx 1 hadoop hadoop   16 Sep 20 13:14 core-site.xml -&gt; ../core-site.xml
lrwxrwxrwx 1 hadoop hadoop   16 Sep 20 13:14 hdfs-site.xml -&gt; ../hdfs-site.xml
lrwxrwxrwx 1 hadoop hadoop   18 Sep 20 13:14 httpfs-site.xml -&gt; ../httpfs-site.xml
lrwxrwxrwx 1 hadoop hadoop   18 Sep 20 13:14 mapred-site.xml -&gt; ../mapred-site.xml
-rw-rw-r-- 1 hadoop hadoop   10 Sep 20 15:36 slaves
lrwxrwxrwx 1 hadoop hadoop   16 Sep 20 13:14 yarn-site.xml -&gt; ../yarn-site.xml
</code></pre>

<p>core-site.xml</p>

<pre><code>&lt;?xml version="1.0"?&gt;
&lt;!-- core-site.xml --&gt;
&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;fs.default.name&lt;/name&gt;
    &lt;value&gt;hdfs://localhost/&lt;/value&gt;
  &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>mapred-site.xml</p>

<pre><code>&lt;?xml version="1.0"?&gt;
&lt;!-- mapred-site.xml --&gt;
&lt;configuration&gt;

  &lt;!-- Same problem whether this (legacy) stanza is included or not.  --&gt;
  &lt;property&gt;
    &lt;name&gt;mapred.job.tracker&lt;/name&gt;
    &lt;value&gt;localhost:8021&lt;/value&gt;
  &lt;/property&gt;

  &lt;property&gt;
    &lt;name&gt;mapreduce.framework.name&lt;/name&gt;

  &lt;/property&gt;

</code></pre>

<p>hdfs-site.xml</p>

<pre><code>&lt;!-- hdfs-site.xml --&gt;
&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;dfs.replication&lt;/name&gt;
    &lt;value&gt;1&lt;/value&gt;
  &lt;/property&gt;
  &lt;property&gt;
    &lt;name&gt;dfs.namenode.name.dir&lt;/name&gt;
    &lt;value&gt;file:/home/myself/APPS.d/APACHE_HADOOP.d/YARN_DATA.d/HDFS.d/NAMENODE.d&lt;/value&gt;

    &lt;name&gt;dfs.datanode.data.dir&lt;/name&gt;

&lt;/configuration&gt;
</code></pre>

<p>yarn-site.xml</p>

<pre><code>&lt;?xml version="1.0"?&gt;
&lt;!-- yarn-site.xml --&gt;
&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;yarn.resourcemanager.address&lt;/name&gt;
    &lt;value&gt;localhost:8032&lt;/value&gt;
  &lt;/property&gt;
  &lt;property&gt;
    &lt;name&gt;yarn.nodemanager.aux-services&lt;/name&gt;
    &lt;value&gt;mapreduce.shuffle&lt;/value&gt;

    hdfs-site.xml):

    hadoop$ hadoop namenode -format

    <div class="post-text" itemprop="text">
<p>After much toil on this problem without success (and trust me I tried it all), I instituted
hadoop using a different solution. Whereas above I downloaded a gzip/tar ball
of the hadoop distribution (again v0.23.3) from one of the download mirrors, this
time I used the Caldera CDH distribution of RPM packages, which I installed via
their YUM repos. In hopes that this will help someone, here are the detailed steps. </p>

<p>Step-1:</p>

<p>For Hadoop 0.20.x (MapReduce version 1):</p>

<pre><code>  # rpm -Uvh http://archive.cloudera.com/redhat/6/x86_64/cdh/cdh3-repository-1.0-1.noarch.rpm
  # rpm --import http://archive.cloudera.com/redhat/6/x86_64/cdh/RPM-GPG-KEY-cloudera
  # yum install hadoop-0.20-conf-pseudo
</code></pre>

<p>-or-</p>

<p>For Hadoop 0.23.x (MapReduce version 2):</p>

<pre><code>  # rpm -Uvh http://archive.cloudera.com/cdh4/one-click-install/redhat/6/x86_64/cloudera-cdh-4-0.noarch.rpm
  # rpm --import http://archive.cloudera.com/cdh4/redhat/6/x86_64/cdh/RPM-GPG-KEY-cloudera
  # yum install hadoop-conf-pseudo
</code></pre>

<p>In both cases above, installing that "psuedo" package (which stands for "pseudo-distributed
Hadoop" mode), will alone conveniently trigger the installation of all the other necessary packages you'll need (via dependency resolution).</p>

<p>Step-2:</p>

<p>Install Sun/Oracle's Java JRE (if you haven't already done so). You can
install it via the RPM that they provide, or the gzip/tar ball portable
version. It doesn't matter which as long as you set and export the "JAVA_HOME"
environment appropriately, and ensure ${JAVA_HOME}/bin/java is in your path.</p>

<pre><code>  # echo $JAVA_HOME; which java
  /home/myself/APPS.d/JAVA-JRE.d/jdk1.7.0_07
  /home/myself/APPS.d/JAVA-JRE.d/jdk1.7.0_07/bin/java
</code></pre>

<p>Note: I actually create a symlink called "latest" and point/re-point it to the JAVA
version specific directory whenever I update the JAVA. I was explicit above for
the reader's understanding.</p>

<p>Step-3:  Format hdfs as the "hdfs" Unix user (created during "yum install" above).  </p>

<pre><code>  # sudo su hdfs -c "hadoop namenode -format"
</code></pre>

<p>Step-4:</p>

<p>Manually start the hadoop daemons.</p>

<pre><code>  for file in `ls /etc/init.d/hadoop*`
  do
  {
     ${file} start
  }
  done
</code></pre>

<p>Step-5:</p>

<p>Check to see if things are working. The following is for MapReduce v1
(It's not that much different for MapReduce v2 at this superficial level).</p>

<pre><code>  root# jps
   23104 DataNode
   23469 TaskTracker
   23361 SecondaryNameNode
   23187 JobTracker
   23267 NameNode
   24754 Jps

   # Do the next commands as yourself (not as "root").
   myself$ hadoop fs -mkdir /foo
   myself$ hadoop fs -rmr /foo
   myself$ hadoop jar /usr/lib/hadoop-0.20/hadoop-0.20.2-cdh3u5-examples.jar pi 2 100000
</code></pre>

<p>I hope this helped!</p>
