    <div class="post-text" itemprop="text">

<p>I am running a Hadoop distributed system with a master node and some worker nodes. The worker nodes need to be able to handle various requests sent from clients on a web app, but only some of the worker nodes are able to handle certain requests since only some of them have processes open to handle these requests. I can't have copies of these processes running on all nodes due to memory constraints and I can't open new processes on the fly since they take too long to start. I need to design a system so that requests get routed to the appropriate worker nodes with the matching processes.</p>

<p>I think it would be possible to use the master node to track information about the running processes on the worker nodes and route all incoming requests based on that information. Are there any Hadoop related systems that can accomplish this task? I would strongly prefer to use something like ZooKeeper or MapReduce but I am not sure if these systems are capable of targeting specific nodes in a cluster. If none of the Hadoop related systems are suitable for this task, are there any other good options (perhaps using sockets to communicate between the master and worker nodes)?</p>
