    private static String getValue(Object obj, List&lt;String&gt; escapes){
        if (obj == null) {
            return "";
        }
        String type = obj.getClass().toString();
        type = type.substring(type.lastIndexOf(".")+1,type.length());

        if (type.toUpperCase().equals(ELEMENT)) {
            Element element = (Element)obj;
            if (element.getTextTrim() != null) {
                return CleanUp(element.getTextTrim(),escapes);                              
            }                       
        }else if (type.toUpperCase().equals(ATTRIBUTE)) {
            Attribute attribute = (Attribute)obj;
            if (attribute.getValue() != null) {
                return CleanUp(attribute.getValue(),escapes);                           
            }                                           
        }else if (type.toUpperCase().equals(STRING)) {
            return CleanUp(obj.toString(),escapes); 
        }else if (type.toUpperCase().equals(TEXT)) {
            Text text = (Text)obj;
            return CleanUp(text.getText(),escapes);
        }else if (type.toUpperCase().equals(DOUBLE)) {
            return CleanUp(obj.toString(),escapes);
        }else if (type.toUpperCase().equals(BOOLEAN)) {
            return CleanUp(obj.toString(),escapes);
        }else {
            return null;
        }
        return "";  
    }

    private static String CleanUp(String value, List&lt;String&gt; listToReplace){

        for (String replace : listToReplace){
            value = value.replace(replace, " ");
        }
        return value;
    }

<p>Using jdk1.8.0_20. --- This causes an error:</p>

<pre><code>import org.jdom2.xpath;
</code></pre>

<p>Removed. --- Then, I get lots of errors </p>

<pre><code>javax/xml/xpath/FieldMap.java:27: error: type XPathExpression does not take parameters
 private XPathExpression&lt;Object&gt; xPathExp;
</code></pre>

<p>Moving the file from javax.xml.xpath`into a "sane" package removes <em>all</em> of those errors, and the file compiles.</p>

<p>That package contains javax.xml.xpath.XPathExpression, and this is in conflict with 
<code>org.jdom2.xpath.XPathExpression</code>.</p>

<p>I don't know what exactly OP has done to get the posted errors - I still think that it's been compiled using some pre-Tiger java. But the findings concerning the package mishap are worth posting.</p>
