    <div class="post-text" itemprop="text">

<p>I know that HDFS stores data using the regular linux file system in the data nodes. My HDFS block size is <code>128 MB</code>. Lets say that I have <code>10 GB</code> of disk space in my hadoop cluster that means, HDFS initially has <code>80 blocks</code> as available storage.</p>

<p>If I create a small file of say <code>12.8 MB</code>, #available HDFS blocks will become 79. What happens if I create another small file of <code>12.8 MB</code>? Will the #availbale blocks stay at 79 or will it come down to 78? In the former case, HDFS basically recalculates the #available blocks after each block allocation based on the available free disk space so, #available blocks will become 78 only after more than 128 MB of disk space is consumed. Please clarify.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The best way to know is to try it, see my results bellow.</p>

<p>But before trying, my guess is that even if you can only allocate 80 <strong>full</strong> blocks in your configuration, you can allocate more than 80 non-empty files.  This is because I think HDFS does not use a full block each time you allocate a non-empty file.  Said in another way, HDFS blocks are not a <em>storage allocation</em> unit, but a <em>replication</em> unit.  I think the storage allocation unit of HDFS is the unit of the underlying filesystem (if you use ext4 with a block size of 4 KB and you create a 1 KB file in a cluster with replication factor of 3, you consume 3 times 4 KB = 12 KB of hard disk space).</p>

<p>Enough guessing and thinking, let's try it.  My lab configuration is as follow:</p>

<ul>
<li>hadoop version 1.0.4</li>
<li>4 data nodes, each with a little less than 5.0G of available space, ext4 block size of 4K</li>
<li>block size of 64 MB, default replication of 1</li>
</ul>

<p>After starting HDFS, I have the following NameNode summary:</p>

<ul>
<li>1 files and directories, 0 blocks = 1 total</li>
<li>DFS Used: 112 KB</li>
<li>DFS Remaining: 19.82 GB</li>
</ul>

<p>Then I do the following commands:</p>

<ul>
<li><code>hadoop fs -mkdir /test</code></li>
<li><code>for f in $(seq 1 10); do hadoop fs -copyFromLocal ./1K_file /test/$f; done</code></li>
</ul>

<p>With these results:</p>

<ul>
<li>12 files and directories, 10 blocks = 22 total</li>
<li>DFS Used: 122.15 KB</li>
<li>DFS Remaining: 19.82 GB</li>
</ul>

<p>So the 10 files did not consume 10 times 64 MB (no modification of "DFS Remaining").</p>
