<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It's likely that you have <code>postgres</code> installed in some other location.
The file it's looking for is <code>pg_hba.conf</code>
You chose the default postgres db for ambari and it will likely look for it under <code>/var/lib/pgsql/data/</code></p>

<p>Looking at the trace, it's trying to rename the file, but just unsure what that file is exactly.<br>
To really know which file and location for sure, simply edit the python script: <code>/usr/lib64/python2.6/fileinput.py</code></p>

<p>Add a line above the code to print the files it's looking for: </p>

<pre><code>print self._filename, ':', self._backupfilename                 
os.rename(self._filename, self._backupfilename)
</code></pre>

<p>Note: I hit a similar situation and it was pointing to <code>/var/lib/pgsql/data</code>, which did not exist (real location was under <code>/data/pghadoop</code>). So what I did was create a symbolic link to my real location, i.e. <code>ln -s /data/pghadoop/ /var/lib/pgsql/data.</code></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If we tried to install Ambari server under Amazon Linux with its bundled postgresql db, you should get error on <code>ambari-server</code> setup. </p>

<p>The first root cause is the startup script postgresql assumes the default data path is at: 
<code>/var/lib/pgsql92/data</code></p>

<p>However, we can see nothing under: <code>/var/lib/pgsql92/data</code>
Instead of, we should see thing under: <code>/var/lib/pgsql9</code></p>

<p>Here is workaround:</p>

<ul>
<li>Fix the default data path for Ambari bundled postgresql:</li>
</ul>

<p><code>sudo ln -s /var/lib/pgsql9 /var/lib/pgsql</code></p>

<ul>
<li>Edit postgresql script</li>
</ul>

<p><code># sudo vim /etc/init.d/postgresql</code></p>

<p>-- comment it:</p>

<p><code>#PGSUFFIX=92</code></p>

<p>-- change below:</p>

<p><code>PGDATA=/var/lib/pgsql${PGSUFFIX}/data</code></p>

<p><code>PGLOG=/var/lib/pgsql${PGSUFFIX}/pgstartup.log</code></p>

<p>To:</p>

<p><code>PGDATA=/var/lib/pgsql/data</code></p>

<p><code>PGLOG=/var/lib/pgsql/pgstartup.log</code></p>
