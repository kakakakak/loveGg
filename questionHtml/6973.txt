<pre><code>export HBASE_HOME=/usr/local/hbase

</code></pre>

    <div class="post-text" itemprop="text">
<p>Try to use Cloudera <code>CDH5</code> and <code>Express Manager</code> distribution. It is free and saves lots of time. It fully supports Ubuntu/Debian system.</p>

<p>I've found that ZooKeeper/HDFS/HBase configuration for distributed cluster is very tricky. For example you have to restart cluster by restarting services in correct order. For example, start  order is kind of the following: <code>ZooKeeper</code> > <code>HDFS</code>/<code>YARN</code> > <code>HBase</code>.</p>

<p><strong>Here is my success story</strong>: I've tried to prepare Hadoop environment with not only HBase, but with Impala, Spark, Ozzie, Hive and Hue. In addition in my case HBase must be with <code>Phoenix</code> module which is not a part of CHD5. I've succeed with single node cluster, make manually. But I've got lost in cluster configuration... I've stuck in xmls, hostnames and so on.</p>

<p>After I decided to spent some time on learning Cloudera's distribution, I've installed CM5 packages, these are <code>Cloudera Manager</code> (i.e. CM5). Next I've started web interface and installed all required deb-packages via web interface(yes, it install full set of packages which is big overhead if you don't need them). That is it! It was fast and confident.</p>

<p>Next is configuration. All was made via web interface as well. You just selecting hosts involved in cluster, an that choose services you need i.e. HBase and so on. After you assign roles, Cloudera Manager just starts cluster. Finished. That is all.</p>

<p>This software also has good collection of best practices such as monitoring, config versioning, tips and tricks, software updates and etc. More important it allow you to get working cluster very quickly, after that you can continue research how to optimize installation procedure.</p>

<p>Good luck!</p>
