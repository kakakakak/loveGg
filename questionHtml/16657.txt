<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I'll share some of my experience. First of all I would analyze whether Hadoop fits into my requirements or not. Hadoop is not a replacement for traditional RDBMSs. It is meant for totally different purpose which is <code>offline batch processing</code> unlike RDBMSs which are used for real time queries. So, if you have real time needs you better think again. In such a case you would require something like <a href="http://hbase.apache.org" rel="nofollow">HBase</a>.</p>

<p>The first link which you have shown here is about <a href="http://hive.apache.org" rel="nofollow">Hive</a>. Hive is a data warehouse which runs on top of an existing Hadoop cluster. Along with that it also provides you an SQL like interface which makes it easier for you to query in an easier and convenient manner, in case you are coming from SQL background. Hive stores the data as Hive tables on top of HDFS(the Hadoop file system). But again Hive is not suitable for real time stuff. If you want to perform real time queries on your Hadoop data you might find <a href="http://hive.apache.org" rel="nofollow">Impala</a> a good fit.</p>

<p>The second link shows you how to use Apache Hadoop with C# and Microsoft Windows Azure. Azure is Microsoft's cloud platform offering. You can find more on this <a href="https://www.hadooponazure.com/" rel="nofollow">here</a>. This link shows you how to write a <a href="http://hadoop.apache.org/docs/stable/mapred_tutorial.html" rel="nofollow">MapReduce</a> program. MapReduce is a computation framework that allows you to process huge amounts of data on a cluster of machines in a distributed and parallel manner.</p>

<p>Originally Hadoop was developed to be used with Linux. But now you have the flexibility to use it with Windows as well, courtesy solutions like Azure and <a href="http://hortonworks.com/products/hortonworksdataplatform/" rel="nofollow">HDP</a>.</p>

<p>Hadoop does not require any special software(apart from the basic things like ssh, appropriate language translator etc) or sophisticated hardware. It was meant to run on commodity hardware. But you might wanna keep the specs prescribed the vendor, you are going to follow, in mind.</p>

<p>You could easily move your SQL data into your Hadoop cluster either by writing your own programs or by more high level tools like <a href="http://sqoop.apache.org" rel="nofollow">Sqoop</a>.</p>

<p>Search is not the only area where Hadoop is used. This is a very good <a href="http://www.slideshare.net/cloudera/20100806-cloudera-10-hadoopable-problems-webinar-4931616" rel="nofollow">link</a> which tells us about the common problems that can be solved using Hadoop.</p>

<p>Hope this answers some of your questions. Let me know if you have any further query.</p>

<hr>

<p>In response to your comment :</p>

<p>Hadoop is basically 2 things-a distributed filesystem(HDFS) and a processing framework(MapReduce). Being a filesystem it lacks random read/write capability. This is where a database, like HBase, comes into picture. Also, Hadoop stores data as files and not as columns which HBase does.</p>

<p>If you want to query your data stored in HDFS through Hive you map your HDFS files as tables in Hive and query them using HiveQL. Suppose you have a file called 'file.txt' in HDFS which has 2 fields number and name, you would do something like this :</p>

<pre><code>hive&gt; CREATE TABLE demo (no INT, name STRING);
hive&gt; LOAD DATA INPATH '/file.txt' INTO TABLE demo;
hive&gt; Select * FROM demo;
</code></pre>
