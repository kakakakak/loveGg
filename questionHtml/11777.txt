<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are lot's of ways on how you can ingest data into HDFS, let me try to illustrate them here:</p>

<ol>
<li><code>hdfs dfs -put</code> - simple way to insert files from local file system to HDFS</li>
<li>HDFS Java API</li>
<li><a href="http://sqoop.apache.org/" rel="nofollow">Sqoop</a> - for bringing data to/from databases</li>
<li><a href="http://flume.apache.org/" rel="nofollow">Flume</a> - streaming files, logs</li>
<li><a href="https://kafka.apache.org/" rel="nofollow">Kafka</a> - distributed queue, mostly for near-real time stream processing</li>
<li><a href="http://nifi.apache.org/" rel="nofollow">Nifi</a> - incubating project at Apache for moving data into HDFS without making lots of changes</li>
</ol>

<p>Best solution for bringing web application logs to HDFS is through Flume.</p>

    <div class="post-text" itemprop="text">
<p>We have three different kinds of <a href="https://jeremyronk.wordpress.com/2014/09/01/structured-semi-structured-and-unstructured-data/" rel="nofollow">data</a> - Structured (schema based systems like Oracle/MySQL etc.), Unstructured (images, weblogs etc.) and Semi-structured data(XML etc.)</p>

<p><strong>Structured data</strong> can be stored in database SQL  in table with rows and columns</p>

<p><strong>Semi-structured data</strong> is information that doesn’t reside in a relational database but that does have some organizational properties that make it easier to analyze. With some process you can store them in relation database (e.g. XML)</p>

<p><strong>Unstructured data</strong> often include text and multimedia content. Examples include e-mail messages, word processing documents, videos, photos, audio files, presentations, webpages and many other kinds of business documents.</p>

<p><strong>Depending on type of your data, you will choose the tools to import data into HDFS</strong>.</p>

<p>Your company may use CRM,ERP tools. But we don't exactly know how the data is organized &amp; structured. </p>

<p>If we leave simple HDFS commands like put, copyFromLocal etc to load data into HDFS compatible format, below are the main tools to load data into HDFS</p>

<p><a href="http://sqoop.apache.org/" rel="nofollow"><strong>Apache Sqoop(TM)</strong></a> is a tool designed for efficiently transferring bulk data between Apache Hadoop and structured datastores such as relational databases. Data from MySQL, SQL Server &amp; Oracle tables can be loaded into HDFS with this tool.</p>

<p><a href="http://flume.apache.org/" rel="nofollow"><strong>Apache Flume</strong></a> is a distributed, reliable, and available service for efficiently collecting, aggregating, and moving large amounts of log data. It has a simple and flexible architecture based on streaming data flows. It is robust and fault tolerant with tunable reliability mechanisms and many failover and recovery mechanisms. </p>

<p><a href="http://thinkbig.teradata.com/leading_big_data_technologies/ingestion-and-streaming-with-storm-kafka-flume/" rel="nofollow"><strong>Other tools</strong></a> include <strong>Chukwa,Storm and Kafka</strong></p>

<p>But other important technology, which is becoming very popular is Spark. It is a Friend &amp; Foe for Hadoop.</p>

<p><a href="http://spark.apache.org/" rel="nofollow"><strong>Spark</strong> </a> is emerging an good alternative to Hadoop for real time data processing, which may or may not use HDFS as data source. </p>
