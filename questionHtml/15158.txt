<p><strong>Update:</strong> <code>RDD.toLocalIterator</code> method that appeared after the original answer has been written is a more efficient way to do the job. It uses <code>runJob</code> to evaluate only a single partition on each step.</p>

<p><strong>TL;DR</strong> And the original answer might give a rough idea how it works:</p>

<p>First of all, get the array of partition indexes:</p>

<pre class="lang-scala prettyprint-override"><code>val parts = rdd.partitions
</code></pre>

<p>Then create smaller rdds filtering out everything but a single partition. Collect the data from smaller rdds and iterate over values of a single partition:</p>

<pre class="lang-scala prettyprint-override"><code>for (p &lt;- parts) {
    val idx = p.index
    val partRdd = rdd.mapPartitionsWithIndex(a =&gt; if (a._1 == idx) a._2 else Iterator(), true)
    //The second argument is true to avoid rdd reshuffling
    val data = partRdd.collect //data contains all values from a single partition 
                               //in the form of array
    //Now you can do with the data whatever you want: iterate, save to a file, etc.
}
</code></pre>

<p>I didn't try this code, but it should work. Please write a comment if it won't compile. Of cause, it will work only if the partitions are small enough. If they aren't, you can always increase the number of partitions with <code>rdd.coalesce(numParts, true)</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Wildfire answer seems semantically correct, but I'm sure you should be able to be vastly more efficient by using the API of Spark.  If you want to process each partition in turn, I don't see why you can't using <code>map</code>/<code>filter</code>/<code>reduce</code>/<code>reduceByKey</code>/<code>mapPartitions</code> operations.  The only time you'd want to have everything in one place in one array is when your going to perform a non-monoidal operation - but that doesn't seem to be what you want.  You should be able to do something like:</p>

<pre><code>rdd.mapPartitions(recordsIterator =&gt; your code that processes a single chunk)
</code></pre>

<p>Or this</p>

<pre><code>rdd.foreachPartition(partition =&gt; {
  partition.toArray
  // Your code
})
</code></pre>

<p>Here is the same approach as <a href="http://stackoverflow.com/a/21801828/1296661">suggested</a> by @Wildlife but written in pyspark. </p>

<p>The nice thing about this approach - it lets user access records in RDD in order. I'm using this code to feed data from RDD into STDIN of the machine learning tool's process.</p>

<pre><code>rdd = sc.parallelize(range(100), 10)
def make_part_filter(index):
    def part_filter(split_index, iterator):
        if split_index == index:
            for el in iterator:
                yield el
    return part_filter

for part_id in range(rdd.getNumPartitions()):
    part_rdd = rdd.mapPartitionsWithIndex(make_part_filter(part_id), True)
    data_from_part_rdd = part_rdd.collect()
    print "partition id: %s elements: %s" % (part_id, data_from_part_rdd)
</code></pre>
