    <div class="post-text" itemprop="text">

<p>My following code does not produce expected output:</p>

<pre><code>public static void main(String[] args) throws MalformedURLException {

    Configuration.addDefaultResource("/home/some_user/conf.xml");
    Configuration conf = new Configuration();
    System.out.println(conf);
    System.out.println(conf.get("color"));
    assertThat(conf.get("color"), is("yellow"));
}
</code></pre>

<p>The property <code>color</code> is set in conf.xml file as follows:</p>

<pre><code>&lt;property&gt;
        &lt;name&gt;color&lt;/name&gt;
        &lt;value&gt;yellow&lt;/value&gt;
        &lt;description&gt;Color&lt;/description&gt;
&lt;/property&gt;
</code></pre>

<p>Looks like file <code>conf.xml</code> isn't getting incorporated in default configuration.</p>

<p>The documentation for <code>Configuration.addDefaultResource(String param)</code> says the param should be in classpath. I don't understand how to add file to the classpath when I am already giving the program full absolute path.</p>

    <div class="post-text" itemprop="text">
<p>First observation: I don't know which version of hadoop you use but the addDefaultResource() has been deprecated for a very long time.</p>

<p>In the later versions of Hadoop the standard way to accomplish what you want is: </p>

<pre><code>Configuration conf = new Configuration()
conf.addResource("path/to/file");
...
</code></pre>

<p>Regarding the classpath issue, you have to simply place the config file in the classpath. So you have to discover what the classpath is to(it is either an environment var or the one which you set with -classpath option). If you didn't use the -classpath option and there is no classpath environment variable then it is automatically set to the current directory (".")</p>
