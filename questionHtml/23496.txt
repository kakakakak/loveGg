    <div class="post-text" itemprop="text">

<p>I am trying to implement a simple wordcount in scala using an immutable map(this is intentional) and the way I am trying to accomplish it is as follows:</p>

<ol>
<li>Create an empty immutable map</li>
<li>Create a scanner that reads through the file.</li>
<li><p>While the scanner.hasNext() is true:</p>

<ul>
<li>Check if the Map contains the word, if it doesn't contain the word, initialize the count to zero</li>
<li>Create a new entry with the key=word and the value=count+1</li>
<li>Update the map</li>
</ul></li>
<li><p>At the end of the iteration, the map is populated with all the values.</p></li>
</ol>

<p>My code is as follows:</p>

<pre><code>val wordMap = Map.empty[String,Int]
val input = new java.util.scanner(new java.io.File("textfile.txt"))
while(input.hasNext()){
  val token = input.next()
  val currentCount = wordMap.getOrElse(token,0) + 1
  val wordMap = wordMap + (token,currentCount)
}
</code></pre>

<p>The ides is that wordMap will have all the wordCounts at the end of the iteration...
Whenever I try to run this snippet, I get the following exception</p>

<blockquote>
  <p>recursive value wordMap needs type.</p>
</blockquote>

<p>Can somebody point out why I am getting this exception and what can I do to remedy it?</p>

<p>Thanks     </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<pre><code>val wordMap = wordMap + (token,currentCount)
</code></pre>

<p>This line is redefining an already-defined variable.  If you want to do this, you need to define <code>wordMap</code> with <code>var</code> and then just use </p>

<pre><code>wordMap = wordMap + (token,currentCount)
</code></pre>

<p>Though how about this instead?:</p>

<pre><code>io.Source.fromFile("textfile.txt")            // read from the file
  .getLines.flatMap{ line =&gt;                  // for each line
     line.split("\\s+")                       // split the line into tokens
       .groupBy(identity).mapValues(_.size)   // count each token in the line
  }                                           // this produces an iterator of token counts
  .toStream                                   // make a Stream so we can groupBy
  .groupBy(_._1).mapValues(_.map(_._2).sum)   // combine all the per-line counts
  .toList
</code></pre>

<p>Note that the per-line pre-aggregation is used to try and reduce the memory required.  Counting across the entire file at once might be too big.</p>

<p>If your file is really massive, I would suggest using doing this in parallel (since word counting is trivial to parallelize) using either Scala's parallel collections or Hadoop (using one of the cool Scala Hadoop wrappers like Scrunch or Scoobi).</p>

<p><strong>EDIT</strong>: Detailed explanation:</p>

<p>Ok, first look at the inner part of the flatMap.  We take a string, and split it apart on whitespace:</p>

<pre><code>val line = "a b c b"
val tokens = line.split("\\s+") // Array(a, b, c, a, b)
</code></pre>

<p>Now <code>identity is a function that just returns its argument, so if we</code>groupBy(identity)`, we map each distinct word <em>type</em>, to each word <em>token</em>:</p>

<pre><code>val grouped = tokens.groupBy(identity) // Map(c -&gt; Array(c), a -&gt; Array(a), b -&gt; Array(b, b))
</code></pre>

<p>And finally, we want to count up the number of tokens for each type:</p>

<pre><code>val counts = grouped.mapValues(_.size) // Map(c -&gt; 1, a -&gt; 1, b -&gt; 2)
</code></pre>

<p>Since we map this over all the lines in the file, we end up with token counts for each line.</p>

<p>So what does <code>flatMap</code> do?  Well, it runs the token-counting function over each line, and then combines all the results into one big collection.</p>

<p>Assume the file is:</p>

<pre><code>a b c b
b c d d d
e f c
</code></pre>

<p>Then we get:</p>

<pre><code>val countsByLine = 
  io.Source.fromFile("textfile.txt")            // read from the file
    .getLines.flatMap{ line =&gt;                  // for each line
       line.split("\\s+")                       // split the line into tokens
         .groupBy(identity).mapValues(_.size)   // count each token in the line
    }                                           // this produces an iterator of token counts
println(countsByLine.toList) // List((c,1), (a,1), (b,2), (c,1), (d,3), (b,1), (c,1), (e,1), (f,1))
</code></pre>

<p>So now we need to combine the counts of each line into one big set of counts.  The <code>countsByLine</code> variable is an <code>Iterator</code>, so it doesn't have a <code>groupBy</code> method.  Instead we can convert it to a <code>Stream</code>, which is basically a lazy list.  We want laziness because we don't want to have to read the entire file into memory before we start.  Then the <code>groupBy</code> groups all counts of the same word type together.</p>

<pre><code>val groupedCounts = countsByLine.toStream.groupBy(_._1)
println(groupedCounts.mapValues(_.toList)) // Map(e -&gt; List((e,1)), f -&gt; List((f,1)), a -&gt; List((a,1)), b -&gt; List((b,2), (b,1)), c -&gt; List((c,1), (c,1), (c,1)), d -&gt; List((d,3)))
</code></pre>

<p>And finally we can sum up the counts from each line for each word type by grabbing the second item from each tuple (the count), and summing:</p>

<pre><code>val totalCounts = groupedCounts.mapValues(_.map(_._2).sum)
println(totalCounts.toList)
List((e,1), (f,1), (a,1), (b,3), (c,3), (d,3))
</code></pre>

<p>And there you have it.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You have a few mistakes: you've defined <code>wordMap</code> twice (<code>val</code> is to declare a value). Also, <code>Map</code> is immutable, so you either have to declare it as a <code>var</code> or use a mutable map (I suggest the former).</p>

<p>Try this: </p>

<pre><code>var wordMap = Map.empty[String,Int] withDefaultValue 0
val input = new java.util.Scanner(new java.io.File("textfile.txt"))
while(input.hasNext()){
  val token = input.next()
  wordMap += token -&gt; (wordMap(token) + 1)
}
</code></pre>
