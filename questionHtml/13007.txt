    <div class="post-text" itemprop="text">

<p>According to new HDInsight <a href="http://azure.microsoft.com/en-us/pricing/details/hdinsight/" rel="nofollow">pricing schema</a> and <a href="http://azure.microsoft.com/en-gb/documentation/articles/hdinsight-release-notes/" rel="nofollow">release notes</a>, you can now specify the size of the instance (A3-A9 and D3-D14). I, however, couldn't find a way of how to specify the instance size when creating a new cluster via powershell New-AzureHDInsightCluster command. I did find that see that now it accepts <a href="https://github.com/Azure/azure-powershell/commit/26f8b60bcc8d55240fedb66c522100bba8aaf22b" rel="nofollow">-DataNodeVMSize</a>, however it doesn't understand keywords like "A7", and when specifying "Extra Large" it creates a standard A3 cluster. Any suggestions whether it's actually possible to specify instance size at the moment? </p>

<p>The "available" VM sizes <a href="https://msdn.microsoft.com/en-us/library/azure/dn197896.aspx" rel="nofollow">https://msdn.microsoft.com/en-us/library/azure/dn197896.aspx</a></p>

<p>Here is the script I am running:</p>

<blockquote>
  <p>$VmSize = "Small";</p>
</blockquote>

<pre><code>New-AzureHDInsightCluster -Name $clusterName `
              -ClusterType Hadoop `
              -Version $MyClusterVersion `
              -Location $MyClusterLocation `
              -ClusterSizeInNodes $NumClusterNodes `
              -Credential $HdInsightCreds `
              -DefaultStorageAccountName $DefaultStorageAccountFqdn `
              -DefaultStorageAccountKey $storageAccountKey `
              -DefaultStorageContainerName $hadoopContainer' `
              -DataNodeVMSize $VmSize `
              -HeadNodeVMSize $HeadNodeVmSize
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As you already mention, as for now, <em>HDInsight</em> clusters only support nodes within <code>A3-A9</code> and <code>D3-D14</code>.</p>

<p>The link you provided shows pretty clear the codes asociated to each node type and size. The code <code>Small</code> belongs to the instance type <code>A1\Small</code>, which is not supported by <em>HDInsight</em>. Thus, it might consider the use of the smallest available instance, which is, in this case, <code>A3\Large</code>.  </p>

<p>You should try specifying other instances that fall under the ranges supported by the <em>HDInsight</em> cluster. I can see you have already tried with <code>extra large</code>. You might want to try typing the code according to the code index available <a href="https://msdn.microsoft.com/en-us/library/azure/dn197896.aspx" rel="nofollow">here</a> (under <em>Sizes for Web and Worker Role Instances</em>), for example.</p>

<pre><code>A3: Large
A4: Extralarge
</code></pre>
