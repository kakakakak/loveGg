<td class="answercell">
    <div class="post-text" itemprop="text">
<p><code>hadoop namenode -format</code> this command deletes all files in your hdfs.</p>

<p>tmp directory contains two folders datanode, namenode in local filesystem. if you format the namenode these two folders becomes empty.</p>

<p>Note : if you want to format your namenode first stop all hadoop services then delete the tmp(contains namenode and datanode) folder in your local file system and start hadoop service surely it will take effect.</p>

<p><strong>Reason for Hadoop namenode -format :</strong></p>

<p>Hadoop NameNode is the centralized place of an HDFS file system which keeps the directory tree of all files in the file system, and tracks where across the cluster the file data is kept. In short, it keeps the metadata related to datanodes. When we format namenode it formats the meta-data related to data-nodes. By doing that, all the information on the datanodes are lost and they can be reused for new data.</p>

<p>By <strong>default the namenode</strong> location will be at <strong>"/tmp/hadoop-myuser/dfs/name"</strong></p>

<p>While you formatting the namenode, this file location was cleared.</p>

<p>To <strong>change the namenode location</strong> add the follwing properties <strong>At hdfs-site.xml</strong></p>

<pre><code>&lt;property&gt;
    &lt;name&gt;dfs.namenode.name.dir&lt;/name&gt;
    &lt;value&gt;file:/search/data/dfs/namenode&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;dfs.datanode.data.dir&lt;/name&gt;
    &lt;value&gt;file:/search/data/dfs/datanode&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>I hope this will help you.. :-)</p>

    <div class="post-text" itemprop="text">
<p>Steps
start all the services using <code>"start-all.sh"</code></p>

<p>check the services are running or not using <code>"JPS"</code>
note: if you use hadoop2.3.0 then following services are need to run</p>

<pre><code>Namenode
Datanode
Resourcemanager
Nodemanager
</code></pre>

<p>Move some file from local to HDFS using hdfs -put  /</p>

<p>Now check at location <strong>"/tmp/hadoop-myuser/dfs/name"</strong> you may find this file split into some BLOCKS conatain 64 MB each.</p>

<p>Then start Formatting using <code>**hadoop namenode -format**</code>
Now the file is not available phisically on that location</p>

<p>Further information <a href="http://stackoverflow.com/a/20142210/3452185">click here</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Actually formatting a Namenode will not format the Datanode.</p>

<p>It will just format the contents of your namenode (which contains details of datanode). Your namenode will no longer know where your data is. Also namenode -format will assign a new namespace ID to the namenode</p>

<p>You will have to change your namespaceID in your datanode to make your datanode work. This will be at dfs/data/current/VERSION</p>

<p>There is a JIRA open now for the same suggesting to format Datanode aswell when you format Namenode. HDFS-107</p>
