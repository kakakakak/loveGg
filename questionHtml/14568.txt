<p>I have set up a multi node Hadoop Cluster. The NameNode and Seconaday namenode runs on the same machine and the cluster has only one Datanode. All the nodes are configured on Amazon EC2 machines.</p>

<p>Following are the configuration files on the master node</p>

<pre><code>masters
54.68.218.192 (public IP of the master node)

slaves
54.68.169.62 (public IP of the slave node)


core-site.xml

&lt;configuration&gt;
&lt;property&gt;
&lt;name&gt;fs.default.name&lt;/name&gt;
&lt;value&gt;hdfs://localhost:9000&lt;/value&gt;
&lt;/property&gt;
&lt;/configuration&gt;


mapred-site.xml

&lt;configuration&gt;
&lt;property&gt;
&lt;name&gt;mapreduce.framework.name&lt;/name&gt;
&lt;value&gt;yarn&lt;/value&gt;
&lt;/property&gt;
&lt;/configuration&gt;


hdfs-site.xml

 &lt;configuration&gt;
 &lt;property&gt;
 &lt;name&gt;dfs.replication&lt;/name&gt;
&lt;value&gt;1&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
&lt;name&gt;dfs.namenode.name.dir&lt;/name&gt;
&lt;value&gt;file:/usr/local/hadoop_store/hdfs/namenode&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
&lt;name&gt;dfs.datanode.name.dir&lt;/name&gt;
&lt;value&gt;file:/usr/local/hadoop_store/hdfs/datanode&lt;/value&gt;
&lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<pre><code>core-site.xml

 &lt;configuration&gt;

&lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>mapred-site.xml</p>

<pre><code>&lt;configuration&gt;
&lt;property&gt;
&lt;name&gt;mapred.job.tracker&lt;/name&gt;
&lt;value&gt;54.68.218.192:10002&lt;/value&gt;
&lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>hdfs-site.xml</p>

<pre><code>&lt;configuration&gt;
&lt;property&gt;
&lt;name&gt;dfs.replication&lt;/name&gt;
&lt;value&gt;1&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
&lt;name&gt;dfs.namenode.name.dir&lt;/name&gt;

</code></pre>

<p>the jps run on the Namenode give the following 
    5696 NameNode
    6504 Jps
    5905 SecondaryNameNode
    6040 ResourceManager</p>

<p>and jps on datanode
    2883 DataNode
    3496 Jps
    3381 NodeManager</p>

<p>which to me seems right.</p>

<p>Now when I try to run a put command:
    hadoop fs -put count_inputfile /test/input/</p>

<p>it gives me the following error:
    put: File /count_inputfile.<em>COPYING</em> could only be replicated to 0 nodes instead of minReplication (=1).  There are 0 datanode(s) running and no node(s) are excluded in this operation.</p>

<p>the logs on the datanode says the following</p>

<pre><code>hadoop-datanode log

<p>yarn-nodemanager log

<p>the web UI of node manager(50070) shows that there are 0 live nodes and 0 dead nodes and the dfs used is 100%</p>

<p>I have also disabled IPV6.
on a few websites I found out that I should also edit the /etc/hosts file

    127.0.0.1 localhost

<p>@Learner, <br>I had this problem of datanodes not shown in the Namenode's web UI. Solved it by these steps in Hadoop 2.4.1. <br><br>do this for all the nodes (master and slaves) <br><BR>1. remove all temporary files ( by default in /tmp) - <code>sudo rm -R /tmp/*</code>.<br>
2. Now try connecting to all nodes through <code>ssh</code> by using <code>ssh username@host</code> and add keys in your master using <code>ssh-copy-id -i ~/.ssh/id_rsa.pub username@host</code> to give unrestricted access of slaves to the master (not doing so might be the problem for refusing connections).
<br>
3. Format the namenode using <code>hadoop namenode -format</code> and try restarting the daemons. </p>

<p>Two things worked for me, </p>

<p><strong>STEP 1 : stop hadoop and clean temp files from <code>hduser</code></strong></p>

<pre><code>sudo rm -R /tmp/*
</code></pre>

<p><a href="https://gist.github.com/prayagupd/8519289#step-7-configure-etchadoop-sitexml" rel="nofollow">also, you may need to delete and recreate</a> <code>/app/hadoop/tmp</code> (mostly when I change hadoop version from <code>2.2.0</code> to <code>2.7.0</code>)</p>

<pre><code>sudo rm -r /app/hadoop/tmp
sudo mkdir -p /app/hadoop/tmp
sudo chown hduser:hadoop /app/hadoop/tmp
sudo chmod 750 /app/hadoop/tmp
</code></pre>

<p><strong>STEP 2: format namenode</strong></p>

<pre><code>hdfs namenode -format
</code></pre>

<p>Now, I can see DataNode</p>

<pre><code>hduser@prayagupd:~$ jps
19135 NameNode
20497 Jps
19477 DataNode
20447 NodeManager
19902 SecondaryNameNode
20106 ResourceManager
</code></pre>
