    <div class="post-text" itemprop="text">

<p>Assume we have a table which has two columns, one column contains the names of some people and the other column contains some values related to each person. One person can have more than one value. Each value has a numeric type. The question is we want to select the top 3 values for each person from the table. If one person has less than 3 values, we select all the values for that person. </p>

<p>The issue can be solved if there are no duplicates in the table by the query provided in this article <a href="http://www.pixelstech.net/article/1369297243_Select_top_3_values_from_each_group_in_a_table_with_SQL">Select top 3 values from each group in a table with SQL </a> . But if there are duplicates, what is the solution? </p>

<p>For example, if for one name John, he has 5 values related to him. They are 20,7,7,7,4. I need to return the name/value pairs as below order by value descending for each name:</p>

<pre>
-----------+-------+
| name      | value |
-----------+-------+
| John   |     20 |
| John   |     7 |
| John   |     7 |
-----------+-------+
</pre>

<p>Only 3 rows should be returned for John even though there are three 7s for John.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In many modern DBMS (e.g. Postgres, Oracle, SQL-Server, DB2 and many others), the following will work just fine. It uses CTEs and ranking function <code>ROW_NUMBER()</code> which is part of the latest SQL standard:</p>

<pre><code> WITH cte AS
  ( SELECT name, value,
           ROW_NUMBER() OVER (PARTITION BY name
                              ORDER BY value DESC
                             )
             AS rn
    FROM t
  )
SELECT name, value, rn
FROM cte
WHERE rn &lt;= 3
ORDER BY name, rn ;
</code></pre>

<p>Without CTE, only <code>ROW_NUMBER()</code>:</p>

<pre><code>SELECT name, value, rn
FROM 
  ( SELECT name, value,
           ROW_NUMBER() OVER (PARTITION BY name
                              ORDER BY value DESC
                             )
             AS rn
    FROM t
  ) tmp 
WHERE rn &lt;= 3
ORDER BY name, rn ; 
</code></pre>

<p>Tested in: </p>

<ul>
<li><strong><a href="http://sqlfiddle.com/#!1/43f97/1">Postgres</a></strong></li>
<li><strong><a href="http://sqlfiddle.com/#!4/43f97/14">Oracle</a></strong></li>
<li><strong><a href="http://sqlfiddle.com/#!3/43f97/1">SQL-Server</a></strong></li>
</ul>

<hr>

<p>In MySQL and other DBMS that do not have ranking functions, one has to use either derived tables, correlated subqueries or self-joins with <code>GROUP BY</code>.</p>

<p>The <code>(tid)</code> is assumed to be the primary key of the table:</p>

<pre><code>SELECT t.tid, t.name, t.value,              -- self join and GROUP BY
       COUNT(*) AS rn
FROM t
  JOIN t AS t2
    ON  t2.name = t.name
    AND ( t2.value &gt; t.value
        OR  t2.value = t.value
        AND t2.tid &lt;= t.tid
        )
GROUP BY t.tid, t.name, t.value
HAVING COUNT(*) &lt;= 3
ORDER BY name, rn ;


SELECT t.tid, t.name, t.value, rn
FROM
  ( SELECT t.tid, t.name, t.value,
           ( SELECT COUNT(*)                -- inline, correlated subquery
             FROM t AS t2
             WHERE t2.name = t.name
              AND ( t2.value &gt; t.value
                 OR  t2.value = t.value
                 AND t2.tid &lt;= t.tid
                  )
           ) AS rn
    FROM t
  ) AS t
WHERE rn &lt;= 3
ORDER BY name, rn ;
</code></pre>

<p>Tested in <strong><a href="http://sqlfiddle.com/#!2/43f97/10">MySQL</a></strong></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I was going to downvote the question.  However, I realized that it might really be asking for a cross-database solution.</p>

<p>Assuming you are looking for a database independent way to do this, the only way I can think of uses correlated subqueries (or non-equijoins).  Here is an example:</p>

<pre><code>select distinct t.personid, val, rank
from (select t.*,
             (select COUNT(distinct val) from t t2 where t2.personid = t.personid and t2.val &gt;= t.val
             ) as rank
      from t
     ) t
where rank in (1, 2, 3)
</code></pre>

<p>However, each database that you mention (and I note, Hadoop is not a database) has a better way of doing this.  Unfortunately, none of them are standard SQL.</p>

<p>Here is an example of it working in SQL Server:</p>

<pre><code>with t as (
      select 1 as personid, 5 as val union all
      select 1 as personid, 6 as val union all
      select 1 as personid, 6 as val union all
      select 1 as personid, 7 as val union all
      select 1 as personid, 8 as val
     )
select distinct t.personid, val, rank
from (select t.*,
             (select COUNT(distinct val) from t t2 where t2.personid = t.personid and t2.val &gt;= t.val
             ) as rank
      from t
     ) t
where rank in (1, 2, 3);
</code></pre>

<p>Using <code>GROUP_CONCAT</code> and <code>FIND_IN_SET</code> you can do that.Check <a href="http://www.sqlfiddle.com/#!2/96438/1" rel="nofollow">SQLFIDDLE</a>.</p>

<pre><code>SELECT *
FROM tbl t
WHERE FIND_IN_SET(t.value,(SELECT
                             SUBSTRING_INDEX(GROUP_CONCAT(t1.value ORDER BY VALUE DESC),',',3)
                           FROM tbl t1
                           WHERE t1.name = t.name
                           GROUP BY t1.name)) &gt; 0
ORDER BY t.name,t.value desc
</code></pre>

INSERT INTO #list VALUES ('John', 20), ('John', 7), ('John', 7), ('John', 7), ('John', 4);

WITH cte
AS (
SELECT NAME
    ,value
    ,ROW_NUMBER() OVER (
        PARTITION BY NAME ORDER BY (value) DESC
        ) RN
FROM #list
)
SELECT NAME
,value
FROM cte
WHERE RN &lt; 4
ORDER BY value DESC
</code></pre>

<p>This works for MS SQL. Should be workable in any other SQL dialect that has the ability to assign row numbers in a group by or over clause (or equivelant)</p>

<pre><code>if object_id('tempdb..#Data') is not null drop table #Data;
GO

create table #data (name varchar(25), value integer);
GO
set nocount on;
insert into #data values ('John', 20);
insert into #data values ('John', 7);
insert into #data values ('John', 7);
insert into #data values ('John', 7);
insert into #data values ('John', 5);
insert into #data values ('Jack', 5);
insert into #data values ('Jane', 30);
insert into #data values ('Jane', 21);
insert into #data values ('John', 5);
insert into #data values ('John', -1);
insert into #data values ('John', -1);
insert into #data values ('Jane', 18);
set nocount off;
GO

with D as (
SELECT
     name
    ,Value
    ,row_number() over (partition by name order by value desc) rn
From
    #Data
)
SELECT Name, Value
FROM D
WHERE RN &lt;= 3
order by Name, Value Desc

Name    Value

Jane    30
Jane    21
Jane    18
John    20
