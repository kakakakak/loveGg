<p>My issue is similar to this one : <a href="http://stackoverflow.com/questions/11479600/runing-a-simple-mahout-program">Runing a simple Mahout program</a> , but a bit more complicated.</p>

<p>I am writing hadoop code, and would like to use Mahout math libs (e.g. SparseRowMatrix &amp; VectorWritable classes).</p>

<p>The problem is that the former example class lies at math folder of latest SNAPSHOT of Mahout, while the latter is placed under core folder - and both share the same path as of the point after the root folder (org.apache.mahout.math).</p>

<p>The problem is that I have found no way to make hadoop jar file to import properly the classes that concern math stuff, but lie under the <em>Core</em> folder of Mahout project.</p>

<p>I am working with Eclipse and to run successfully my jobs, I have ,each time I export a new jar, to search down the classpath tree of core folder &amp; bundle the respective math folder into my project.</p>

<p>If anyone could help me avoid this procedure, I would be grateful. Thanks in advance :-)</p>

    <div class="post-text" itemprop="text">
<p>Are you using Mahout code directly in Eclipse project, or you're using other build tool?
I would recommend to use maven-based project, and declare dependency on needed Mahout version in it. Plus you'll get ability to create striped job jar, like this was done for <a href="https://github.com/tdunning/MiA" rel="nofollow">examples from book 'Mahout in Action'</a> - look onto <code>pom.xml</code> there...</p>

<p>Maven support in eclipse is implemented through <code>m2eclipse</code> plugin...</p>
