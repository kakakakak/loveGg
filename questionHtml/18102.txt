Will nodes who have done maps will do too reduce tasks?</p>

<p>Thank you.</p>

    <div class="post-text" itemprop="text">
<p><strong>How Many Maps?</strong></p>

<blockquote>
  <p>The number of maps is usually driven by the total size of the inputs, that is, the total number of blocks of the input files.</p>
  
  <p>The right level of parallelism for maps seems to be around 10-100 maps per-node, although it has been set up to 300 maps for very cpu-light map tasks. Task setup takes a while, so it is best if the maps take at least a minute to execute.</p>
</blockquote>

<p>If you havve 10TB of input data and a blocksize of 128MB, you’ll end up with 82,000 maps, unless Configuration.set(MRJobConfig.NUM_MAPS, int) (which only provides a hint to the framework) is used to set it even higher.</p>

<p><strong>How Many Reduces?</strong></p>

<p>The right number of reduces seems to be 0.95 or 1.75 multiplied by (  &lt; no. of nodes > * &lt; no. of maximum containers per node > ).</p>

<p>With 0.95 all of the reduces can launch immediately and start transferring map outputs as the maps finish. With 1.75 the faster nodes will finish their first round of reduces and launch a second wave of reduces doing a much better job of load balancing.</p>

<p>Increasing the number of reduces increases the framework overhead, but increases load balancing and lowers the cost of failures.</p>

<p><strong>Reducer NONE</strong></p>

<p>It is legal to set the number of reduce-tasks to zero if no reduction is desired</p>

<p><strong>Which nodes for Reduce tasks?</strong></p>

<p>You can configure number of mappers and number of reducers per node as per <a href="https://hadoop.apache.org/docs/r2.4.1/hadoop-mapreduce-client/hadoop-mapreduce-client-core/mapred-default.xml" rel="nofollow">Configuration parameters</a> like <strong>mapreduce.tasktracker.reduce.tasks.maximum</strong></p>

<p>if you set this parameter as zero, that node won't be considered for Reduce tasks. Otherwise, all nodes in the cluster are eligible for Reduce tasks.</p>

<p>Source : <a href="https://hadoop.apache.org/docs/current/hadoop-mapreduce-client/hadoop-mapreduce-client-core/MapReduceTutorial.html" rel="nofollow"><strong>Map Reduce Tutorial</strong></a> from Apache.</p>

<p><strong>Note:</strong> For a given Job, you can set mapreduce.job.maps &amp; mapreduce.job.reduces. But it may not be effective. We should leave the decisions to Map Reduce Framework to decide on number of Map &amp; Reduce tasks</p>

<p><strong>EDIT:</strong></p>

<p>How to decide which Reducer node?</p>

<p>Assume that you have equal reduce slots available on two nodes N1 and N2 and current load on N1 > N2, then , Reduce task will be assigned to N2. If both load and number of slots are same, whoever sends first heartbeat to resource manager will get the task. This is the code block for reduce assignment:<a href="http://grepcode.com/file/repository.cloudera.com/content/repositories/releases/com.cloudera.hadoop/hadoop-core/0.20.2-320/org/apache/hadoop/mapred/JobQueueTaskScheduler.java#207" rel="nofollow">http://grepcode.com/file/repository.cloudera.com/content/repositories/releases/com.cloudera.hadoop/hadoop-core/0.20.2-320/org/apache/hadoop/mapred/JobQueueTaskScheduler.java#207</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<blockquote>
  <p>how does Hadoop decides how many nodes will do map tasks</p>
</blockquote>

<p>By default, the number of mappers will be same as the number of split (blocks) of the input to the mapreduce. </p>

<p>Now about the nodes, In the Hadoop 2, each node runs it own <code>NodeManager</code> (NM). The job of the NM is to manage the application container assigned to it by the <code>Resourcemanager</code> (RM). So basically, each of the task will be running in the individual container. To run the mapper tasks, <code>ApplicationMaster</code> negotiate the container from the <code>ResourceManager</code>. Once the containers are allocated, the <code>NodeManager</code> will launch the task and monitor it.</p>

<blockquote>
  <p>which nodes will do the reduce tasks?</p>
</blockquote>

<p>Again the reduce tasks will also runs in the containers. The <code>ApplicationMaster</code> (per-application (job)) will negotiate the containers from the RM and launch the reducer tasks. Mostly they run on the different nodes then the Mapper nodes.</p>

<p>The default number of reducers for any job is 1. The number of reducers can be set in the job configuration.</p>
