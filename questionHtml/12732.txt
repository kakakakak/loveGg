    <div class="post-text" itemprop="text">

<p>It is kind of obvious and we will all agree that we can call <code>HDFS + YARN + MapReduce</code> as <em>Hadoop</em>. But what happens with different other combinations and other products in the <em>Hadoop</em> ecosystem?</p>

<p>Is, for example, <code>HDFS + YARN + Spark</code> still <em>Hadoop</em>? Is <code>HBASE</code> <em>Hadoop?</em> I guess we consider <code>HDFS + YARN + Pig</code> <em>Hadoop</em>, since <em>Pig</em> uses <em>MapReduce.</em></p>

<p>Are only the <em>MapReduce</em> tools considered <em>Hadoop</em>, but anything else run on <code>HDFS + YARN</code>(like <em>Spark</em>) is <em>non Hadoop</em>?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I agree with your impression that the <em>"Hadoop"</em> term does not have a useful definition. <em>"We have a Hadoop cluster"</em> may mean various things.</p>

<p>There is an official answer though at <a href="http://hadoop.apache.org/#What+Is+Apache+Hadoop%3F" rel="nofollow">http://hadoop.apache.org/#What+Is+Apache+Hadoop%3F</a>:</p>

<blockquote>
  <p>The Apache™ Hadoop® project develops open-source software for
  reliable, scalable, distributed computing.</p>
  
  <p>The Apache Hadoop software library is a framework that allows for the
  distributed processing of large data sets across clusters of computers
  using simple programming models.</p>
</blockquote>

<p>So <em>"Hadoop"</em> is the name of a project and a software library. Any other usage is ill-defined.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In addition to Apache hadoop definition from <a href="https://hadoop.apache.org/" rel="nofollow">Official website</a>, I would like to highlight that <strong>Hadoop is a framework</strong> and there are many sub-systems in <a href="https://hadoopecosystemtable.github.io/" rel="nofollow">Hadoop ecosystem</a> </p>

<p><strong>Quoting this content from official website so that broken links in future does not cause any issue to this answer.</strong> </p>

<p>The project includes these modules:</p>

<p><code>Hadoop Common:</code> The common utilities that support the other Hadoop modules.</p>

<p><code>Hadoop Distributed File System (HDFS™):</code> A distributed file system that provides high-throughput access to application data.</p>

<p><code>Hadoop YARN:</code> A framework for job scheduling and cluster resource management.</p>

<p><code>Hadoop MapReduce:</code> A YARN-based system for parallel processing of large data sets.</p>

<p>But these four modules does not cover complete Hadoop Ecosystem. There are many Hadoop related projects and 40+ subsystems in Hadoop Ecosystems.</p>

<p>Other Hadoop-related projects at Apache include:</p>

<p><code>Ambari™:</code> A web-based tool for provisioning, managing, and monitoring Apache Hadoop clusters which includes support for Hadoop HDFS, Hadoop MapReduce, Hive, HCatalog, HBase, ZooKeeper, Oozie, Pig and Sqoop.</p>

<p><code>Avro™</code>: A data serialization system.</p>

<p><code>Cassandra™:</code> A scalable multi-master database with no single points of failure.</p>

<p><code>Chukwa™:</code> A data collection system for managing large distributed systems.</p>

<p><code>HBase™:</code> A scalable, distributed database that supports structured data storage for large tables.</p>

<p><code>Hive™:</code> A data warehouse infrastructure that provides data summarization and ad hoc querying.</p>

<p><code>Mahout™:</code> A Scalable machine learning and data mining library.</p>

<p><code>Pig™:</code> A high-level data-flow language and execution framework for parallel computation.</p>

<p><code>Spark™:</code> A fast and general compute engine for Hadoop data. Spark provides a simple and expressive programming model that supports a wide range of applications, including ETL, machine learning, stream processing, and graph computation.</p>

<p><code>Tez™:</code> A generalized data-flow programming framework, built on Hadoop YARN, which provides a powerful and flexible engine to execute an arbitrary DAG of tasks to process data for both batch and interactive use-cases. Tez is being adopted by Hive™, Pig™ and other frameworks in the Hadoop ecosystem, and also by other commercial software (e.g. ETL tools), to replace Hadoop™ MapReduce as the underlying execution engine.</p>

<p><code>ZooKeeper™:</code> A high-performance coordination service for distributed applications.</p>

<p><strong><em>Coming back to your question:</em></strong></p>

<p>Just have a look at 40+ sub systems in Hadoop eco system. Every thing you have quoted may not be Hadoop but most them are related to Hadoop. </p>

<p><strong>Spark</strong> is part of Hadoop ecosystem. But it can neither use <strong>HDFS</strong> nor <strong>YARN</strong>. HDFS data sets can be replaced with <strong>RDD</strong> ( resilient distributed dataset) and can run in <strong>Standalone</strong> mode without <strong>YARN</strong>.  </p>
