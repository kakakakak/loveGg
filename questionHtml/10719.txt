<p>I have the following scenario-</p>

<p>Pig version used 0.70</p>

<p>Sample HDFS directory structure:</p>

<pre><code>/user/training/test/20100810/&lt;data files&gt;
/user/training/test/20100811/&lt;data files&gt;
/user/training/test/20100812/&lt;data files&gt;
/user/training/test/20100813/&lt;data files&gt;
/user/training/test/20100814/&lt;data files&gt;
</code></pre>

<p>As you can see in the paths listed above, one of the directory names is a date stamp.</p>

<p><strong>Problem:</strong> I want to load files from a date range say from 20100810 to 20100813.</p>

<p>I can pass the 'from' and 'to' of the date range as parameters to the Pig script but how do I make use of these parameters in the LOAD statement. I am able to do the following</p>

<pre><code>temp = LOAD '/user/training/test/{20100810,20100811,20100812}' USING SomeLoader() AS (...);
</code></pre>

<p>The following works with hadoop:</p>

<pre><code>hadoop fs -ls /user/training/test/{20100810..20100813}
</code></pre>

<p>But it fails when I try the same with LOAD inside the pig script. How do I make use of the parameters passed to the Pig script to load data from a date range? </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Pig is processing your file name pattern using the hadoop file glob utilities, not the shell's glob utilities.  Hadoop's are documented <strong><a href="http://hadoop.apache.org/docs/stable/api/org/apache/hadoop/fs/FileSystem.html#globStatus%28org.apache.hadoop.fs.Path%29">here</a></strong>.  As you can see, hadoop does not support the '..' operator for a range.  It seems to me you have two options - either write out the <code>{date1,date2,date2,...,dateN}</code> list by hand, which if this is a rare use case is probably the way to go, or write a wrapper script which generates that list for you.  Building such a list from a date range should be a trivial task for the scripting language of your choice.  For my application, I've gone with the generated list route, and it's working fine (CHD3 distribution).</p>

<p>script.pig:</p>

<pre><code>temp = LOAD '$input' USING SomeLoader() AS (...);

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>i ran across this answer when i was having trouble trying to create a file glob in a script and then pass it as a parameter into a pig script.</p>

<p>none of the current answers applied to my situation, but i did find a general answer that might be helpful here.</p>

<p>in my case, the shell expansion was happening and then passing that into the script - causing complete problems with the pig parser, understandably.</p>

<p>so by simply surrounding the glob in double-quotes protects it from being expanded by the shell, and passes it as is into the command.</p>

<p>WON'T WORK:</p>

<pre class="lang-cpp prettyprint-override"><code>$ pig -f my-pig-file.pig -p INPUTFILEMASK='/logs/file{01,02,06}.log' -p OTHERPARAM=6
</code></pre>

<p>WILL WORK</p>

<pre class="lang-cpp prettyprint-override"><code>$ pig -f my-pig-file.pig -p INPUTFILEMASK="/logs/file{01,02,06}.log" -p OTHERPARAM=6
</code></pre>

<p>i hope this saves someone some pain and agony.</p>

<p>So since this works:</p>

<pre><code>temp = LOAD '/user/training/test/{20100810,20100811,20100812}' USING SomeLoader()
</code></pre>

<p>but this does not work:</p>

<pre><code>temp = LOAD '/user/training/test/{20100810..20100812}' USING SomeLoader()
</code></pre>

<p>but if you want a date range that spans say 300 days and passing a full list to LOAD is not elegant to say the least. I came up with this and it works.</p>

<p>Say you want to load data from 2012-10-08 to today 2013-02-14, what you can do is</p>

<pre><code>temp = LOAD '/user/training/test/{201210*,201211*,201212,2013*}' USING SomeLoader()
</code></pre>

<p>then do a filter after that</p>

<pre><code>filtered = FILTER temp BY (the_date&gt;='2012-10-08')
</code></pre>
