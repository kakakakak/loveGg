<p>I was able to successfully start single-node cluster on 2 computers on my home network, but I am having trouble starting them as a multi-node cluster. When I run the command <code>start-dfs.sh</code> I get the output</p>

<pre><code>hduser@eric-T5082:/usr/local/hadoop/sbin$ start-dfs.sh
Starting namenodes on [master]
master: starting namenode, logging to /usr/local/hadoop/logs/hadoop-hduser-namenode-eric-T5082.out
slave: starting datanode, logging to /usr/local/hadoop/logs/hadoop-hduser-datanode-Study-Linux.out
master: starting datanode, logging to /usr/local/hadoop/logs/hadoop-hduser-datanode-eric-T5082.out
Starting secondary namenodes [0.0.0.0]
0.0.0.0: starting secondarynamenode, logging to /usr/local/hadoop/logs/hadoop-hduser-secondarynamenode-eric-T5082.out
</code></pre>

<p>When I run jps, I get the following output:</p>

<pre><code>hduser@eric-T5082:/usr/local/hadoop/sbin$ jps
The program 'jps' can be found in the following packages:
 * openjdk-7-jdk
 * openjdk-6-jdk
Try: sudo apt-get install &lt;selected package&gt;
</code></pre>

<p>yet <code>jps</code> is returning the correct result for the slave node:</p>

<pre><code>hduser@Study-Linux:/usr/local/hadoop/etc/hadoop$ jps
6401 Jps
6300 DataNode
</code></pre>

<p>I suspect this may be due to (a) a port problem, i.e. the port is already occupied; (b) a problem with temporary files being generated and interfering with the <code>hdfs namenode -format</code> command. But I have tried to address problem (a) by trying different ports for the namenode and (b) erasing the temporary files before running <code>hdfs</code>. </p>
