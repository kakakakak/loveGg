<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You haven't written anything about your interest.
I know algorithms in graph mining has been implemented over hadoop framework. This software <a href="http://www.cs.cmu.edu/~pegasus/">http://www.cs.cmu.edu/~pegasus/</a> and paper : "PEGASUS: A Peta-Scale Graph Mining System - Implementation and Observations" may give you starting point.</p>

<p>Further, this link discusses something similar to your question: <a href="http://atbrox.com/2010/02/08/parallel-machine-learning-for-hadoopmapreduce-a-python-example/">http://atbrox.com/2010/02/08/parallel-machine-learning-for-hadoopmapreduce-a-python-example/</a> but it is in python.
And, there is a very good paper by Andrew Ng "Map-Reduce for Machine Learning on Multicore".</p>

<p>There was a NIPS 2009  workshop on similar topic "Large-Scale Machine Learning: Parallelism and Massive Datasets". You can browse some of the paper and get an idea. </p>

<p>Edit : Also there is Apache Mahout <a href="http://mahout.apache.org/">http://mahout.apache.org/</a>   -->" Our core algorithms for clustering, classfication and batch based collaborative filtering are implemented on top of Apache Hadoop using the map/reduce paradigm"</p>
