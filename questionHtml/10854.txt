<div>
    <div class="post-text" itemprop="text">

<p>It seems to me that, at the end of the day, most NoSQL databases are at their core key/value stores, which means one should be able to build a layer which could be NoSQL database agnostic.</p>

<p>That layer would only use CRUD operations (put, set, delete), but would expose more advanced features, and you'd be able to switch with minimal effort  the underlying DB whether it's Mongo, Redis, Cassandra, etc.</p>

<p><strong>Would building something like this have value to many people, and does it already exist?</strong></p>

<p>Thanks</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It's next to impossible to build such thing. </p>

<p>As a thought experiment, I suggest that you take, for example, Redis, MongoDB and Cassandra, and design an API of such layer.</p>

<p>These NoSQL solutions have drastically different characteristics and they serve different purposes. Trying to build a common API for them is like building a common API for SQL database, spreadsheet document, plain text file and gmail. </p>

<p>While you can certainly come up with <em>something</em>, it will completely pointless.</p>

<p>Different needs call for different tools.</p>
