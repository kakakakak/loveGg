<p>Try this </p>

<p>Comment 127.0.1.1 in /etc/hosts file using # 
then put your ip  and computer name  in new line
if you want to use localhost make sure that 127.0.0.1 localhost is there in your hosts file
then replace all occurance of ip in configuration files replace with localhost</p>

<p>if you want to use ip instead of localhost then make sure that ip and equivalent domain name is there in your hosts file and replace all occurance of localhost with your ip.</p>

<p>problem related to namenode generally occurs due to misconfiguration of host or IP</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Take a look at  java.io.FileNotFoundException: /hadoop/tmp/dfs/name/current/VERSION (Permission denied)</p>

<p>So, first of all - please see what do you have set as hbase.rootdir indeed - whether it is pointing to the HDFS or local filesystem. My example (with localhost for pseudo-distributed mode):</p>

<pre><code>    &lt;configuration&gt;
        &lt;property&gt;
            &lt;name&gt;hbase.rootdir&lt;/name&gt;
            &lt;value&gt;hdfs://localhost:54310/hbase&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
            &lt;name&gt;hbase.master&lt;/name&gt;
            &lt;value&gt;127.0.0.1:60000&lt;/value&gt;
        &lt;/property&gt;
    &lt;/configuration&gt;
</code></pre>

<p>Next, looking at your log it seems that most likely you're running using the local filesystem and you don't have the read/write access to the directory where HBase stores its data - check it with </p>

<pre><code>mcbatyuk:/ bam$ ls -l / |grep hadoop
drwxr-xr-x   3 bam   wheel       102 Feb 29 21:34 hadoop
</code></pre>

<p>If your base.rootdir is on HDFS you seem to have broken permissions, so you will need to change it with</p>

<pre><code># hadoop fs -chmod -R MODE /hadoop/
</code></pre>

<p>or change the property dfs.permissions to false in your $HADOOP_HOME/conf/hdfs-site.xml</p>
