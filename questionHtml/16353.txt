<blockquote>
  <p><strong>Possible Duplicate:</strong><br>
  <a href="http://stackoverflow.com/questions/11399140/equivalent-of-mongos-outreduce-option-in-hadoop">Equivalent of mongo&#39;s out:reduce option in hadoop</a>  </p>

<p>I have 2 datasets, one is a complement of the other. It looks something like this (not actual fields):</p>

<pre><code>Question
========
id(key)
name
description

Answer
========
id(key)
type
question_id

Output
======
question_id (key)
name
description
type_a_count
type_b_count
</code></pre>

<p>I want to know how many answers of a certain type there are for each question. I used to do this with mongodb's map reduce engine by emitting the same fields(but zeroed) of my question mapper except a one in the type_count field and just add everything up in my reducer. <b>The problem I have now is that when I run the Answer mapper, the values from my question mapper are overwritten by the ones from the answer mapper.</b></p>

<p><b>I'm looking for the equivalent of mongodb's {out: "reduce"} option.</b></p>

<h2>More details:</h2>

<ul>
<li>I'm only using a mapper for my question mapper</li>
<li>The outputURI for both jobs is the same since I want it to merge</li>
<li>I want to use the question mapper's output and the answer mapper's output as my reducer's input</li>
</ul>

    <div class="post-text" itemprop="text">
<p>This answer may or may not be to your liking. I know you tagged java, but there is a library called cascalog (written in clojure) that can be used to write hadoop queries. It's pretty simple:</p>

<pre><code>$ lein repl
REPL started; server listening on localhost port 16309
myapp=&gt; (use 'cascalog.playground)
nil
myapp=&gt; (bootstrap)
nil
myapp=&gt; (def questions [["1" "what?" "desc what"] ["2" "where?" "Desc where"]])
#'myapp/questions
myapp=&gt; (def answers [["1" "a" "1"]["2" "a" "1"]["3" "a" "1"]["4" "b" "2"]])
#'myapp/answers
myapp=&gt; (?&lt;- (stdout) [?type ?name ?desc ?count] (questions ?qid ?name ?desc) (answers ?aid ?type ?qid) (c/count ?count))

RESULTS
-----------------------
a       what?   desc what   3
b       where?  Desc where  1
</code></pre>

<p>The following is a good starting place to learn about cascalog: <a href="http://nathanmarz.com/blog/introducing-cascalog-a-clojure-based-query-language-for-hado.html" rel="nofollow">http://nathanmarz.com/blog/introducing-cascalog-a-clojure-based-query-language-for-hado.html</a>. </p>
