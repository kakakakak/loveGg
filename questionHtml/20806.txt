<p>I am trying to start to HBase in IDEA by running HMaster class, but I got this error:</p>

<pre><code>java.lang.RuntimeException: Failed suppression of fs shutdown hook: Thread[Thread-8,5,main]
    at org.apache.hadoop.hbase.regionserver.ShutdownHook.suppressHdfsShutdownHook(ShutdownHook.java:196)
    at org.apache.hadoop.hbase.regionserver.ShutdownHook.install(ShutdownHook.java:83)
    at org.apache.hadoop.hbase.util.JVMClusterUtil.startup(JVMClusterUtil.java:191)
    at org.apache.hadoop.hbase.LocalHBaseCluster.startup(LocalHBaseCluster.java:420)
    at org.apache.hadoop.hbase.master.HMasterCommandLine.startMaster(HMasterCommandLine.java:149)
    at org.apache.hadoop.hbase.master.HMasterCommandLine.run(HMasterCommandLine.java:104)
    at org.apache.hadoop.util.ToolRunner.run(ToolRunner.java:65)
    at org.apache.hadoop.hbase.util.ServerCommandLine.doMain(ServerCommandLine.java:76)
    at org.apache.hadoop.hbase.master.HMaster.main(HMaster.java:2100)
</code></pre>

<p>FYI /etc/hosts file only contains </p>

<blockquote>
  <p>127.0.0.1 - machine name</p>
</blockquote>

<p>Here is core-site.xml</p>

<pre><code>&lt;configuration&gt;
   &lt;property&gt;
       &lt;name&gt;fs.default.name&lt;/name&gt;
       &lt;value&gt;hdfs://localhost:8020&lt;/value&gt;
   &lt;/property&gt;

   &lt;property&gt;
       &lt;name&gt;hadoop.tmp.dir&lt;/name&gt;
       &lt;value&gt;/home/ubuntu/Desktop/hdfs/tmp&lt;/value&gt;
   &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>and </p>

<p>hbase-site.xml</p>

<pre><code>&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;hbase.rootdir&lt;/name&gt;
    &lt;value&gt;/usr/lib/hbase&lt;/value&gt;
  &lt;/property&gt;
  &lt;property&gt;
    &lt;name&gt;hbase.zookeeper.property.dataDir&lt;/name&gt;
    &lt;value&gt;/usr/lib/zookeeper&lt;/value&gt;
  &lt;/property&gt;
  &lt;property&gt;
    &lt;name&gt;hbase.zookeeper.property.clientPort&lt;/name&gt;
    &lt;value&gt;2181&lt;/value&gt;
  &lt;/property&gt;
  &lt;property&gt;
    &lt;name&gt;hbase.zookeeper.quorum&lt;/name&gt;

 &lt;/property&gt;

<p>I met similar problem, but in standalone mode. Here is how I resolved the problem:</p>

<ol>
<li>check the output of <code>"hostname"</code> and "hostname -f", make sure the
output of the command "hostname" only contains the host name, while
the "hostname -f" contains the FQDN. If not, resolve this first.</li>
<li>I ran it as root using command "<code>sudo bin/start-hbase.sh</code>"</li>
</ol>

<p>I still don't know the root cause of this issue, but the Hbase instance is running after I took above steps. Hope it helps!</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>This error happens when an HBase compiled against Hadoop 1 has Hadoop 2 jars on the classpath. One way of getting into this situation would be to just dump the Hadoop jars in the lib/ folder, another way is by having an external tool putting those jars on the system's classpath, which HBase will pickup.</p>

<p>This is easy to verify, simply run <code>bin/hbase classpath</code> and it will print it out. </p>

<p>In Standalone mode, or when Zookeeper is managed by HBase, it also gets printed on a line like this at the beginning of the log:</p>

<pre><code>INFO org.apache.zookeeper.server.ZooKeeperServer: Server environment:java.class.path=
</code></pre>

<p>Resolve by removing the Hadoop 2 jars.</p>
