<p>I will answer you based on MySQL, because I see a tag with that RDMS name in this post. So then, you have three options. </p>

<h2>1.- Filtering on Sqoop query side</h2>

<p>I assume here that you <strong>import your data using Sqoop</strong>. This tool have an option that allows you to export the result of a SQL query. In that query you could use this MySQL method, <em>UNIX_TIMESTAMP(date, format)</em>,  that transform Date to Timestamp. The Sqoop instruction will be something like this:</p>

<pre><code>sqoop import --connect jdbc:mysql://mysqlhost/mysqldb --username user --password 
passwd --query "SELECT col_1, ..., UNIX_TIMESTAMP(STR_TO_DATE(date_col, '%Y-%m-%d')) FROM table1 WHERE \$CONDITIONS" -m 1 --target-dir hive_table1_data`
</code></pre>

<p>Notice that <strong>WHERE \$CONDITIONS  is mandatory</strong>. Furthermore I've assumes here your date colum is a string. But if it is a date type, then the method <em>str_to_date</em> is not needed.</p>

<h2>2.- On the RDMS side</h2>

<p>In this point I will assume that <strong>you have permisions to CREATE VIEWS</strong> in that database.</p>

<p>Other solution is create a view that contains the field date transformed to timestamp. As we saw below the function is <em>UNIX_TIMESTAMP(date, format)</em>. <a href="http://stackoverflow.com/questions/11133760/mysql-convert-datetime-to-unix-timestamp">Here</a> you have more details. In your case you should write something like this:</p>

<pre><code>CREATE VIEW view_to_export AS 
  SELECT 
      fields1,
      ...,
      UNIX_TIMESTAMP(STR_TO_DATE(field_date, '%Y-%m-%d'));
</code></pre>

<p>Assuming your date field is a string data type, if it is a date data type, the method <em>str_to_date</em> is not needed. The sqoop instruction will be something like this:</p>

<pre><code>sqoop import --connect jdbc:mysql://mysqlhost/mysqldb --username user --password 
    passwd  --table mysql_table_name -m 1 --target-dir hive_table1_data
</code></pre>

<p>So then, with that view, you could use it with sqoop without problems.</p>

<h2>3. After import the data</h2>

<p>In case you use other kind of import system, but you have the data stored into your HDFS. You could create another new table with that field transformed. To do that you could use something like this:</p>

<pre><code>INSERT INTO TABLE my_table
SELECT
   col_1,
   ...,
   unix_timestamp(date_col, 'yyyy-MM-dd')
)
FROM exported_table
;
</code></pre>

<p>In this case you need more time to process. But it is independent on the way you import your data, and <strong>use less resources from MySQL server</strong>, because you don't have to calculate the transformation from date to timestamp for each row. <strong>You delegate that to Hadoop</strong> which mission is process huge amount of data.</p>
