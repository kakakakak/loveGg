    <div class="post-text" itemprop="text">

<p>Intuitively, hadoop is doing something like this to distribute keys to mappers, using python-esque pseudocode.</p>

<pre><code># data is a dict with many key-value pairs
keys = data.keys()
key_set_size = len(keys) / num_mappers
index = 0
mapper_keys = []
for i in range(num_mappers):
  end_index = index + key_set_size
  send_to_mapper(keys[int(index):int(end_index)], i)
  index = end_index
# And something vaguely similar for the reducer (but not exactly).
</code></pre>

<p>It seems like somewhere hadoop knows the index of each key it is passing around, since it distributes them evenly among the mappers (or reducers).  My question is: how can I access this index?  I'm looking for a range of integers [0, n) mapping to all my n keys; this is what I mean by an "index".</p>

<p>I'm interested in the ability to get the index from within either the mapper or reducer.</p>

    <div class="post-text" itemprop="text">
<p>Actually, your reducer (each individual one) gets an array of items back that correspond to the reduce key. So do you want the offset of items within the reduce key in your reducer, or do you want the overall offset of the particular item in the global array of all lines being processed? To get an indeex in your mapper, you can simply prepend a line number to each line of the file before the file gets to the mapper. This will tell you the "global index". However keep in mind that with 1 000 000 items, item 662 345 could be processed before item 10 000.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you are using the new MR API then the <strong>org.apache.hadoop.mapreduce.lib.partition.HashPartitioner</strong> is the default partitioner or else <strong>org.apache.hadoop.mapred.lib.HashPartitioner</strong> is the default partitioner. You can call the <strong>getPartition()</strong> on either of the HashPartitioner to get the partition number for the key (which you mentioned as index).</p>

<p><em>Note that the HashPartitioner class is only used to distribute the keys to the Reducer. When it comes to a mapper, each input split is processed by a map task and the keys are not distributed.</em></p>

<p>Here is the code from HashPartitioner for the getPartition(). You can write a simple Java program for the same.</p>

<blockquote>
  <p>public int getPartition(K key, V value, int numReduceTasks) {<br>
  return (key.hashCode() &amp; Integer.MAX_VALUE) % numReduceTasks;<br>
  }</p>
</blockquote>

<p><strong>Edit:</strong> Including another way to get the index.</p>

<p>The following code <a href="http://old.nabble.com/How-to-find-partition-number-in-reducer-td20927911.html" rel="nofollow">from</a> should also work. To be included in the map or the reduce function.</p>

<blockquote>
  <p>public void configure(JobConf job) {<br>
  partition = job.getInt( "mapred.task.partition", 0);<br>
  }</p>
</blockquote>
