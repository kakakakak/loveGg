<p>I am using 
Cassandra-1.2 with patch 5234 - Table created through CQL3 are not accessible to Pig
Hadoop - 1.1.2
pig 0.11.1</p>

<p>I have a table in Cassandra</p>

<p>datatypetest (num int PRIMARY KEY, ascii ascii, blob blob, text text, varnum varint);</p>

<p>and the test data in datatypetest is </p>

<pre><code> num | ascii | blob   | text | varnum
-----+-------+--------+--------+------
  13 |   126 | 0x0003 | John | null
</code></pre>

<p>I ran the following PIG Script </p>

<pre><code>test1 = LOAD 'cassandra://keyspace1/datatypetest' USING CassandraStorage() AS 
(num:int, columns: bag {T: tuple(name, value)});
</code></pre>

<p>And the output is as follows in the alias test1</p>

<pre><code>(12,{((),),((ascii),125),((blob),��),((text),deepak)})
</code></pre>

<p>As you can see in the output, it is not in the following format </p>

<pre><code>(&lt;row_key&gt;,{(&lt;column_name1&gt;,&lt;value1&gt;),(&lt;column_name2&gt;,&lt;value2&gt;)})
</code></pre>

<p>The inner bag has tuple which has another inner tuple and the first inner tuple which I presume is the key is empty.</p>

<p>I cannot use columns.ascii or columns.blob or columns.text to access the columns tuples like below and get an exception</p>

<pre><code>test2 = FOREACH test1 GENERATE num, columns.text;
2013-07-29 09:11:58,488 [main] ERROR org.apache.pig.tools.grunt.Grunt - 
ERROR 1200: Pig script failed to parse: 
&lt;line 3, column 8&gt; pig script failed to validate:    
org.apache.pig.impl.logicalLayer.FrontendException: ERROR 1128: 
Cannot find field text in name:tuple(),value:bytearray
</code></pre>

<p>How can I access the column tuples.
Thanks in advance.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You should not use <code>CassandraStorage</code> when referencing tables created using CQL3. <code>CassandraStorage</code> is analogous to the Thrift API. When accessing CQL3 tables, use <code>CqlStorage</code>:</p>

<pre><code>test1 = LOAD 'cql://keyspace1/datatypetest' USING CqlStorage();
</code></pre>

<p>This should give you name/value tuples for the columns and their contents. The response should look something like this:</p>

<pre><code>((name,13),(ascii,126),(blob,"blobvalue"),(text,John))
</code></pre>

<p>However, there does seem to be a mismatch between the returned collection and the schema that <code>CqlStorage</code> generates. (<a href="http://stackoverflow.com/questions/18391552/cqlstorage-generates-wrong-pig-schema">See this question</a>.)</p>
