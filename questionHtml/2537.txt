<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Note that pig decides the hadoop version depending on which context var you have set
HADOOP_HOME -> v1
HADOOP_PREFIX -> v2</p>

<p>If you use hadoop2, you need to recompile the piggybank (which is by default compiled for hadoop1)</p>

<ol>
<li>go to pig/contrib/piggybank/java</li>
<li>$ ant -Dhadoopversion=23</li>
<li>then copy that jar over pig/lib/piggybank.jar</li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>A few more details because the other answers didn't work for me:</p>

<ol>
<li>Git clone the pig git mirror <a href="https://github.com/apache/pig" rel="nofollow">https://github.com/apache/pig</a></li>
<li>cd into the cloned directory</li>
<li><p>if you've already built pig in the past in this directory, you should run a clean</p>

<pre><code>ant clean
</code></pre></li>
<li><p>build pig for hadoop 2</p>

<pre><code>ant -Dhadoopversion=23
</code></pre></li>
<li><p>cd into piggybank </p>

<pre><code>cd contrib/piggybank/java
</code></pre></li>
<li><p>again, if you've build piggybank before, make sure to clean out the old build files</p>

<pre><code>ant clean
</code></pre></li>
<li><p>build piggybank for hadoop 2 (same command, different directory)</p>

<pre><code>ant -Dhadoopversion=23
</code></pre></li>
</ol>

<p>If you don't build pig first, piggybank will throw a bunch of "symbol not found" exceptions while compiling. In addition, since I had previously built pig for Hadoop 1 (accidentally), without running a clean, I ran into runtime errors.</p>

<p>Some times you may get problem after installing Pig like below:-</p>

<pre><code>java.lang.IncompatibleClassChangeError: Found interface org.apache.hadoop.mapreduce.JobContext, but class was expected
 at org.apache.hcatalog.common.HCatUtil.checkJobContextIfRunningFromBackend(HCatUtil.java:88)
 at org.apache.hcatalog.pig.HCatLoader.setLocation(HCatLoader.java:162)
 at org.apache.pig.backend.hadoop.executionengine.mapReduceLayer.JobControlCompiler.getJob(JobControlCompiler.java:540)
 at org.apache.pig.backend.hadoop.executionengine.mapReduceLayer.JobControlCompiler.compile(JobControlCompiler.java:322)
 at org.apache.pig.backend.hadoop.executionengine.mapReduceLayer.MapReduceLauncher.launchPig(MapReduceLauncher.java:199)
 at org.apache.pig.backend.hadoop.executionengine.HExecutionEngine.launchPig(HExecutionEngine.java:277)
 at org.apache.pig.PigServer.launchPlan(PigServer.java:1367)
 at org.apache.pig.PigServer.executeCompiledLogicalPlan(PigServer.java:1352)
 at org.apache.pig.PigServer.execute(PigServer.java:1341)
</code></pre>

<p>Many blogs suggest you to recompile the Pig by executing command:</p>

<pre><code>ant clean jar-all -Dhadoopversion=23
</code></pre>

<p>or recompile piggybank.jar by executing below steps</p>

<pre><code>cd contrib/piggybank/java

ant clean

ant -Dhadoopversion=23
</code></pre>

<p><strong>But this may not solve your problem big time</strong>. The actual cause here is related to HCatalog. Try updating it!!. In my case, I was using Hive0.13 and Pig.0.13. And I was using HCatalog provided with Hive0.13.</p>

<p>Then I updated Pig to 0.15 and used separate hive-hcatalog-0.13.0.2.1.1.0-385 library jars. And problem was resolved....</p>

<p>Because later I identified it was not Pig who was creating problem rather it was Hive-HCatalog libraries. Hope this may help. </p>

<p>Even i faced the same error with Hadoop version 2.2.0.
The work around is, we have to register following jar files using the grunt shell.</p>

<p>The paths that i am gonna paste below will be according to the hadoop-2.2.0 version. Kindly find the jars according to your version.</p>

<p>/hadoop-2.2.0/share/hadoop/mapreduce/ hadoop-mapreduce-client-core-2.2.0.jar </p>

<p>/hadoop-2.2.0/share/hadoop/common/hadoop-common-2.2.0.jar </p>

<p>Using the REGISTER command we have to register these jars along with piggybank.</p>
