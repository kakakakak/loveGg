<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Since your cluster is a tens of nodes, it is fine to have a single node with <code>Namenode</code> and <code>JobTracker</code>. <code>Secondary Namenode</code> should be on the other node as it requires more memory to perform periodic backups.<br>
Coming to your question,<br>
It could be the configuration file copying causing conflits.Same problem <a href="http://stackoverflow.com/questions/17975144/only-one-datanode-can-run-in-a-multinode-hadoop-setup">Answered here</a><br>
And you can try copying the working datanode configuration with the appropriate changes.<br>
If there is no data on the cluster, You can format the namenode, before this stop all the the daemons and then start again.<br>
Hope it helps!</p>

    <div class="post-text" itemprop="text">
<p>The fix would be as follows. This will not cause any data deletion as you would have set replication factor to more than 1. 
The fix would be to check for the services running on the box. In this case, you will have <code>tasktracker</code> running on the box. Stop task tracker by running<br>
<br><code>hadoop-daemon stop tasktracker</code></p>

<p>Once this is done, go to the location you might have mentioned under <code>dfs.data.dir</code> property and delete all the files\folders here. Once this is done, run <br>
<br><code>hadoop-daemon start datanode</code>
<br><code>hadoop-daemon start tasktracker</code></p>

<p>This should bring up the datanode and the tasktracker. If you are successful in doing this, go to the namenode and run</p>

<p><br><code>hadoop dfsadmin -refreshNodes</code></p>
