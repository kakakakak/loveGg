<p>I have installed Hadoop and HDFS using this tutorial </p>

<pre><code>http://codesfusion.blogspot.com/2013/10/setup-hadoop-2x-220-on-ubuntu.html
</code></pre>

<p>Everything is fine.</p>

<p>I am also able to create directories and use them using</p>

<pre><code>hadoop fs -mkdir /tmp
hadoop fs -mkdir /small
</code></pre>

<p>I can also say</p>

<pre><code>hadoop fs -ls /
</code></pre>

<p>However I am following a tutorial in which the trainer does</p>

<pre><code>hadoop fs -mkdir temp
hadoop fs -ls
</code></pre>

<p>now on my machine when I issue the above command it says </p>

<pre><code>ls: `.': No such file or directory
</code></pre>

<p>In my training video the command hadoop fs -ls works perfectly. Why should I specify the "/"?</p>

<p>Also I am getting this warning in all my commands</p>

<pre><code>13/12/28 20:23:52 WARN util.NativeCodeLoader: Unable to load native-hadoop library for your platform... using builtin-java classes where applicable
</code></pre>

<p>But in my trainers video there is no such warning.</p>

<p>My configuration file is exactly as the tutorial above and I can also see all management UIs at</p>

<pre><code>http://abhishek-pc:8042/
http://abhishek-pc:50070/
http://abhishek-pc:8088/
</code></pre>

<p>So my question is what is wrong with my configuration and why is my system behaving differently than the training video?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>well, your problem regarding <code>ls: '.': No such file or directory</code>' is because there is not home dir on HDFS for your current user. Try</p>

<pre><code>hadoop fs -mkdir -p /user/[current login user]
</code></pre>

<p>Then you will be able to <code>hadoop fs -ls</code></p>

<p>As for this warning <code>WARN util.NativeCodeLoader: Unable to load native-hadoop library for your platform... using builtin-java classes where applicable</code>, please see my answer at <a href="http://stackoverflow.com/questions/19943766/hadoop-unable-to-load-native-hadoop-library-for-your-platform-error-on-centos/19993403#19993403">this question</a></p>
