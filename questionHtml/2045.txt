<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Your cluster is definitely running, since I can browse to the Spark master web UI at <code>http://yourMasterHostname:8080</code>.  </p>

<p>It looks like you're using Spark 0.8.0, which is affected by a (rare) bug where <code>spark-ec2</code> may erroneously see empty security groups and fail to find an existing cluster.  This was fixed in <a href="https://github.com/mesos/spark/pull/938" rel="nofollow">a pull request</a>, but the patch hasn't been backported to any of the Spark 0.8.x releases. </p>

<p>In the meantime, you can manually apply <a href="https://github.com/apache/incubator-spark/commit/f589ce771a53dcbfda5f62fe0ac77164688ecc76" rel="nofollow">the fix</a> to spark-ec2, since it only changes four lines.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I had the same problem with <code>Spark 0.9.1</code> and updated <code>spark-ec2</code> script. So after successful deploy I tried to login:</p>

<pre><code>./spark-ec2 -k my-key-pair -i my-key-pair.pem login MY_SPARK_CLUSTER
</code></pre>

<p>and this gives error:</p>

<pre><code>Searching for existing cluster Spark...
ERROR: Could not find any existing cluster
</code></pre>

<p>The problem was that my cluster is in region <code>eu-west-1</code>, but default region is <code>us-east-1</code>. So while login you should add <code>--region</code> key:</p>

<pre><code>./spark-ec2 -k my-key-pair -i my-key-pair.pem login MY_SPARK_CLUSTER --region=YOUR_REGION
</code></pre>
