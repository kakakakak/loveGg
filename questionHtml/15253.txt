<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Eclipse does not allow us to create a Runnable Jar with the code with warnings. </p>

<p>For instance, </p>

<p>1) Unused imports(packages)
2) Deprecated warnings, etc.</p>

<p>Thus, make sure, you take care of all of these before creating the jar. Also, in the sample  WordCount code of hadoop, an instance of Job class is created. Which is deprecated. And thus, you will definitely face issues in that matter. Thus, to overcome that, please delete the line:
 <i>Delete the Job job = new Job (conf, "basic word count");</i></p>

<p>and include the line below:
<i>Job job = Job.getInstance();</i></p>

<p>Export does not get stopped if you had compile warnings. even if you have errors other class files will get packaged as its an independent ant script</p>

<p>Sample project with saved generated files <a href="http://sel2in.com/pages/prog/java/Run1.zip%20source%20files" rel="nofollow">Run sample</a></p>

<p>Bld.xml made by IDE export (eclipse) </p>

<pre><code>&lt;?xml version="1.0" encoding="UTF-8" standalone="no"?&gt;
&lt;project default="create_run_jar" name="Create Runnable Jar for Project Run1 with libraries in sub-folder"&gt;
    &lt;!--this file was created by Eclipse Runnable JAR Export Wizard--&gt;
    &lt;!--ANT 1.7 is required                                        --&gt;
    &lt;target name="create_run_jar"&gt;
        &lt;jar destfile="D:/U/t/workspace/1/Run1/o/RunApp.jar"&gt;
            &lt;manifest&gt;
                &lt;attribute name="Main-Class" value="m.TT"/&gt;
                &lt;attribute name="Class-Path" value="."/&gt;
            &lt;/manifest&gt;
            &lt;fileset dir="D:/U/t/workspace/1/Run1/bin"/&gt;
        &lt;/jar&gt;
        &lt;delete dir="D:/U/t/workspace/1/Run1/o/RunApp_lib"/&gt;
        &lt;mkdir dir="D:/U/t/workspace/1/Run1/o/RunApp_lib"/&gt;
    &lt;/target&gt;
&lt;/project&gt;
