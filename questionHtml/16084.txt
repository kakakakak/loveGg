<p>I have installed Apache Samza, that uses Yarn to manage the jobs. It is running on two Debian servers on virtual machines. Samza is version 0.9.1. Hadoop is version 2.6.0. I am seeing two different problems that I am not sure if they are related, but both look like Yarn is not doing what it should.</p>

<ul>
<li>When I try to kill the jobs with the script provided by samza (kill-yarn-job.sh) I see in the web interface that jobs change the state from Running or Accepted to Killed but the java process is still running. After long time, the only way to kill them is to do it the hard way: kill -9 .</li>
<li>Although I have been changing yarn-site.xml values I cannot get more than a single job running.
My machines have 4 Gb ram and 4 cpu cores. This is the content of </li>
</ul>

<p>yarn-site.xml:</p>

<pre><code>&lt;configuration&gt;
&lt;property&gt;
 &lt;name&gt;yarn.resourcemanager.hostname&lt;/name&gt;
 &lt;value&gt;kfk-samza01&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
&lt;name&gt;yarn.nodemanager.resource.memory-mb&lt;/name&gt;
&lt;value&gt;2048&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
&lt;name&gt;yarn.scheduler.minimum-allocation-mb&lt;/name&gt;
&lt;value&gt;128&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
&lt;name&gt;yarn.nodemanager.resource.cpu-vcores&lt;/name&gt;
&lt;value&gt;3&lt;/value&gt;
&lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>In the job options file I have configured added the following:</p>

<pre><code>yarn.container.memory.mb=256
yarn.am.container.memory.mb=256

task.opts= -Xms128M -Xmx128M
</code></pre>

<p>When the jobs are running I can see that -Xms128M -Xmx128M options are ignored and go with default values.</p>

<p>I have seen the following error. It looks like some memory limit is preventing jobs going from Accepted to Running, but I can not find how to solve it.</p>

<pre><code>Container [pid=23007,containerID=container_1443454508386_0003_01_000001] is running beyond virtual memory limits. Current usage: 13.9 MB of 256 MB physical memory used; 1.1 GB of 537.6 MB virtual memory used. Killing container
</code></pre>

<p>Actually jobs are just clean functions, so none of my code should be introducing noise.</p>

<p>Any idea of what is the problem?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Please try the below job configuration properties to limit the container memory allocation.</p>

<pre><code>mapreduce.map.memory.mb
mapreduce.reduce.memory.mb
</code></pre>

<p>These two properties values can be <code>256MB</code> as per your case.</p>

<p>and also configure the below two properties,</p>

<pre><code>mapreduce.map.java.opts
mapreduce.reduce.java.opts
</code></pre>

<p>The value for these 2 properties shall be <code>128MB</code> as per your case.</p>

<p>[Note: the above two <code>*.java.opts</code> value must be little lower than respective <code>*.memory.mb</code> properties]</p>

<p>If you still continue to get the virtual memory issue then try reducing the ration value of the virtual memory allocation by configuring the below property.</p>

<pre><code>yarn.nodemanager.vmem-pmem-ratio
</code></pre>

<p>Default is <code>2.1</code>, try reducing it if you still get the virtual memory issue.</p>

<p>Once these properties are correctly set then you will get the containers cleared upon successful completion.</p>

<p>Hope this helps.</p>
