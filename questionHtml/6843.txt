<p>It looks like a silly problem, but I can´t find a way to filter null values from my rows. 
This is the result when I dump the object geoinfo:</p>

<blockquote>
  <p>DUMP geoinfo;
  <br/>([longitude#70.95853,latitude#30.9773])
  <br/>([longitude#-9.37944507,latitude#38.91780853])
  <br/>(null)
  <br/>(null)
  <br/>(null)
  <br/>([longitude#-92.64416,latitude#16.73326])
  <br/>(null)
  <br/>(null)
  <br/>([longitude#-9.15199849,latitude#38.71179122])
  <br/>([longitude#-9.15210796,latitude#38.71195131])</p>
</blockquote>

<p>here is the description</p>

<blockquote>
  <p>DESCRIBE geoinfo;
  <br/>geoinfo: {geoLocation: bytearray}</p>
</blockquote>

<p>What I'm trying to do is to filter null values like this:</p>

<blockquote>
  <p>geoinfo_no_nulls = FILTER geoinfo BY geoLocation is not null;</p>
</blockquote>

<p>but the result remains the same. nothing is filtered.</p>

<p>I also tried something like this</p>

<blockquote>
  <p>geoinfo_no_nulls = FILTER geoinfo BY geoLocation != 'null';</p>
</blockquote>

<p>and I got an error</p>

<blockquote>
  <p>org.apache.pig.backend.executionengine.ExecException: ERROR 1071: Cannot convert a map to a String</p>
</blockquote>

<p>What am I doing wrong?</p>

<p>details, running on ubuntu, hadoop-1.0.3 with pig 0.9.3</p>

<p>pig -version

compiled Oct 24 2012, 19:04:03</p>

    <div class="post-text" itemprop="text">
<p>Thank you for your answers guys. 
It help me to find the path.</p>

<p>In the end the problem seams to be with the JsonLoader I was using.
I don't know why exactly, but it seams to have a bug with null strings.</p>

<p>I finally changed my code to use <a href="https://github.com/kevinweil/elephant-bird" rel="nofollow">https://github.com/kevinweil/elephant-bird</a>.</p>

<p>the code now looks like this:</p>

<pre><code>    register 'elephant-bird-core-3.0.0.jar'
    register 'elephant-bird-pig-3.0.0.jar'
    register 'google-collections-1.0.jar'
    register 'json-simple-1.1.jar'

    json_lines = LOAD '/twitterecho/tweets/stream/v1/json/2012_10_10/08' USING com.twitter.elephantbird.pig.load.JsonLoader();

    geo_tweets = FOREACH json_lines GENERATE (CHARARRAY) $0#'id' AS id, (CHARARRAY) $0#'geoLocation' AS geoLocation;

    tweets_grp = GROUP geo_tweets BY id;
    unique_tweets = FOREACH tweets_grp {
          first_tweet = LIMIT inpt 1;
          GENERATE FLATTEN(first_tweet);
    };

    only_not_nulls = FILTER geo_tweets BY geoLocation is not null;
    store only_not_nulls into '/twitter_data/results/geo_tweets';
</code></pre>

<p>cheers</p>
