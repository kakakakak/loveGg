    <div class="post-text" itemprop="text">

<p>Is it possible to write map/reduce jobs for Amazon Elastic MapReduce (<a href="http://aws.amazon.com/elasticmapreduce/">http://aws.amazon.com/elasticmapreduce/</a>) using .NET languages? In particular I would like to use C#.</p>

<p>Preliminary research suggests not. The above URL's marketing text suggests you have a "choice of Java, Ruby, Perl, Python, PHP, R, or C++", without mentioning .NET languages. This Amazon thread (<a href="http://developer.amazonwebservices.com/connect/thread.jspa?messageID=136051">http://developer.amazonwebservices.com/connect/thread.jspa?messageID=136051</a> -- "Support for C# / F# map/reducers") explicitly says that "currently Amazon Elastic MapReduce does not support Mono platform or languages such as C# or F#."</p>

<p>The above suggests that it can't be done. I'm wondering if there are any workarounds, though. For example, can I modify the Elastic MapReduce machine image for my account, and install Mono on there?</p>

<p>An alternative, suggested by Amazon FAQs "Using Other Software Required by Your Jar" (<a href="http://docs.amazonwebservices.com/ElasticMapReduce/latest/DeveloperGuide/index.html?CHAP_AdvancedTopics.html">http://docs.amazonwebservices.com/ElasticMapReduce/latest/DeveloperGuide/index.html?CHAP_AdvancedTopics.html</a>) and "How to Use Additional Files and Libraries With the Mapper or Reducer" (<a href="http://docs.amazonwebservices.com/ElasticMapReduce/latest/DeveloperGuide/index.html?addl_files.html">http://docs.amazonwebservices.com/ElasticMapReduce/latest/DeveloperGuide/index.html?addl_files.html</a>), is to make the first step of the Map/Reduce job be to install Mono on the local instance. That sounds kind of inefficient, but maybe it could work?</p>

<p>Maybe a saner alternative would be to try to forgo the convenience of Elastic MapReduce, and manually set up my own Hadoop cluster on EC2. Then I assume I could install Mono without difficulty.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Elastic MapReduce now has a "bootstrap actions" feature, which Amazon currently explains as follows:</p>

<blockquote>
  <p>A bootstrap action is a mechanism that allows you to run a script on Elastic MapReduce instances
  prior to Hadoop starting. Bootstrap action scripts are stored in Amazon S3 and passed to Amazon
  Elastic MapReduce when creating a new job flow. Bootstrap action scripts are downloaded from
  Amazon S3 and executed on each instance before the job flow is executed.</p>
  
  <p>Bootstrap action scripts can be written in any language already installed on the job flow
  instance, including Ruby, Python, Perl, and bash.</p>
</blockquote>

<p>(See <a href="http://docs.amazonwebservices.com/ElasticMapReduce/latest/DeveloperGuide/index.html?introduction.html" rel="nofollow">http://docs.amazonwebservices.com/ElasticMapReduce/latest/DeveloperGuide/index.html?introduction.html</a>)</p>

<p>One suggested use of this is to install software on your cluster machines. You could potentially use this to install a .NET runtime environment (probably Mono rather than Microsoft's, because because I imagine all the Elastic MapReduce machines are running Linux) on your cluster machines. (Not sure how hard the unattended install would be. Any ideas?) Having done so, you can call out to your .NET mappers/reducers using Hadoop streaming, which Elastic MapReduce does seem to support.</p>
