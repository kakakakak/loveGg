    <div class="post-text" itemprop="text">

<p>Here is my problem: I have a file in HDFS which can potentially be huge (=not enough to fit all in memory)</p>

<p>What I would like to do is avoid having to cache this file in memory, and only process it line by line like I would do with a regular file:</p>

<pre><code>for line in open("myfile", "r"):
    # do some processing
</code></pre>

<p>I am looking to see if there is an easy way to get this done right without using external libraries. I can probably make it work with <a href="http://code.google.com/p/libpyhdfs/">libpyhdfs</a> or <a href="https://github.com/traviscrawford/python-hdfs">python-hdfs</a> but I'd like if possible to avoid introducing new dependencies and untested libs in the system, especially since both of these don't seem heavily maintained and state that they shouldn't be used in production.</p>

<p>I was thinking to do this using the standard "hadoop" command line tools using the Python <code>subprocess</code> module, but I can't seem to be able to do what I need since there is no command line tools that would do my processing and I would like to execute a Python function for every linein a streaming fashion.</p>

<p>Is there a way to apply Python functions as right operands of the pipes using the subprocess module? Or even better, open it like a file as a generator so I could process each line easily?</p>

<pre><code>cat = subprocess.Popen(["hadoop", "fs", "-cat", "/path/to/myfile"], stdout=subprocess.PIPE)
</code></pre>

<p>If there is another way to achieve what I described above without using  an external library, I'm also pretty open.</p>

<p>Thanks for any help !</p>

<p>You want <a href="http://docs.python.org/release/2.3/lib/module-xreadlines.html">xreadlines</a>, it reads lines from a file without loading the whole file into memory.</p>

<p><strong>Edit</strong>:</p>

<p>Now I see your question, you just need to get the stdout pipe from your <code>Popen</code> object:</p>

<pre><code>cat = subprocess.Popen(["hadoop", "fs", "-cat", "/path/to/myfile"], stdout=subprocess.PIPE)
for line in cat.stdout:
    print line
</code></pre>

    <div class="post-text" itemprop="text">
<p>If you want to avoid adding external dependencies at any cost, Keith's answer is the way to go.  <a href="http://pydoop.sourceforge.net">Pydoop</a>, on the other hand, could make your life much easier:</p>

<pre class="lang-py prettyprint-override"><code>import pydoop.hdfs as hdfs
with hdfs.open('/user/myuser/filename') as f:
    for line in f:
        do_something(line)
</code></pre>

<p>Regarding your concerns, Pydoop is actively developed and has been used in production for years at <a href="http://www.crs4.it">CRS4</a>, mostly for computational biology applications.</p>
