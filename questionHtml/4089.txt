<p>I need to write a program that will write to a file the difference between two files.
The program has to loop through a 600 MB file with over 13.464.448 lines, check if a grep returns true on another file and then write the result onto another file.
I wrote a quick test with about 1.000.000 records and it took over an hour, so i'm guessing this approach could take 9+ hours.</p>

<p>Do you have any recommendations on how to make this faster? Any particular language i should use? I was planning on doing it in bash or python.</p>

<p>Thanks a lot in advance.</p>

<p><strong>[EDIT 1]:</strong> Sorry, when i say difference between two files i did not mean a diff. The result file is in a different format.</p>

<p>The logic is a bit like this:</p>

<p>File A has 297.599 lines
File B has over 13 million lines</p>

<p>I pick the current line being read from FILE A, grep it on FILE B, and if the line is present in File B i will write it to the result file. By the way, File A and File B have different formats. Result file will have the format of File A.</p>

<p><strong>[EDIT 2]:</strong> I was asked at work to create a bash solution ideally so that we don't have to install python on all the machines this has to run on.</p>

<p>This is my curent implementation:</p>

<pre><code>#!/bin/bash

LAST_TTP=`ls -ltr TTP_*.txt | tail -1 | awk '{ print $9 }'`
LAST_EXP=`ls -ltr *.SSMT | tail -1 | awk '{ print $9 }'`

while read -r line; do
   MATCH="$(grep $line $LAST_EXP)"
   echo "line: $line, match: $MATCH"

   # if not empty
   if [ ! -z "$MATCH" ]
   then
      echo $MATCH &gt;&gt; result
   fi

done &lt; $LAST_TTP
</code></pre>

<p>This bash approach is taking over 10 hours to complete. Do you have any suggestions on how to make it more efficient in bash?</p>

<p>Thanks a lot in advance!</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You're probably looking in a list instead of a set, leading to an O(n²) performance. Try:</p>

<pre><code>with open('b') as b:
  blines = set(b)
with open('a') as a:
  with open('result', 'w') as result:
    for line in a:
      if line not in blines:
        result.write(line)
</code></pre>

<p>Assuming uniformly long (and not overly long lines), the performance of this implementation is in <code>O(|A| + |B|)</code> (amortized, due to <a href="http://wiki.python.org/moin/TimeComplexity#set" rel="nofollow">Pyton's <code>set</code> being extremely fast</a>). The memory demand is in <code>O(|B|)</code>, but with a factor significantly greater than 1.</p>

<p>If the order of lines in the output does not matter, you can also sort both files and then compare them line-by-line. This will have a performance in the order of <code>O(|A| log |A| + B log |B|)</code>. The memory demand will be in <code>O(|A|+|B|)</code>, or more precisely, <code>|A|</code> + <code>|B|</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Sort each of the input files. Now read one line from each. If one line compares less than the other, output it as a difference and read the next line from that file. If both lines compare equal, read the next line from both files. If you read to the end of one file, all the lines in the other file are differences.</p>

<p>This is an O(n log n) algorithm as opposed to the O(n^2) algorithm you started with.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The join command also does what you want. join requires both files to be sorted up front. The option -v prints a line for each unpairable  line.</p>

<p>So you will want something like</p>

<p>join -v 1 sortedfile1 sortedfile2</p>

<p>(you will need to set the join options according to your file format, see manpage of join)</p>

<p>The below example joins the files test1.txt and test2.txt using the second resp. first field for the join. Assuming the files were sorted upfront using the sort command.
The -v 1 option only outputs the lines of test1.txt could not be joined.</p>

<pre>
>cat test1.txt
a 1 2
b 2 3

> cat test2.txt
1 x
4 x

> join -v 1  -1 2  -2 1  test1.txt test2.txt
2 b 3

> join -v 1 -1 2 -2 1 -o 1.1 1.2 1.3 test1.txt test2.txt
b 2 3
</pre>

<p>sort and join both work fine with large files. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can speed up your script slightly by stopping the <code>grep</code> when it finds the first match if that's appropriate for your needs.</p>

<p>If your <code>grep</code> supports it use <code>grep -m 1</code>.</p>

<p>Your problem is that you're spawning <code>grep</code> almost 300,000 times and it's reading over 13,000,000 lines each time.</p>

<p>Stopping <code>grep</code> at the first match will help a little, but the overhead of all those execs is a big factor as well. An implementation in Python will eliminate this issue.</p>

<p>As for choosing the files in your script, please see <a href="http://mywiki.wooledge.org/BashFAQ/003" rel="nofollow">BashFAQ/003</a> and <a href="http://mywiki.wooledge.org/ParsingLs" rel="nofollow">Parsing <code>ls</code></a>.</p>
