    <div class="post-text" itemprop="text">

<p>I am trying to write via JDBC:</p>

<pre><code>df.write.jdbc("jdbc:postgresql://123.123.123.123:5432/myDatabase", "myTable", props)
</code></pre>

<p>The <a href="https://spark.apache.org/docs/1.5.1/configuration.html" rel="nofollow">Spark docs</a> explain that the configuration option <code>spark.driver.extraClassPath</code> cannot be used to add JDBC Driver JARs if running in client mode (which is the mode Dataproc runs in) since the JVM has already been started.</p>

<p>I tried adding the JAR path in Dataproc's submit command:</p>

<pre><code>gcloud beta dataproc jobs submit spark ... 
     --jars file:///home/bryan/org.postgresql.postgresql-9.4-1203-jdbc41.jar
</code></pre>

<p>I also added the command to load the driver:</p>

<pre><code>  Class.forName("org.postgresql.Driver")
</code></pre>

<p>But I still get the error:</p>

<pre><code>java.sql.SQLException: No suitable driver found for jdbc:postgresql://123.123.123.123:5432/myDatabase 
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You may want to try adding <code>--driver-class-path</code> to the very end of your command arguments:</p>

<pre><code>gcloud beta dataproc jobs submit spark ... 
    --jars file:///home/bryan/org.postgresql.postgresql-9.4-1203-jdbc41.jar \
    --driver-class-path /home/bryan/org.postgresql.postgresql-9.4-1203-jdbc41.jar
</code></pre>

<p>Another approach if you're staging the jarfile onto the cluster before the job anyway is to dump the jarfile you need into <code>/usr/lib/hadoop/lib/</code> where it should automatically be part of the driver classpath for both Hadoop and Spark jobs.</p>
