    <div class="post-text" itemprop="text">

<p>I'm running hadoop in a single-machine, local-only setup, and I'm looking for a nice, painless way to debug mappers and reducers in eclipse.  <em>Eclipse</em> has no problem running mapreduce tasks.  However, when I go to debug, it gives me this error :</p>

<blockquote>
  <p>12/03/28 14:03:23 WARN mapred.JobClient: No job jar file set.  User classes may not be found. See JobConf(Class) or JobConf#setJar(String).</p>
</blockquote>

<p>Okay, so I do some research.  Apparently, I should use eclipse's remote debugging facility, and add this to my <code>hadoop-env.sh</code> :</p>

<pre><code>-agentlib:jdwp=transport=dt_socket,server=y,suspend=y,address=5000
</code></pre>

<p>I do that and I can step through my code in eclipse.  Only problem is that, because of the "suspend=y", I can't use the "hadoop" command from the command line to do things like look at the job queue; it hangs, I'm imagining because it's waiting for a debugger to attach.  Also, I can't run "hbase shell" when I'm in this mode, probably for the same reason.</p>

<p>So basically, if I want to <strong>flip back and forth between "debug mode" and "normal mode"</strong>, I need to update <code>hadoop-env.sh</code> and restart my machine.  Major pain.  So I have a few questions :</p>

<ol>
<li><p>Is there an easier way to do debug mapreduce jobs in eclipse?</p></li>
<li><p>How come eclipse can run my mapreduce jobs just fine, but for debugging I need to use remote debugging?</p></li>
<li><p>Is there a way to tell hadoop to use remote debugging for mapreduce jobs, but to operate in normal mode for all other tasks?  (such as <em>"hadoop queue"</em> or <em>"hbase shell"</em>).</p></li>
<li><p>Is there an easier way to switch <code>hadoop-env.sh</code> configurations without rebooting my machine?  hadoop-env.sh is not executable by default.</p></li>
<li><p>This is a more general question : what exactly is happening when I run hadoop in local-only mode?  Are there any processes on my machine that are "always on" and executing hadoop jobs?  Or does hadoop only do things when I run the "hadoop" command from the command line?  What is eclipse doing when I run a mapreduce job from eclipse?  I had to reference <code>hadoop-core</code> in my <code>pom.xml</code> in order to make my project work.  Is eclipse submitting jobs to my installed hadoop instance, or is it somehow running it all from the <code>hadoop-core-1.0.0.jar</code> in my maven cache?</p></li>
</ol>

<p>Here is my Main class :</p>

<pre><code>public class Main {
      public static void main(String[] args) throws Exception {     
        Job job = new Job();
        job.setJarByClass(Main.class);
        job.setJobName("FirstStage");

        FileInputFormat.addInputPath(job, new Path("/home/sangfroid/project/in"));
        FileOutputFormat.setOutputPath(job, new Path("/home/sangfroid/project/out"));

        job.setMapperClass(FirstStageMapper.class);
        job.setReducerClass(FirstStageReducer.class);

        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(Text.class);

        System.exit(job.waitForCompletion(true) ? 0 : 1);
      }
}
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The only way you can debug hadoop in eclipse is running hadoop in local mode. The reason being, each map reduce task run in ist own JVM and when you don't hadoop in local mode, eclipse won't be able to debug. </p>

<p>When you set hadoop to local mode, instead of using <em>hdfs API</em>(which is default), hadoop file system changes to <code>file:///</code>. Thus, running <code>hadoop fs -ls</code> will not be a hdfs command, but more of <code>hadoop fs -ls file:///</code>, a path to your local directory. None of the JobTracker or NameNode runs. </p>

<p>These blogposts might help:</p>

<ul>
<li><a href="http://let-them-c.blogspot.com/2011/07/running-hadoop-locally-on-eclipse.html" rel="nofollow">http://let-them-c.blogspot.com/2011/07/running-hadoop-locally-on-eclipse.html</a></li>
<li><a href="http://let-them-c.blogspot.com/2011/07/configurations-of-running-hadoop.html" rel="nofollow">http://let-them-c.blogspot.com/2011/07/configurations-of-running-hadoop.html</a></li>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Jumbune's debugger will do all these with minimal effort.</p>

<p>The debugger provides code level control flow statistics of the MapReduce job.</p>

<p>User may apply regex validations or its own user defined validation classes. As per validations applied, Flow Debugger checks the flow of  data for mapper and reducer respectively.</p>

<p>It also provides a comprehensive table/chart view where the flow of input records is displayed at job level, MR level, and instance level.
Unmatched keys/values represent the number of erroneous key/value data in job execution result. Debugger drills down into the code to examine the flow of data for various counters like loops and conditions if, else-if, etc.</p>

<p>Jumbune is open source and available at www.jumbune.org and <a href="https://github.com/impetus-opensource/jumbune" rel="nofollow">https://github.com/impetus-opensource/jumbune</a></p>

    <div class="post-text" itemprop="text">
<p>I also like to debug via unit test w/MRUnit. I will use this in combination with approvaltests which creates an easy visualization of the Map Reduce process, and makes it easy to pass in scenarios that are failing. It also runs seamlessly from eclipse.</p>

<p>For example:</p>

<pre><code>HadoopApprovals.verifyMapReduce(new WordCountMapper(), 
                         new WordCountReducer(), 0, "cat cat dog");
</code></pre>

<p>Will produce the output:</p>

<pre><code>[cat cat dog] 
-&gt; maps via WordCountMapper to -&gt;
(cat, 1) 
(cat, 1) 
(dog, 1)

-&gt; reduces via WordCountReducer to -&gt;
(cat, 2) 
(dog, 1)
</code></pre>

<p>There's a video on the process here: <a href="http://t.co/leExFVrf">http://t.co/leExFVrf</a></p>
