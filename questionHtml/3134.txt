    <div class="post-text" itemprop="text">

<p>I am using Pig (0.9.1) with UDFs written in Python. The Python scripts import modules from the standard Python library. I have been able to run the Pig scrips that call the Python UDFs successfully in local mode, but when I run on the cluster it appears Pig's generated Hadoop job is unable to find the imported modules. What needs to be done?</p>

<p>For example:</p>

<ul>
<li>Does python (or jython) need to be installed on each task tracker node?</li>
<li>Do the python (or jython) modules need to be installed on each task tracker node?</li>
<li>Do the task tracker nodes need to know how to find the modules?</li>
<li>If so, how do you specify the path (via an environment variable - how is that done for the task tracker)?</li>
</ul>

    <div class="post-text" itemprop="text">
<blockquote>
  <p>Does python (or jython) need to be installed on each task tracker
  node?</p>
</blockquote>

<p>Yes, since it's executed in task trackers.</p>

<blockquote>
  <p>Do the python (or jython) modules need to be installed on each task
  tracker node?</p>
</blockquote>

<p>If you are using a 3rd party module, it should be installed in task trackers as well (like geoip, etc).</p>

<blockquote>
  <p>Do the task tracker nodes need to know how to find the modules?
  If so, how do you specify the path (via an environment variable - how
  is that done for the task tracker)?</p>
</blockquote>

<p>As an answer from the book "<a href="http://ofps.oreilly.com/titles/9781449302641/intro_pig_latin.html#udfs">Programming Pig</a>" : </p>

<blockquote>
  <p>register is also used to locate resources for Python UDFs that you use
  in your Pig Latin scripts. In this case you do not register a jar, but
  rather a Python script that contains your UDF. The Python script must
  be in your current directory.</p>
</blockquote>

<p>And also this one is important :</p>

<blockquote>
  <p>A caveat, Pig does not trace dependencies inside your Python scripts
  and send the needed Python modules to your Hadoop cluster. You are
  required to make sure the modules you need reside on the task nodes in
  your cluster and that the PYTHONPATH environment variable is set on
  those nodes such that your UDFs will be able to find them for import.
  This issue has been fixed after 0.9, but as of this writing not yet
  released.</p>
</blockquote>

<p>And if you are using jython :</p>

<blockquote>
  <p>Pig does not know where on your system the Jython interpreter is, so
  you must include jython.jar in your classpath when invoking Pig. This
  can be done by setting the PIG_CLASSPATH environment variable.</p>
</blockquote>

<p>As a summary, if you are using streaming then you can use "SHIP" command in pig which would send your executable files to cluster. if you are using UDF, as long as it can be compiled(check the note about jython) and doesn't have 3rd party dependency in it (which you didn't already put in PYTHONPATH / or installed in cluster), the UDF would be shipped to cluster when executed. (As a tip, it would make your life much more easier if you put your simple UDF dependencies in the same folder with pig script when registering)</p>

<p>Hope these would clear things.</p>
