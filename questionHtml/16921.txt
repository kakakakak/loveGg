<p>How to setup Cloudera Hadoop Cluster on EC2 - S3 or EBS Instances?

When I start creating Hadoop Cluster from Cloudera Manager it starts creating new EC2 Instances as per the number of node you specify. It will request the Instance but the issue is it generates "Instance Store" Instances.</p>

<p>How can I provide my already existing instances which has EBS or S3 storage?</p>

<p>Any Idea?</p>

    <div class="post-text" itemprop="text">
<p>This is by design:</p>

<blockquote>
  <p>Why does Cloudera Manager prefer instance store-backed over EBS-backed
  AMIs? Although EBS volumes offer persistent storage, they are
  network-attached and charge per I/O request, so they are not suitable
  for Hadoop deployments. If you wish to experiment with EBS-backed
  instances, you can always use a custom EBS AMI.</p>
</blockquote>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Hadoop provide way to read and write data on S3 and EBS both in case of EBS you just need to mount the disk on your instance and use it in hadoop.</p>

<p>But in case of S3 there will be some extra step to provide <strong>Access key and Secrate key</strong> to hadoop by using property <strong>fs.s3n.awsAccessKeyId and fs.s3n.awsSecretAccessKey</strong> in core-site.xml and hdfs-site.xml as mention below</p>

<pre><code>  &lt;property&gt;
    &lt;name&gt;fs.s3n.awsAccessKeyId&lt;/name&gt;
    &lt;value&gt;Access Key&lt;/value&gt;
  &lt;/property&gt;
  &lt;property&gt;
    &lt;name&gt;fs.s3n.awsSecretAccessKey&lt;/name&gt;
    &lt;value&gt;Secret key&lt;/value&gt;
  &lt;/property&gt;
</code></pre>

<p>Then after setting this one can do following:</p>

<pre><code>hadoop fs -ls s3n://Bucket_Name/
</code></pre>

<p>And this will print the available folders in your S3 Bucket.</p>
