<td class="answercell">
    <div class="post-text" itemprop="text">
<p><a href="https://github.com/shinyplasticbag/openkeyval" rel="nofollow">OpenKeyval is open-source</a>, so you can host it yourself internally and add whatever access limitations you like.</p>

<p><a href="https://couchdb.apache.org/" rel="nofollow">CouchDB</a> is basically OpenKeyval on steroids. It provides a RESTful JSON API than can be accessed from any environment that allows HTTP requests (i.e., JavaScript) but you'll have to host it yourself since there aren't any public-facing CouchDB servers (that I know of).</p>

<p>And since you're looking for a list:</p>

<ul>
<li><a href="https://aws.amazon.com/simpledb/" rel="nofollow">Amazon SimpleDB</a></li>
<li><a href="http://hypertable.org/" rel="nofollow">Hypertable</a></li>
<li><a href="https://cassandra.apache.org/" rel="nofollow">Cassandra</a></li>
<li><a href="https://hadoop.apache.org/" rel="nofollow">Hadoop</a></li>
<li><a href="https://github.com/tuulos/ringo" rel="nofollow">Ringo</a></li>
<li><a href="https://code.google.com/p/thrudb/" rel="nofollow">ThruDB</a></li>
<li><a href="https://couchdb.apache.org/" rel="nofollow">CouchDB</a></li>
<li><a href="https://hbase.apache.org/" rel="nofollow">HBase</a></li>
</ul>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><a href="https://www.stackmob.com" rel="nofollow">StackMob</a> or <a href="https://parse.com" rel="nofollow">Parse</a> if you want a (client-side) JavaScript API with user management, facebook/twitter integration, data store (with geospatial), and push notifications.</p>

<p>StackMob also lets you host your website.</p>

<p>For more flexibility, less service lock-in, and cheaper scalability: I would suggest <a href="http://couchdb.apache.org" rel="nofollow">CouchDB</a> (though you would likely still use a hosting service like <a href="https://cloudant.com" rel="nofollow">Cloudant</a>). CouchDB can host your website, and provides a HTTP API for storing data, to which your client-side JavaScript can make REST calls.</p>
