<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I'm not entirely clear what your problem / error is, but if you want the contents of stopwords-flat.zip to be available in the current working directory at runtime use the <code>-archives</code> flag rather than <code>-files</code> (which could be your problem as you're using <code>-file</code>).</p>

<p>Hadoop will unpack the named archive file (zip), and the contents will be available as if they were in the local directory of your running mapper:</p>

<pre><code>bin/hadoop jar /home/../streaming/hadoop-0.21.0-streaming.jar \
  -input  /user/root/input/voa.txt 
  -output /user/root/output \
  -mapper /home/../mapper.py \
  -reducer /home/../reducer.py \
  -files /home/../nltkandyaml.mod \
  -archives /home/../stopwords-flat.zip
</code></pre>

<pre><code>    zip -r [your-nltk-package-name/nltk] nltk.zip

    zip -r [your-yaml-package-name/lib/yaml] yaml.zip
</code></pre>

<p>then in your script, add:</p>

<pre><code>    importer = zipimport.zipimporter('nltk.zip')
    importer2=zipimport.zipimporter('yaml.zip')
    yaml = importer2.load_module('yaml')
    nltk = importer.load_module('nltk')
</code></pre>

<p>in your command, add:</p>

<pre><code>    -file [path-to-your-zip-file]
</code></pre>
