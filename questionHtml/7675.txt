<p>download osquery <a href="https://code.facebook.com/projects/658950180885092" rel="nofollow">https://code.facebook.com/projects/658950180885092</a></p>

<p>and install </p>

<p>issue this command osqueryi </p>

<p>when prompt appears use this sql command to see all running java processes and find the pids</p>

<p>SELECT name, path, pid FROM processes where name= "java";</p>

<p>you will get something that looks like this on a mac</p>

<pre><code>  +------+--------------------------------------------------------------------------+-------+
  | name | path                                                                     | pid   |
  +------+--------------------------------------------------------------------------+-------+
  | java | /Library/Java/JavaVirtualMachines/jdk1.8.0_45.jdk/Contents/Home/bin/java | 59446 |
  | java | /Library/Java/JavaVirtualMachines/jdk1.8.0_45.jdk/Contents/Home/bin/java | 59584 |
  | java | /Library/Java/JavaVirtualMachines/jdk1.8.0_45.jdk/Contents/Home/bin/java | 59676 |
  | java | /Library/Java/JavaVirtualMachines/jdk1.8.0_45.jdk/Contents/Home/bin/java | 59790 |
  +------+--------------------------------------------------------------------------+-------+
</code></pre>

<p>issue the sudo kill - PID command on all the processes to make sure that it kills the port in use 0.0.0.0:50070</p>

<p>then after all this retry the sbin/start-dfs.sh and the namenode should now show </p>
