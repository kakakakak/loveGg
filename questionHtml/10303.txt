<p>I have Spark and Hadoop installed on OS X. I successfully worked through an example where Hadoop ran locally, had files stored in HDFS and I ran spark with</p>

<pre class="lang-sh prettyprint-override"><code>spark-shell --master yarn-client
</code></pre>

<p>and from within the shell worked with HDFS. I'm having problems, however, trying to get Spark to run without HDFS, just locally on my machine. I looked at <a href="http://stackoverflow.com/questions/22129321/connection-refused-when-running-sparkpi-locally">this answer</a> but it doesn't feel right messing around with environment variables when the <a href="http://spark.apache.org/docs/latest/index.html" rel="nofollow">Spark documentation</a> says </p>

<blockquote>
  <p>It’s easy to run locally on one machine — all you need is to have java
  installed on your system PATH, or the JAVA_HOME environment variable
  pointing to a Java installation.</p>
</blockquote>

<p>If I run the basic <code>SparkPi</code> example I get the correct output.</p>

<p>If I try run the sample Java app <a href="http://spark.apache.org/docs/latest/quick-start.html#tab_java_3" rel="nofollow">they provide</a>, again, I get output, but this time with <strong>connection refused</strong> errors relating to <strong>port 9000</strong>, which sounds like it's trying to connect to <strong>Hadoop</strong>, but I don't know why because I'm not specifying that</p>

<h1>UPDATE</h1>

<p>It seems that the fact I have the environment variable <code>HADOOP_CONF_DIR</code> set is causing some issues. Under that directory, I have <code>core-site.xml</code> which contains the following</p>

<pre class="lang-xml prettyprint-override"><code>&lt;property&gt;
     &lt;name&gt;fs.default.name&lt;/name&gt;                                     
     &lt;value&gt;hdfs://localhost:9000&lt;/value&gt;                             
&lt;/property&gt; 
</code></pre>

<p>If I change the value e.g. <code>&lt;value&gt;hdfs://localhost:9100&lt;/value&gt;</code> then when I attempt to run the spark job, the connection refused error refers to this changed port</p>

<pre class="lang-sh prettyprint-override"><code>Exception in thread "main" java.net.ConnectException: Call From 37-2-37-10.tssg.org/10.37.2.37 to localhost:9100 failed on connection exception: java.net.ConnectException: Connection refused; For more details see:  http://wiki.apache.org/hadoop/ConnectionRefused 
</code></pre>

<p>So for some reason, despite instructing it to run locally, it is trying to connect to HDFS. If I remove the <code>HADOOP_CONF_DIR</code> environment variable, the job works fine. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Apache Spark uses the Hadoop client libraries for file access when you use <code>sc.textFile</code>. This makes it possible to use an <code>hdfs://</code> or <code>s3n://</code> path for example. You can also use local paths as <code>file:/home/robocode/foo.txt</code>.</p>

<p>If you specify a file name without a schema, <code>fs.default.name</code> is used. It defaults to <code>file:</code>, but you explicitly override it to <code>hdfs://localhost:9000</code> in your <code>core-site.xml</code>. So if you don't specify the schema, it's trying to read from HDFS.</p>

<p>The simplest solution is to specify the schema:</p>

<pre><code>JavaRDD&lt;String&gt; logData = sc.textFile("file:/home/robocode/foo.txt").cache();
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I think the environment variables that you have defined before for the Hadoop related example are still interfering with your test.</p>

<p>Given that you are using the official Spark Java example:</p>

<pre><code>public static void main(String[] args) {
    String logFile = "YOUR_SPARK_HOME/README.md"; // Should be some file on your system
    SparkConf conf = new SparkConf().setAppName("Simple Application");
    JavaSparkContext sc = new JavaSparkContext(conf);
    JavaRDD&lt;String&gt; logData = sc.textFile(logFile).cache();
...
}
</code></pre>

<p>I would suggest to explictly set the Master, as default values are taken otherwise.</p>

<pre><code>SparkConf conf = new SparkConf().setMaster("local").setAppName("Simple Application")
</code></pre>

<p>Check the <a href="http://spark.apache.org/docs/latest/api/scala/#org.apache.spark.SparkConf" rel="nofollow">SparkConf doc</a> for more information:</p>
