<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Actually this answer is not so simple, as you would expect.</p>

<p>I will make many assumptions, first that you use <code>sbt</code>, second is that you are working in a linux based computer, third is the last is that you have two <code>classes</code> in your project, let's say <code>RunMe</code> and <code>Globals</code>, and the last assumption will be that you want to set up the settings inside the program. Thus, somewhere in your runnable code you must have something like this:</p>

<pre><code>object RunMe {
  def main(args: Array[String]) {
    val conf = new SparkConf()
      .setMaster("mesos://master:5050") //If you use Mesos, and if your network resolves the hostname master to its IP.
      .setAppName("my-app")
      .set("spark.executor.memory", "10g")
    val sc = new SparkContext(conf)
    val sqlContext = new SQLContext()

    //your code comes here
  }
}
</code></pre>

<p>The steps you must follow are:</p>

<ul>
<li><p>Compile the project, in the root of it, by using: </p>

<p><code>$ sbt assembly</code></p></li>
<li><p>Send the job to the master node, this is the interesting part (assuming you have the next structure in your project <code>target/scala/</code>, and inside you have a file <code>.jar</code>, which corresponds to the compiled project)</p>

<p><code>$ spark-submit --class RunMe target/scala/app.jar</code> </p></li>
</ul>

<p>Notice that, because I assumed that the project has two or more classes you would have to identify which class you want to run. Furthermore, I bet that both approaches, for <code>Yarn</code> and <code>Mesos</code> are very similar. </p>
