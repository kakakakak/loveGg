<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Have you copied the jar containing the <strong>JDBC driver</strong> for your metadata db into Hive's lib dir? </p>

<p>For instance, if you're using <strong>MySQL</strong> to hold your metadata db, you wll need to copy </p>

<p><strong><code>mysql-connector-java-5.1.22-bin.jar</code></strong> into <strong><code>$HIVE_HOME/lib</code></strong>.</p>

<p>This fixed that same error for me.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>One possibility is that your hadoop is not running. I ran into this same problem when doing  a simple one node hadoop cluster on my local. Each time you start up your computer you have to manually start your hadoop nodes before hive can connect to them. </p>

<p>Try running $HADOOP_HOME/bin/stop-all.sh</p>

<p>If the output says there were no nodes to stop, that means your hadoop wasn't up, in this case, run</p>

<p>$HADOOP_HOME/bin/start-all.sh </p>

<p>and then hive and you should be good to go.</p>

<p>(this was using the default derby database. Apparently if you are using mysql then you can also get this error from some connector configuration (see other answer)).</p>

<p>EDIT:
Actually another thing I discovered is that Hive needs to create a metastore_db directory whenever you start hive, but it creates it in the path that hive was run from. You can encounter the above error if you don't have write permission in the directory that you run hive from. To fix this, there's a config you can set in hive-site.xml to make this directory absolute rather than relative.</p>

<p>See here:</p>

<p><a href="http://stackoverflow.com/questions/13624893/metastore-db-created-wherever-i-run-hive">metastore_db created wherever I run Hive</a></p>

<p>In general, this indicates that the Hive <strong>metastore</strong> service isn't started properly. But the error message you paste doesn't contain sufficient information to locate the exact cause of the problem.<br>
Indeed, there are quite a lot reasons which may lead to this error, as the answers posted by other authors each relate to a specific aspect.<br>
My suggestion is: run <code>hive --service metastore</code> command to get a more detailed and targeted error message, and then address the problem accordingly. You may refer to <em>/tmp/user_name/hive.log</em> log file, too.  </p>
