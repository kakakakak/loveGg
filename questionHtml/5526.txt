<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You need a High Availability configured ResourceManager. Read <a href="http://docs.hortonworks.com/HDPDocuments/HDP2/HDP-2.1.2/bk_system-admin-guide/content/ch_hadoop-ha-rm-3z.html" rel="nofollow">Deploy ResourceManager HA Cluster</a> for how to configure such a cluster. Then you'll be able to failover, manually or automatically, the RM.</p>

<p>This link explains more: <a href="http://hadoop.apache.org/docs/r2.4.1/hadoop-yarn/hadoop-yarn-site/ResourceManagerHA.html" rel="nofollow">ResourceManager High Availability</a></p>

<p>Actually, since 2.4.0 onward, is possible to restart RM and keep accepted applications (MR jobs) w/o a secondary HA RM. See <a href="http://hadoop.apache.org/docs/current/hadoop-yarn/hadoop-yarn-site/ResourceManagerRestart.html" rel="nofollow">ResourceManger Restart</a>:</p>

<blockquote>
  <p>ResourceManager is the central authority that manages resources and schedules applications running atop of YARN. Hence, it is potentially a single point of failure in a Apache YARN cluster.</p>
  
  <p>This document gives an overview of ResourceManager Restart, a feature that enhances ResourceManager to keep functioning across restarts and also makes ResourceManager down-time invisible to end-users.</p>
  
  <p>ResourceManager Restart feature is divided into two phases:</p>
  
  <ul>
  <li><p><strong>Phase 1</strong>: Enhance RM to persist application/attempt state and other credentials information in a pluggable state-store. RM will reload this information from state-store upon restart and re-kick the previously running applications. Users are not required to re-submit the applications.</p></li>
  <li><p><strong>Phase 2</strong>: Focus on re-constructing the running state of ResourceManger by reading back the container statuses from NodeMangers and container requests from ApplicationMasters upon restart. The key difference from phase 1 is that previously running applications will not be killed after RM restarts, and so applications won't lose its work because of RM outage.</p></li>
  </ul>
  
  <p>As of Hadoop 2.4.0 release, only ResourceManager Restart Phase 1 is implemented which is described below.</p>
</blockquote>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Simple answer: <strong>No</strong>! </p>

<p>When you stop the resource manager, tasktrackers and datanodes cannot communicate with the master node, and hence with each other (as they don't know where to ask their input data from). Moreover, the nodes don't know where the data is stored. These are all information stored in the (master). While a job is running, all this information is needed in order to proceed, so stopping the resource manager, the running job will fail.</p>

<p><strong>EDIT</strong>: Apparently, things are not so simple anymore, as @Remus Rusanu's answer proved :)</p>
