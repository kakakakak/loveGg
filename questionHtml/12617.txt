<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It seems you are using a pretty old version of Cygnus. Packages such as <code>es.tid.fiware.orionconnectors.cosmosinjector...</code> were replaced some time ago.</p>

<p>Please, refer to the last version in the master branch at <a href="https://github.com/telefonicaid/fiware-cygnus" rel="nofollow">https://github.com/telefonicaid/fiware-cygnus</a> (it matches 0.8.1) or configure the FIWARE repo in order to get the RPM for 0.7.1 as described at <a href="https://github.com/telefonicaid/fiware-cygnus#section4.1" rel="nofollow">https://github.com/telefonicaid/fiware-cygnus#section4.1</a></p>

<p><b>EDIT 1</b></p>

<p>Please observe this log:</p>

<pre><code>msg=org.apache.flume.conf.FlumeConfiguration$AgentConfiguration[336] : Agent configuration for 'cygnusagent' does not contain any valid channels. Marking it as invalid.
</code></pre>

<p>It you have a look on your <code>agent_umanick.conf</code> file, you will see it is missing something like:</p>

<pre><code>cygnusagent.sinks.hdfs-channel.type = memory
cygnusagent.sinks.hdfs-channel.capacity = 10000
cygnusagent.sinks.hdfs-channel.transactionCapacity = 100
</code></pre>

<p>Check it at <a href="https://github.com/telefonicaid/fiware-cygnus/blob/master/README.md#section5.2" rel="nofollow">https://github.com/telefonicaid/fiware-cygnus/blob/master/README.md#section5.2</a></p>

<p><b>EDIT 2</b></p>

<p>Again, if you have a look on <a href="https://github.com/telefonicaid/fiware-cygnus/blob/master/README.md#section5.2" rel="nofollow">https://github.com/telefonicaid/fiware-cygnus/blob/master/README.md#section5.2</a> you will see you are missing some important configuration parameters in the source and the sink:</p>

<pre><code># Default service (service semantic depends on the persistence sink)
cygnusagent.sources.http-source.handler.default_service = def_serv
# Default service path (service path semantic depends on the persistence sink)
cygnusagent.sources.http-source.handler.default_service_path = def_servpath
# Source interceptors, do not change
cygnusagent.sources.http-source.interceptors = ts gi
# TimestampInterceptor, do not change
cygnusagent.sources.http-source.interceptors.ts.type = timestamp
# GroupingInterceptor, do not change
cygnusagent.sources.http-source.interceptors.gi.type = com.telefonica.iot.cygnus.interceptors.GroupingInterceptor$Builder
# Grouping rules for the GroupingIntercetor, put the right absolute path to the file if necessary
# See the doc/design/interceptors document for more details
cygnusagent.sources.http-source.interceptors.gi.grouping_rules_conf_file = /usr/cygnus/conf/grouping_rules.conf

# Hive FQDN/IP address of the Hive server
cygnusagent.sinks.hdfs-sink.hive_host = cosmos.lab.fi-ware.org
# Hive port for Hive external table provisioning
cygnusagent.sinks.hdfs-sink.hive_port = 10000
# Kerberos-based authentication enabling
cygnusagent.sinks.hdfs-sink.krb5_auth = false
</code></pre>

<p>Among them, it is very important <code>/usr/cygnus/conf/grouping_rules.conf</code> exists, even if it is empty. The software provides a template called <code>grouping_rules.conf.template</code>; please rename/copy it in order to have the appropriate file.</p>

<p><b>EDIT 3</b></p>

<p>Command to be executed in order to read the persisted data:</p>

<pre><code>hadoop fs -cat /user/ilko.garcia/def_serv/def_servpath/613cb7464a7950ba_umanick_technologies_s_l/613cb7464a7950ba_umanick_technologies_s_l.txt
</code></pre>
