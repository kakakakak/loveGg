    <div class="post-text" itemprop="text">

<p>I work on text mining and statistical modeling, mostly course and research projects at school. I have primarily been using the Windows GUI versions of R and Python. I will be done soon and I realize that going into the industry, most work is done on Unix/Linux machines. </p>

<p>I wanted to get some hands on experience working on Unix before I start looking for jobs (in about 6 months), especially at the command line. I wanted to ask you guys for two things - </p>

<p>a. Which unix/linux distribution would be most beneficial in getting familiar with. I understand that most of the knowledge will scale across distributions, but I still wanted to know which one would be the best to invest time on. </p>

<p>b. Is there any resource or book to help me pick up speed on working from the command line instead of a GUI as in Gnome or KDE. </p>

<p>I am not sure if it matters, but I also wanted to mention that alongside I also want to invest some time in learning the basics of Hadoop, Pig and Mahout. </p>

    <div class="post-text" itemprop="text">
<p>I use Ubuntu myself, but for your purposes, it doesn't matter too much which one you choose - as long as the chosen one doesn't eat up all your time learning UNIX itself - you want to focus on tools, not on system administration.</p>

<p><strong>Better to spend time learning an editor (vim/emacs), a scripting language (Python, Ruby), and mapreduce (Hadoop, Pig, and Mahout).</strong></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I wouldn't worry much about the specific Linux distribution you end up learning. It almost certainly won't be the same as what they use at your eventual employer. Instead, pick a distribution that your friends and fellow students use. If no one else is using Linux, then <a href="http://www.ubuntu.com/" rel="nofollow">Ubuntu</a> is a good place to start.</p>

<p>You should also consider learning Mac OS X. The differences aren't huge, but more and more developers prefer to use OS X as their desktop unix environment.</p>

<p>You should also take some time to learn the basics of SQL. At the very least, grab <a href="http://www.sqlite.org/" rel="nofollow">SQLite</a> so that you can make a database and run some queries. If you want to go deeper, give <a href="http://www.mysql.com/" rel="nofollow">MySQL</a> a try. Big statistical analysis projects often have SQL databases to manage the data sets. Even with medium sized projects, you may find it much easier to work with your data in a database than in flat files.</p>
