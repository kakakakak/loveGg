    <div class="post-text" itemprop="text">
<p>The owner of the folder (most likely a file in the folder) must have made it accessible to anonymous reader.
If that is the case, s3n://x/y... is translated to 
<a href="http://s3.amazonaws.com/x/y" rel="nofollow">http://s3.amazonaws.com/x/y</a>...
or 
<a href="http://x.s3.amazonaws.com/y" rel="nofollow">http://x.s3.amazonaws.com/y</a>...</p>

<p>x is the name of the bucket.
y... is the path wihtin the bucket.</p>

<p>If you want to make sure the file exists, e.g. if you suspect the name was misspelled, you can  in your browser to open
<a href="http://s3.amazonaws.com/x" rel="nofollow">http://s3.amazonaws.com/x</a>
and you'll see XML describing "files" that is S3 objects, available.
Try this:
<a href="http://s3.amazonaws.com/elasticmapreduce" rel="nofollow">http://s3.amazonaws.com/elasticmapreduce</a></p>

<p>I tried this, and seems that the path you want is not public.
AWS EBS documentation quotes s3://elasticmapreduce/samples/wordcount/input in one of the "getting started" examples. But s3 is different from s3n, so input might be available to EMR, but not to HTTP access.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The files in s3n://elasticmapreduce/samples/wordcount/input are public, and made available as input by Amazon to the sample word count Hadoop program. The best way to fetch them is to</p>

<ol>
<li>Start a new Amazon Elastic MapReduce Job Flow (it doesn't matter which one) from the Amazon Web Services console, and make sure that you keep the the job alive with the Keep Alive option</li>
<li>Once the EC2 machines have started, find the instances on EC2 from the Amazon Web Services console</li>
<li>ssh into one of the running EC2 instances, using the hadoop user, for example
<code>ssh -i keypair.pem hadoop@ec2-IPADDRESS.compute-1.amazonaws.com</code></li>
<li>Obtain the files you need, using <code>hadoop dfs -copyToLocal s3://elasticmapreduce/samples/wordcount/input/0002 .</code></li>
<li>sftp the files to your local system</li>
</ol>

<p>Then you can download them by specifying the whole name, e.g.
<a href="http://s3.amazonaws.com/elasticmapreduce/samples/wordcount/input/0001" rel="nofollow">s3.amazonaws.com/elasticmapreduce/samples/wordcount/input/0001</a></p>
