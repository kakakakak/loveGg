<p>Q: How can you evaluate a product if you can't try it out</p>

<p><strong>Welcome to the "cloud"</strong>  </p>

<p>Use an open-source Platform-as-a-service setup. This way, you can test things on your own machines and migrate to a host if you like it. I dare say, using a freely available language (Python, Java) and running in something (GAE) that resembles a normal API is better. Hadoop might even be an alternative, and use something like Cloudera.</p>

    <div class="post-text" itemprop="text">
<p>MSDN Premium account holders get a special deal that provides quite a bit for free, and unless you try to launch a commercial site with it, I think you're going to have plenty of headroom:</p>

<ul>
<li>3 1GB SQL Azure databases</li>
<li>7 GB in, 14 GB out monthly</li>
<li>750 hours (on a "small" compute instance)</li>
<li>1M access control transactions</li>
<li>5 service bus connections</li>
<li>10GB Azure storage + 1M storage transactions</li>
</ul>

<p>I'd say that's pretty generous.</p>
