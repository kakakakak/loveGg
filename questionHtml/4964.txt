<p>We have currently one running project which uses RDBMS database( with lots of tables and stored procedures for manipulating data). The current flow is like : the data access layer will call stored procedures, which will insert/delete/update or fetch data from RDBMS(please note that these stored procedures are not doing any bulk proccesses.). The current data structure contains lots of primary key, foreign key relation ship and have lots of updates to existing database tables.a I just want to know whether we can use HBase for our purpose? then how can we use Hadoop with HBase replacing RDBMS?</p>

    <div class="post-text" itemprop="text">
<p>You need to ask yourself, what is the RDBMS not doing for you, and what is it that you hope to achieve by moving to Hadoop/HBase?</p>

<p>This article may help. There are a lot more.
<a href="http://it.toolbox.com/blogs/madgreek/nosql-vs-rdbms-apples-and-oranges-37713" rel="nofollow">http://it.toolbox.com/blogs/madgreek/nosql-vs-rdbms-apples-and-oranges-37713</a></p>

<p>If the purpose is trying new technology, I suggest trying their tutorial/getting started.</p>

<p>If it's a clear problem you're trying to solve, then you may want to articulate the problem.</p>

<p>Good Luck!</p>

Consider that your organization probably has no employees with the needed experience for hbase. Moving to hbase with the attendant data conversion and application rewriting will be very expensive and risky.</p>
