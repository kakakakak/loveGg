<td class="answercell">
    <div class="post-text" itemprop="text">
<p>What fixed this for me was including a couple of extra settings in the <code>SparkConf</code>, which seem to make sure the workers get access to the PySpark and Py4J modules:</p>

<pre><code>conf = (SparkConf()
     .setMaster("yarn-client")
     .setAppName("HDFS Filter")
     .set("spark.executor.memory", "1g")
     .set('spark.yarn.dist.files','file:/usr/hdp/2.3.2.0-2950/spark/python/lib/pyspark.zip,file:/usr/hdp/2.3.2.0-2950/spark/python/lib/py4j-0.8.2.1-src.zip')
     .setExecutorEnv('PYTHONPATH','pyspark.zip:py4j-0.8.2.1-src.zip'))
</code></pre>

<p>You'll need to edit the paths as appropriate for your system.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I think you need to set the <code>PYSPARK_PYTHON</code> environment variable to point to whichever installation of python you're using. It seems you're not using <code>/usr/bin/python2.7</code> to launch the job.</p>

<p>I usually call this function before importing and running pyspark to make sure things are set correctly:</p>

<pre><code>def configure_spark(spark_home=None, pyspark_python=None):
    spark_home = spark_home or "/path/to/default/spark/home"
    os.environ['SPARK_HOME'] = spark_home

    # Add the PySpark directories to the Python path:
    sys.path.insert(1, os.path.join(spark_home, 'python'))
    sys.path.insert(1, os.path.join(spark_home, 'python', 'pyspark'))
    sys.path.insert(1, os.path.join(spark_home, 'python', 'build'))

    # If PySpark isn't specified, use currently running Python binary:
    pyspark_python = pyspark_python or sys.executable
    os.environ['PYSPARK_PYTHON'] = pyspark_python
</code></pre>
