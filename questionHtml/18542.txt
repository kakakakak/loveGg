<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The command means that a connection is conducted to the own machine, to the current user.</p>

<p>In general, that means a new login shell with (perhaps) new privileges, when the list of groups the user is member in has changed.</p>

<p>In the case given, this does not apply, as the user and groups are created beforehand.</p>

<p>Other reasons for <code>ssh localhost</code> may include</p>

<ul>
<li>the <a href="http://stackoverflow.com/a/21926362/296974">already mentioned</a> addition to <code>.ssh/known_hosts</code></li>
<li>a clean environment</li>
</ul>

<p>But be warned that after <code>ssh localhost</code> you are in a kind of "subshell" (it is not really a subshell, as we have a "thicker layer" in-between) so you have to <code>logout</code> / <code>exit</code> / <code>^D</code> twice.</p>

<p>For adding to <code>known_hosts</code> a mere <code>ssh localhost true</code> would have been enough; the connection is terminated immediately again.</p>
