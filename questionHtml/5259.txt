    <div class="post-text" itemprop="text">

<p>I have been working with installing Hadoop. I followed some instruction on a Udemy course, and I installed Hadoop on pseudo distributed mode, on my laptop. It was fairly straightforward. </p>

<p>After that, I started to wonder if I could set up Hadoop on a desktop computer. So went out and bought an empty case and put in a 64 bit, 8 core AMD processor, along with a 50GB SSD hard drive and 4 inexpensive 500GB hard drives. I installed Ubuntu 14.04 on the SSD drive, and put virtual machines on the other drives.</p>

<p>I'm envisioning using my SSD as the master and using my 4 hard drives as nodes. Again, everything is living in the same case.</p>

<p>Unfortunately, and I've been searching everywhere, and I can't find any tutorials, guides, books, etc, that describe setting up Hadoop in this manner. It seems like most everything I've found that details installation of Hadoop is either a simple pseudo distributed setup (which I've already done), or else the instructions jump straight to large scale commercial applications. I'm still learning the basics, clearly, but I'd like to play in this sort-of in between place.</p>

<p>Has anyone done this before, and/or come across any documentation / tutorials / etc that describe how to set Hadoop up in this way? Many thanks in advance for the help.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>if you done the hadoop installation in pseduo mode means then follow the below steps this may help you.</p>

<p>MULTINODE :</p>

<p>Configure the hosts in the network using the following settings in the host file. This has to be done in all machine [in namenode too].</p>

<h1>sudo vi /etc/hosts</h1>

<p>add the following lines in the file:</p>

<p>yourip1 master</p>

<p>yourip2 slave01</p>

<p>yourip3 slave02</p>

<p>yourip4 slave03</p>

<p>yourip5 slave04</p>

<p>[Save and exit – type ESC then :wq ]</p>

<p>Change the hostname for the namenode and datanodes.</p>

<h1>sudo vi /etc/hostname</h1>

<p>For master machine [namenode ] – master</p>

<p>For other machines – slave01 and slave02 and slave03 and slave04 and slave 05</p>

<p>Restart the machines in order to get the settings related to the network applied.</p>

<h1>sudo shutdown -r now</h1>

<p>Copy the keys from the master node to all datanodes, so as this will help to access the machines without asking for permissions everytime.</p>

<pre><code>    #ssh-copy-id –i ~/.ssh/id_rsa.pub hduser@slave01
    #ssh-copy-id –i ~/.ssh/id_rsa.pub hduser@slave02
    #ssh-copy-id –i  ~/.ssh/id_rsa.pub hduser@slave03
    #ssh-copy-id –i  ~/.ssh/id_rsa.pub hduser@slave04
</code></pre>

<p>Now we are about to configure the hadoop configuration settings, so navigate to the ‘conf’ folder.</p>

<h1>cd ~/hadoop/etc</h1>

<p>Edit the slaves file within the hadoop directory.</p>

<h1>vi ~/hadoop/conf/slaves</h1>

<p>And add the below :</p>

<pre><code>master

slave01

slave02

slave03

slave04
</code></pre>

<p>Now update localhost to master in core-site.xml,hdfs-site.xml,mapred-site.xml and yarn-site.xml</p>

<p>Now copy the files from the hadoop/etc/hadoop folder from master to slave machines.</p>

<p>then format you namenode in all machines.</p>

<p>and start the hadoop services.</p>

<p>I given you the some clues for how to configure the hadoop multinode cluster.</p>

<ul>
<li>You can <strong>run</strong> hadoop in different VM's which are located in different drives in the same system.</li>
<li>But you need to allocate <strong>same configurations</strong> for all the master and slave nodes</li>
<li>Also ensure that all the VM's having <strong>different ip</strong> addresses.</li>
<li>You can get different IP addresses by <strong>connecting</strong> your master computer to the <strong>LAN</strong> or you need to disable some functionality in VM machines in order to get different IP addresses.</li>
</ul>
