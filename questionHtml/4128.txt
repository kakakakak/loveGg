    <div class="post-text" itemprop="text">
<p>Ok, i found the solution myself.</p>

<p>similar to this (<a href="http://social.msdn.microsoft.com/Forums/de-DE/baf613fc-3b24-4f47-9800-38499965e9dc/vs2010-is-using-a-wrong-compiler-Version" rel="nofollow">http://social.msdn.microsoft.com/Forums/de-DE/baf613fc-3b24-4f47-9800-38499965e9dc/vs2010-is-using-a-wrong-compiler-Version</a>) question, my "Microsoft.Cpp.Win32.user" property sheet contained references to the vs2008 directories. i have no clue how they got there, but removing them solved my problem.</p>

<p>open <strong>View -> Other Windows -> Property Manager</strong></p>

<p>double click on <strong>Microsoft.Cpp.Win32.user</strong></p>

<p>go to <strong>VC++ Directories</strong></p>

<p>rows which have been changed are bold. on all bold rows, click inside the text field, then on the Dropdown menu select <strong>inherit from parent or Project Defaults</strong></p>

<p>I verified that it now uses the correct Compiler by compiling the following snippet:</p>

<pre><code>auto asd = std::make_unique&lt;int&gt;();
</code></pre>

<p>make_unique has been added with VS2013.</p>

<p>Anyone have a clue, what changed These values?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The difference between <code>libwinutils.vcxproj</code> and <code>winutils.vcxproj</code> shows the problem.</p>

<p><code>&lt;Import Project="$(VCTargetsPath)\Microsoft.Cpp.props" /&gt;</code> appears before <code>Release|Win32</code> PropertyGroup.</p>

<pre><code>&lt;Import Project="$(VCTargetsPath)\Microsoft.Cpp.props" /&gt;
&lt;PropertyGroup Condition="'$(Configuration)|$(Platform)'=='Release|Win32'" Label="Configuration"&gt;
  &lt;ConfigurationType&gt;Application&lt;/ConfigurationType&gt;
  &lt;UseDebugLibraries&gt;false&lt;/UseDebugLibraries&gt;
  &lt;WholeProgramOptimization&gt;true&lt;/WholeProgramOptimization&gt;
  &lt;CharacterSet&gt;Unicode&lt;/CharacterSet&gt;
  &lt;PlatformToolset&gt;v110&lt;/PlatformToolset&gt;
&lt;/PropertyGroup&gt;
</code></pre>

<p>Open <code>winutils.vcxproj</code>, and move <code>&lt;Import ...&gt;</code> after <code>Release|Win32</code> PropertyGroup, like the following quote:</p>

<pre><code>&lt;PropertyGroup Condition="'$(Configuration)|$(Platform)'=='Release|Win32'" Label="Configuration"&gt;
  &lt;ConfigurationType&gt;Application&lt;/ConfigurationType&gt;
  &lt;UseDebugLibraries&gt;false&lt;/UseDebugLibraries&gt;
  &lt;WholeProgramOptimization&gt;true&lt;/WholeProgramOptimization&gt;
  &lt;CharacterSet&gt;Unicode&lt;/CharacterSet&gt;
  &lt;PlatformToolset&gt;v110&lt;/PlatformToolset&gt;
&lt;/PropertyGroup&gt;
&lt;Import Project="$(VCTargetsPath)\Microsoft.Cpp.props" /&gt;
</code></pre>

<p>I have successfully built winutils of hadoop-2.7.0-src with VS2012 toolset.</p>
