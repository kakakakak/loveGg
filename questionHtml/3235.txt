<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are a couple of issues. You're on Windows and things are different on this OS comparing to other POSIX-compliant OSes.</p>

<p>Start by reading <a href="https://wiki.apache.org/hadoop/WindowsProblems" rel="nofollow">Problems running Hadoop on Windows</a> document and see if "missing WINUTILS.EXE" is the issue. Make sure you run <code>spark-shell</code> in console with admin rights.</p>

<p>You may also want to read the answers to a similar question <a href="http://stackoverflow.com/q/32721647/1305344">Why does starting spark-shell fail with NullPointerException on Windows?</a></p>

<p>Also, you may have started <code>spark-shell</code> inside <code>bin</code> subdirectory and hence the errors like:</p>

<blockquote>
  <p>15/11/18 17:51:39 WARN General: Plugin (Bundle)
  "org.datanucleus.api.jdo" is already registered. Ensure you dont have
  multiple JAR versions of the same plugin in the classpath. The URL
  "file:/C:/spark-1.5.2-bin-hadoop2.4/bin/../lib/datanucleus-api-jdo-3.2.6.jar"
  is already registered, and you are trying to register an identical
  plugin located at URL
  "file:/C:/spark-1.5.2-bin-hadoop2.4/lib/datanucleus-api-jdo-3.2.6.jar."</p>
</blockquote>

<p>And the last issue:</p>

<blockquote>
  <p>15/11/18 17:51:47 WARN : Your hostname, Lenovo-PC resolves to a
  loopback/non-reachable address: fe80:0:0:0:297a:e76d:828:59dc%wlan2,
  but we couldn't find any external IP address!</p>
</blockquote>

<p>One <em>workaround</em> is to set <code>SPARK_LOCAL_HOSTNAME</code> to some resolvable host name and be done with it.</p>

<ul>
<li><code>SPARK_LOCAL_HOSTNAME</code> is the custom host name that overrides any other candidates for hostname when driver, master, workers, and executors are created.</li>
</ul>

<p>In your case, using <code>spark-shell</code>, just execute the following:</p>

<pre><code>SPARK_LOCAL_HOSTNAME=localhost ./bin/spark-shell
</code></pre>

<p>You can also use:</p>

<pre><code>./bin/spark-shell -c spark.driver.host=localhost
</code></pre>

<p>Refer also to <a href="https://spark.apache.org/docs/latest/configuration.html#environment-variables" rel="nofollow">Environment Variables</a> in the official documentation of Spark.</p>
