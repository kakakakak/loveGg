<p>When I am plotting, I often plot to an <code>eps</code> file and a <code>png</code> file like this:</p>

<pre><code>postscript(file=paste(dir, output, "_ggplot.eps", sep=""), onefile=FALSE, horizontal=FALSE, width=4.8, height=4.0)
# Plotting code
dev.off()

png(paste(dir, output, "_ggplot.png", sep=""), width=450, height=300)
# Plotting code
dev.off()
</code></pre>

<p>The problem is that the plotting code is repeated twice. Is it possible to specify multiple devices for plotting?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can combine them using <code>dev.copy()</code>. For example,</p>

<pre><code>  X11 ()
  plot (x,y)
  dev.copy (jpeg,filename="test.jpg");
  dev.off ();
</code></pre>

<p>Lookup <code>help(dev.copy)</code> for more details.</p>

<pre><code>Usage:

     dev.copy(device, ..., which = dev.next())
     dev.print(device = postscript, ...)
     dev.copy2eps(...)
     dev.copy2pdf(..., out.type = "pdf")
     dev.control(displaylist = c("inhibit", "enable"))
</code></pre>

<p>Tyler is correct from the standard usage.  However, to make life easier, you can try an alternate method: wrap your plotting code as a function, so that you can then wrap a sequence of outputs.  That can at least simplify your code for producing output.</p>

<p>Another possibility that may work is to fork your process, via <code>foreach</code>, and each iteration produces a different type of output, depending on the index associated with the iteration.  I have done this to produce a lot of plots in parallel (though maybe I used Hadoop, I can't remember at the moment).</p>

<p>You could use a for-loop:</p>

<pre><code>devices &lt;- c("pdf", "png")

for (i in seq_along(devices)) {

    if (devices[i] == "png") {
    ppi &lt;- 600
    png(file = "Plots/regression.png",
        width = 8.4 * ppi, height = 6.5 * ppi, res = ppi,
        family = "Latin Modern Roman")
    }

    if (devices[i] == "pdf") {
        cairo_pdf(file = "Plots/regression.pdf", width = 8.4, height = 6.5,
                  family = "Latin Modern Roman")
    }

    # Insert plotting code

    graphics.off()

}
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Using the <a href="http://cran.r-project.org/web/packages/R.devices/index.html" rel="nofollow">R.devices</a> package, you can do:</p>

<pre><code>library('R.devices')
library('ggplot2')

devEval(c("eps", "png"), name="myfig", tags="ggplot", sep="_", aspectRatio=1.2, {
  gg &lt;- qplot(mpg, wt, data=mtcars, colour=cyl)
  print(gg)
})
</code></pre>

<p>This will generate 'myfig_ggplot.eps' and 'myfig_ggplot.png'.  The default <code>sep</code> is a comma and the default output directory is figures/.</p>
