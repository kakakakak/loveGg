    <div class="post-text" itemprop="text">

<p>I am trying to import cloudera's <strong>org.apache.hadoop:hadoop-client:2.0.0-cdh4.0.0</strong> <a href="https://ccp.cloudera.com/display/CDH4DOC/Using+the+CDH4+Maven+Repository">from cdh4 maven repo</a> in a maven project in eclipse 3.81, m2e plugin, with oracle's jdk 1.7.0_05 on win7 using </p>

<pre><code>&lt;dependency&gt;
    &lt;groupId&gt;org.apache.hadoop&lt;/groupId&gt;
    &lt;artifactId&gt;hadoop-client&lt;/artifactId&gt;
    &lt;version&gt;2.0.0-cdh4.0.0&lt;/version&gt;
&lt;/dependency&gt;
</code></pre>

<p>however, I get the following error:</p>

<pre><code>The container 'Maven Dependencies' references non existing library 'C:\Users\MyUserId\.m2\repository\jdk\tools\jdk.tools\1.6\jdk.tools-1.6.jar'
</code></pre>

<p>more  specific, maven states that the following artifact is missing</p>

<pre><code>Missing artifact jdk.tools:jdk.tools:jar:1.6
</code></pre>

<p>any idea on how to solve this?
Many thanks in advance.</p>

    <div class="post-text" itemprop="text">
<p><code>jdk.tools:jdk.tools</code> (or <code>com.sun:tools</code>, or whatever you name it) is a JAR file that is distributed with JDK. Usually you add it to maven projects like this:</p>

<pre><code>&lt;dependency&gt;
    &lt;groupId&gt;jdk.tools&lt;/groupId&gt;
    &lt;artifactId&gt;jdk.tools&lt;/artifactId&gt;
    &lt;scope&gt;system&lt;/scope&gt;
    &lt;systemPath&gt;${java.home}/../lib/tools.jar&lt;/systemPath&gt;
&lt;/dependency&gt;
</code></pre>

<p>See, the <a href="http://maven.apache.org/general.html#tools-jar-dependency">Maven FAQ for adding dependencies to <code>tools.jar</code></a></p>

<p>Or, you can manually install <code>tools.jar</code> in the local repository using:</p>

<pre><code>mvn install:install-file -DgroupId=jdk.tools -DartifactId=jdk.tools -Dpackaging=jar -Dversion=1.6 -Dfile=tools.jar -DgeneratePom=true
</code></pre>

<p>and then reference it like Cloudera did, using:</p>

<pre><code>&lt;dependency&gt;
    &lt;groupId&gt;jdk.tools&lt;/groupId&gt;
    &lt;artifactId&gt;jdk.tools&lt;/artifactId&gt;
    &lt;version&gt;1.6&lt;/version&gt;
&lt;/dependency&gt;
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The problem is in the Eclipse Maven support, the related question is <a href="http://stackoverflow.com/questions/13288735/maven-not-picking-java-home-correctly/23129154">here</a>.</p>

<p>Under Eclipse, the <code>java.home</code> variable is set to the JRE that was used to start Eclipse, not the build JRE. The default system JRE from <code>C:\Program Files</code> doesn't include the JDK so <code>tools.jar</code> is not being found.</p>

<p>To fix the issue you need to start Eclipse using the JRE from the JDK by adding something like this to <code>eclipse.ini</code> (<em>before</em> <code>-vmargs</code>!):</p>

<pre><code>-vm
C:/&lt;your_path_to_jdk170&gt;/jre/bin/server/jvm.dll
</code></pre>

<p>Then refresh the Maven dependencies (Alt-F5) (Just refreshing the project isn't sufficient).</p>

<p>Ok, if you are using Windows OS</p>

<ol>
<li><p>Go to C:\Program Files\Java\jdk1.8.0_40\lib
(jdk Version might be different for you)</p></li>
<li><p>Make sure tools.jar is present (otherwise download it)</p></li>
<li><p>Copy this path "C:\Program Files\Java\jdk1.8.0_40"</p></li>
<li><p>In pom.xml</p>

<pre><code>&lt;dependency&gt;
&lt;groupId&gt;jdk.tools&lt;/groupId&gt;
&lt;artifactId&gt;jdk.tools&lt;/artifactId&gt;
&lt;version&gt;1.8.0_40&lt;/version&gt;
&lt;scope&gt;system&lt;/scope&gt;
&lt;systemPath&gt;C:/Program Files/Java/jdk1.8.0_40/lib/tools.jar&lt;/systemPath&gt;
&lt;/dependency&gt;
</code></pre></li>
<li><p>Rebuild and run! BINGO!</p></li>
</ol>
