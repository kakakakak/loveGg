    <div class="post-text" itemprop="text">

<p>In multiple projects we have to store, aggregate, evaluate simple measurement values. One row typcially consists of a time stamp, a value and some attributes to the value. In some applications we would like to store 1000 values per second and more. These values must not only be inserted but also deleted at the same rate, since the lifetime of a value is restricted to a year or so (in different aggregation steps, we do not store 1000/s for the whole year).</p>

<p>Until now, we have developped different solutions. One based on Firebird, one on Oracle and one on some self-made storage mechanism. But none of these are very satisfying solutions.</p>

<p>Both RDBMS solutions cannot handle the desired data flow. Besides that, the applications that deliver the values (e.g. device drivers) cannot be easily attached to databases, the insert statements are cumbersome. And finally, while having an SQL interface to the data is strongly desired, typical evaluations are hard to formulate in SQL and slow in the execution. E.g. find the maximum value with time stamp per 15 minutes for all measurements during the last month.</p>

<p>The self-made solution can handle the insertion rate and has a client-friendly API to do it, but it has nothing like a query language and cannot be used by other applications via some standard interface e.g. for reporting.</p>

<p>The best solution in my dreams would be a database system that:  </p>

<ul>
<li>has an API for very fast insertion  </li>
<li>is able to remove/truncate the values in the same speed  </li>
<li>provides a standard SQL interface with specific support for typical time series data  </li>
</ul>

<p>Do you know some database that comes near those requirements or would you approach the problem in a different way?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Most other answers seem to mention SQL based databases. NoSQL based databases are far superior at this kind of thing.</p>

<p>Some Open source time-series databases:</p>

<ul>
<li><a href="http://influxdb.com/">http://influxdb.com/</a> - "An open-source distributed time series database with no external dependencies."</li>
<li><a href="http://square.github.io/cube/">http://square.github.io/cube/</a> - Written ontop of <a href="http://www.mongodb.org/">MongoDB</a></li>
<li><a href="http://opentsdb.net/">http://opentsdb.net/</a> - Written on top of <a href="http://hbase.apache.org/">Apache HBase</a></li>
<li><a href="https://github.com/kairosdb/kairosdb">https://github.com/kairosdb/kairosdb</a> - A rewrite of OpenTSDB that also enables using <a href="http://cassandra.apache.org/">Cassandra</a> instead of Hadoop</li>
<li><a href="http://www.gocircuit.org/vena.html">http://www.gocircuit.org/vena.html</a> - A tutorial on writing a substitute of OpenTSDB using Go-circuits</li>
</ul>

<p>Cloud-based:</p>

<ul>
<li><a href="https://www.tempoiq.com">https://www.tempoiq.com</a></li>
</ul>
