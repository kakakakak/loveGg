<p>This might be a very special case but after pounding on my head for a while I thought to get help from Stackoverflow community. </p>

<p>I am building an inverted index for large data set (One day worth of data from large system). The building of inverted index get executed as a map reduce job on Hadoop. Inverted index is build with the help of scala. Structure of the inverted index as follows: <code>{key:"New", ProductID:[1,2,3,4,5,...]}</code> these get written in to avro files.</p>

<p>During this process I run into Java Heap Size issue. I think the reason is that terms like "New" as I showed above contain large number of productId(s). I have a, rough idea where the issue could take place in my Scala code:</p>

<pre><code>  def toIndexedRecord(ids: List[Long], token: String): IndexRecord = {
    val javaList = ids.map(l =&gt; l: java.lang.Long).asJava //need to convert from scala long to java long
    new IndexRecord(token, javaList)
  }
</code></pre>

<p>And this is how I use this method (It get used in many locations but same code structure and login get use)</p>

<pre><code>  val titles = textPipeDump.map(vf =&gt; (vf.itemId, normalizer.customNormalizer(vf.title + " " + vf.subTitle).trim))
    .flatMap {
    case (id, title) =&gt;
      val ss = title.split("\\s+")
      ss.map(word =&gt; (word, List(id)))
  }
    .filter(f =&gt; f._2.nonEmpty)
    .group
    .sum
    .map {
    case (token, ids) =&gt;
      toIndexedRecord(ids, token)
  } 
</code></pre>

<p><code>textPipeDump</code> is scalding <code>MultipleTextLine</code> field object </p>

<pre><code>case class MultipleTextLineFiles(p : String*) extends FixedPathSource(p:_*) with TextLineScheme
</code></pre>

<p>I have a case class to split and grab the fields that I want from that text line and thats the object <code>ss</code></p>
