    <div class="post-text" itemprop="text">

<p>When following the tutorial instructions for connecting to my JobFlow in EMR, I type following:</p>

<pre><code> ./elastic-mapreduce --jobflow j-3FLVMX9CYE5L6 --ssh
</code></pre>

<p>and get this error:</p>

<pre><code>Permission denied (publickey)
</code></pre>

<p>I'm already able to run other elastic-mapreduce commands just fine to create flows etc, so I'm assuming there's security settings required on the actual master instance for the flow, but nothing in the tutorial explains how to configure this (after all, I need to SSH into it to do the configuration in the first place!) </p>

<p>I found that I need to login as user "hadoop" using the EC2 keypair, and not any of the regular suspects (ec2-user, root, etc.) Like:</p>

<pre><code>ssh -i privatekey.pem hadoop@masternode
</code></pre>

<p>Hope this is useful to someone.</p>
