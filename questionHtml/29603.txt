<p>I'm using AWS Elastic MapReduce and I would like to be able to set the logging level. For example, I would like for log.isDebugEnabled() to return true. A bit of googling led me to find this blog article:</p>

<p><a href="http://vangjee.wordpress.com/2012/03/24/an-approach-to-controlling-logging-on-amazon-web-services-aws-elastic-mapreduce-emr/" rel="nofollow">http://vangjee.wordpress.com/2012/03/24/an-approach-to-controlling-logging-on-amazon-web-services-aws-elastic-mapreduce-emr/</a></p>

<p>which basically suggests writing a shell script to copy and overwrite the local log4j.properties file. This seems like a complicated approach. I would prefer a simpler way of setting the debug level. Is there any way?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are 2 other ways: </p>

<ol>
<li>Using <code>hadoop daemonlog -setlevel</code> command one can set the logging level for a given Hadoop daemon and classname.</li>
<li>Visit the jobtracker's web UI and set the log name's level. 
The web UI url would be :
<code>http://&lt;host:port&gt;/logLevel</code></li>
</ol>

<p>but both these ways only set the log levels for the timespan till the daemons are running, as soon as they are restarted, they will pick-up the one's in log4j.properties.</p>

<p>Read more <a href="http://books.google.co.in/books?id=Wu_xeGdU4G8C&amp;pg=PA352&amp;lpg=PA352&amp;dq=hadoop%20set%20log%20level&amp;source=bl&amp;ots=i7BNVwXfXq&amp;sig=51bNyjX1KFLzqPzXLCwcviU0tG8&amp;hl=en&amp;sa=X&amp;ei=e7FZUsDpOIeZiQfM24GoDg&amp;ved=0CFcQ6AEwBA#v=onepage&amp;q=hadoop%20set%20log%20level&amp;f=false" rel="nofollow">here</a>.</p>
