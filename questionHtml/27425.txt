    <div class="post-text" itemprop="text">

<p>I am using the python yelp/mrjob framework for my mapreduce jobs. There are only about 4G of data and I don't want to go through the trouble of setting up Hadoop or EMR. I have a 64 core machine and it takes about 2 hours to process the data with mrjob. I notice mrjob assigns 54 mappers for my job, but it seems to run only one at a time. Is there a way to make mrjob run all tasks in parallel with all my cpu cores?</p>

<p>I manually changed number of tasks but didn't help much.
<code>--jobconf mapred.map.tasks=10 --jobconf mapred.reduce.tasks=10</code></p>

<p>EDIT:
I have <code>-r local</code> when I execute the job, however, looking at the <a href="https://github.com/Yelp/mrjob/blob/master/mrjob/local.py#L33" rel="nofollow">code</a>, it seems it defaults to run one process at a time. Please tell me I am wrong.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The local job runner for mrjob just spawns one subprocess for each MR stage, one for the mapper, one for the combiner (optionally), and one for the reducer, and passes data between them via a pipe. It is not designed to have any parallelism at all, so it will never take advantage of your 64 cores.</p>

<p>My suggestion would be to run hadoop on your local machine and submit the job with the <code>-r hadoop</code> option. A hadoop cluster running on your local machine in pseduo-distributed mode should be able to take advantage of your multiple cores.</p>

<p>See this question which addresses that topic: <a href="http://stackoverflow.com/questions/8357296/full-utilization-of-all-cores-in-hadoop-pseudo-distributed-mode">Full utilization of all cores in Hadoop pseudo-distributed mode</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The runner for a job can be specified via the command line with the <code>-r</code> option.</p>

<p>When you run a <code>mrjob</code> script from the command line, the default run mode is <code>inline</code> which runs your job on your local machine in a single process. The other obvious options for running jobs are <code>emr</code> and <code>hadoop</code>.</p>

<p>You can make the job run in parallel on you local machine by setting the runner as <code>local</code></p>

<pre><code>$ python myjob.py -r local
</code></pre>

<p>Those <code>--jobconf</code> options are only recognised by Hadoop (i.e. on EMR or a Hadoop cluster).</p>
