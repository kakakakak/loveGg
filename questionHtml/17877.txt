<p>I'm using <a href="https://www.npmjs.com/package/kafka-node" rel="nofollow">kafka-node</a> to send messages between the services in my application. I'm using the <a href="https://hub.docker.com/r/wurstmeister/kafka/" rel="nofollow">wurstmeister/kafka</a> and <a href="https://hub.docker.com/r/wurstmeister/zookeeper/" rel="nofollow">wurstmeister/zookeeper</a> containers to run Kafka and Zookeeper (for simplicity's sake).</p>

<p>Every time I try to <code>send()</code> a message to the broker, I get a <code>LeaderNotAvailable</code> error. I used <a href="https://github.com/claudemamo/kafka-web-console" rel="nofollow">kafka-web-console</a> to inspect the broker, and it shows the two topics that I had set up, but it shows no leader (see <a href="http://cl.ly/image/1f3g0K2S1J0F" rel="nofollow">http://cl.ly/image/1f3g0K2S1J0F</a>).</p>

<p>Neither from reading the container logs (below), nor any documentation, nor trying to search for this problem can I work out why it's not working (or how to fix it).</p>

<p><strong>Update</strong> sending seems to work if I destroy the containers and start them up again (rather than just restarting them using <code>docker-compose up</code>).</p>
