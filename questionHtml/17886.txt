<p>Here's the root cause of the exception <code>org.apache.giraph.utils.UnsafeReads.ensureRemaining</code>.  Notice this is called by the giraph utils.</p>

<p>The exception means that the reader insists it needs that much more input from the input stream, but the input stream doesn't have that much input left (i.e. it hit EOF).</p>

    <div class="post-text" itemprop="text">
<p>The problem was in the class TextAndDoubleComplexWritable. I wasn't aware of the importance of methods <code>readFields</code> and <code>write</code> when we are implementing the <code>Writable</code> interface. This are crucial because are the methods that let us send and receive information in giraph. I was writing a empty string in the <code>readFields</code> method, and i should use that method for writing both values of my vertex. I updated both methods in the following way:</p>

<pre><code>public void write(DataOutput out) throws IOException {
        out.writeDouble(this.vertexData);
        out.writeUTF(this.idsVerticesAnteriores != "" ? "hola"
                : this.idsVerticesAnteriores);
}

public void readFields(DataInput in) throws IOException {
    this.vertexData = in.readDouble();
    this.idsVerticesAnteriores = in.readUTF();
    // idsVerticesAnteriores = in.readLine();
}
</code></pre>

<p>and this is working, finally!!</p>
