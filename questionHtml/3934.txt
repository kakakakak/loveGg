<p>I have a problem with my Ambari server, it is not able to start the Namenode. I'm using HDP 2.0.6, Ambari 1.4.1. It is worth to mention this is happening once I've enabled the Kerberos security, I mean, when it is disabled there is no error.</p>

<p>The error is:</p>

<p><code>
2015-02-04 16:01:48,680 ERROR namenode.EditLogInputStream (EditLogFileInputStream.java:nextOpImpl(173)) - caught exception initializing <a href="http://int-iot-hadoop-fe-02.novalocal:8480/getJournal?jid=integration&amp;segmentTxId=1&amp;storageInfo=-47%3A1493795199%3A0%3ACID-a5152e6c-64ab-4978-9f1c-e4613a09454d" rel="nofollow">http://int-iot-hadoop-fe-02.novalocal:8480/getJournal?jid=integration&amp;segmentTxId=1&amp;storageInfo=-47%3A1493795199%3A0%3ACID-a5152e6c-64ab-4978-9f1c-e4613a09454d</a>
org.apache.hadoop.hdfs.server.namenode.TransferFsImage$HttpGetFailedException: Fetch of <a href="http://int-iot-hadoop-fe-02.novalocal:8480/getJournal?jid=integration&amp;segmentTxId=1&amp;storageInfo=-47%3A1493795199%3A0%3ACID-a5152e6c-64ab-4978-9f1c-e4613a09454d" rel="nofollow">http://int-iot-hadoop-fe-02.novalocal:8480/getJournal?jid=integration&amp;segmentTxId=1&amp;storageInfo=-47%3A1493795199%3A0%3ACID-a5152e6c-64ab-4978-9f1c-e4613a09454d</a> failed with status code 500
Response message:
getedit failed. java.lang.IllegalArgumentException: Does not contain a valid host:port authority: null  at org.apache.hadoop.net.NetUtils.createSocketAddr(NetUtils.java:211)  at org.apache.hadoop.net.NetUtils.createSocketAddr(NetUtils.java:163)  at org.apache.hadoop.net.NetUtils.createSocketAddr(NetUtils.java:152)  at org.apache.hadoop.hdfs.server.namenode.SecondaryNameNode.getHttpAddress(SecondaryNameNode.java:210)  at org.apache.hadoop.hdfs.qjournal.server.GetJournalEditServlet.isValidRequestor(GetJournalEditServlet.java:93)  at org.apache.hadoop.hdfs.qjournal.server.GetJournalEditServlet.checkRequestorOrSendError(GetJournalEditServlet.java:128)  at org.apache.hadoop.hdfs.qjournal.server.GetJournalEditServlet.doGet(GetJournalEditServlet.java:174)  at
...
</code></p>

<p>It seems the problem is about retrieving the Secondary Namenode http address, which in fact is set to null in <code>hdfs-site-xml</code> (I do not know why):</p>

<pre><code>&lt;property&gt;
  &lt;name&gt;dfs.namenode.secondary.http-address&lt;/name&gt;
  &lt;value&gt;null&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>I've tried to set that parameter's value to the appropriate one, but nothing works:</p>

<ul>
<li>By manually editing the <code>hdfs-site.xml</code> files and running <code>hdfs namenode</code>, but nothing occurs.</li>
<li>By manually editing the <code>hdfs-site.xml</code> files and starting the whole HDFS from Ambari, but nothing occurs. Even, the <code>dfs.namenode.secondary.http-address</code> parameter is set to null again!</li>
<li>Through Ambari UI > HDFS services > config tab > <code>hdfs-site.xml</code> list > add new property... the problem is that <code>dfs.namenode.secondary.http-address</code> is not listed by the UI does not allow me to add it because it says... it is already existing! :)</li>
<li>I've tried to add the value in /usr/lib/ambari-server/web/data/configuration/hdfs-site.json, thinking this could be the place where Ambari stores the values that are show in the UI, but no success.</li>
</ul>

<p>I've also noted that a <code>site-XXXX.pp</code> file is created under <code>/var/lib/ambari-agent/data/</code> each time the HDFS service is restarted from the Amabri UI, and I've found each one of these files has:</p>

<pre><code>[root@int-iot-hadoop-fe-02 ~]# cat /var/lib/ambari-agent/data/site-3228.pp | grep dfs.namenode.secondary.http-address
"dfs.namenode.secondary.http-address" =&gt; 'null',
</code></pre>

<p>I think other candidate file for configuring this property could be <code>/var/lib/ambari-agent/puppet/modules/hdp-hadoop/manifests/params.pp</code>. There is a <code>### hdfs-site</code> section, but I'm not able to figure out which is the name of the puppet variable associated to the <code>dfs.namenode.secondary.http-address</code> property.</p>

<p>Any ideas? Thanks!</p>

    <div class="post-text" itemprop="text">
<p>I have a workaround to make it work under ambari environment:</p>

<ul>
<li><p>In the ambari node modify:</p>

<ul>
<li>/usr/lib/ambari-server/web/javascripts/app.js</li>
<li>/usr/lib/ambari-server/web/javascripts/app.js.map</li>
</ul></li>
</ul>

<p>changing from:</p>

<pre><code>{
"name": "dfs.namenode.secondary.http-address",
"templateName": ["snamenode_host"],
"foreignKey": null,
"value": "&lt;templateName[0]&gt;:50090",
"filename": "hdfs-site.xml"
},
</code></pre>

<p>to the specific value for your secondary namenode and not the template one:</p>

<pre><code>{
"name": "dfs.namenode.secondary.http-address",
"templateName": ["snamenode_host"],
"foreignKey": null,
"value": "my.secondary.namenode.domain:50090",
"filename": "hdfs-site.xml"
},
</code></pre>

<ul>
<li><p>rename /usr/lib/ambari-server/web/javascripts/app.js.gz to /usr/lib/ambari-server/web/javascripts/app.js.gz.old</p></li>
<li><p>gzip the app.js so a new app.js.gz is generated in the same directory</p></li>
<li><p>Refresh your ambari web and force an HDFS restart, this will regenerate the appropiate /etc/hadoop/conf/hdfs-site.xml, if it does not, you coud add in the ambari web a new property and then delete it in order to force the changes when you press the save button.</p></li>
</ul>

<p>Hope this helps.</p>

<p>--mLG</p>
