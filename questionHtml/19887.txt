    <div class="post-text" itemprop="text">

<p>Using Gradle 1.0 milestone 8.</p>

<p>My project uses slf4j+Logback for logging, so I want to prevent any transitive deps on log4j  from polluting my classpath. Thus, I added a global exclusion, like so:</p>

<pre><code>configurations {
    all*.exclude group: "log4j", module: "log4j"
}
</code></pre>

<p>However, I am using a test library (<code>hadoop-minicluster</code>) which has a runtime dependency on log4j, so I now need to allow a log4j dependency for my test runtime.  I tried adding a direct dependency on log4j:</p>

<pre><code>testRuntime group: "log4j", name: "log4j", version: "1.2.15"
</code></pre>

<p>and editing my exclusion code (a bit of a hack):</p>

<pre><code>configurations.findAll {!it.name.endsWith('testRuntime')}.each { conf -&gt;
    conf.exclude group: "log4j", module: "log4j"
}
</code></pre>

<p>But this does not work. Adding the exclusion to the testCompile conf automatically adds it to all inheriting configurations as well, including testRuntime. And it seems that this exclusion overrides even the explicit dependency that I added.</p>

<p>It appears that this is expected behaviour for Gradle. From <a href="http://gradle.org/docs/current/userguide/dependency_management.html">the docs</a>:</p>

<blockquote>
  <p>If you define an exclude for a particular configuration, the excluded transitive dependency will be filtered for all dependencies when resolving this configuration <strong>or any inheriting configuration</strong>.</p>
</blockquote>

<p>So is there any other way to do what I want to achieve?</p>

<p>Ideas:</p>

<ul>
<li>Create a new conf <code>myTestRuntime</code> that does not extend from testCompile, and use that for my test classpath.
<ul>
<li>But then I have to duplicate all dependencies for both testCompile and myTestRuntime.</li>
</ul></li>
<li>Remove config-level exclusions. For all confs apart from testRuntime, loop through dependencies and manually remove log4j (or add a dep-level exclusion on log4j).
<ul>
<li>Is this even possible? Configuration.allDependencies is read-only.</li>
</ul></li>
</ul>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>For now I have managed to work around the problem, but I still welcome any better solutions.</p>

<p>Here is what I ended up doing:</p>

<ul>
<li><p>Add a new configuration just for log4j:</p>

<pre><code>log4j(group: 'log4j', name: 'log4j', version: '1.2.15') {
    transitive = false
}
</code></pre></li>
<li><p>Leave the config-level exclusion for all configurations apart from that one:</p>

<pre><code>configurations.findAll {!it.name.endsWith('log4j')}.each { conf -&gt;
    conf.exclude group: "log4j", module: "log4j"
}
</code></pre></li>
<li><p>Add the log4j configuration to my tests' classpath:</p>

<pre><code>test {
    classpath += configurations.log4j
}
</code></pre></li>
</ul>

<p>This way we can get log4j.jar onto the classpath, even though it is excluded from the testRuntime configuration.</p>

<p>Even i encountered similar situation where i need to exclude spark jars from including in fat jar but test cases requires spark jars to execute.so below configuration worked for me. So basically i am adding compile time dependencies to test classpath. so for your problem below solution should work</p>

<pre><code>configurations{
    runtime.exclude group: 'log4j'
}

test {
        classpath += configurations.compile
}
</code></pre>
