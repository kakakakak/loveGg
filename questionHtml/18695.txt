<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Bitly developed and released a Python package called <a href="https://github.com/bitly/data_hacks">data_hacks</a> that provides some ASCII visualization tools.</p>

<p>A utility that parses input data points and outputs a text histogram:</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I was dissatisfied with the AWK script answer below. It's really cool (actually REALLY cool), but it's limited on its inputs. So I wrote my own in Perl that has some better characteristics. Here's the project page: <a href="https://github.com/philovivero/distribution" rel="nofollow">https://github.com/philovivero/distribution</a></p>

<p>Notably: you can just pass in any arbitrary strings, and it'll tally up everything for you (doing the <code>| sort | uniq -c</code> bit) and then output. You can take the output of my script and sort it if you want the histogram by values rather than by frequency of values. So check it out:</p>

<pre><code>cat /usr/share/dict/words | awk '{print length($1)}' | distribution --char=# | sort -n
Value Count Percentile
1  52  0.05%     #
2  182  0.18%    #
3  845  0.85%    ####
4  3346  3.37%   ##############
5  6788  6.84%   ###########################
6  11278  11.37% ############################################
7  14787  14.91% ##########################################################
8  15674  15.81% ##############################################################
9  14262  14.38% ########################################################
10 11546  11.64% #############################################
11 8415  8.49%   #################################
12 5508  5.55%   ######################
13 3236  3.26%   #############
14 1679  1.69%   #######
15 893  0.90%    ####
16 382  0.39%    ##
17 176  0.18%    #
18 72  0.07%     #
19 31  0.03%     #
20 10  0.01%     #
21 3  0.00%      #
22 5  0.01%      #
23 1  0.00%      #
</code></pre>

<p>Yeah, that's just like the awk script answer, but look at this:</p>

<pre><code>$ cat /var/log/syslog.1 | awk '{print $5}' | distribution --height=10 --char=#
Value                 Count Percentile
kernel:               2012  49.94% ############################################
NetworkManager[1197]: 949  23.55%  #####################
ovpn-client[11447]:   337  8.36%   ########
wpa_supplicant[3695]: 191  4.74%   #####
avahi-daemon[1172]:   160  3.97%   ####
dhclient:             143  3.55%   ####
dbus[1111]:           44  1.09%    #
last                  12  0.30%    #
rtkit-daemon[2656]:   7  0.17%     #
dnsmasq[27313]:       6  0.15%     #
</code></pre>

<p>And regexp tokenizing/matching of the input:</p>

<pre><code>zcat /var/log/syslog*gz \
    | awk '{print $5" "$6}' \
    | distribution --tokenize=word --match=word --height=10 --char=o
Val           |Ct (Pct)       Histogram
kernel        |12112 (32.99%) ooooooooooooooooooooooooooooooooooooooooooooooooo
NetworkManager|5695 (15.51%)  ooooooooooooooooooooooo
info          |5371 (14.63%)  oooooooooooooooooooooo
client        |1633 (4.45%)   ooooooo
ovpn          |1633 (4.45%)   ooooooo
daemon        |868 (2.36%)    oooo
avahi         |853 (2.32%)    oooo
dhclient      |736 (2.00%)    ooo
Trying        |667 (1.82%)    ooo
dnsmasq       |562 (1.53%)    ooo
</code></pre>

<p>I wrote this because I was doing some log analysis on large Hadoop logfiles. I've searched for this sort of tool dozens of times in the past. I'm glad I finally just wrote it myself, because it was only a few hours' work, and is paying off immensely already. Notes:</p>

<ol>
<li>If something else already tallied up the numbers for you, you pass
<code>--graph</code> to my script. eg: <code>du -sb /etc/* | distribution --graph</code>.</li>
<li>If you pass the <code>--verbose</code> option while parsing an extremely large file,
it'll give you progress and at the end some statistics about run-time.</li>
<li>If you pass the <code>--color</code> option it'll colourise your output. I like bling.</li>
<li>You must have <code>Time::HiRes</code>, which is fine on some distributions
(Ubuntu 12.04), but CentOS (for example) doesn't include this (?!). On
my to-do is to dynamically note if that module is available and only
use it if so. For now, you have to know how to comment out the
time-related lines in the script</li>
</ol>
