<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Start your terminal and login as super user or root using</p>

<ul>
<li><code>sudo su root</code></li>
</ul>

<p>then <code>start R</code> in terminal and install the rhadoop packages using following commands</p>

<ul>
<li><p>install.packages(c("codetools", "R", "Rcpp", "RJSONIO", "bitops",
   "digest", "functional", "stringr", "plyr", "reshape2", "rJava"))
   install.packages(c("dplyr","R.methodsS3"))
   install.packages(c("Hmisc")) install.packages(c("caTools"))
   Sys.setenv(HADOOP_HOME="/usr/local/hadoop")
   Sys.setenv(HADOOP_CMD="/usr/local/hadoop/bin/hadoop")</p></li>
<li><p><code>Sys.setenv(HADOOP_STREAMING="/usr/local/hadoop/share/hadoop/tools/lib/hadoopversiomentionhere.jar")</code></p></li>
<li><p>after that install the <code>rmr2 rhdfs2</code> <a href="https://github.com/RevolutionAnalytics/RHadoop/wiki/Downloads" rel="nofollow">here</a></p></li>
<li><p>after that install these downloaded source file using this command</p></li>
<li><p><code>install.packages(path_to_file, repos = NULL, type="source")</code></p></li>
<li><p>now after installing close the terminal R and then the terminal open
<code>rstudio</code> run the R code for streaming error will be solved as the
above steps will install the R libraries in global folders.</p></li>
</ul>

<p>Optionally if u want u can install R itself being a super user for being on the safer side hope this helps</p>

<ol>
<li><p>Have a look at your R libraries</p>

<pre><code>.libPaths()
</code></pre></li>
<li><p>Check which library package functional was installed to with commands below:</p>

<pre><code>system.file(package="functional")
</code></pre></li>
<li><p>If it is installed in a personal library, instead of in a library common to all users, jobs will fail with error saying the package cannot loaded.</p></li>
