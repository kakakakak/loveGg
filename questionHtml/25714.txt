    <div class="post-text" itemprop="text">

<p>I am working on creating a k-mer frequency counter (similar to word count in Hadoop) written in Scala. I'm fairly new to Scala, but I have some programming experience.</p>

<p>The input is a text file containing a gene sequence and my task is to get the frequency of each k-mer where <code>k</code> is some specified length of the sequence.</p>

<p>Therefore, the sequence <code>AGCTTTC</code> has three <strong>5-mers</strong> (AGCTT, GCTTT, CTTTC)</p>

<p>I've parsed through the input and created a huge string which is the entire sequence, the new lines throw off the k-mer counting as the end of one line's sequence should still form a k-mer with the beginning of the next line's sequence.</p>

<p>Now I am trying to write a function that will generate a list of maps <code>List[Map[String, Int]]</code> with which it should be easy to use scala's <code>groupBy</code> function to get the count of the common k-mers</p>

<pre><code>import scala.io.Source

object Main {
  def main(args: Array[String]) {

    // Get all of the lines from the input file
    val input = Source.fromFile("input.txt").getLines.toArray

    // Create one huge string which contains all the lines but the first
    val lines = input.tail.mkString.replace("\n","")

    val mappedKmers: List[Map[String,Int]] = getMappedKmers(5, lines)

  }

  def getMappedKmers(k: Int, seq: String): List[Map[String, Int]] = {
    for (i &lt;- 0 until seq.length - k) {
      Map(seq.substring(i, i+k), 1) // Map the k-mer to a count of 1
    }
  }
}
</code></pre>

<p>Couple of questions:</p>

<ul>
<li>How to create/generate <code>List[Map[String,Int]]</code>?</li>
<li>How would you do it?</li>
</ul>

<p>Any help and/or advice is definitely appreciated!</p>

    <div class="post-text" itemprop="text">
<p>You're pretty close—there are three fairly minor problems with your code.</p>

<p>The first is that <code>for (i &lt;- whatever) foo(i)</code> is syntactic sugar for <code>whatever.foreach(i =&gt; foo(i))</code>, which means you're not actually doing anything with the contents of <code>whatever</code>. What you want is <code>for (i &lt;- whatever) yield foo(i)</code>, which is sugar for <code>whatever.map(i =&gt; foo(i))</code> and returns the transformed collection.</p>

<p>The second issue is that <code>0 until seq.length - k</code> is a <code>Range</code>, not a <code>List</code>, so even once you've added the <code>yield</code>, the result still won't line up with the declared return type.</p>

<p>The third issue is that <code>Map(k, v)</code> tries to create a map with two key-value pairs, <code>k</code> and <code>v</code>. You want <code>Map(k -&gt; v)</code> or <code>Map((k, v))</code>, either of which is explicit about the fact that you have a single argument pair.</p>

<p>So the following should work:</p>

<pre><code>def getMappedKmers(k: Int, seq: String): IndexedSeq[Map[String, Int]] = {
  for (i &lt;- 0 until seq.length - k) yield {
    Map(seq.substring(i, i + k) -&gt; 1) // Map the k-mer to a count of 1
  }
}
</code></pre>

<p>You could also convert either the range or the entire result to a list with <code>.toList</code> if you'd prefer a list at the end.</p>

<p>It's worth noting, by the way, that the <code>sliding</code> method on <code>Seq</code> does exactly what you want:</p>

<pre><code>scala&gt; "AGCTTTC".sliding(5).foreach(println)
AGCTT
GCTTT
CTTTC
</code></pre>

<p>I'd definitely suggest something like <code>"AGCTTTC".sliding(5).toList.groupBy(identity)</code> for real code.</p>
