<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As an update to this (and to address <a href="http://stackoverflow.com/users/1698695/david-williams">David Williams</a>' point about Java 1.7), I experienced that only setting the <code>.realm</code> and <code>.kdc</code> properties was insufficient to stop the offending message.</p>

<p>However, by examining the <a href="http://hg.openjdk.java.net/macosx-port/macosx-port/jdk/file/0785fa5940f6/src/share/classes/sun/security/krb5/Config.java">source file</a> that is omitting the message I was able to determine that setting the <code>.krb5.conf</code> property to <code>/dev/null</code> was enough to suppress the message. Obviously if you actually have a krb5 configuration, better to specify the actual path to it.</p>

<p>In total, my <code>hadoop-env.sh</code> snippet is as follows:</p>

<pre><code>HADOOP_OPTS="${HADOOP_OPTS} -Djava.security.krb5.realm= -Djava.security.krb5.kdc="
HADOOP_OPTS="${HADOOP_OPTS} -Djava.security.krb5.conf=/dev/null"
</code></pre>

<p>I'm having the same issue on OS X 10.8.2, Java version 1.7.0_21. Unfortunately, the above solution does not fix the problem with this version :(</p>

<p><strong>Edit:</strong> I found the solution to this, based on a hint I saw <a href="http://stackoverflow.com/questions/14716910/still-getting-unable-to-load-realm-info-from-scdynamicstore-after-bug-fix">here</a>. In the <code>hadoop-env.sh</code> file, change the <code>JAVA_HOME</code> setting to:</p>

<pre><code>export JAVA_HOME=`/usr/libexec/java_home -v 1.6`
</code></pre>

<p>I had similar problem on MacOS and after trying different combinations this is what worked for me universally (both Hadoop 1.2 and 2.2):</p>

<p>in <code>$HADOOP_HOME/conf/hadoop-env.sh</code> set the following lines:</p>

<pre><code># Set Hadoop-specific environment variables here.
export HADOOP_OPTS="-Djava.security.krb5.realm= -Djava.security.krb5.kdc="

# The java implementation to use.
export JAVA_HOME=`/usr/libexec/java_home -v 1.6`
</code></pre>

<p>Hope this will help</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I had this error when debugging MapReduce from Eclipse, but it was a red herring. The real problem was that I should have been remote debugging by adding debugging parameters to the JAVA_OPTS</p>

<pre><code>-Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=y,address=1044
</code></pre>

<p>And then creating a new "Remote Java Application" profile in the debug configuration that pointed to port 1044.</p>

<p><a href="http://www.opensourceconnections.com/2013/04/13/how-to-debug-solr-with-eclipse/" rel="nofollow">This article</a> has some more in-depth information about the debugging side of things. It's talking about Solr, but works much the same with Hadoop. If you have trouble, stick a message below and I'll try to help.</p>
