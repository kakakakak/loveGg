<p>I have gone through lot of blogs on stackoverflow and also apache wiki for getting to know the way the mappers are set in Hadoop. I also went through [<a href="http://stackoverflow.com/questions/17195708/hadoop-how-total-mappers-are-determined]">hadoop - how total mappers are determined</a> [this] post.
Some say its based on InputFormat and some posts say its based on the number of blocks the input file id split into.</p>

<p>Some how I am confused by the default setting. </p>

<p>When I run a wordcount example I see the mappers are low as 2. What is really happening in the setting ? Also this post [<a href="http://svn.apache.org/repos/asf/hadoop/common/trunk/hadoop-mapreduce-project/hadoop-mapreduce-examples/src/main/java/org/apache/hadoop/examples/QuasiMonteCarlo.java]" rel="nofollow">http://svn.apache.org/repos/asf/hadoop/common/trunk/hadoop-mapreduce-project/hadoop-mapreduce-examples/src/main/java/org/apache/hadoop/examples/QuasiMonteCarlo.java]</a> [example program]. Here they set the mappers based on user input. How can one manually  do this setting ?</p>

<p>I would really appreciate for some help and understanding of how mappers work.</p>

<p>Thanks in advance  </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Use the java system properties <code>mapred.min.split.size</code> and <code>mapred.max.split.size</code> to guide Hadoop to use the split size you want.  This won't always work - particularly when your data is in a compression format that is not splittable (e.g. gz, but bzip2 is splittable).</p>

<p>So if you want more mappers, use a smaller split size. Simple!</p>

<p>(Updated as requested) Now this won't work for many small files, in particular you'll end up with more mappers than you want.  For this situation use <code>CombineFileInputFormat</code> ... in Scalding this SO explains: <a href="http://stackoverflow.com/questions/23917404/create-scalding-source-like-textline-that-combines-multiple-files-into-single-ma">Create Scalding Source like TextLine that combines multiple files into single mappers</a></p>
