    public int run(String[] args) throws Exception

        job.getConfiguration().setStrings(ratingsFile.getName(), "rating");

        job.setOutputFormatClass(TextOutputFormat.class);

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Make sure you have valid following configuration files in hadoop classpath. By default configuration files are taken from the directory /etc/hadoop/conf. This activity should be performed a part of hadoop client node setup.</p>

<pre><code>mapred-site.xml
yarn-site.xml
core-site.xml
</code></pre>

<p>If the above mentioned configuration files are empty. You got to pupulate the above files with right properties. Population can be achieved in two ways</p>

<p>In Cloudera Manager when click on service yarn, in action portion, there is an option <strong><code>Deploy client configuration</code></strong> along with start,stop etc. Use that option to deploy the client configuration.</p>

<p>Sometimes above option maynot work if the node is not managed by CM and yarn gateway is not configured on the node. use the option <strong><code>Download client configuration</code></strong> instead of deploy client Configuration. Extract the downloaded zip configuration file(above files) and copy those files to the location /etc/hadoop/conf manually. </p>

<p>For executing the jar either <code>hadoop</code> or <code>yarn</code> can be used.</p>
