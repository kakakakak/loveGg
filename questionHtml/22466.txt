<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Does step #3 went without hitch? Upto step 3 you are downloading the binaries from SVN to your machine and step 4 is setting the binaries with your Hadoop Environment.  </p>

<p>Step 4  suggests you the following:</p>

<p>export PATH=$PATH:/usr/src/hive/build/dist/bin/</p>

<p>-> Means you are adding directory /usr/src/hive/build/dist/bin/ in your PATH environment. You must have installed hive in this folder -> /usr/src/hive/build/dist/bin/ so adding this folder (actually Hive Binaries) in your path will let you run Hive in your machine.</p>

<p>export PATH=$PATH:/usr/src/hive/build/dist/lib/</p>

<p>-> Means you are adding directory /usr/src/hive/build/dist/lib/ in your PATH environment. When you have installed Hive on your machine, Hive related libraries are located in this folder > /usr/src/hive/build/dist/lib/ so adding this directory to your PATH, will help Hive to run successfully.</p>

<p>export PATH=$PATH:/usr/local/hadoop/bin</p>

<p>-> If you already have Hadoop running in your machine, this should already set otherwise this command is just setting Hadoop Binary folder in your machine path.</p>

<p>If you dont know what is PATH, just look for "PATH in Linux" at internet.</p>

    <div class="post-text" itemprop="text">
<p>Step 1 : Download and Extract Hadoop</p>

<p>Step 2 : Set JAVA_HOME path to conf/hadoop-env.sh  //This step is to set java path for hadoop </p>

<p>Step 3 : conf/core-site.xml:</p>

<pre><code>&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;fs.default.name&lt;/name&gt;  //Place your home folder here for using hadoop 
    &lt;value&gt;hdfs://localhost:9000&lt;/value&gt;
  &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>Step 4 : conf/hdfs-site.xml:</p>

<pre><code>&lt;configuration&gt;          //This setting for the number of replications of the file or you can add data node for the save the file
  &lt;property&gt;
    &lt;name&gt;dfs.replication&lt;/name&gt;
    &lt;value&gt;1&lt;/value&gt;
  &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>Step 5 : conf/mapred-site.xml:</p>

<pre><code>&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;mapred.job.tracker&lt;/name&gt;
    &lt;value&gt;localhost:9001&lt;/value&gt;        // add your master host in the place of localhost here
  &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>Step 6 : Login SSH localhost and Format a new distributed-filesystem</p>

<pre><code>bin/hadoop namenode -format
</code></pre>

<p>Step 7 : Start the hadoop daemons:</p>

<pre><code>bin/start-all.sh
</code></pre>

<p>Step 8 : Check the NameNode &amp; JobTracker below port</p>

<pre><code>http://localhost:50070/       //master
http://localhost:50030/       //slave  
</code></pre>

<p>// Its is also better to try the ssh for check your working nodes      </p>

<p>Step 9 : Download and Extract Hive</p>

<p>Step 10 : Set below Env variables.</p>

<pre><code>export HADOOP_HOME=&lt;hadoop-install-dir&gt;
export HIVE_HOME=&lt;hive-install-dir&gt;
export PATH=$HIVE_HOME/bin:$PATH
$HIVE_HOME/bin/hive
</code></pre>
