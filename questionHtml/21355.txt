    <div class="post-text" itemprop="text">

<p>I would like to ask about a processing task I am trying to complete using a data pipeline in AWS, but I have not been able to get it to work.</p>

<p>Basically, I have 2 data nodes representing 2 MySQL databases, where the data is supposed to be extracted from periodically and placed in an S3 bucket. This copy activity is working fine selecting daily every row that has been added, let's say today - 1 day. </p>

<p>However, that bucket containing the collected data as CSVs should become the input for an EMR activity, which will be processing those files and aggregating the information. The problem is that I do not know how to remove or move the already processed files to a different bucket so I do not have to process all the files every day.</p>

<p>To clarify, I am looking for a way to move or remove already processed files in an S3 bucket from a pipeline. Can I do that? Is there any other way I can only process some files in an EMR activity based on a naming convention or something else?</p>

    <div class="post-text" itemprop="text">
<p>Even better, create a DataPipeline ShellCommandActivity and use the aws command line tools.  </p>

<p>Create a script with these two lines:</p>

<pre><code>    sudo yum -y upgrade aws-cli 
    aws s3 rm $1 --recursive
</code></pre>

<p>The first line ensures you have the latest aws tools.  </p>

<p>The second one removes a directory and all its contents.  The $1 is an argument passed to the script.</p>

<p>In your ShellCommandActivity:</p>

<pre><code>    "scriptUri": "s3://myBucket/scripts/theScriptAbove.sh",
    "scriptArgument": "s3://myBucket/myDirectoryToBeDeleted"
</code></pre>

<p>The details on how the aws s3 command works are at: </p>

<pre><code>    http://docs.aws.amazon.com/cli/latest/reference/s3/index.html
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Another approach without using EMR is to install <a href="http://s3tools.org/s3cmd" rel="nofollow">s3cmd</a> tool through ShellCommandActivity in a small EC2 instance, then you can use s3cmd in pipeline to operate your S3 repo in whatever way you want.</p>

<p>A tricky part of this approach is to configure s3cmd through a configuration file safely (basically pass access key and secret), as you can't just ssh into the EC2 instance and use 's3cmd --configure' interactively in a pipeline. </p>

<p>To do that, you create a config file in the ShellCommandActivity using 'cat'. For example:</p>

<pre><code>cat &lt;&lt;EOT &gt;&gt; s3.cfg
blah
blah
blah
EOT
</code></pre>

<p>Then use '-c' option to attach the config file every time you call s3cmd like this:</p>

<pre><code>s3cmd -c s3.cfg ls
</code></pre>

<p>Sounds complicated, but works. </p>
