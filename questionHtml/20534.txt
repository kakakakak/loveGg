    <div class="post-text" itemprop="text">

<p>I have around <code>1000 files</code> and each file is of the size of <code>1GB</code>. And I need to find a String in all these <code>1000 files</code> and also which files contains that particular String. I am working with Hadoop File System and all those <code>1000 files</code> are in Hadoop File System.</p>

<p>All the <code>1000 files</code> are under real folder, so If I do like this below, I will be getting all the <code>1000 files</code>. And I need to find which files contains a particular String <strong>hello</strong> under real folder.</p>

<pre><code>bash-3.00$ hadoop fs -ls /technology/dps/real
</code></pre>

<p>And this is my data structure in hdfs-</p>

<pre><code>row format delimited 
fields terminated by '\29'
collection items terminated by ','
map keys terminated by ':'
stored as textfile
</code></pre>

<p>How I can write MapReduce jobs to do this particular problem so that I can find which files contains a particular string? Any simple example will be of great help to me.</p>

<p><strong>Update:-</strong></p>

<p>With the use of grep in Unix I can solve the above problem scenario, but it is very very slow and it takes lot of time to get the actual output-</p>

<pre><code>hadoop fs -ls /technology/dps/real | awk '{print $8}' | while read f; do hadoop fs -cat $f | grep cec7051a1380a47a4497a107fecb84c1 &gt;/dev/null &amp;&amp; echo $f; done
</code></pre>

<p>So that is the reason I was looking for some MapReduce jobs to do this kind of problem...</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It sounds like you're looking for a grep-like program, which is easy to implement using <a href="http://hadoop.apache.org/mapreduce/docs/r0.21.0/streaming.html" rel="nofollow">Hadoop Streaming</a> (the Hadoop Java API would work too):</p>

<p>First, write a mapper that outputs the name of the file being processed if the line being processed contains your search string.  I used Python, but any language would work:</p>

<pre><code>#!/usr/bin/env python
import os
import sys

SEARCH_STRING = os.environ["SEARCH_STRING"]

for line in sys.stdin:
    if SEARCH_STRING in line.split():
        print os.environ["map_input_file"]
</code></pre>

<p>This code reads the search string from the <code>SEARCH_STRING</code> environmental variable.  Here, I split the input line and check whether the search string matches any of the splits; you could change this to perform a substring search or use regular expressions to check for matches.</p>

<p>Next, run a Hadoop streaming job using this mapper and no reducers:</p>

<pre><code>$ bin/hadoop jar contrib/streaming/hadoop-streaming-*.jar \
    -D mapred.reduce.tasks=0
    -input hdfs:///data \
    -mapper search.py \
    -file search.py \
    -output /search_results \
    -cmdenv SEARCH_STRING="Apache"
</code></pre>

<p>The output will be written in several parts; to obtain a list of matches, you can simply cat the files (provided they aren't too big):</p>

<pre><code>$ bin/hadoop fs -cat /search_results/part-*
hdfs://localhost/data/CHANGES.txt
hdfs://localhost/data/CHANGES.txt
hdfs://localhost/data/ivy.xml   
hdfs://localhost/data/README.txt
... 
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can try something like this, though I'm not sure if it's an efficient way to go about it. Let me know if it works - I haven't tested it or anything.</p>

<p>You can use it like this: <strong><em>java SearchFiles /technology/dps/real hello</em></strong> making sure you run it from the appropriate directory of course.</p>

<pre><code>import java.io.BufferedReader;
import java.io.FileReader;
import java.io.File;
import java.io.IOException;
import java.util.ArrayList;
import java.util.Scanner;

public class SearchFiles {

    public static void main(String[] args) throws IOException {
        if (args.length &lt; 2) {
            System.err.println("Usage: [search-dir] [search-string]");
            return;
        }
        File searchDir = new File(args[0]);
        String searchString = args[1];
        ArrayList&lt;File&gt; matches = checkFiles(searchDir.listFiles(), searchString, new ArrayList&lt;File&gt;());
        System.out.println("These files contain '" + searchString + "':");
        for (File file : matches) {
            System.out.println(file.getPath());
        }
    }

    private static ArrayList&lt;File&gt; checkFiles(File[] files, String search, ArrayList&lt;File&gt; acc) throws IOException {
        for (File file : files) {
            if (file.isDirectory()) {
                checkFiles(file.listFiles(), search, acc);
            } else {
                if (fileContainsString(file, search)) {
                    acc.add(file);
                }
            }
        }
        return acc;
    }

    private static boolean fileContainsString(File file, String search) throws IOException {
        BufferedReader in = new BufferedReader(new FileReader(file));
        String line;
        while ((line = in.readLine()) != null) {
            if (line.contains(search)) {
                in.close();
                return true;
            }
        }
        in.close();
        return false;
    }
}
</code></pre>
