    <div class="post-text" itemprop="text">

<p>I have a large set of files for which some heavy processing needs to be done.
This processing in single threaded, uses a few hundred MiB of RAM (on the machine used to start the job) and takes a few minutes to run.
My current usecase is to start a hadoop job on the input data, but I've had this same problem in other cases before.</p>

<p>In order to fully utilize the available CPU power I want to be able to run several those tasks in paralell.</p>

<p>However a very simple example shell script like this will trash the system performance due to excessive load and swapping:</p>

<pre><code>find . -type f | while read name ; 
do 
   some_heavy_processing_command ${name} &amp;
done
</code></pre>

<p>So what I want is essentially similar to what "gmake -j4" does.</p>

<p>I know bash supports the "wait" command but that only waits untill all child processes have completed. In the past I've created scripting that does a 'ps' command and then grep the child processes out by name (yes, i know ... ugly).</p>

<p>What is the simplest/cleanest/best solution to do what I want?</p>

<hr>

<p>Edit: Thanks to Frederik: Yes indeed this is a duplicate of <a href="http://stackoverflow.com/questions/6511884/how-to-limit-number-of-threads-used-in-a-function-in-bash">How to limit number of threads used in a function in bash</a>
The "xargs --max-procs=4" works like a charm.
(So I voted to close my own question)</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<pre><code>#! /usr/bin/env bash

set -o monitor 
# means: run background processes in a separate processes...
trap add_next_job CHLD 
# execute add_next_job when we receive a child complete signal

todo_array=($(find . -type f)) # places output into an array

index=0
max_jobs=2

function add_next_job {
    # if still jobs to do then add one
    if [[ $index -lt ${#todo_array[*]} ]]
    # apparently stackoverflow doesn't like bash syntax
    # the hash in the if is not a comment - rather it's bash awkward way of getting its length
    then
        echo adding job ${todo_array[$index]}
        do_job ${todo_array[$index]} &amp; 
        # replace the line above with the command you want
        index=$(($index+1))
    fi
}

function do_job {
    echo "starting job $1"
    sleep 2
}

# add initial set of jobs
while [[ $index -lt $max_jobs ]]
do
    add_next_job
done

# wait for all jobs to complete
wait
echo "done"
</code></pre>

<p>Having said that Fredrik makes the excellent point that xargs does exactly what you want... </p>

   while [ `jobs | wc -l` -ge 2 ]

find . -type f | while read name ;

   max2; some_heavy_processing_command ${name} &amp;

<pre><code>if [ $max_jobs -gt ${#todo_array[*]} ];
    then
           # there are more elements found in the array than max jobs, setting max jobs to #of array elements"
            max_jobs=${#todo_array[*]}
 fi
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I think I found a more handy solution using <a href="/questions/tagged/make" class="post-tag" title="show questions tagged &#39;make&#39;" rel="tag">make</a>:</p>

<pre><code>#!/usr/bin/make -f

THIS := $(lastword $(MAKEFILE_LIST))
TARGETS := $(shell find . -name '*.sh' -type f)

.PHONY: all $(TARGETS)

all: $(TARGETS)

$(TARGETS):
        some_heavy_processing_command $@

$(THIS): ; # Avoid to try to remake this makefile
</code></pre>

<p>Call it as e.g. 'test.mak', and add execute rights. If You call <code>./test.mak</code> it will call the <code>some_heavy_processing_command</code> one-by-one. But You can call as <code>./test.mak -j 4</code>, then it will run four subprocesses at once. Also You can use it on a more sophisticated way: run as <code>./test.mak -j 5 -l 1.5</code>, then it will run maximum 5 sub-processes while the system load is under 1.5, but it will limit the number of processes if the system load exceeds 1.5.</p>

<p>It is more flexible than <a href="/questions/tagged/xargs" class="post-tag" title="show questions tagged &#39;xargs&#39;" rel="tag">xargs</a>, and <a href="/questions/tagged/make" class="post-tag" title="show questions tagged &#39;make&#39;" rel="tag">make</a> is part of the standard distribution, not like <code>parallel</code>.</p>

<p>Here is a very good function I used to control the maximum # of jobs from bash or ksh. NOTE: the - 1 in the pgrep subtracts the wc -l subprocess.</p>

<pre><code>function jobmax
{
    typeset -i MAXJOBS=$1
    sleep .1
    while (( ($(pgrep -P $$ | wc -l) - 1) &gt;= $MAXJOBS ))
    do
        sleep .1
    done
}

nproc=5
for i in {1..100}
do
    sleep 1 &amp;
    jobmax $nproc
done
wait # Wait for the rest
</code></pre>
