<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You have multiple versions of the library containing <code>org.eclipse.jdt.internal.compiler.CompilationResult</code> on your classpath, and they are ordered so that a version in which the method <code>CompilationResult.getProblems()</code> does not exist (or was incompatibly changed) comes first on the classpath, probably because it is provided by whatever container you're running in.</p>

<p>Possible solutions:</p>

<ul>
<li>Reorder the classpath to avoid the problem</li>
<li>If it is a Maven project, exclude the duplicate JARs as dependencies (in NetBeans, expand the <em>Libraries</em> node and right click the library and choose <em>Exclude Dependency</em> from the popup menu)</li>
</ul>
