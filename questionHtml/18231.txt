    <div class="post-text" itemprop="text">

<p>I am running Apache Hadoop 2.6.0 on Ubuntu 14.0 and I have a table created in Hive 0.13.0 as:</p>

<pre><code>CREATE TABLE IF NOT EXISTS recipes_hive.cuisine (
ID INT COMMENT 'Cuisine ID.', 
name STRING COMMENT 'Cusine name - primary key.', 
area STRING COMMENT 'Name of the area of origin - foreign key.', 
scope STRING COMMENT 'Either country or area.') 
COMMENT 'Table containing cuisines data.'
ROW FORMAT DELIMITED FIELDS TERMINATED BY ','
STORED AS TEXTFILE;
</code></pre>

<p>And I fill it with data with statement:</p>

<pre><code>LOAD DATA LOCAL INPATH 'path_to_file/CUISINE.csv'
OVERWRITE INTO TABLE recipes_hive.cuisine;
</code></pre>

<p>My database has several such tables all created and filled with same procedure. When running simple queries like:</p>

<pre><code>SELECT * FROM cuisine
</code></pre>

<p>or even with certain conditions in WHERE clause I get the results as expected but running more complex queries I get squat. For example:</p>

<pre><code>SELECT cuisine.name, SUM(IF (ingredient.category = "fruit",1,2))/count(*) AS      PERC 
FROM cuisine JOIN recipe ON recipe.cuisine = cuisine.name JOIN part_of ON part_of.id_recipe = recipe.id JOIN ingredient ON ingredient.name = part_of.ingredient 
GROUP BY cuisine.name 
ORDER BY PERC DESC
</code></pre>

<p>, or:</p>

<pre><code>SELECT ingredient.id, ingredient.name 
FROM cuisine JOIN recipe ON recipe.cuisine = cuisine.name JOIN part_of ON part_of.id_recipe = recipe.id JOIN ingredient ON ingredient.name = part_of.ingredient 
WHERE ingredient.id IN (
SELECT ingredient.id 
FROM cuisine c JOIN recipe ON recipe.cuisine = c.name JOIN part_of ON part_of.id_recipe = recipe.id JOIN ingredient ON ingredient.name = part_of.ingredient 
WHERE c.name = "Pakistan") AND cuisine.name = "Bangladesh"
</code></pre>

<p>The first example calculates some percentage and the second one checks for mutual elements.</p>

<p>The MapReduce and Hadoop get invoked properly and they return no error. The output ends with:</p>

<pre><code>Execution completed successfully
MapredLocal task succeeded
OK
Time taken: 122.119 seconds
</code></pre>

<p>I have checked the web and people had similar issues as I do. I checked:</p>

<p><a href="http://stackoverflow.com/questions/23094261/hive-table-returning-empty-result-set-on-all-queries">Hive Table returning empty result set on all queries</a></p>

<p><a href="http://stackoverflow.com/questions/11347699/simple-hive-query-is-empty">Simple Hive query is empty</a></p>

<p>but failed to get my issue resolved. The data is in fact in the HDFS and as mentioned before it works on simple queries.</p>

<p>So either there is something wrong with my Hive instance or my queries are not written correctly.</p>

<p>Any help would be greatly appreciated.
Best regards.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If we have Cuisine table containing ID = {1,2,3} and Recipe table containing ID = {5,6,7} , then even though these tables are non-empty, we still get no rows returned when we do INNER JOIN Cuisine.ID = Recipe.ID (as the ID's are different in the 2 tables)
Can you please check that there is no such condition. </p>

<pre><code>SELECT count(1)
FROM cuisine c JOIN recipe ON recipe.cuisine = c.name WHERE c.name = "Pakistan";

--- must return &gt; 0 

select count(1) from recipe as recipe
JOIN part_of ON part_of.id_recipe = recipe.id ;

--- must return &gt; 0 

select count(1) from part_of as part_of
JOIN ingredient ON ingredient.name = part_of.ingredient ;

--- must return &gt; 0 
</code></pre>

<p>So the inner query returns a row when all the count(*) are non-zero. Now test the outer select :</p>

<pre><code>SELECT ingredient.id, ingredient.name 
FROM cuisine JOIN recipe ON recipe.cuisine = cuisine.name JOIN part_of ON part_of.id_recipe = recipe.id JOIN ingredient ON ingredient.name = part_of.ingredient 
WHERE ingredient.id = &lt;inner query result&gt; and cuisine.name = "Bangladesh";
</code></pre>
