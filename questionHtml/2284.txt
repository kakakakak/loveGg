<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Let me rephrase your question a bit and see if this addresses it.</p>

<p><strong>What does Spring Batch provide that I'd have to handle myself when building a batch application?</strong></p>

<p>Spring Batch served as the basis for JSR-352 (the java batch specification) and since that specification has come out, there is a lot of Spring Batch now available within the java space.  That being said, there is still a lot that Spring Batch provides outside of the scope of what basic Java does:</p>

<p><strong>Within a "basic" batch job</strong></p>

<p>Within the scope of a simple batch job, Spring Batch provides a collection of utilities and implementations that have been battle tested in all enterprise verticals.  Some examples are:</p>

<ul>
<li>Over 17 <code>ItemReader</code> and 15 <code>ItemWriter</code> implementations covering vast options for input and output (File, JDBC, NoSQL, JMS, etc).  All of these provide declarative I/O options so that you don't have to write and test code for stateful readers and writers.</li>
<li>A collection of <code>Tasklet</code> (Spring Batch's equivalent to JSR-352's <code>Batchlet</code>) implementations including ones for executing shell commands and interfacing with Hadoop.</li>
<li>The ability to stop/start/restart jobs and maintain state between executions.</li>
<li>The ability to skip and retry records as they are being processed.</li>
<li>Transaction management.  Spring Batch handles transactions for you.</li>
<li>The ability to notify other systems when errors occur via messaging by integrating Spring Integration.</li>
<li>Java or XML based configuration.</li>
<li>All the Spring features like DI, AOP, testability, etc.</li>
<li>Vendor independence - By using Spring Batch, you get to use a framework that open source and not tied to any one vendor.</li>
</ul>

<p><strong>Additional advantages</strong></p>

<p>Beyond the above examples of what Spring Batch brings to the table, it goes much further:</p>

<ul>
<li>Scalability options - Spring Batch provides a number of scalability options that range from within a single JVM via threads (multithreaded step, local partitioning, and splits) to multi-JVM scalability (remote partitioning and remote chunking).</li>
<li>Integration with Spring Integration - Spring Integration provides a number of useful elements that allow you to build robust batch applications to handle things like error messages, poling directories for files, automatically FTPing files, etc.</li>
<li>Big data support - Through the Spring for Apache Hadoop project, there are a number of extensions to Spring Batch that allow it to work well with Hadoop.  You can run Spring Batch jobs on YARN, you can execute Pig, Hive, MapReduce, etc jobs.</li>
<li>Integration with Spring XD - Spring XD provides a distributed runtime for the deployment, management, and execution of batch jobs.</li>
</ul>

<p>I personally view batch processing as the "set it and forget it" model of programming.  While it isn't sexy, batch processing is a very useful model of processing and is more useful in places than most people realize.  Spring Batch provides an environment that makes developing robust batch jobs as easily as possible.</p>
