    <div class="post-text" itemprop="text">

<p>I'm using the new Hadoop API to write a sequence of map-reduce jobs.  I plan to use <a href="http://yahoo.github.com/oozie/" rel="nofollow">Oozie</a> to pipeline all of these together, but I can't seem to find a way to do multiple output streams from a <code>map-reduce</code> node in the workflow.</p>

<p>Normally to write multiple outputs I would use code similar to the code given in the <a href="http://hadoop.apache.org/mapreduce/docs/r0.21.0/api/org/apache/hadoop/mapreduce/lib/output/MultipleOutputs.html" rel="nofollow">MultipleOutputs javadoc</a>, but oozie gets all its configuration from <code>workflow.xml</code> file so the named outputs cannot be configured like they are in the example.</p>

<p>I've come across a <a href="http://mail-archives.apache.org/mod_mbox/incubator-oozie-users/201201.mbox/%3C1328043046.74499.YahooMailNeo@web161305.mail.bf1.yahoo.com%3E" rel="nofollow">thread</a> discussing the use of multiple outputs in Oozie, but there was no solution presented beyond creating a Java task and adding it to the Oozie pipline directly.</p>

<p>Is there a way to this via a <code>map-reduce</code> node in the <code>workflow.xml</code>?</p>

<p>Edit:</p>

<p>Chris's solution did work, though I wish there was a better way.  Here are the exact changes I made.</p>

<p>I added the following to the workflow.xml file:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs&lt;/name&gt;
   &lt;value&gt;${output1} ${output2}&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs.namedOutput.${output1}.key&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.Text&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs.namedOutput.${output1}.value&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.LongWritable&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs.namedOutput.${output1}.format&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.mapreduce.lib.output.TextOutputFormat&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs.namedOutput.${output2}.key&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.Text&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs.namedOutput.${output2}.value&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.LongWritable&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mapreduce.multipleoutputs.namedOutput.${output2}.format&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.mapreduce.lib.output.TextOutputFormat&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>I added the following to the job.properties file that is fed to oozie at startup:</p>

<pre><code>output1=totals
output2=uniques
</code></pre>

<p>Then in the reducer I wrote to the named outputs <code>totals</code> and <code>uniques</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>the <code>addNamedOutput</code> utility methods for MultipleOutputs is just configuring configuration properties - so go look at an instance of your job that has run and extract the properties for MultipleOutputs (look in the job.xml, lined from the JobTracker page). </p>

<p>Alternatively, look through the source for MultipleOutputs and see what configuration properties are being set when you call this method.</p>

<p>Once you know the properties being set, add them to the configuration section of map-reduce element in your Oozie workflow.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As of Hadoop 2.x the property names have changed from mapreduce.multipleoutputs.* to mo.*, thus to new configuration properties would now look like this:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;mo.namedOutputs&lt;/name&gt;
   &lt;value&gt;${output1} ${output2}&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mo.namedOutput.${output1}.key&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.Text&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mo.namedOutput.${output1}.value&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.LongWritable&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mo.namedOutput.${output1}.format&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.mapreduce.lib.output.TextOutputFormat&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mo.namedOutput.${output2}.key&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.Text&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mo.namedOutput.${output2}.value&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.io.LongWritable&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
    &lt;name&gt;mo.namedOutput.${output2}.format&lt;/name&gt;
   &lt;value&gt;org.apache.hadoop.mapreduce.lib.output.TextOutputFormat&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>Tested and verified on Hadoop 2.4.x, Ooize 4.0.0</p>
