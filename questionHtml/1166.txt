    <div class="post-text" itemprop="text">

<p>I have been trying to process a good chunk of data (a few GBs) but my personal computer resists to do it in a reasonable time span, so I was wondering what options do I have? I was using python's <a href="http://docs.python.org/library/csv.html" rel="nofollow">csv.reader</a> but it was painfully slow even to fetch 200,000 lines. Then I migrated this data to an <a href="http://www.sqlite.org/faq.html" rel="nofollow">sqlite</a> database which retrieved results a bit faster and without using so much memory but slowness was still a major issue.</p>

<p>So, again... what options do I have to process this data? I was wondering about using amazon's <a href="http://aws.amazon.com/ec2/spot-instances/" rel="nofollow">spot instances</a> which seem useful for this kind of purpose but maybe there are other solutions to explore.</p>

<p>Supposing that spot instances is a good option and considering I have never used them before, I'd like to ask what can I expect from them? Does anyone have experience using them for this kind of thing? If so, What is your workflow? I thought I could find a few blog posts detailing workflows for scientific computing, image processing or that kind of thing but I didn't find anything so if you can explain a bit of that or point out some links, I'd appreciate it.</p>

<p>Thanks in advance.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I would try to use <a href="http://numpy.scipy.org/"><code>numpy</code></a> to work with your large datasets localy. Numpy arrays should use less memory compared <code>csv.reader</code> and computation times should be much faster when using vectorised numpy functions.</p>

<p>However there may be a memory problem when reading the file.
<a href="http://docs.scipy.org/doc/numpy/reference/generated/numpy.loadtxt.html"><code>numpy.loadtxt</code></a> or <a href="http://docs.scipy.org/doc/numpy/reference/generated/numpy.genfromtxt.html#numpy.genfromtxt"><code>numpy.genfromtxt</code></a> also consume a lot of memory when reading files.
If this is a problem some (brand new) alternative parser engines are compared <a href="http://wesmckinney.com/blog/?p=543">here</a>. According to this post, the new <a href="http://pandas.pydata.org/"><code>pandas</code></a> (a library which is built on top of numpy) parser seems to be an option. </p>

<p>As mentioned in the comments I would also suggest to store your data in a binary format like HDF5 when you have read your files once. Loading the data from a HDF5 file is really fast from my experience (would be interesting to know how fast it is compared to sqlite in your case). The simplest way I know to save your array as HDF5 is with pandas</p>

<pre><code>import pandas as pd

data = pd.read_csv(filename, options...)
store = pd.HDFStore('data.h5')
store['mydata'] = data
store.close()
</code></pre>

<p>loading your data is than as simple as</p>

<pre><code>import pandas as pd

store = pd.HDFStore('data.h5')
data = store['mydata']
store.close()
</code></pre>
