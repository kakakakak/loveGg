<pre><code>chmod +s myscript &amp;&amp; chown root myscript &amp;&amp; chgrp root myscript

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As guys already mentioned you can use <code>sudo</code> only if your current use is configured at the system as one that can run <code>sudo</code>. Otherwise (in general case) you have to type password. </p>

<p>You have to type password if you want to run commands on remote computers as well using <code>ssh</code> or copy files remotely using <code>scp</code>. </p>

<p>Some commands can accept password as a parameter of command line. For example <code>mysql</code> utility can accept password after switch <code>-p</code>. Some other commands are more strict: they do not exept password in command line and even do not allow you to supply password via redirecting of stdin of the process. Examples are <code>ssh</code> and <code>scp</code>. </p>

<p>The only way to be sure that you can always programmatically run any command that requires typing the password is to emulate the terminal. This is what utility <code>expect</code> does. Take a look on its man page: <a href="http://linux.die.net/man/1/expect" rel="nofollow">http://linux.die.net/man/1/expect</a></p>

<p>I used <code>expect</code> in past. it is script driven utility that allows simulating user's actions exactly as human user is doing when he is working with terminal. We ran the expect script from java as regular external process using <code>ProcessBuilder</code>.</p>
    </div>
