    <div class="post-text" itemprop="text">

<p>Normally, we write the mapper in the form : </p>

<pre><code>public static class Map extends Mapper&lt;**LongWritable**, Text, Text, IntWritable&gt;
</code></pre>

<p>Here the input key-value pair for the mapper is <code>&lt;LongWritable, Text&gt;</code> - as far as I know when the mapper gets the input data its goes through line by line - so the Key for the mapper signifies the line number - please correct me if I am wrong.</p>

<p>My question is : If I give the input key-value pair for mapper as <code>&lt;Text, Text&gt;</code> then it is giving the error  </p>

<pre><code> java.lang.ClassCastException: org.apache.hadoop.io.LongWritable cannot be cast to org.apache.hadoop.io.Text
</code></pre>

<p>Is it a mandatory to give the input key-value pair of mapper as <code>&lt;LongWritable, Text&gt;</code> - if yes then why ? if no then what the reason of the error ? Can you please help me understand the proper reasoning of the error ?</p>

<p>Thanks in advance.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The input to the mapper depends on what InputFormat is used. The InputFormat is responsible for reading the incoming data and shaping it into whatever format the Mapper expects.The default InputFormat is <a href="http://hadoop.apache.org/docs/current/api/org/apache/hadoop/mapreduce/lib/input/TextInputFormat.html">TextInputFormat</a>, which extends <code>FileInputFormat&lt;LongWritable, Text&gt;</code>.</p>

<p>If you do not change the InputFormat, using a Mapper with different Key-Value type signature than <code>&lt;LongWritable, Text&gt;</code> will cause this error. If you expect <code>&lt;Text, Text&gt;</code> input, you will have to choose an appropiate InputFormat. You can set the InputFormat in Job setup:</p>

<pre><code>job.setInputFormat(MyInputFormat.class);
</code></pre>

<p>And like I said, by default this is set to TextInputFormat.</p>

<p>Now, let's say your input data is a bunch of newline-separated records delimited by a comma:</p>

<ul>
<li>"A,value1"  </li>
<li>"B,value2"</li>
</ul>

<p>If you want the input key to the mapper to be ("A", "value1"), ("B", "value2") you will have to implement a custom InputFormat and RecordReader with the <code>&lt;Text, Text&gt;</code> signature. <strong>Fortunately</strong>, this is pretty easy. There is <a href="http://developer.yahoo.com/hadoop/tutorial/module5.html#fileformat">an example here</a> and probably a few examples floating around StackOverflow as well.</p>

<p>In short, add a class which extends <code>FileInputFormat&lt;Text, Text&gt;</code> and a class which extends <code>RecordReader&lt;Text, Text&gt;</code>. Override the <code>FileInputFormat#getRecordReader</code> method, and have it return an instance of your custom RecordReader.</p>

<p>Then you will have to implement the required RecordReader logic. The simplest way to do this is to create an instance of <a href="http://hadoop.apache.org/docs/r1.1.2/api/org/apache/hadoop/mapreduce/lib/input/LineRecordReader.html">LineRecordReader</a> in your custom RecordReader, and delegate all basic responsibilities to this instance. In the getCurrentKey and getCurrentValue-methods you will implement the logic for extracting the comma delimited Text contents by calling <code>LineRecordReader#getCurrentValue</code> and splitting it on comma.</p>

<p>Finally, set your new InputFormat as Job InputFormat as shown after the second paragraph above.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In the book "Hadoop: The Difinitive Guide" by Tom White I think he has an appropriate answer to this(pg. 197):</p>

<p>"TextInputFormat’s 
keys, being simply the offset within the file, are not normally very
useful. It is common for each line in a file to be a key-value pair, separated by a delimiter
such as a tab character. For example, this is the output produced by 
TextOutputFormat, Hadoop’s default 
OutputFormat. To interpret such files correctly, 
KeyValueTextInputFormat
 is appropriate.</p>

<p>You can specify the separator via the 
key.value.separator.in.input.line
 property. It
is a tab character by default."</p>

record reader reads a single line in first cycle. And o/p of the mapper can be whatever u want (it can be (Text,Text) or (Text, IntWritable) or ......)</p>
