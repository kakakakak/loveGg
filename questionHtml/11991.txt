<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Java is a very good start for big data as most of the technologies revolve around Hadoop and are written in Java.</p>

<p>Some use Python too but often Python has only second-class support.</p>

<p>Scala is another language that has a lot of big data libraries. For example <a href="http://twitter.github.io/" rel="nofollow">Twitter has a lot of big data technologies written in Scala</a>.</p>

<p>It really depends on what you want to do. As I said, Java is a good start. You can try to implement simple MapReduce "hello-worlds" such as <a href="https://hadoop.apache.org/docs/r1.2.1/mapred_tutorial.html#Example%3a+WordCount+v1.0" rel="nofollow">word counting</a> and then quickly move to learn libraries and languages such as <a href="http://www.cascading.org/" rel="nofollow">Cascading</a> and <a href="https://pig.apache.org/" rel="nofollow">Pig</a>. The reason is that your productivity will be likely higher in such higher-level languages. The resulting jobs might not be as efficient as hand-crafted mapreduce code but you can worry  about that later.</p>
