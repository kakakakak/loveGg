<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Jacky, there are different ports for different services. </p>

<p><strong>50070</strong> is HDFS service which every Hadoop system has (therefore I suggested it). </p>

<p><strong>8888</strong> is likely just a hortonworks' port for some specific web service. </p>

<p><strong>50030</strong> is Jobtracker port</p>

<p>First of all, make sure that you can connect/communicate with your VirtualBox (host os can access guest services). If so, find out what service/port you need e.g. <a href="http://stackoverflow.com/questions/12813365/jobtracker-api-error-call-to-localhost-127-0-0-150030-failed-on-local-excepti">Jobtracker API error - Call to localhost/127.0.0.1:50030 failed on local exception: java.io.EOFException</a> </p>

<p>It can be a lot of work - so if you know Cloudera distribution has all things you need - go for Cloudera</p>

    <div class="post-text" itemprop="text">
<p>Hadoop 2.2.0 version don't use jobtracker as a whole. The yarn has split the work of job tracker in two half see apache hadoop documentation.</p>

<p>First of all go to the mapred-site.xml add the below properties with the other mentioned in all installation steps:-</p>

<p>mapreduce.jobtracker.address</p>

<p>localhost:54311</p>

<p>mapreduce.jobtracker.http.address</p>

<p>0.0.0.0:50030</p>

<p>mapreduce.jobhistory.address</p>

<p>0.0.0.0:10020</p>

<p>mapreduce.jobhistory.webapp.address</p>

<p>0.0.0.0:19888</p>

<p>and after that configure your hadoop location as:-</p>

<p>Map/Reduce (V2) Master</p>

<p>Host: localhost</p>

<p>port: 54311</p>

<p>DFS Master</p>

<p>check the checkbox use M/R master host</p>

<p>port: 9000**</p>

<p>Now every thing will be fine.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I found myself in similar situation when I was unable to connect to Hive server in HortonWorks sandbox. What I found that the Virtual Image you use for SandBox uses NAT for networking. So what it means that the IP address of the Guest OS (Sandbox in this case) is same as the machine it runs on. To enable effective communication, there is Port Forwarding in visualization software. The ports that are default configured for Hadoop in Sandbox are mapped to different (or sometime same) ports by default.
You can check the port forwarding and the configured rules to reach to specific service/port from the Host operating system.
Now, about the eclipse plugin for Hadoop 2.2.0, I am yet to be successful to find how to do it. So I will post more as I go along my Hadoop development journey.</p>
