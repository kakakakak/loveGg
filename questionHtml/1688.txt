<p>I am newbie to Apache Spark and Cluster Computing and I implemented Spark in Standalone Mode (Same Machine with Master and Worker), it worked fine for me.</p>

<p>Then, I downloaded pre-built version of spark, and followed these instructions and placed in every nodes of my cluster: <a href="http://spark.apache.org/docs/latest/spark-standalone.html#installing-spark-standalone-to-a-cluster" rel="nofollow">http://spark.apache.org/docs/latest/spark-standalone.html#installing-spark-standalone-to-a-cluster</a>.</p>

<p>My Master node has IP address: 172.17.0.224 and my Slave nodes has IP Address: 172.17.0.221, 172.17.0.222 and 172.17.0.223. </p>

<p>And I edited <code>slaves</code> and <code>spark-env.sh</code> files to add the IP addresses of my slaves and IP address of my master respectively.</p>

<p>I started the master node <code>start-master.sh</code> and started the slave nodes with <code>start-slaves.sh</code>, everything worked fine.</p>

<p>I submitted my spark-job using the command <code>spark-submit --class "Rice" --master spark://172.17.0.224:7077 cs453project/target/scala-2.11/simple-project_2.11-1.0.jar cs453project/input.txt cs453project/ouput2 cs453project/ouput3</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The path you used is probably only local to the driver.  You have to use a path that is accessible to all of the workers.  The driver does not send the actual data to the workers - that would be unfortunately slow.  The workers will try to read the data using the path you gave them.  In this case, they will fail because the don't have the files locally. </p>
