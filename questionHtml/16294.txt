<p>If you specify nothing after <code>-ls</code>, then the folders will be those in your "home" directory. If you want to give a path relative to your home folder, you can do so</p>

<pre><code>hadoop fs ls tmp/someTmpStuff
</code></pre>

<p>(assuming tmp is a folder in your home directory ) or use a fully qualified path</p>

<pre><code>hadoop fs ls /user/me/tmp/someTmpStuff
</code></pre>

<blockquote>
  <p>I need to go inside that tmp folder, took a look at Hadoop shell
  commands in here but still didn't find the command for it.
  <a href="http://hadoop.apache.org/docs/r0.18.3/hdfs_shell.html" rel="nofollow">http://hadoop.apache.org/docs/r0.18.3/hdfs_shell.html</a></p>
</blockquote>

<p>There is nothing like <strong>cd</strong> which can take us inside a directory. So you can't go to that folder like you can do in your local FS. You could probably use <strong>ls</strong> a others have suggested, but that just list the content inside a directory and doesn't take you to that directory. If you really want to go inside a particular directory, you could make use of the HDFS WebUI. You can point your web browser to <strong>NameNode_Machine:50070</strong> to go there. It allows you to browse the entire HDFS. You can view and download the files as well.</p>
