    <div class="post-text" itemprop="text">

<p>I am learning Hadoop, and would like to try the pseudo-distributed operation</p>

<p>When I try to use <code>start-all.sh</code> to start the Hadoop daemons, should I use a non-root user like foo-user or use root.</p>

<p>Using root has no problem, however, I am a little bit concerned about it. </p>

<p>Using a non-root user, foo-user, it complains that it doesn't have permission to files</p>

<pre><code>/var/run/hadoop/hadoop-foo-user-namenode.pid: permission denied
/var/run/hadoop/hadoop-foo-user-tasktracker-foohost.pid: permission denied
</code></pre>

<p>It was trying to create these two files in the directory <code>/var/run/hadoop</code></p>

<p>I tried <code>vim /var/run/hadoop/testfile</code>, and couldn't save. So turns out that foo-user doesn't have permission to write at <code>/var/run/hadoop</code></p>

<p>I checked the permission of <code>/var/run/hadoop</code></p>

<pre><code>drwxrwxr-x root hadoop 4096 Feb 8 23:42 hadoop
</code></pre>

<p>foo-user is in group hadoop, so should have write permission to <code>/var/run/hadoop</code>. Indeed, several other id files are created there, like the ...jobtracker.pid</p>

<p>So should I use root for <code>start-all.sh</code> or there is something wrong with the permission ( I am really confused)?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It's not recommended to start Hadoop as the root, below is quoted from Yahoo's Hadoop tutorial:</p>

<blockquote>
  <p>The user who owns the Hadoop instances will need to have read and
  write access to each of these directories. It is not necessary for all
  users to have access to these directories. Set permissions with chmod
  as appropriate. In a large-scale environment, it is recommended that
  you create a user named "hadoop" on each node for the express purpose
  of owning and running Hadoop tasks. For a single individual's machine,
  it is perfectly acceptable to run Hadoop under your own username. It
  is not recommended that you run Hadoop as root.</p>
</blockquote>

<p>Even though <code>foo-user</code> is in the group <code>hadoop</code> in the Linux filesystem, you still need to make sure 
 that <code>foo-user</code> is also a group member in HDFS (by default the group is called <code>supergroup</code>), you'll see what the group is when you do <code>hadoop fs -ls path_to_your_data</code>.</p>
