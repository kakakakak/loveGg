for line in sys.stdin:

    for sent in array:

# sort

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You are missing a lot of configurations and you need to define directories and such.  See here:</p>

<p><a href="http://wiki.apache.org/hadoop/QuickStart" rel="nofollow">http://wiki.apache.org/hadoop/QuickStart</a></p>

<p>Distributed operation is just like the pseudo-distributed operation described above, except:</p>

<ol>
<li>Specify hostname or IP address of the master server in the values for fs.default.name and mapred.job.tracker in conf/hadoop-site.xml. These are specified as host:port pairs.</li>
<li>Specify directories for dfs.name.dir and dfs.data.dir in conf/hadoop-site.xml. These are used to hold distributed filesystem data on the master node and slave nodes respectively. Note that dfs.data.dir may contain a space- or comma-separated list of directory names, so that data may be stored on multiple devices.</li>
<li>Specify mapred.local.dir in conf/hadoop-site.xml. This determines where temporary MapReduce data is written. It also may be a list of directories.</li>
<li>Specify mapred.map.tasks and mapred.reduce.tasks in conf/mapred-default.xml. As a rule of thumb, use 10x the number of slave processors for mapred.map.tasks, and 2x the number of slave processors for mapred.reduce.tasks.</li>
<li>List all slave hostnames or IP addresses in your conf/slaves file, one per line and make sure jobtracker is in your /etc/hosts file pointing to your jobtracker node</li>
</ol>

<pre><code>./bin/hadoop jar contrib/streaming/hadoop-0.20.2-streaming.jar \
