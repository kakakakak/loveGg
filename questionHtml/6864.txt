<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I'm using AWS EMR Hadoop 1.0.3, and it is possible to specify different directories and files based on k/v pairs. Use either of the following functions from the <code>MultipleOutputs</code> class:</p>

<pre class="lang-java prettyprint-override"><code>public void write(KEYOUT key, VALUEOUT value, String baseOutputPath)
</code></pre>

<p>or</p>

<pre class="lang-java prettyprint-override"><code>public &lt;K,V&gt; void write(String namedOutput, K key, V value,
                        String baseOutputPath)
</code></pre>

<p>The former <code>write</code> method requires the key to be the same type as the map output key (in case you are using this in the mapper) or the same type as the reduce output key (in case you are using this in the reducer). The value must also be typed in similar fashion.</p>

<p>The latter <code>write</code> method requires the key/value types to match the types specified when you setup the MultipleObjects static properties using the <code>addNamedOutput</code> function:</p>

<pre class="lang-java prettyprint-override"><code>public static void addNamedOutput(Job job,
                              String namedOutput,
                              Class&lt;? extends OutputFormat&gt; outputFormatClass,
                              Class&lt;?&gt; keyClass,
                              Class&lt;?&gt; valueClass)
</code></pre>

<p>So if you need different output types than the <code>Context</code> is using, you must use the latter <code>write</code> method.</p>

<p>The trick to getting different output directories is to pass a <code>baseOutputPath</code> that contains a directory separator, like this:</p>

<pre class="lang-java prettyprint-override"><code>multipleOutputs.write("output1", key, value, "dir1/part");
</code></pre>

<p>In my case, this created files named "dir1/part-r-00000".</p>

<p>I was not successful in using a <code>baseOutputPath</code> that contains the <code>..</code> directory, so all <code>baseOutputPath</code>s are strictly contained in the path passed to the <code>-output</code> parameter.</p>

<p>For more details on how to setup and properly use MultipleOutputs, see this code I found (not mine, but I found it very helpful; does not use different output directories). <a href="https://github.com/rystsov/learning-hadoop/blob/master/src/main/java/com/twitter/rystsov/mr/MultipulOutputExample.java" rel="nofollow">https://github.com/rystsov/learning-hadoop/blob/master/src/main/java/com/twitter/rystsov/mr/MultipulOutputExample.java</a></p>

<p>An Excerpt from the HDG book - </p>

<p>"In the old MapReduce API, there are two classes for producing multiple outputs: MultipleOutputFormat and MultipleOutputs. In a nutshell,  MultipleOutputs is more fully featured, but  MultipleOutputFormat has more control over the output directory structure and file naming.  MultipleOutputs in the new API combines the best features of the two multiple output classes in the old API."</p>

<p>It has an example on how you can control directory structure,file naming and output format using MultipleOutputs  API.</p>

<p>Similar to: <a href="http://stackoverflow.com/questions/14824215/hadoop-reducer-how-can-i-output-to-multiple-directories-using-speculative-execu">Hadoop Reducer: How can I output to multiple directories using speculative execution?</a></p>

<p>Basically you can write to HDFS directly from your reducer - you'll just need to be wary of speculative execution and name your files uniquely, then you'll need to implement you own OutputCommitter to clean up the aborted attempts (this is the most difficult part if you have truely dynamic output folders - you'll need to step through each folder and delete the attemps associated with aborted / failed tasks). A simple solution to this is to turn off speculative execution</p>
