    <div class="post-text" itemprop="text">

<p>I am new to Hadoop and have run into problems trying to run it on my Windows 7 machine. Particularly I am interested in running Hadoop 2.1.0 as its <a href="http://hadoop.apache.org/releases.html#25+August%2C+2013%3A+Release+2.1.0-beta+available">release notes</a> mention that running on Windows is supported. I know that I can try to run 1.x versions on Windows with Cygwin or even use prepared VM by for example Cloudera, but these options are in some reasons less convenient for me.</p>

<p>Having examined a tarball from <a href="http://apache-mirror.rbc.ru/pub/apache/hadoop/common/hadoop-2.1.0-beta/">http://apache-mirror.rbc.ru/pub/apache/hadoop/common/hadoop-2.1.0-beta/</a> I found that there really are some *.cmd scripts that can be run without Cygwin. Everything worked fine when I formated HDFS partition but when I tried to run hdfs namenode daemon I faced two errors: first, non fatal, was that winutils.exe could not be found (it really wasn't present in the tarball downloaded). I found the sources of this component in the Apache Hadoop sources tree and compiled it with Microsoft SDK and MSbuild. Thanks to detailed error message it was clear where to put the executable to satisfy Hadoop. But the second error which is fatal doesn't contain enough information for me to solve:</p>

<pre><code>13/09/05 10:20:09 FATAL namenode.NameNode: Exception in namenode join
java.lang.UnsatisfiedLinkError: org.apache.hadoop.io.nativeio.NativeIO$Windows.access0(Ljava/lang/String;I)Z
    at org.apache.hadoop.io.nativeio.NativeIO$Windows.access0(Native Method)
    at org.apache.hadoop.io.nativeio.NativeIO$Windows.access(NativeIO.java:423)
    at org.apache.hadoop.fs.FileUtil.canWrite(FileUtil.java:952)
    at org.apache.hadoop.hdfs.server.common.Storage$StorageDirectory.analyzeStorage(Storage.java:451)
    at org.apache.hadoop.hdfs.server.namenode.FSImage.recoverStorageDirs(FSImage.java:282)
    at org.apache.hadoop.hdfs.server.namenode.FSImage.recoverTransitionRead(FSImage.java:200)
...
13/09/05 10:20:09 INFO util.ExitUtil: Exiting with status 1
</code></pre>

<p>Looks like something else should be compiled. I'm going to try to build Hadoop from the source with Maven but isn't there a simpler way? Isn't there some option-I-know-not-of that can disable native code and make that tarball usable on Windows?</p>

<p>Thank you.</p>

<p>UPDATED. Yes, indeed. "Homebrew" package contained some extra files, most importantly winutils.exe and hadoop.dll. With this files namenode and datanode started successfully. I think the question can be closed. I didn't delete it in case someone face the same difficulty.</p>

<p>UPDATED 2. To build the "homebrew" package I did the following:</p>

<ol>
<li>Got sources, and unpacked them.</li>
<li>Read carefully BUILDING.txt.</li>
<li>Installed dependencies:<br>
3a) Windows SDK 7.1<br>
3b) Maven (I used 3.0.5)
3c) JDK (I used 1.7.25)<br>
3d) ProtocolBuffer (I used 2.5.0 - <a href="http://protobuf.googlecode.com/files/protoc-2.5.0-win32.zip">http://protobuf.googlecode.com/files/protoc-2.5.0-win32.zip</a>). It is enough just to put compiler (protoc.exe) into some of the PATH folders.<br>
3e) A set of UNIX command line tools (I installed Cygwin)  </li>
<li>Started command line of Windows SDK. Start | All programs | Microsoft Windows SDK v7.1 | ... Command Prompt (I modified this shortcut, adding option /release in the command line to build release versions of native code). All the next steps are made from inside SDK command line window)</li>
<li><p>Set up the environment:</p>

<p>set JAVA_HOME={path_to_JDK_root}  </p></li>
</ol>

<p>It seems that JAVA_HOME <em>MUST NOT</em> contain space!  </p>

<pre><code>set PATH={path_to_maven_bin};%PATH%  
set Platform=x64  
set PATH={path_to_cygwin_bin};%PATH%  
set PATH={path_to_protoc.exe};%PATH%  
</code></pre>

<ol>
<li>Changed dir to sources root folder (BUILDING.txt warns that there are some limitations on the path length so sources root should have short name - I used D:\hds)</li>
<li><p>Ran building process:</p>

<p>mvn package -Pdist -DskipTests  </p></li>
</ol>

<p>You can try without 'skipTests' but on my machine some tests failed and building was terminated. It may be connected to sybolic link issues mentioned in BUILDING .txt.
8. Picked the result in hadoop-dist\target\hadoop-2.1.0-beta (windows executables and dlls are in 'bin' folder) </p>

    <div class="post-text" itemprop="text">
<p>I had the same problem but with recent hadoop v. 2.2.0. Here are my steps for solving that problem:</p>

<ol>
<li><p>I've built <code>winutils.exe</code> from sources. Project directory:</p>

<p><code>hadoop-2.2.0-src\hadoop-common-project\hadoop-common\src\main\winutils</code></p>

<p>My OS: Windows 7. Tool for building: MS Visual Studio Express 2013 for Windows Desktop (it's free and can be loaded from <a href="http://www.microsoft.com/visualstudio/">http://www.microsoft.com/visualstudio/</a>).
Open Studio, <code>File -&gt; Open -&gt; winutils.sln</code>. Right click on solution on the right side -> <code>Build</code>.
There were a couple errors in my case (you might need to fix project properties, specify output folder).
Viola! You get <code>winutils.exe</code> - put it into hadoop's bin.</p></li>
<li><p>Next we need to build <code>hadoop.dll</code>.
Some woodoo magic here goes: open</p>

<p><code>hadoop-2.2.0-src\hadoop-common-project\hadoop-common\src\main\native\native.sln</code></p>

<p>in MS VS; right click on solution -> build.
I got a bunch of errors. I created manually several missed header files (don't ask me why they are missed in source tarball!):</p>

<p><a href="https://github.com/jerishsd/hadoop-experiments/tree/master/sources">https://github.com/jerishsd/hadoop-experiments/tree/master/sources</a></p>

<p>(and don't ask me what this project on git is for! I don't know - google pointed it out by searching header file names)
I've copied</p>

<p><code>hadoop-2.2.0-src\hadoop-common-project\hadoop-common\target\winutils\Debug\libwinutils.lib</code></p>

<p>(result of step # 1) into</p>

<p><code>hadoop-2.2.0-src\hadoop-common-project\hadoop-common\target\bin</code></p>

<p>And finally build operation produces hadoop.dll!
Put it again into hadoop's bin and happily run namenode!</p></li>
</ol>

<p>Hope my steps will help somebody.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I have followed following steps to install Hadoop 2.2.0</p>

<p>Steps to build Hadoop bin distribution for Windows</p>

<ol>
<li><p>Download and install Microsoft Windows SDK v7.1.</p></li>
<li><p>Download and install Unix command-line tool Cygwin.</p></li>
<li><p>Download and install Maven 3.1.1.</p></li>
<li><p>Download Protocol Buffers 2.5.0 and extract to a folder (say c:\protobuf).</p></li>
<li><p>Add Environment Variables JAVA_HOME, M2_HOME and Platform if not added already. 
Note : Variable name Platform is case sensitive. And value will be either x64 or Win32 for building on a 64-bit or 32-bit system. 
Edit Path Variable to add bin directory of Cygwin (say C:\cygwin64\bin), bin directory of Maven (say C:\maven\bin) and installation path of Protocol Buffers (say c:\protobuf).</p></li>
<li><p>Download hadoop-2.2.0-src.tar.gz and extract to a folder having short path (say c:\hdfs) to avoid runtime problem due to maximum path length limitation in Windows.</p></li>
<li><p>Select Start --> All Programs --> Microsoft Windows SDK v7.1 and open Windows SDK 7.1 Command Prompt. Change directory to Hadoop source code folder (c:\hdfs). Execute mvn package with options -Pdist,native-win -DskipTests -Dtar to create Windows binary tar distribution. </p></li>
<li><p>If everything goes well in the previous step, then native distribution hadoop-2.2.0.tar.gz will be created inside C:\hdfs\hadoop-dist\target\hadoop-2.2.0 directory.</p></li>
</ol>

<p><strong>Install Hadoop</strong></p>

<ol>
<li><p>Extract hadoop-2.2.0.tar.gz to a folder (say c:\hadoop).</p></li>
<li><p>Add Environment Variable HADOOP_HOME and edit Path Variable to add bin directory of HADOOP_HOME (say C:\hadoop\bin).</p></li>
</ol>

<p><strong>Configure Hadoop</strong></p>

<p><strong>C:\hadoop\etc\hadoop\core-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
                &lt;name&gt;fs.defaultFS&lt;/name&gt;
                &lt;value&gt;hdfs://localhost:9000&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>C:\hadoop\etc\hadoop\hdfs-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
                &lt;name&gt;dfs.replication&lt;/name&gt;
                &lt;value&gt;1&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
                &lt;name&gt;dfs.namenode.name.dir&lt;/name&gt;
                &lt;value&gt;file:/hadoop/data/dfs/namenode&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
                &lt;name&gt;dfs.datanode.data.dir&lt;/name&gt;
                &lt;value&gt;file:/hadoop/data/dfs/datanode&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>C:\hadoop\etc\hadoop\mapred-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
           &lt;name&gt;mapreduce.framework.name&lt;/name&gt;
           &lt;value&gt;yarn&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>C:\hadoop\etc\hadoop\ yarn-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
           &lt;name&gt;yarn.nodemanager.aux-services&lt;/name&gt;
           &lt;value&gt;mapreduce_shuffle&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
           &lt;name&gt;yarn.nodemanager.aux-services.mapreduce.shuffle.class&lt;/name&gt;
           &lt;value&gt;org.apache.hadoop.mapred.ShuffleHandler&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>Format namenode</strong></p>

<p>For the first time only, namenode needs to be formatted.</p>

<pre><code>C:\Users\abhijitg&gt;cd c:\hadoop\bin 
c:\hadoop\bin&gt;hdfs namenode –format
</code></pre>

<p><strong>Start HDFS (Namenode and Datanode)</strong></p>

<pre><code>C:\Users\abhijitg&gt;cd c:\hadoop\sbin
c:\hadoop\sbin&gt;start-dfs
</code></pre>

<p><strong>Start MapReduce aka YARN (Resource Manager and Node Manager)</strong></p>

<pre><code>C:\Users\abhijitg&gt;cd c:\hadoop\sbin
c:\hadoop\sbin&gt;start-yarn
starting yarn daemons
</code></pre>

<p>Total four separate Command Prompt windows will be opened automatically to run <strong>Namenode, Datanode, Resource Manager, Node Manager</strong></p>

<p>Reference : <a href="http://www.srccodes.com/p/article/38/build-install-configure-run-apache-hadoop-2.2.0-microsoft-windows-os">Build, Install, Configure and Run Apache Hadoop 2.2.0 in Microsoft Windows OS</a></p>

<p>Han has prepared the Hadoop 2.2 Windows x64 binaries (see his <a href="http://blog.newitfarmer.com/big_data/big-data-platform/hadoop/13878/repost-error-util-shell-failed-to-locate-the-winutils-binary-in-the-hadoop-binary-path">blog</a>) and <a href="https://github.com/srccodes/hadoop-common-2.2.0-bin">uploaded them to Github</a>.</p>

<p>After putting the two binaries <code>winutils.exe</code> and <code>hadoop.dll</code> into the <code>%hadoop_prefix%\bin</code> folder, I got the same <code>UnsatisfiedLinkError</code>.</p>

<p>The problem was that some dependency of <code>hadoop.dll</code> was missing. I used <a href="http://www.dependencywalker.com/">Dependency Walker</a> to check the dependencies of the binaries and the <a href="http://www.microsoft.com/en-us/download/details.aspx?id=14632">Microsoft Visual C++ 2010 Redistributables</a> were missing.</p>

<p>So besides building all the components yourself, the answer to the problem is</p>

<ul>
<li>make sure to use the same architecture for Java and the native code. <code>java -version</code> tells you if you use 32 or x64.</li>
<li>then use Dependency Walker to make sure all native binaries are pure and of the same architecture. Sometimes a x64 dependency is missing and Windows falls back to x86, which does not work. See <a href="http://stackoverflow.com/questions/19943766/hadoop-unable-to-load-native-hadoop-library-for-your-platform-error-on-centos">answer of another question</a>.</li>
<li>also check if all dependencies of the native binaries are satisfied.</li>
</ul>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I ran into same problem with Hadoop 2.4.1 on Windows 8.1; there were a few differences with the resulting solution caused mostly by the newer OS.</p>

<p>I first installed Hadoop 2.4.1 binary, unpacking it into %<strong>HADOOP_HOME</strong>%.</p>

<p>The previous answers describe how to set up Java, protobuf, cygwin, and maven, and the needed environment variables. I had to change my <strong>Platform</strong> environment variable from HP's odd 'BCD' value.</p>

<p>I downloaded the source from an Apache mirror, and unpacked it in a short directory (<strong>HADOOP_SRC</strong> = C:\hsrc). Maven ran fine from a standard Windows command prompt in that directory: <strong>mvn package -DskipTests</strong>.</p>

<p>Instead of using the Windows 7 SDK (which I could not get to load) or the Windows 8.1 SDK (which doesn't have the command line build tools), I used the free <strong>Microsoft Visual Studio Express 2013 for Windows Desktop</strong>. Hadoop's build needed the MSBuild location (C:\Program Files (x86)\MSBuild\12.0) in the PATH, and required that the various Hadoop native source projects be upgraded to the newer (MS VS 2013) format. The maven build failures were nice enough to point out the absolute path of each project as it failed, making it easy to load the project into Visual Studio (which automatically converts, after asking).</p>

<p>Once built, I copied the native executables and libraries into the Hadoop bin directory. They were built in %<strong>HADOOP_SRC</strong>%\hadoop-common-project\hadoop-common\target\bin, and needed to be copied into %<strong>HADOOP_HOME</strong>%\bin.</p>
