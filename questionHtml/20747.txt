<p>I have spent weeks trying to resolve different errors in building Hadoop. SO was helpful in pointing me towards the answer to an occasional problem, but after a lot of searching here on SO, I was never able to get the whole thing to build. </p>

<p>It’s been a couple of weeks since all this started so I have forgotten most of the explicit error messages, but the problems I had included </p>

<ul>
<li>Protobuff versions being wrong</li>
<li>SSH connections not working</li>
<li>Mojofailure Exceptions during build</li>
<li>Incorrect Java versions being used</li>
<li>C++ sanity checks failing</li>
<li>a host of other crap that made no sense to me and I couldn't decipher root causes for</li>
</ul>

<p>Today I finally got Hadoop to build from the git repo source and wanted to record the process for the SO community members that face similar problems.</p>

<p>For those of you trying to build Hadoop from source, here is how I got everything to compile from source.</p>

<p>Some notes on configuration:</p>

<ul>
<li>I am installing Hadoop in a virtual environment, in my case VirtualBox. </li>
<li>The Host machine runs Windows 7 x64</li>
<li>The Guest VM runs CentOS 7 x64</li>
<li>I am aiming for the bare minimum installation</li>
</ul>

<p><strong>How to Build Hadoop From Source Without Errors</strong></p>

<p>Preliminary Downloads:
You need to download the following before you begin.</p>

<ul>
<li>Virtual Box (I used version 4.3.16 r95972 available here: <a href="https://www.virtualbox.org/wiki/Download_Old_Builds_4_3_pre24" rel="nofollow">old VB builds</a>)</li>
<li>CentOS 7 minimal iso file from - <a href="http://www.centos.org/download/" rel="nofollow">http://www.centos.org/download/</a></li>
<li>WinSCP (version 5.7.4) - <a href="https://winscp.net/eng/download.php" rel="nofollow">https://winscp.net/eng/download.php</a></li>
</ul>

<p>This walk through consists of 4 Phases</p>

<ol>
<li>Create a CentOS Appliance inside VirtualBox that can support
building Hadoop</li>
<li>Add SSH capabilities to the Appliance so that downloaded prerequisites can be scp’ed from the Host to the Guest VM</li>
<li>Install all the things (utilities and dependencies) needed to build Hadoop</li>
<li>Build Hadoop without errors</li>
</ol>

<p><strong>Phase 1 - Creating a CentOS Appliance for VirtualBox</strong></p>

<p>Start by opening VirtualBox and clicking on the “New” button in the top left corner. This will open a new window asking for some information about the virtual machine appliance you want to create. </p>

<ul>
<li>Name it “CentOS x64 – Hadoop Base” </li>
<li>Select Linux as the “Type” of operating system</li>
<li>Select RedHat (64 Bit) as the “Version.” </li>
<li>Click “Next”</li>
</ul>

<p><img src="http://i.stack.imgur.com/z2Cpz.png" alt="Create New Appliance"></p>

<p>Follow the remaining prompts in the VM creation wizard. The only things I changed from the defaults where on the “Memory size” passage (I used 4096 MB) and the “File location and size” passage (I used 128 GB). I would encourage you to do the same if your system can support it. <strong>Leave all other defaults alone</strong></p>

<ul>
<li>Click “Create” on the last passage of the VM creation wizard</li>
</ul>

<p>Once created, the VM will show up on the left hand pane of the VirtualBox Window.</p>

<p><img src="http://i.stack.imgur.com/Itupp.png" alt="new VM"></p>

<ul>
<li>Double click on the VM you just created and wait for the dialog to come up asking you for the iso file you want to use. </li>
<li>When the dialog appears, click on the folder icon on the right and navigate to / select the “CentOS minimal iso” you downloaded during the Preliminary steps. </li>
<li>Once the iso is listed in the drop down box Click “Start”</li>
</ul>

<p><img src="http://i.stack.imgur.com/yKaUA.png" alt="iso selection"></p>

<p>When prompted, after the VM boots, select “Install CentOS 7” (this is not the default, you have to press the “up” arrow) and press “Enter”. When the setup program loads, the first thing it will ask you about is your keyboard layout. I leave the defaults in place and just click the “Continue” button in the lower right corner. This brings up the Installation Summary page on which you need to make changes to 2 areas: “Installation Destination” and “Network &amp; Host Name”</p>

<p><img src="http://i.stack.imgur.com/Aum0N.png" alt="installation summary"></p>

<ul>
<li>Click “Installation Destination” </li>
<li>Double Click the virtual disk (make sure that the background is blue <strong>and</strong> the check mark is there) </li>
<li>Click “Done” to go back to the "Installation Summary" page. </li>
</ul>

<p><img src="http://i.stack.imgur.com/m2O5q.png" alt="installation destination"></p>

<p>Back on the Installation Summary page, 
 - Click “Network and Host Name” 
 - In this menu screen turn on Ethernet networking by clicking the toggle switch on the right. 
 - Click “Done” in the top left corner.</p>

<p><img src="http://i.stack.imgur.com/rjiz8.png" alt="network enabling"></p>

<p>With both modifications complete you can click the “Begin Installation” button in the bottom right corner. As the iso installs to your system you should take the time to provide a root password by </p>

<ul>
<li>Clicking on that option at the top left of the page</li>
<li>Filling out the form it brings up</li>
<li>Clicking “Done” (if you select a password considered weak, you have to double click “Done” to accept anyway). </li>
</ul>

<p>I added a password, but I did not bother to add any non-root users.</p>

<p>Once everything is installed click on the “Reboot” button that appears in the bottom right of the screen.</p>

<p><img src="http://i.stack.imgur.com/92v1U.png" alt="CentOS installed"></p>

<p>Once the system reboots select CentOS 7 and allow it to boot. Check your credentials by logging in as root, and then close the CentOS VM by clicking on the red X button at the top right of the window and selecting “Power off the machine” when prompted.</p>

<p><strong>This completes Phase 1</strong></p>

<p>You should now be looking at just VirtualBox</p>

<p><strong>Phase 2 - Adding SSH capabilities to the VM to support download transfers</strong></p>

<ul>
<li>Open the settings of your CentOS Appliance by first clicking the appliance</li>
<li>Next, click the “Settings” button on the top left of VirtualBox’s main menu. This will bring up a new window. </li>
<li>In the left hand pane of the new window, click on “Network” which will display a set of adapter tabs. </li>
<li>Now click on the Triangle to the left of the label “Advanced”. </li>
<li>This will reveal a series of options, but the one you need to click on is the button labeled “Port Forwarding”</li>
</ul>

<p><img src="http://i.stack.imgur.com/39qLy.png" alt="Port Forwarding"></p>

<p>This will bring up another window where you can set port forwarding rules. </p>

<ul>
<li>Click the green plus sign in the top right corner. This will produce a row where you can enter in a port forwarding rule. </li>
<li>Add the following rule to the row</li>
</ul>

<blockquote>
  <p>Name= ssh, Host port =2222, Guest port = 22</p>
</blockquote>

<ul>
<li>Click the “OK” button on the Port Forwarding window</li>
<li>Click the “OK” button on the Appliance Settings window.</li>
</ul>

<p><img src="http://i.stack.imgur.com/ezuC9.png" alt="Port forwarding rules"></p>

<p>With this rule in place you should now be able to ssh from your Windows Host to the CentOS Guest on port 2222 and avoid the following error:</p>

<blockquote>
  <p>ssh: connect to host localhost port 22: Connection refused</p>
</blockquote>

<p>You should now be looking at just VirtualBox again.</p>

<ul>
<li>Start the CentOS VM appliance and log in as root. </li>
<li>Once logged in, execute the following line from the command prompt.</li>
</ul>

<p><code>yum –y install openssh-server openssh-client</code></p>

<p>This command will install a ssh server on the CentOS VM. After the install, confirm that the ssh server is running by typing the following command.</p>

<p><code>ps –aux | grep sshd</code></p>

<p>This command should return 2 processes showing sshd (the ssh daemon). One is the grep command itself. The other is your server running in the background.</p>

<p>Now we need to make sure that ssh did in fact generate the keys it will need to communicate with WinSCP.  Issue the following command and make sure that all keys’ byte size values are non-zero. </p>

<p><code>ls -l /etc/ssh</code></p>

<p><img src="http://i.stack.imgur.com/lZFw9.png" alt="good ssh keys"></p>

<blockquote>
  <p><em>If the sizes of the keys are 0 bytes, you need to remove them, restart the sshd daemon, and validate that the keys were regenerated when sshd restarted. To do all that, execute the following commands</em></p>
</blockquote>

<p><code>rm –rf /etc/ssh/ssh*key*
 systemctl restart sshd
 ls -l /etc/ssh</code></p>

<blockquote>
  <p>This processes will help avoid unexpected “connection closed by 127.0.0.1” errors.</p>
</blockquote>

<p>Now that we have an ssh daemon up and keys generated, we are going to test the connection. Start by opening WinSCP. And entering in the following values on the start menu that pops up.</p>

<blockquote>
  <p>Host name = localhost, Port number = 2222, User name = root, Password = , File Protocol = SCP.</p>
</blockquote>

<p>Note that you need to set “File Protocol” last. If you don’t, it will try to outsmart you when you enter in a “Port number” that it isn’t expecting. When all the values are entered. Click the “Login” Button and accept / click Update or OK to any security warnings you get.</p>

<p><img src="http://i.stack.imgur.com/Y2tet.png" alt="WinSCP Startup"></p>

<p>Once you have logged in, move a file between the Host and VM Guest to confirm everything is working. </p>

<p>Though I won’t focus on it here, you can also us Cygwin to connect to the VM, and it is useful for diagnosing connection problems. The command you need to enter to get verbose diagnostic output is</p>

<p><code>ssh –vvv –p 2222 root@localhost</code></p>

<p><strong>This completes Phase 2</strong></p>

<p><strong>Phase 3 - Install Utilities and Dependencies Needed to Build Hadoop</strong></p>

<p>Our CentOS distribution really is “barebones” and so we need to install everything required to build Hadoop. We will do this by downloading most things in Windows and then moving them over to the VM via WinSCP.</p>

<p>Before we start, we need to add a “downloads” directory to the home directory of the root user on the CentOS VM by issuing the following command at the CentOS command line.</p>

<p><code>mkdir ~/downloads/</code></p>

<p>We can now begin downloading Hadoop dependencies. We will download everything to Windows and then use WinSCP to move it over to the VM.</p>

<p>Start by downloading the Java 7 JDK from - <a href="http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html" rel="nofollow">http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html</a></p>

<p>Ignore the “End of Public Updates” error message at the top of the page. Java 7 is what Apache recommends.</p>

<blockquote>
  <p>You want to download the jdk-7u79-linux-x64.rpm file </p>
</blockquote>

<p>Once downloaded use WinSCP to navigate to the Downloads directory of the Host computer and the newly created “downloads” directory of the Guest VM (you may need to click the refresh icon on the VM side of the WinSCP pane to see the directory). Drag and drop the jdk file from the Host over to the VM Guest.</p>

<p><img src="http://i.stack.imgur.com/06ZvN.png" alt="WinSCP java transfer"></p>

<p>Now we just need to install the JDK on the CentOS VM. From the CentOS command line change your directory to the “downloads” folder we created under root’s home, once in the “downloads” directory use rpm to install java 7.</p>

<p><code>cd ~/downloads</code>
<code>rpm –ihv jdk-7u79-linux-x64.rpm</code></p>

<p>Once installation is complete, you can verify it by typing </p>

<p><code>java –version</code></p>

<p>Which will produce output stating that you have a Java Run Time Environment installed.</p>

<p>Next we are going to install a subset of the packages Hadoop needs to build successfully. The list is taken straight from the Apache website: <a href="https://wiki.apache.org/hadoop/HowToContribute" rel="nofollow">https://wiki.apache.org/hadoop/HowToContribute</a> and the command we need to enter on the command line to retrieve them is:</p>

<p><code>yum -y install  lzo-devel  zlib-devel  gcc autoconf automake libtool openssl-devel fuse-devel</code></p>

<p>Next we are going to install Apache’s Maven. You can download it here:<br>
<a href="https://archive.apache.org/dist/maven/binaries/" rel="nofollow">https://archive.apache.org/dist/maven/binaries/</a></p>

<p>Apache’s website says you can use version 3+. I used version 3.2.2 so download this file to follow along:</p>

<blockquote>
  <p>apache-maven-3.2.2-bin.tar.gz </p>
</blockquote>

<p>Once you have the file downloaded, use WinSCP to move it from your host computer to the Guest VM ‘s “downloads” folder just like you did with the JDK file. We then untar the file into the /usr/local/ directory, and create a symbolic link in the /usr/local/ directory that points to the maven folder with the following three commands.</p>

<p><code>tar xzf apache-maven-3.2.2-bin.tar.gz -C /usr/local</code></p>

<p><code>cd /usr/local</code></p>

<p><code>ln -s apache-maven-3.2.2 maven</code></p>

<p>We now need to add Maven’s bin directory to the $PATH variable. We do so by editing the .bashrc file in root’s home directory. Open the file for editing in vi by using the following command</p>

<p><code>vi ~/.bashrc</code></p>

<p>This will bring up the bash file in the vi editor ( if you need it, a tutorial on vi can be found here: <a href="http://www.unix-manuals.com/tutorials/vi/vi-in-10-1.html" rel="nofollow">http://www.unix-manuals.com/tutorials/vi/vi-in-10-1.html</a> ) follow these instructions to correctly update the file. </p>

<ul>
<li>Enter Edit mode by pressing the “a” key</li>
<li>Add the following lines to the file:

<ul>
<li>export M2_HOME=/usr/local/maven</li>
<li>export PATH=$M2_HOME/bin:$PATH</li>
</ul></li>
<li>Press the “Esc” key to leave Edit mode</li>
<li>Type “:wq” – it will automatically show up at the vi command line (bottom left of the screen)</li>
<li>Press “Enter”</li>
</ul>

<p>Now log out of CentOS. Log back into CentOS, and check to make sure that the new PATH variable is appropriately set using the following commands.</p>

<p><code>exit</code></p>

<p><code>&lt;log back in as root&gt;</code></p>

<p><code>mvn –version</code></p>

<p>you should see output indicating that maven is currently installed</p>

<p><img src="http://i.stack.imgur.com/SDCh5.png" alt="maven installed"></p>

<p>Next we need to install C++ support for gcc. We do that with the following one line command</p>

<p><code>yum –y install gcc-c++.x86_64</code></p>

<p>Next we need to install git so that we can pull down the Hadoop source code.</p>

<p><code>yum –y install git</code></p>

<p>Once you have git. Go ahead and pull down the Hadoop source. There is still one more thing (ProtocolBuffer) we need before we can build the source code, but we need to see the BUILDING.txt file in the repo before we download ProtocolBuffer to make sure that we get the right version.</p>

<p>To get the Hadoop source we run the git clone command. Simply execute the following commands from the CentOS command line to download the Hadoop repo.</p>

<p><code>cd /usr/local</code></p>

<p><code>git clone git://git.apache.org/hadoop.git</code></p>

<p>The clone operation will place a “hadoop” directory in your /usr/local directory. When the operation has completed and you have the command prompt back, take a look at the BUILDING.txt file in your new hadoop directory using the following command:</p>

<p><code>less /usr/local/hadoop/BUILDING.txt</code></p>

<p>In the “Requirements” section of the file it states the version of ProtocolBuffer we need for Hadoop to build correctly. In this case it’s ProtocolBuffer 2.5.0. With this information in hand we go back to the command prompt by pressing “q” for quit.</p>

<p><img src="http://i.stack.imgur.com/wwbVK.png" alt="protobuff version"></p>

<p>Now we can finally, install the last of the things Hadoop needs: ProtocolBuffer. To get the right version of ProtocolBuffer, we visit the ProtocolBuffer release page:</p>

<p><a href="https://github.com/google/protobuf/releases" rel="nofollow">https://github.com/google/protobuf/releases</a></p>

<p>and scroll down until we see the version needed for Hadoop to compile.  For this walkthrough you want to download the following file.</p>

<blockquote>
  <p>protobuf-2.5.0.tar.gz</p>
</blockquote>

<p>Once downloaded, use WinSCP and transfer it to the VM’s “downloads” folder like you did earlier for the other downloads. Once the file is sitting in the VM’s “downloads” folder, issue the following commands to install ProtocolBuffer on CentOS</p>

<p><code>cd ~/downloads</code></p>

<p><code>tar xzf protobuf-2.5.0.tar.gz -C /usr/local</code></p>

<p><code>cd /usr/local/protobuf-2.5.0</code></p>

<p><code>./configure</code></p>

<p>`make'</p>

<p><code>make install</code></p>

<p>Once this is done all the prerequisite utilities and dependencies needed for building Hadoop will be installed.</p>

<p><strong>This completes Phase 3</strong></p>

<p><strong>Phase 4 - Build Hadoop Without Errors</strong></p>

<p>Go to the Hadoop directory, and run Maven skipping the tests using the following commands:</p>

<p><code>cd /usr/local/hadoop</code></p>

<p><code>mvn clean install -DskipTests</code></p>

<p>The build should now occur without any problems and when everything is finished, you should see a screen like the one below.</p>

<p><img src="http://i.stack.imgur.com/ARlYA.png" alt="Hadoop Built"></p>

<p><strong>This completes the walk through</strong></p>

<p>I hope some of you find it helpful.</p>
