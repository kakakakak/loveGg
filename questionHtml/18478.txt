<blockquote>
  <h1>Summary:</h1>
  
  <p>Is it possible to:</p>
  
  <ol>
  <li>Import data into Hadoop with the «MongoDB Connector for Hadoop».</li>
  <li>Process it with Hadoop MapReduce.</li>
  <li>Export it with Sqoop in a single transaction.</li>
  </ol>
</blockquote>

<p>I am building a web application with MongoDB. While MongoDB work well for most of the work,  in some parts I need stronger transactional guarantees, for which I use a MySQL database.</p>

<p>My problem is that I want to read a big MongoDB collection for data analysis, but the size of the collection means that the analytic job would take too long to process. Unfortunately, MongoDB's built-in map-reduce framework would not work well for this job, so I would prefer to carry out the analysis with <a href="http://hadoop.apache.org/" rel="nofollow">Apache Hadoop</a>.</p>

<p>I understand that it is possible read data from MongoDB into Hadoop by using the <a href="https://github.com/mongodb/mongo-hadoop" rel="nofollow">«MongoDB Connector for Hadoop»</a>, which reads data from MongoDB, processes it with MapReduce in Hadoop, and finally outputs the results back into a MongoDB database. </p>

<p>The problem is that I want the output of the MapReduce to go into a MySQL database, rather than MongoDB, because the results must be merged with other MySQL tables. </p>

<p>For this purpose I know that Sqoop can export result of a Hadoop MapReduce into MySQL.</p>

<p>Ultimately, I want too read MongoDB data then process it with Hadoop and finally output the result into a MySQL database.</p>

<p>Is this possible? Which tools are available to do this?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<blockquote>
  <p><strong>TL;DR:</strong> Set an an output formatter that writes to a RDBS in your Hadoop job:</p>

<pre><code> job.setOutputFormatClass( DBOutputFormat.class );
</code></pre>
</blockquote>

<p>Several things to note:</p>

<ol>
<li><p>Exporting data from MongoDB to Hadoop using Sqoop is not possible. This is because Sqoop uses <a href="http://www.oracle.com/technetwork/java/javase/jdbc/index.html" rel="nofollow">JDBC</a> which provides a call-level API for <em>SQL-based database</em>, but MongoDB <strong>is not an SQL-based database</strong>. You can look at the <a href="https://docs.mongodb.org/ecosystem/tools/hadoop/" rel="nofollow">«MongoDB Connector for Hadoop»</a> to do this job. The connector is available <a href="https://github.com/mongodb/mongo-hadoop" rel="nofollow">on GitHub</a>. (Edit: as you point out in your update.)</p></li>
<li><p>Sqoop exports are not made in a single transaction by default. Instead, according to the <a href="http://archive.cloudera.com/cdh/3/sqoop/SqoopUserGuide.html#_literal_sqoop_export_literal" rel="nofollow">Sqoop docs</a>:</p>

<blockquote>
  <p>Since Sqoop breaks down export process into multiple transactions, it is possible that a failed export job may result in partial data being committed to the database. This can further lead to subsequent jobs failing due to insert collisions in some cases, or lead to duplicated data in others. You can overcome this problem by specifying a staging table via the <code>--staging-table</code> option which acts as an auxiliary table that is used to stage exported data. The staged data is finally moved to the destination table in a single transaction.</p>
</blockquote></li>
<li><p>The «MongoDB Connector for Hadoop» does not seem to force the workflow you describe. According to the docs:</p>

<blockquote>
  <p>This connectivity takes the form of allowing both reading MongoDB data into Hadoop (for use in MapReduce jobs as well as other components of the Hadoop ecosystem), as well as writing the results of Hadoop jobs out to MongoDB.</p>
</blockquote></li>
<li><p>Indeed, as far as I understand from the <a href="https://github.com/mongodb/mongo-hadoop/tree/master/examples/" rel="nofollow">«MongoDB Connector for Hadoop»: examples</a>, it would be possible to specify a <a href="http://hadoop.apache.org/common/docs/current/api/org/apache/hadoop/mapred/lib/db/DBOutputFormat.html" rel="nofollow"><code>org.apache.hadoop.mapred.lib.db.DBOutputFormat</code></a> into your Hadoop MapReduce job to write the output to a MySQL database. Following the example from the connector repository:</p>

<pre><code>job.setMapperClass( TokenizerMapper.class );
job.setCombinerClass( IntSumReducer.class );
job.setReducerClass( IntSumReducer.class );
job.setOutputKeyClass( Text.class );
job.setOutputValueClass( IntWritable.class );
job.setInputFormatClass( MongoInputFormat.class );
/* Instead of:
 * job.setOutputFormatClass( MongoOutputFormat.class );
 * we use an OutputFormatClass that writes the job results 
 * to a MySQL database. Beware that the following OutputFormat 
 * will only write the *key* to the database, but the principle
 * remains the same for all output formatters
 */
job.setOutputFormatClass( DBOutputFormat.class );
</code></pre></li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Use MongoHadoop connector to read data from MongoDB and process it using Hadoop.</p>

<p>Link:
<a href="https://github.com/mongodb/mongo-hadoop/blob/master/hive/README.md" rel="nofollow">https://github.com/mongodb/mongo-hadoop/blob/master/hive/README.md</a></p>

<p>Using this connector you can use Pig and Hive to read data from Mongo db and process it using Hadoop.</p>

<p>Example of Mongo Hive table:</p>

<pre><code>  CREATE EXTERNAL TABLE TestMongoHiveTable
    ( 
    id STRING,
    Name STRING
    )
    STORED BY 'com.mongodb.hadoop.hive.MongoStorageHandler'
    WITH SERDEPROPERTIES('mongo.columns.mapping'='{"id":"_id","Name":"Name"}')
    LOCATION '/tmp/test/TestMongoHiveTable/' 
    TBLPROPERTIES('mongo.uri'='mongodb://{MONGO_DB_IP}/userDetails.json');
</code></pre>

<p>Once it is exported to hive table you can use Sqoop or Pig to export data to mysql.</p>

<p>Here is a flow.</p>

<p>Mongo DB -> Process data using Mongo DB hadoop connector (Pig) -> Store it to hive table/HDFS -> Export data to mysql using sqoop.</p>
