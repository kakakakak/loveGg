    <div class="post-text" itemprop="text">

<p>While Hive supports positive like queries: ex. </p>

<blockquote>
  <p>select * from table_name where column_name <strong>like 'root~%'</strong>;</p>
</blockquote>

<p>Hive <strong>Does not</strong> support negative like queries: ex.</p>

<blockquote>
  <p>select * from table_name where column_name <strong>not like 'root~%'</strong>;</p>
</blockquote>

<p>Does anyone know an equivalent solution that Hive does support?</p>

<p>Check out <a href="https://cwiki.apache.org/confluence/display/Hive/LanguageManual" rel="nofollow">https://cwiki.apache.org/confluence/display/Hive/LanguageManual</a> if you haven't. I reference it all the time when I'm writing queries for hive.</p>

<p>I haven't done anything where I'm trying to match part of a word, but you might check out <code>RLIKE</code> (in this section <a href="https://cwiki.apache.org/confluence/display/Hive/LanguageManual+UDF#Relational_Operators" rel="nofollow">https://cwiki.apache.org/confluence/display/Hive/LanguageManual+UDF#Relational_Operators</a>)</p>

<p>This is probably a bit of a hack job, but you could do a sub query where you check if it matches the positive value and do a <code>CASE</code> (<a href="http://wiki.apache.org/hadoop/Hive/LanguageManual/UDF#Conditional_Functions" rel="nofollow">http://wiki.apache.org/hadoop/Hive/LanguageManual/UDF#Conditional_Functions</a>) to have a known value for the main query to check against to see if it matches or not.</p>

<p>Another option is to write a UDF which does the checking. </p>

<p>I'm just brainstorming while sitting at home with no access to Hive, so I may be missing something obvious. :)</p>

<p>Hope that helps in some fashion or another. \^_^/</p>

<p>EDIT: Adding in additional method from my comment below.</p>

<p>For your provided example colName RLIKE '[^r][^o][^o][^t]~\w' That may not be the optimal REGEX, but something to look into instead of sub-queries</p>
