<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Are you sure by specifying a file name instead of directory's name?</p>

<p>This doesn't make sense to me:</p>

<pre><code>-Dmapred.input.dir=input/testDataFileModel.csv

mapred.input.dir=${wordcount.input.path}testDataFileModel.csv
</code></pre>

<p>Try to set <strong><em>-Dmapred.input.dir</em></strong> to the real dir where your file is in.</p>

<p>Hadoop is smart enough to iterate over all the files in the directory (not recursively, you have to configure it to do so).</p>

<p>Try this instead:</p>

<pre><code>&lt;jar-runner jar="recommender-engine-mapred-1.0.jar" main-class="org.apache.mahout.cf.taste.hadoop.item.RecommenderJob" run-at-startup="true" pre-action="setupScript" configuration-ref="hadoopConfiguration"&gt;
        &lt;arg value="-btrue"/&gt;
        &lt;arg value="-sorg.apache.mahout.math.hadoop.similarity.cooccurrence.measures.LoglikelihoodSimilarity"/&gt;
        &lt;arg value="-i${wordcount.input.path}"/&gt;
        &lt;arg value="-o${wordcount.input.path}"/&gt;    
&lt;/jar-runner&gt;
</code></pre>

<p>This is how you pass on the parameters (&lt; arg / >). Be careful not to put space after the '-b,-s,-i,-o' characters, because I have found that it makes a difference with and without space (it worked for me only without space).</p>
