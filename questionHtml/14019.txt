<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you just want to run hbase without going into zookeeper management for standalone Hbase, then remove the all the <code>property</code> blocks from <b>hbase-site.xml</b> except the property block with <code>hbase.rootdir</code> as name. <br>Now run <code>/bin/start-hbase.sh</code>. <br>Hbase comes with its own zookeeper , which gets started when you run <code>/bin/start-hbase.sh</code>, which will suffice if you are trying to get around things for the first time. Later you can put distributed mode configurations for zookeeper.<br>
You only need to run /sbin/start-dfs.sh for running hbase since value of <code>hbase.rootdir</code> is set to <code>hdfs://127.0.0.1:9000/hbase</code> in you hbase-site.xml. If you change it to some location on local filesystem using <code>file:///some_location_on_local_filesystem</code>, then you don't even need to run  /sbin/start-dfs.sh. <br><code>hdfs://127.0.0.1:9000/hbase</code> says its a place on hdfs and <code>/sbin/start-dfs.sh</code> starts namenode and datanode which provides underlying API to access the hdfs file system. For knowing about yarn , please look at <a href="http://hadoop.apache.org/docs/r2.3.0/hadoop-yarn/hadoop-yarn-site/YARN.html" rel="nofollow">http://hadoop.apache.org/docs/r2.3.0/hadoop-yarn/hadoop-yarn-site/YARN.html</a>.</p>

    <div class="post-text" itemprop="text">
<p>You need to start zookeeper and then run Hbase-shell </p>

<pre><code>{HBASE_HOME}/bin/hbase-daemons.sh {start,stop} zookeeper
</code></pre>

<p>and you may want to check this property in hbase-env.sh</p>

<pre><code># Tell HBase whether it should manage its own instance of Zookeeper or not.
export HBASE_MANAGES_ZK=false
</code></pre>

<p>Refer to <a href="http://hbase.apache.org/book/zookeeper.html#d248e13764" rel="nofollow">Source - Zookeeper</a></p>
