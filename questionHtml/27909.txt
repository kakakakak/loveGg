<td class="answercell">
    <div class="post-text" itemprop="text">
<p>One of the reason could be that default native library in Hadoop is built for 32-bit. </p>

<p>Add these lines to your .bashrc or .bash_profile:</p>

<pre><code>export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_INSTALL/lib/native
export HADOOP_OPTS="-Djava.library.path=$HADOOP_INSTALL/lib"
</code></pre>

<p>This worked for me on Ubuntu 12.04.</p>

<p>$HADOOP_INSTALL is the location where you install Hadoop (generally it is /usr/local/hadoop).</p>

<p>or even there may be HADOOP_HOME instead of HADOOP_INSTALL.</p>

<p>I think that the only problem here is the same as in <a href="http://stackoverflow.com/q/20011252/2693875">this question</a>, so the solution is also the same:</p>

<p><br/>
Stop JVM from printing the stack guard warning to stdout/stderr, because this is what breaks the HDFS starting script.</p>

<p><br/>
Do it by replacing in your <code>etc/hadoop/hadoop-env.sh</code> line:</p>

<pre><code>export HADOOP_OPTS="$HADOOP_OPTS -Djava.net.preferIPv4Stack=true"
</code></pre>

<p>with:</p>

<pre><code>export HADOOP_OPTS="$HADOOP_OPTS -XX:-PrintWarnings -Djava.net.preferIPv4Stack=true"
</code></pre>

<p><br/></p>

<p>(<a href="https://chawlasumit.wordpress.com/2014/06/17/hadoop-java-hotspottm-execstack-warning/" rel="nofollow">This solution</a> has been found on <a href="https://chawlasumit.wordpress.com/" rel="nofollow">Sumit Chawla's blog</a>)</p>

<p>This error is mostly happen due to mis-configuration of .bashrc. </p>

<p>add these below code in your .bashrc file at the end. </p>

<pre><code>#HADOOP VARIABLES START
