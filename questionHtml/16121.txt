<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You use Apache Kafka as a distributed and robust queue that can handle high volume data and enables you to pass messages from one end-point to another.</p>

<p>Storm is not a queue. It is a system that has distributed real time processing abilities, meaning you can execute all kind of manipulations on real time data in parallel.</p>

<p>The common flow of these tools (as I know it) goes as follows:</p>

<p>real-time-system --> Kafka --> Storm --> NoSql --> BI(optional)</p>

<p>So you have your real time app handling high volume data, sends it to Kafka queue. Storm pulls the data from kafka and applies some required manipulation. At this point you usually like to get some benefits from this data, so you either send it to some Nosql db for additional BI calculations, or you could simply query this NoSql from any other system.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Kafka and Storm have a slightly different purpose:</p>

<p><strong>Kafka</strong> is a distributed message broker which can handle big amount of messages per second. It uses publish-subscribe paradigm and relies on topics and partitions. Kafka uses Zookeeper to share and save state between brokers. So Kafka is basically responsible for transferring messages from one machine to another.</p>

<p><strong>Storm</strong> is a scalable, fault-tolerant, real-time analytic system (think like Hadoop in realtime). It consumes data from sources (Spouts) and passes it to pipeline (Bolts). You can combine them in the topology. So Storm is basically a computation unit (aggregation, machine learning).</p>

<hr>

<p>But you can use them together: for example your application uses kafka to send data to other servers which uses storm to make some computation on it.</p>

<h2>This is how it works</h2>

<p><strong>Kafka</strong> - To provide a realtime stream</p>

<p><strong>Storm</strong> - To perform some operations on that stream</p>

<p>You might take a look at the GitHub project <a href="https://github.com/abhishekgoel137/kafka-nodejs-d3js" rel="nofollow">https://github.com/abhishekgoel137/kafka-nodejs-d3js</a>.</p>

<p><strong>(D3js is a graph-representation library)</strong></p>

<p><strong>Ideal case:</strong></p>

<pre><code>Realtime application -&gt; Kafka -&gt; Storm -&gt; NoSQL -&gt; d3js
</code></pre>

<p><strong>This repository is based on:</strong></p>

<pre><code>Realtime application -&gt; Kafka -&gt; &lt;plain Node.js&gt; -&gt; NoSQL -&gt; d3js
</code></pre>
