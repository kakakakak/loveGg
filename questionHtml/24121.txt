    <div class="post-text" itemprop="text">

<p>I tried a simple example in Hadoop which is as follows:</p>

<p>I created a table:</p>

<pre><code>CREATE TABLE Empl(EmpID INT,EmpName STRING,SALARY FLOAT)
PARTITIONED BY (DOJ DATE)
ROW FORMAT DELIMITED 
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n';
</code></pre>

<p>Loaded the data:</p>

<pre><code>LOAD DATA LOCAL INPATH '/home/jugal/Desktop/example' INTO TABLE empl PARTITION(DOJ='2013-04-05');
</code></pre>

<p>Data in the file:</p>

<pre><code>101,Jugal,8000,'2015-04-04'
102,Kannappan,9000,'2015-04-04'
103,Prasanna,10000,'2014-04-01'
104,Hafiez,7000,'2015-04-09'
105,Ashwin,9500,'2013-04-05'
</code></pre>

<p>Every time I load the partitioned data the complete DOJ column is getting overwritten with the data I partition with. </p>

<p>To my understanding, when data is partitioned depending on a particular date, a directory is created, in which a file is created only with the matching partitioned date given.</p>

    <div class="post-text" itemprop="text">
<p>The LOAD DATA command only copy the files to the destination directory. It doesn't read the records of the input file, so it cannot do partitioning based on record values.</p>

<p>That is why your record gets overwritten with the static value you mentioned in partition.</p>

<p>In your case , I guess you are looking for dynamic partitioning refer <a href="https://cwiki.apache.org/confluence/display/Hive/Tutorial#Tutorial-Dynamic-PartitionInsert" rel="nofollow">this</a> </p>
