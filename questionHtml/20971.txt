    <div class="post-text" itemprop="text">

<p>I'm looking for a command line wrapper for the DEFLATE algorithm.</p>

<p>I have a file (git blob) that is compressed using DEFLATE, and I want to uncompress it. The gzip command does not seem to have an option to directly use the DEFLATE algorithm, rather than the gzip format.</p>

<p>Ideally I'm looking for a standard Unix/Linux tool that can do this.</p>

<p>edit: This is the output I get when trying to use gzip for my problem:</p>

<pre><code>$ cat .git/objects/c0/fb67ab3fda7909000da003f4b2ce50a53f43e7 | gunzip

gzip: stdin: not in gzip format
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can do this with the OpenSSL command line tool:</p>

<pre><code>openssl zlib -d &lt; $IN &gt; $OUT
</code></pre>

<p>Unfortunately, at least on Ubuntu, the <code>zlib</code> subcommand is disabled in the default build configuration (<code>--no-zlib</code> <code>--no-zlib-dynamic</code>), so you would need to compile <code>openssl</code> from source to use it. But it is enabled by default on Arch, for example.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Here's a example of breaking open a commit object in Python:</p>

<pre><code>$ git show
commit 0972d7651ff85bedf464fba868c2ef434543916a
# all the junk in my commit...
$ python
&gt;&gt;&gt; import zlib
&gt;&gt;&gt; file = open(".git/objects/09/72d7651ff85bedf464fba868c2ef434543916a")
&gt;&gt;&gt; data = file.read()
&gt;&gt;&gt; print data
# binary garbage
&gt;&gt;&gt; unzipped_data = zlib.decompress(data)
&gt;&gt;&gt; print unzipped_data
# all the junk in my commit!
</code></pre>

<p>What you will see there is almost identical to the output of 'git cat-file -p [hash]', except that command doesn't print the header ('commit' followed by the size of the content and a null byte).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I found this question looking for a work-around with a bug with the <code>-text</code> utility in the new version of the <code>hadoop dfs</code> client I just installed. The <code>-text</code> utility works like <code>cat</code>, except if the file being read is compressed, it transparently decompresses and outputs the plain-text (hence the name).</p>

<p>The answers already posted were definitely helpful, but some of them have one problem when dealing with Hadoop-sized amounts of data - they read everything into memory before decompressing.</p>

<p>So, here are my variations on the <code>Perl</code> and <code>Python</code> answers above that do not have that limitation:</p>

<p>Python:</p>

<pre><code>hadoop fs -cat /path/to/example.deflate |
  python -c 'import zlib,sys;map(lambda b:sys.stdout.write(zlib.decompress(b)),iter(lambda:sys.stdin.read(4096),""))'
</code></pre>

<p>Perl:</p>

<pre><code>hadoop fs -cat /path/to/example.deflate |
  perl -MCompress::Zlib -e 'print uncompress($buf) while sysread(STDIN,$buf,4096)'
</code></pre>

<p>Note the use of the <code>-cat</code> sub-command, instead of <code>-text</code>. This is so that my work-around does not break after they've fixed the bug. Apologies for the readability of the python version.</p>
