<p>I developed application which read data from Kinesis, performs calculations and store result to Redshift. Spark 1.5 + Python<br>
It works fine on one machine.<br>
I tried deploy on standalone cluster but I received message "Error: Cluster deploy mode is currently not supported for python applications on standalone clusters."<br>
I created cluster on EMR with<br>
Release label:emr-4.1.0<br>
Hadoop distribution:Amazon 2.6.0<br>
Applications:Spark 1.5.0  </p>

<p>I created "step" with "cluster" deploy mode  </p>

<p>spark-submit --deploy-mode cluster --driver-class-path /home/hadoop/jars/RedshiftJDBC4-1.1.7.1007.jar --py-files /home/hadoop/jars/deploy.zip --jars /home/hadoop/jars/spark-streaming-kinesis-asl-assembly_2.11-1.5.0.jar /home/hadoop/jars/Main_K_JDBC.py mvp5 5 /home/hadoop/jvm-error   </p>

<p>It's finished with "Completed" status within 22 seconds. No any error... No calculation.  </p>

<p>I created "step" with "client" deploy mode  </p>

<p>spark-submit --deploy-mode client --driver-class-path /home/hadoop/jars/RedshiftJDBC4-1.1.7.1007.jar --py-files /home/hadoop/jars/deploy.zip --jars /home/hadoop/jars/spark-streaming-kinesis-asl-assembly_2.11-1.5.0.jar /home/hadoop/jars/Main_K_JDBC.py mvp5 5 /home/hadoop/jvm-error </p>

<p>It started. Running.. But he did not read data from Kinesis and did not wrote data to Redshift. No errors...<br>
