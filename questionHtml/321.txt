<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In order to keep everything on the grid use hadoop streaming with a single reducer and cat as the mapper and reducer (basically a noop) - add compression using MR flags.</p>

<pre><code>hadoop jar \
    $HADOOP_PREFIX/share/hadoop/tools/lib/hadoop-streaming.jar \&lt;br&gt;
    -Dmapred.reduce.tasks=1 \
    -Dmapred.job.queue.name=$QUEUE \
    -input "$INPUT" \
    -output "$OUTPUT" \
    -mapper cat \
    -reducer cat
</code></pre>

<p>If you want compression add<br>
  -Dmapred.output.compress=true \
-Dmapred.output.compression.codec=org.apache.hadoop.io.compress.GzipCodec</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you set up fuse to mount your HDFS to a local directory, then your output can be the mounted filesystem. </p>

<p>For example, I have our HDFS mounted to <code>/mnt/hdfs</code> locally. I run the following command and it works great:</p>

<pre><code>hadoop fs -getmerge /reports/some_output /mnt/hdfs/reports/some_output.txt
</code></pre>

<p>Of course, there are other reasons to use fuse to mount HDFS to a local directory, but this was a nice side effect for us.</p>

<p>1) Get all the files from hdfs to a local file system</p>

<p>2) merge all files into a single file(using cat or other commands)</p>

<p>3) push the single file to hdfs again</p>

<p>Or we can use the following commands</p>

<p>1) hadoop fs -getmerge hadoop_input_files_dir mergedfile_on_local_filesystem</p>

<p>It will get all files from hdfs directory and merged to a single file on local fs</p>

<p>2) Push mergedfile_on_local_filesystem to hdfs using -copyFromLocal</p>

<p>hadoop fs -copyFromLocal mergedfile_on_local_filesystem hdfs_path_of_single_merged_file</p>
