<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As mentioned by miguno, Hadoop only considers the value of mapred.map.tasks as a hint.</p>

<p>That being said, when I was messing around with MapReduce I was able to increase the map count by specifying a max count. This might not work for you, but you might give it a shot.</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;mapred.tasktracker.map.tasks.maximum&lt;/name&gt;
    &lt;value&gt;60&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>NOTE: This value represents the TOTAL amount of maps. So if you want each of your (3) nodes to run 20 maps you have to specify mapred.map.tasks, like so:</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;mapred.map.tasks&lt;/name&gt;
    &lt;value&gt;20&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><em>This question seems to be a duplicate of <a href="http://stackoverflow.com/questions/6885441/setting-the-number-of-map-tasks-and-reduce-tasks">Setting the number of map tasks and reduce tasks</a>.</em></p>

<p>Hadoop <a href="http://markmail.org/message/k32nrcb2ncsq67ef?q=mapred.map.tasks%20" rel="nofollow">does not honor <code>mapred.map.tasks</code> beyond considering it a hint</a>.</p>

<p>See this <a href="http://wiki.apache.org/hadoop/HowManyMapsAndReduces" rel="nofollow">information on the Hadoop wiki</a>:</p>

<blockquote>
  <p>Actually controlling the number of maps is subtle. The mapred.map.tasks parameter is just a hint to the InputFormat for the number of maps. The default InputFormat behavior is to split the total number of bytes into the right number of fragments. However, in the default case the DFS block size of the input files is treated as an upper bound for input splits. A lower bound on the split size can be set via mapred.min.split.size. Thus, if you expect 10TB of input data and have 128MB DFS blocks, you'll end up with 82k maps, unless your mapred.map.tasks is even larger. Ultimately the InputFormat determines the number of maps.</p>
</blockquote>

<p>That said Hadoop does accept the user specified <code>mapred.reduce.tasks</code> and doesn't manipulate that.</p>

<p>In summary you cannot force <code>mapred.map.tasks</code> for a given MapReduce job but you can force <code>mapred.reduce.tasks</code>.</p>

<p>Edit: Going slightly beyond your direct question there is a way to indirectly force Hadoop to use more mappers.  This involves setting the combination of <code>mapred.min.split.size</code>, <code>dfs.block.size</code> and <code>mapred.max.split.size</code> appropriately.  Note that the actual sizes of the input files also play a role here. See <a href="http://stackoverflow.com/a/10083814/1743580">this answer for details</a>, which basically quotes from <em>Tom White's Hadoop: The Definite Guide</em> book.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Its primarily the input format that determines the number of map tasks. 
<a href="http://wiki.apache.org/hadoop/HowManyMapsAndReduces" rel="nofollow">http://wiki.apache.org/hadoop/HowManyMapsAndReduces</a>    </p>

<p>To your question, by default, the task tracker runs two map | reduce  tasks concurrently.<br>
To change that , set the property mapred.map.tasks.maximum  in /conf/mapred-site.xml</p>

<p>. Advised to go with the formula, (CPUS > 2) ? (CPUS * 0.75) : 1  while setting this.</p>
