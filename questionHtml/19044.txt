<p>I am connecting to a sockets API that is very inflexible.  It will return rows such as:</p>

<pre><code>NAME, CITY, STATE, JOB, MONTH
</code></pre>

<p>But will have duplicates because it does not do any aggregation.  I need to count the duplicate rows (which would be very easy in SQL, but not, as far as I know, in Java).</p>

<p>Example source data:</p>

<pre><code>NAME,     CITY, STATE, JOB,         MONTH
John Doe, Denver, CO, INSTALLATION, 090301
John Doe, Denver, CO, INSTALLATION, 090301
John Doe, Denver, CO, INSTALLATION, 090301
Jane Doe, Phoenix, AZ, SUPPORT, 090301
</code></pre>

<p>Intended:</p>

<pre><code>    NAME,    CITY, STATE,          JOB,  MONTH, COUNT
John Doe,  Denver,    CO, INSTALLATION, 090301,   3
Jane Doe, Phoenix,    AZ,      SUPPORT, 090301,   1
</code></pre>

<p>I can easily do this for approximately 100,000 return rows, but I am dealing with about 60 million in a month.  Any ideas?</p>

<p>Edit: Unfortunately, the rows are not returned sorted... nor is there an option through the API to sort them.  I get this giant mess of stuff that needs to be aggregated.  Right now I use an ArrayList and do indexOf(new row) to find if the item already exists, but it gets slower the more rows that there are. </p>

<p>Edit: For clarification, this would only need to be run once a month, at the end of the month.  Thank you for all of the responses</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You could use a HashSet to store the previous row with the same contents. (assuming your Row objects have proper .hashValue() and .equals() methods implemented.</p>

<p>Something like this perhaps:</p>

<pre><code>Set&lt;Row&gt; previousRows = new HashSet&lt;Row&gt;();
List&lt;Row&gt; rowsInOrder = new LinkedList&lt;Row&gt;();
</code></pre>

<p>Then in use (assuming further that you have an incrementCount() method to the Row class):</p>

<pre><code>Row newRow = getNextRow();
if(!previousRows.contains(newRow)){
    previousRows.put(newRow);
    rowsInOrder.add(newRow);
} 
previousRows.get(newRow).incrementCount();
</code></pre>

<p>If you don't care about the order in which the rows came in, you can get rid of the List and just use the Set.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Are you able to fit all the data in memory at once?  If you are putting it in an ArrayList, it sounds like you can.</p>

<p>If that is the case, you can just use an implementation of MultiSet, such as the one in <a href="http://google-collections.googlecode.com/svn/trunk/javadoc/index.html?com/google/common/collect/Multiset.html" rel="nofollow">Google collections</a></p>

<p>Then, you could just do insert all your rows into the multiset as follows</p>

<pre><code>Multiset&lt;Row&gt; rowMultiset = HashMultiset.create();
for (Row row: rows) {
  rowMultiset.add(row);
}
</code></pre>

<p>And you can iterate through, with a count, using something like:</p>

<pre><code>for (Multiset.Entry entry : rowMultiset.entrySet()) {
  System.out.println("row: "+entry.getElement()+", count: "+entry.getCount());
}
</code></pre>

<p>If you don't want to use an external library, you can do something similar using a HashMap mapping rows to integers. </p>

<p>If it is NOT the case that all your rows fit into memory, I think the simplest approach is just to insert the data into a database and do a query.  Databases are designed and optimized for large datasets which don't fit into memory.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Are the rows always returned sorted? ie. are the rows to be grouped always returned one after another? If the answer is yes:</p>

<p>1) Initialize a counter.</p>

<p>2) Keep track of the previous row that you just read and compare it to the current row. If it's the same, increment your counter. If it's different, record your row with the current counter value and reset the counter.</p>

<p>3) When you reach the last record, make sure to record the row with the current count.</p>

<p>This strategy will allow you to read in the large data sets in a stream and keep your program's memory footprint low while producing the more compact aggregate data you're after.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I can think of four ways to do this:</p>

<ul>
<li><p>If you have enough memory to hold representations of 60 million rows in memory (less duplicates), use a <code>HashMap&lt;Row, Integer&gt;</code> to represent the counts.</p></li>
<li><p>Store the rows in an RDB, and then use SQL to aggregate and count.</p></li>
<li><p>Write the rows to a big file and use classical merge sort it before counting the
 rows in a single pass.</p></li>
<li><p>Use something like Hadoop to spread the rows across multiple machine.</p></li>
</ul>

<p>The fact that you are expecting to be accumulating counts over the period of a month or more suggests that you need to consider the possibility that your application will need to be restarted.  That suggests that an RDB or file-based solution is required.</p>
