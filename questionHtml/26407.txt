    <div class="post-text" itemprop="text">

<p>I have access to a computer running Linux with 20 cores, 92 GB of RAM, and 100 GB of storage on an HDD. I would like to use Hadoop for a task involving a large amount of data (over 1M words, over 1B word combinations). <strong>Would pseudo-distributed mode or fully distributed mode be the best way to leverage the power of Hadoop on a single computer?</strong></p>

<p>For my intended use of Hadoop, experiencing data loss and having to re-run the job because of node failure are not large issues.</p>

<p><a href="https://ofirm.wordpress.com/2014/01/05/creating-a-virtualized-fully-distributed-hadoop-cluster-using-linux-containers/">This project</a> involving <a href="https://linuxcontainers.org/">Linux Containers</a> uses fully distributed mode. <a href="https://github.com/jweese/thrax/wiki/Pseudodistributed-hadoop">This article</a> describes pseudo-distributed mode; more detail can be found <a href="https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-common/SingleCluster.html#Pseudo-Distributed_Operation">here</a>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As per my understanding, you have a single machine with 20 cores. In this case no need of virtualizing it, because the VMs that you create will consume some resources from the total resources. The best option is to install Linux OS in the laptop, install hadoop in pseudo distributed mode and configure the available resources for container allocation. </p>

<p>You need CPU cores as well as memory for getting good performance. So 20 cores alone will not help you. You need good amount of physical memory also. 
You can refer this <a href="http://docs.hortonworks.com/HDPDocuments/HDP2/HDP-2.0.9.1/bk_installing_manually_book/content/rpm-chap1-11.html" rel="nofollow">document</a> for allocating memory.</p>

<p>The fundamental behind hadoop is distributed computing and storage for processing large data in a cost effective manner. So if you try to achieve multiple machines in the same parent machine (small machines) by using virtualization, it won't help you because lot of resources will be consumed by the OS of individual machines. Instead if you install hadoop in the machine and configure the resources properly to hadoop, the jobs will execute in multiple containers (depending upon the availability and requirement) and hence parallel processing will happen. Thus you can achieve the maximum performance out of the existing machine.</p>

<p>So the <strong>best option</strong> is to set up a <strong>pseudo distributed cluster</strong> and allocate resources properly. Pseudo distributed mode is a mode in which all the daemons run in a single machine. </p>

<p>With the hardware configuration that you shared, you can use the below configuration for your hadoop set up. This can handle enough load.</p>

<pre><code>(yarn-site.xml)    yarn.nodemanager.resource.memory-mb  = 81920
(yarn-site.xml)    yarn.scheduler.minimum-allocation-mb = 1024
(yarn-site.xml)    yarn.scheduler.maximum-allocation-mb = 81920
(yarn-site.xml)    yarn.nodemanager.resource.cpu-vcores = 16
(yarn-site.xml)    yarn.scheduler.minimum-allocation-vcores = 1
(yarn-site.xml)    yarn.scheduler.increment-allocation-vcores = 1
(yarn-site.xml)    yarn.scheduler.maximum-allocation-vcores = 16
(mapred-site.xml)  mapreduce.map.memory.mb  = 4096
(mapred-site.xml)  mapreduce.reduce.memory.mb   = 8192
(mapred-site.xml)  mapreduce.map.java.opts  = 3072
(mapred-site.xml)  mapreduce.reduce.java.opts   = 6144
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I don't think you can leverage the real benefits of Hadoop considering you have only one machine in your cluster. In my opinion, an easier and better alternative would be to:</p>

<ol>
<li><p>Please go ahead with Pseudo distributed mode and store your data in Hadoop.</p></li>
<li><p>Use any in memory database (<strong>Impala or Presto or Spark</strong>) on top of the data stored in hadoop.</p></li>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The best way to utilize all the cores is<br>
<strong>Method 1:</strong> to use virtualization if the hardware supports (install esxi or any of the hypervisors ) and create the VM instances of linux machines or install openstack cloud and create VMs so that you can  fully utilize the hardware.<br>
<strong>Method 2:</strong> Though this can be achieved by a easier method of installing a host OS on a machine and installing VMware or Virtualbox,but as there is two layers b/w hardware and the hadoop performance is little reduced compared to the <strong>Method 1</strong>.</p>

<p>after this you can install the hadoop flavour you like.</p>

<p>Its always better to use distributed mode because in pseudo distributed mode there is a chance that data loss in case of crash of the system as the replication factor is 1 whereas in distributed mode the replication factor by default is 3.
And as in the pseudo distributed mode each deamon spins up with one java thread even a loss in single thread may cause entire MR job to run again.</p>

<p>Edit:
Looking at the configuration of storage of 100 gb and almost equal memory(assuming data is less than 100 gb) you can go ahead with pseudo cluster and u can leverage the advantage of in memory processing using <strong>Spark</strong> as  spark supports mapreduce equalent operations along with SQL, dataframes,mllib,graphX (Python is also comes natively), or if u know R u can directly use R using spark 1.4 (as spark is faster than hadoop)</p>
