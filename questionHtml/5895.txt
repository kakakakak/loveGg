<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Apache Mahout is an OpenSource Machile Learning library, that can be used with or without MapReduce (Apache Hadoop).</p>

<p>It provides the folloeing algorithms implementation in Java:</p>

<ul>
<li>Collaborative Filtering</li>
<li>User and Item based recommenders</li>
<li>K-Means, Fuzzy K-Means clustering</li>
<li>Mean Shift clustering</li>
<li>Dirichlet process clustering</li>
<li>Latent Dirichlet Allocation</li>
<li>Singular value decomposition</li>
<li>Parallel Frequent Pattern mining</li>
<li>Complementary Naive Bayes classifier</li>
<li>Random forest decision tree based classifier</li>
</ul>

<p>You can read more:
<a href="http://mahout.apache.org/" rel="nofollow">http://mahout.apache.org/</a></p>

<p><a href="http://girlincomputerscience.blogspot.com.br/2010/11/apache-mahout.html" rel="nofollow">http://girlincomputerscience.blogspot.com.br/2010/11/apache-mahout.html</a></p>

<p><a href="http://www.ibm.com/developerworks/java/library/j-mahout/" rel="nofollow">http://www.ibm.com/developerworks/java/library/j-mahout/</a></p>

<p>I'm the author of a Java open-source software for frequent pattern mining. It offers algorithms for mining sequential patterns, association rules, frequent itemsets, etc. </p>

<p>Although it is not specifically designed for text mining, some of the algorithms could be applied in to mine frequent patterns in text. For example, if you want to find some sequences of words that appear often together in several sentences you could apply a sequential pattern mining algorithm. But to do that you would need to to some pre-processing before applying my software so that your text file are in the proper format.</p>

<p>You can check the software here:
<a href="http://www.philippe-fournier-viger.com/spmf/" rel="nofollow">http://www.philippe-fournier-viger.com/spmf/</a></p>
