    <div class="post-text" itemprop="text">
<p>I figured this out for people having issues with this.  For my scenario, it was a combination of issues.</p>

<pre><code>./bin/hadoop jar ~/myjar/cdf-11-16.jar CdfInputDriver -libjars cdfjava.jar -files libcdf.so,libcdfNativeLibrary.so input output
</code></pre>

<p>A couple things had thrown me in a loop.  Here are some things I checked.  If anyone has factual info on why these contributed to it working, it would be appreciated.</p>

<p>(for the linux newbie) If you are running hadoop using sudo make sure to include -E to include environment variables.</p>

<p>Make sure the third party .jar library is located on your masternode.  (seemed to be necessary but haven't confirmed with documentation... maybe my enviro variables were incorrect otherwise)</p>

<p>I was able to run this using Amazon EMR.  I uploaded .so files and .jars to s3, ssh'd into the master node of the cluster, installed s3cmd via <a href="http://blog.adaptovate.com/2013/06/installing-s3cmd-on-ec2-so-that-yum.html" rel="nofollow">http://blog.adaptovate.com/2013/06/installing-s3cmd-on-ec2-so-that-yum.html</a> , copied cdf-11-16.jar (mapreduce jar) and cdfjava.jar (third party jar) to the masternode with s3cmd get, and ran the job. I was able to reference to .so files on S3.</p>
