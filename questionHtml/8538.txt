<td class="answercell">
    <div class="post-text" itemprop="text">
<blockquote>
  <p>When I run maven test, java.lang.OutOfMemoryError happens. I google it for solutions and have tried to export MAVEN_OPTS=-Xmx1024m, but it did not work. </p>
</blockquote>

<p>Setting the <code>Xmx</code> options using <code>MAVEN_OPTS</code> does work, it does configure the JVM used to start Maven. That being said, the maven-surefire-plugin <a href="http://maven.apache.org/plugins/maven-surefire-plugin/test-mojo.html#forkMode">forks</a> a new JVM by default, and your <code>MAVEN_OPTS</code> are thus not passed.</p>

<p>To configure the sizing of the JVM used by the maven-surefire-plugin, you would either have to: </p>

<ul>
<li>change the <a href="http://maven.apache.org/plugins/maven-surefire-plugin/test-mojo.html#forkMode"><code>forkMode</code></a> to <code>never</code> (which is be a not so good idea because Maven won't be isolated from the test) ~or~ </li>
<li>use the <a href="http://maven.apache.org/plugins/maven-surefire-plugin/test-mojo.html#argLine"><code>argLine</code></a> parameter (the right way):</li>
</ul>

<p>In the later case, something like this:</p>

<pre><code>&lt;configuration&gt;
  &lt;argLine&gt;-Xmx1024m&lt;/argLine&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>But</strong> I have to say that I tend to agree with Stephen here, there is very likely something wrong with one of your test and I'm not sure that giving more memory is the right solution to "solve" (hide?) your problem.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The chances are that the problem is in one of the unit tests that you've asked Maven to run.</p>

<p>As such fiddling with the heap size is the wrong approach.  Instead, you should be looking at the unit test that has caused the OOME, and trying to figure out if it is the fault of the unit test or the code that it is testing.</p>

<p>Start by looking at the stack trace.  If there isn't one, run <code>mvn ... test</code> again with the <code>-e</code> option.</p>
