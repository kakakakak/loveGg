<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It is likely that you're using a pager such as <code>less</code> or <code>most</code> since the output of <code>ps aux</code> is longer than a screenful. If so, the following options will cause (or force) long lines to wrap instead of being truncated.</p>

<pre><code>ps aux | less -+S

ps aux | most -w
</code></pre>

<p>If you use either of the following commands, lines won't be wrapped but you can use your arrow keys or other movement keys to scroll left and right.</p>

<pre><code>ps aux | less -S    # use arrow keys, or Esc-( and Esc-), or Alt-( and Alt-) 

ps aux | most       # use arrow keys, or &lt; and &gt; (Tab can also be used to scroll right)
</code></pre>

<p>Lines are always wrapped for <code>more</code> and <code>pg</code>.</p>

<p>When <code>ps aux</code> is used in a pipe, the <code>w</code> option is unnecessary since <code>ps</code> only uses screen width when output is to the terminal.</p>

<p>Using the -auxww option I've been able to get the full path to output on my putty terminal window.</p>

<pre><code>darragh@darraghserver ~ $uname -a
SunOS darraghserver 5.10 Generic_142901-13 i86pc i386 i86pc

darragh@darraghserver ~ $which ps
/usr/bin/ps&lt;br&gt;

darragh@darraghserver ~ $/usr/ucb/ps -auxww | grep ps
darragh 13680  0.0  0.0 3872 3152 pts/1    O 14:39:32  0:00 /usr/ucb/ps -auxww
darragh 13681  0.0  0.0 1420  852 pts/1    S 14:39:32  0:00 grep ps
</code></pre>

<p>"ps aux" lists all processes executed by all users.  See man 'ps' for details.  The 'ww' sets unlimited width.</p>

<pre><code>-w         Wide output. Use this option twice for unlimited width.
w          Wide output. Use this option twice for unlimited width.
</code></pre>

<p>I found the answer on the following blog:<br>
<a href="http://www.snowfrog.net/2010/06/10/solaris-ps-output-truncated-at-80-columns/">http://www.snowfrog.net/2010/06/10/solaris-ps-output-truncated-at-80-columns/</a></p>

<p>you can set output format,eg to see only the command and the process id.</p>

<pre><code>ps -eo pid,args
</code></pre>

<p>see the man page of ps for more output format. alternatively, you can use the <code>-w</code> or <code>--width n</code> options.</p>

<p>If all else fails, here's another workaround, (just to see your long cmds)</p>

<pre><code>awk '{ split(FILENAME,f,"/") ; printf "%s: %s\n", f[3],$0 }' /proc/[0-9]*/cmdline
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Sorry to be late to the party but just found this solution to the problem.</p>

<p>The lines are truncated because ps insists on using the value of $COLUMNS, even if the output is not the screen at that moment. Which is a bug, IMHO. But easy to work around, just make
ps think you have a superwide screen, i.e. set COLUMNS high for the duration of the ps command. An example:</p>

<pre><code>$ ps -edalf                 # truncates lines to screen width
$ COLUMNS=1000 ps -edalf    # wraps lines regardless of screen width
</code></pre>

<p>I hope this is still useful to someone. All the other ideas seemed much too complicated :)</p>
