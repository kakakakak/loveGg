<p>I have been working on this problem for two days and still have not find the way.</p>

<p><strong>Problem</strong>: Our Spark installed via newest CDH 5 always complains about the lost of LzoCodec class, even after I install the HADOOP_LZO through Parcels in cloudera manager. <strong>We are running MR1 on CDH 5.0.0-1.cdh5.0.0.p0.47</strong>.</p>

<p><strong>Try to fix</strong>:
The configurations in <a href="https://www.cloudera.com/content/cloudera-content/cloudera-docs/CM4Ent/4.7.3/Cloudera-Manager-Installation-Guide/cmig_install_LZO_Compression.html">official CDH documentation about 'Using the LZO Parcel</a>' are also added but the problem is still there.</p>

<p>Most of the googled posts give similar advices to the above. I also suspect that the spark is trying to run against YARN that is not activated there; but I can not find the configuration in CMF or other posts about this topic.</p>

<p>Please give me some help if you know how to deal with it.</p>

    <div class="post-text" itemprop="text">
<p>Solved!! May the solution help others who encounter the same problem.</p>

<hr>

<p>In this tutorial, I will show you how to enable LZO compression on Hadoop, Pig
and Spark. I suppose that you have set up a basic hadoop installation
successfully (if not, please refer to other tutorials for <a href="http://hadoop.apache.org/docs/stable/">Hadoop installation
</a>).</p>

<p>You reach this page possibly because you encounter the same problem
as I encountered, usually starting with Java exception:</p>

<pre><code>Caused by: java.lang.ClassNotFoundException: Class com.hadoop.compression.lzo.LzoCodec not found.
</code></pre>

<p>As the Apache and Cloudera distributions are two of the most popular
distributions, configurations for both contexts are shown. Briefly, three
main steps would be walked through towards the final success:</p>

<ul>
<li>Installing <code>native-lzo</code> libraries</li>
<li>Installing <code>hadoop-lzo</code> library</li>
<li>Setting up environment variables correctly (the right part
consuming my most time)</li>
</ul>

<h2>Step1: Installing <code>native-lzo</code> libraries</h2>

<p>The <a href="http://www.oberhumer.com/opensource/lzo/">native-lzo library</a> is
required for the installation of <code>hadoop-lzo</code>. You can install them manually
or by facilitating the Package Manager (<strong>NOTE:</strong> Make sure all nodes in the
cluster have <code>native-lzo</code> installed.):</p>

<ul>
<li><p>On Mac OS:</p>

<pre><code>sudo port install lzop lzo2
</code></pre></li>
<li><p>On RH or CentOS:</p>

<pre><code>sudo yum install lzo liblzo-devel
</code></pre></li>
<li><p>On Debian or ubuntu:</p>

<pre><code>sudo apt-get install liblzo2-dev
</code></pre></li>
</ul>

<h2>Step2: Installing <code>hadoop-lzo</code> library</h2>

<h3>For Apache Hadoop</h3>

<p>As the LZO is GPL'ed, it not shipped with official Hadoop distribution which
takes Apache Software License. I recommend the <a href="https://github.com/twitter/hadoop-lzo&gt;">Twitter version</a> which is a forked version of
<a href="https://code.google.com/a/apache-extras.org/p/hadoop-gpl-compression&gt;">hadoop-gpl-compression</a> with
remarkable improvements. If you are running the official Hadoop, some
installation structures are provided the <a href="https://github.com/twitter/hadoop-lzo/blob/master/README.md&gt;">the documentation</a>.</p>

<h3>For Cloudera Distribution</h3>

<p>In Cloudera's CDH, <code>hadoop-lzo</code> is shipped to customers as parcels and you
can download and distribute it conviniently using the Cloudera Manager. By
default, the <code>hadoop-lzo</code> will be installed in
<code>/opt/cloudera/parcels/HADOOP_LZO</code>.</p>

<p>Here we show the configuration on our cluster:</p>

<ul>
<li>Cloudera CDH 5</li>
<li>HADOOP_LZO version 0.4.15</li>
</ul>

<h2>Step3: Setting up env variables</h2>

<h3>For Apache Hadoop/Pig</h3>

<p>The basic configuration is for Apache Hadoop, while Pig is piggying upon its
functionality.</p>

<ul>
<li><p>Set compression codecs libraries in <code>core-site.xml</code>:</p>

<pre><code>&lt;property&gt;
  &lt;name&gt;io.compression.codecs&lt;/name&gt;
  &lt;value&gt;org.apache.hadoop.io.compress.GzipCodec,
      org.apache.hadoop.io.compress.DefaultCodec,
      org.apache.hadoop.io.compress.BZip2Codec,
      com.hadoop.compression.lzo.LzoCodec,
      com.hadoop.compression.lzo.LzopCodec
  &lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
  &lt;name&gt;io.compression.codec.lzo.class&lt;/name&gt;
  &lt;value&gt;com.hadoop.compression.lzo.LzoCodec&lt;/value&gt;
&lt;/property&gt;
</code></pre></li>
<li><p>Set MapReduce compression configuration in <code>mapred-site.xml</code>:</p>

<pre><code>&lt;property&gt;
  &lt;name&gt;mapred.compress.map.output&lt;/name&gt;
  &lt;value&gt;true&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
  &lt;name&gt;mapred.map.output.compression.codec&lt;/name&gt;
  &lt;value&gt;com.hadoop.compression.lzo.LzoCodec&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
  &lt;name&gt;mapred.child.env&lt;/name&gt;
  &lt;value&gt;JAVA_LIBRARY_PATH=$JAVA_LIBRARY_PATH:/path/to/your/hadoop-lzo/libs/native&lt;/value&gt;
&lt;/property&gt;
</code></pre></li>
<li><p>Append <code>HADOOP_CLASSPATH</code> to <code>hadoop-env.sh</code>:</p>

<pre><code>HADOOP_CLASSPATH=$HADOOP_CLASSPATH:/opt/cloudera/parcels/CDH/lib/hadoop/lib/*
</code></pre></li>
</ul>

<h3>For Cloudera Distribution</h3>

<p>You can use the Cloudera Manager to enable the same previous settings via GUI
interface:</p>

<ul>
<li><p>For MapReduce component, change the configuration of corresponding keys as
above:</p>

<pre><code>&gt; **io.compression.codecs**
&gt; **mapred.compress.map.output**
&gt; **mapred.map.output.compression.codec**
&gt; **MapReduce Client safety valve for mapred-site.xml**
</code></pre></li>
<li><p>Edit <strong>MapReduce Client Environment Snippet for hadoop-env.sh</strong> to append the
<code>HADOOP_CLASSPATH</code> variable.</p></li>
</ul>

<p>At last, restart dependent services in right order and deploy the
configurations among all nodes. That's it!!. Then you can test the
functionality with command and get successful messages similar to below:</p>

<pre><code>   $ hadoop jar /path/to/hadoop-lzo.jar com.hadoop.compression.lzo.LzoIndexer lzo_logs
   $ 14/05/04 01:13:13 INFO lzo.GPLNativeCodeLoader: Loaded native gpl library
   $ 14/05/04 01:13:13 INFO lzo.LzoCodec: Successfully loaded &amp; initialized native-lzo library [hadoop-lzo rev 49753b4b5a029410c3bd91278c360c2241328387]
   $ 14/05/04 01:13:14 INFO lzo.LzoIndexer: [INDEX] LZO Indexing file datasets/lzo_logs size 0.00 GB...
   $ 14/05/04 01:13:14 INFO Configuration.deprecation: hadoop.native.lib is deprecated. Instead, use io.native.lib.available
   $ 14/05/04 01:13:14 INFO lzo.LzoIndexer: Completed LZO Indexing in 0.39 seconds (0.02 MB/s).  Index size is 0.01 KB.
</code></pre>

<h3>For Spark</h3>

<p>This consumes me much time because there are less information in previous
posts. But the solution is strightforward with previous experience.</p>

<p>No matter the Spark is installed via tar or the Cloudera Manager, you need
merely to append two path values to <code>spark-env.sh</code>:</p>

<pre><code>   SPARK_LIBRARY_PATH=$SPARK_LIBRARY_PATH:/path/to/your/hadoop-lzo/libs/native
   SPARK_CLASSPATH=$SPARK_CLASSPATH:/path/to/your/hadoop-lzo/java/libs
</code></pre>

<h2>Ralated posts and questions</h2>

<p>A comparison of LZO performance is given in <a href="http://blog.cloudera.com/blog/2009/11/hadoop-at-twitter-part-1-splittable-lzo-compression/">another place</a>. A
related question is also asked on <a href="http://stackoverflow.com/q/23441142/1320284">StackOverflow</a> but there are no solutions
about this up to the finish of this tutorial. You maybe also interested in how
to <a href="http://www.cloudera.com/content/cloudera-content/cloudera-docs/CM4Ent/latest/Cloudera-Manager-Installation-Guide/cmig_install_LZO_Compression.html">use the LZO Parcel from Cloudera</a>.</p>

    <div class="post-text" itemprop="text">
<p>For Hortonworks 2.3.0 with Ambari for Spark to work with LZO you need to add <strong>Custom spark-defaults</strong> properties.  I added:</p>

<ul>
<li>spark.driver.extraClassPath <code>/usr/hdp/current/hadoop-client/lib/hadoop-lzo-0.6.0.{{hdp_full_version}}.jar</code></li>
<li>spark.driver.extraLibraryPath <code>/usr/hdp/current/hadoop-client/lib/native:/usr/hdp/current/hadoop-client/lib/native/Linux-amd64-64</code></li>
</ul>

<p>This is based on the <a href="http://docs.hortonworks.com/HDPDocuments/HDP2/HDP-2.3.0/bk_upgrading_hdp_manually/content/upgrade-spark-22.html" rel="nofollow">HDP 2.3.0 upgrading SPARK 2.2</a> page (it has some typos).</p>
