<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I blogged about it for myself:</p>

<p><a href="http://codingwiththomas.blogspot.de/2011/04/graph-exploration-with-hadoop-mapreduce.html">http://codingwiththomas.blogspot.de/2011/04/graph-exploration-with-hadoop-mapreduce.html</a></p>

<p>But MapReduce isn't a good fit for these Graph analysis things. Better use BSP (bulk synchronous parallel) for that, Apache Hama provides a good graph API on top of Hadoop HDFS.</p>

<p>I've written a connected components algorithm with MapReduce here: (Mindist search) </p>

<p><a href="https://github.com/thomasjungblut/tjungblut-graph/tree/master/src/de/jungblut/graph/mapreduce">https://github.com/thomasjungblut/tjungblut-graph/tree/master/src/de/jungblut/graph/mapreduce</a></p>

<p>Also a BSP version for Apache Hama can be found here:</p>

<p><a href="https://github.com/thomasjungblut/tjungblut-graph/blob/master/src/de/jungblut/graph/bsp/MindistSearch.java">https://github.com/thomasjungblut/tjungblut-graph/blob/master/src/de/jungblut/graph/bsp/MindistSearch.java</a></p>

<p>The implementation isn't as difficult as in MapReduce and it is at least 10 times faster.
If you're interested, checkout the latest version in TRUNK and visit our mailing list.</p>

<p><a href="http://hama.apache.org/">http://hama.apache.org/</a></p>

<p><a href="http://apache.org/hama/mail-lists.html">http://apache.org/hama/mail-lists.html</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I don't really know if an API is available which has methods to find strongly connected components. But, I implemented the BFS algorithm to find distance from source node to all other nodes in the graph (the graph was a directed graph as big as 65 million nodes).</p>

<p>The idea was to explore the neighbors (distance of 1) for each node in one iteration and  feeding the output of reduce back to map, until the distances converge. The map emits the shortest distances possible from each node, and reduce updated the node with the shortest distance from the list. </p>

<p>I would suggest to check <a href="http://www.johnandcailin.com/blog/cailin/breadth-first-graph-search-using-iterative-map-reduce-algorithm" rel="nofollow">this out</a>. Also, <a href="http://blog.piccolboni.info/2010/07/map-reduce-algorithm-for-connected.html" rel="nofollow">this could help</a>. These two links would give you the basic idea about graph algorithms in map reduce paradigm (if you are already not familiar). Essentially, you need to twist the algorithm to use DFS instead of BFS. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You may want to look at the <a href="http://www.cs.cmu.edu/~pegasus/index.htm" rel="nofollow">Pegasus project</a> from Carnegie Mellon University. They provide an efficient - and elegant - implementation using MapReduce. They also provide binaries, samples and a very detailed documentation. </p>

<p>The implementation itself is based on the Generalized Iterative Matrix-Vector multiplication (GIM-V) proposed by U Kang in 2009. </p>

<blockquote>
  <p><a href="http://www.cs.cmu.edu/~ukang/papers/PegasusICDM2009.pdf" rel="nofollow">PEGASUS: A Peta-Scale Graph Mining System</a> - Implementation and
  Observations U Kang, Charalampos E. Tsourakakis, Christos Faloutsos In
  IEEE International Conference on Data Mining (ICDM 2009)</p>
</blockquote>

<p>EDIT:
The official implementation is actually limited to 2.1 billions nodes (node id are stored as integers). I'm creating a fork on github (<a href="https://github.com/placeiq/pegasus" rel="nofollow">https://github.com/placeiq/pegasus</a>) to share my patch and other enhancements (eg. Snappy compression).</p>
