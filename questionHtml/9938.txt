<p>as explained in the title, when i execute my Hadoop Program (and debug it in local mode) the following happens:</p>

<p><em><strong>1. All 10 csv-lines in my test data are handled correctly in the Mapper, the Partitioner and the RawComperator(OutputKeyComparatorClass) that is called after the map-step. But the OutputValueGroupingComparatorClass's and the ReduceClass's functions do NOT get executed afterwards.</em></strong> </p>

<p><strong>UPDATE</strong>
With the help of Chris (see comments) i found out, that my program was NOT started in localMode as i expected it: the <code>isLocal</code> variable in the <code>ReduceTask</code> class is set to <code>false</code>, though it should be <code>true</code>.</p>

<p>To me it is absolutely unclear why this happens, since the 3 options that have to be set to enable the standalone mode were set the right way. Surprisingly: tho the <code>local</code> setting was ignored, the "read from normal disc" setting wasnt, which is very strange imho, because i thought <code>local</code> mode and the <code>file:///</code> protocol are coupled. </p>

<p>During debugging <code>ReduceTask</code> i set the <code>isLocal</code> variable to true by evaluating <code>isLocal=true</code> in my debug view and then tried to execute the rest of the program. It did not work out and this is the stacktrace:</p>

<p>Sorry i didn't see this before, but you appear to have two important configuration properties set to final in your conf xml files, as denoted by the following log statements:</p>

<blockquote>
<pre><code>12/05/22 14:28:29 WARN conf.Configuration: file:/tmp/hadoop-ema/mapred/local/localRunner/job_local_0001.xml:a attempt to override final parameter: fs.default.name;  Ignoring.
12/05/22 14:28:29 WARN conf.Configuration: file:/tmp/hadoop-ema/mapred/local/localRunner/job_local_0001.xml:a attempt to override final parameter: mapred.job.tracker;  Ignoring.
</code></pre>
</blockquote>

<p>This means that your job is unable to actually run in local mode, it starts in local mode, but the reducer reads the serialized job configuration and determines it is not in local mode, and tried to fetch map outputs via the task tracker ports.</p>

<p>You said your fix was to rename the conf folder - this will default hadoop back to the default configuration, where these two properties are not marked as 'final'</p>
