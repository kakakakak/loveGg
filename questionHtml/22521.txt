    <div class="post-text" itemprop="text">

<p>I have been trying to run a Avro map-reduce on oozie. I specify the mapper and reducer class in the workflow.xml and provide other configs too. But it gives out an </p>

<pre><code>java.lang.RunTime Exception - class mr.sales.avro.etl.SalesMapper not org.apache.hadoop.mapred.Mapper
</code></pre>

<p>The same job when run directly on a hadoop cluster (and not via oozie) gets completed and gives the desired output. So it seems probable that I may be missing some oozie config. What I guess from the exception is that oozie requires the mapper to be a subclass of <code>org.apache.hadoop.mapred.Mapper</code> but Avro mappers have a different signature - they extend org.apache.avro.mapred.AvroMapper and this may be reason for the error.</p>

<p>So my question is how do I confiure oozie workflow/properties file to  allow it to run an Avro map-reduce job.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>With AVRO, you'll need to configure a few extra properties:</p>

<ul>
<li><code>org.apache.avro.mapred.HadoopMapper</code> is the actual mapper class you need to set (this implements the Mapper interface)</li>
<li><code>avro.mapper</code> property should name your <code>SalesMapper</code> class</li>
</ul>

<p>There are other properties for the combiner and reducer too - check the AvroJob source and the utility methods.</p>

<p>Another way of doing this is to examine the job.xml from a job you manually submitted, and copy over the relevant configuration properties to your oozie workflow.xml</p>
