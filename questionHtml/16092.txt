    <div class="post-text" itemprop="text">

<p>I'm trying to set my JAVA_HOME in Cygwin with this command:</p>

<pre><code>export JAVA_HOME="/cygdrive/c/Program Files/Java/jdk1.7.0_10"
</code></pre>

<p>But when I do <code>cd $JAVA_HOME</code>, I'd get this error:</p>

<pre><code>$ cd $JAVA_HOME
-bash: cd: /cygdrive/c/Program: No such file or directory
</code></pre>

<p>I tried quoting, and escaping the space (ie., <code>\</code>), but none worked.
Any idea what else would?</p>

<p>Thanks,</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I faced this problem too and I saw many posts but nothing really worked. There is a small trick that I did and things started working.</p>

<p>My JAVA_HOME was set to C:/Program Files/Java/jdk1.7.0_23.
The problem was with Program Files directory and I was getting the same error.</p>

<p>In Windows there is a short name created for every directory with a space which is without a space.
You can see it by running dir /X command on the command prompt.
The Short name for Program Files was PROGRA~1.</p>

<p>In the Windows env variable through My Computer I changed the JAVA_HOME to C:/PROGR~1/Java/jdk1.7.0_23 and in hadoop-env.sh I changed JAVA_HOME to /cygdrv/c/PROGRA~1/Java/jdk1.7.0_23.</p>

<p>It worked fine.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>To avoid using the tedious Windows environment variables, and also use the actual path string copied from Windows explorer, I suggest adding the following to your startup script:</p>

<pre><code>             TMP=`cygpath -sw "C:\Program Files\Java\jdk1.8.0_31"`
export JAVA_HOME=`cygpath -u $TMP`
</code></pre>

<p>The first <code>cygpath</code> invocation obtains a <strong>s</strong>hort, <strong>w</strong>indows path; the second converts it to <strong>u</strong>nix format, which works fine in cygwin.</p>

<p>This will also now work fine:</p>

<pre><code>$ cd $JAVA_HOME
</code></pre>

<p>Try using the DOS subst command to take the spaces of the JAVA_HOME  path name.

<p>In a DOS shell</p>

<pre><code>subst j: "C:/Program Files/Java/jdk1.7.0_45"
</code></pre>

<p>J:  is now an abbreviation for C:/Program Files/Java/jdk1.7.0_45</p>

<p>You can now cd to J:</p>

<p>now run Cygwin and </p>

<pre><code>export JAVA_HOME="J:"
</code></pre>
