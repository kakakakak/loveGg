<td class="answercell">
    <div class="post-text" itemprop="text">
<p>When you installed <code>Cygwin</code> and ran <code>ssh-host-config</code>, you might have created user <code>cyg_server</code>. So when you are connecting to <code>ssh</code>, you need to use the user <code>cyg_server@localhost</code>.</p>

<p>When you run ssh with that user, it will ask you for the password that you provided when you ran <code>ssh-host-config</code>.</p>

<pre><code>$ ssh cyg_server@localhost
cyg_server@localhost's password:
Last login: Mon Dec 31 01:14:44 2012 from ::1
cyg_server@polorumpus ~
$
</code></pre>

<p>I hope it will be useful.</p>

<p>I had to do these to fix the problem:</p>

<pre><code>chown &lt;USERNAME&gt; /var/log/sshd.log
chown -R &lt;USERNAME&gt; /var/empty
chown &lt;USERNAME&gt; /etc/ssh*
chmod 755 /var/empty
chmod 644 /var/log/sshd.log
</code></pre>

<p>Note that the  is my desired user name, not cyg_server.</p>

<p>I had the same problem, and it was generated by Putty authentication agent (Pageant): it was running with a private key. For some reason, it tried to use it to connect to localhost. If I closed Pageant (or removed the key), I could ssh to localhost.
But as I wanted to have Pageant running (to be a client of other ssh servers), I managed to modify the local sshd service, so that it 'ignored' in some way Pageant, to accept connections:
Modify /etc/ssh/sshd_config:</p>

<p>PubkeyAuthentication no</p>

<p>Now, it works</p>
