    <div class="post-text" itemprop="text">

<p>I have a bash-tab-completion script for Apache's Hadoop.  Normally, I use zsh as my day-to-day shell.  It tends to be pretty bash-like when I need it to be, but it looks like the tab-completion systems are radically different between them.  Is there a simple way to "convert" the existing bash-tab-completion definitions to work in zsh?  I don't want to invest a ton of time in this, but if it's easy I'd save a moderate amount of effort.</p>

    <div class="post-text" itemprop="text">
<p>From <a href="http://zshwiki.org/home/convert/bash">this page</a> (dated 2010/01/05):</p>

<blockquote>
  <p>Zsh can handle bash completions functions. The latest development version of zsh has a function bashcompinit, that when run will allow zsh to read bash completion specifications and functions. This is documented in the zshcompsys man page. To use it all you need to do is run bashcompinit at any time after compinit. It will define complete and compgen functions corresponding to the bash builtins.</p>
</blockquote>

<td class="answercell">

<pre><code>autoload -U +X compinit &amp;&amp; compinit
autoload -U +X bashcompinit &amp;&amp; bashcompinit
source /path/to/your/bash_completion_script
</code></pre>

<p>I am running zsh <code>zsh 5.0.2 (x86_64-apple-darwin13.0)</code> <strong>without any ~/.zshrc</strong> and the above sequence worked in a freshly spawned zsh shell.</p>

<p>Thanks to git-completion.bash script for the hint :D</p>

<hr>

<h2>Read-on for more details on above 3 lines:</h2>

<p>Bash has awesome in built auto completion support but the bash autocomplete scripts don't work directly zsh as zsh environment doesn't have the essential bash autocomplete helper functions like <code>compgen</code>, <code>complete</code>. It does so in an effort to keep zsh session fast.</p>

<p>These days zsh is shipped with appropriate completion scripts like <code>compinit</code> and <code>bashcompinit</code> which have the required functions to support bash autocomplete scripts.</p>

<p><code>autoload &lt;func_name&gt;</code>: Note that autoload is defined in zsh and not bash. <code>autoload</code> looks for a file named  in the directory paths returned by <code>fpath</code> command and marks a function to load the same when it is first invoked. </p>

<ul>
<li><strong>-U:</strong> Ignore any aliases when loading a function like compinit or bashcompinit</li>
<li><strong>+X:</strong> Just load the named function fow now and don't execute it</li>
</ul>

<p>For example on my system <code>echo $fpath</code> returns <code>/usr/share/zsh/site-functions</code> and <code>/usr/share/zsh/5.0.5/functions</code> and both <code>compinit</code> and <code>bashcompinit</code> are available at <code>/usr/share/zsh/5.0.5/functions</code>.</p>

<p>Also for most people may be only <code>autoload -U +X bashcompinit &amp;&amp; bashcompinit</code> is required because some other script like git autocomplete or their own <code>~/.zshrc</code> may be doing <code>autoload -U +X compinit &amp;&amp; compinit</code>, but it's safe to just run both of them.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>For <code>zsh</code> use:</p>

<ul>
<li><code>compdef</code></li>
<li><code>compadd</code></li>
</ul>

<p>My example:</p>

<pre><code># bash completion for bxrun (/home/ecuomo/projects/bashx/bxrun)
_bxrun_methods() {
    grep "^\s*\(function\s\+\)\?__.\+()\s*{.*$" "${1}" | while read line ; do
        echo "$line" | sed "s/()\s*{.*//g" | sed "s/\s*\(function\s\+\)\?__//g"
    done
}
_bxrun_lst() {
    if [ -d "/home/ecuomo/projects/bashx/src/actions" ]; then
        for f in /home/ecuomo/projects/bashx/src/actions/* ; do
            if [ -f "${f}" ]; then
                basename "${f}" | sed 's/\..*$//g'
            fi
        done
    fi
    _bxrun_methods "/home/ecuomo/projects/bashx/bxrun"
    _bxrun_methods "/home/ecuomo/projects/bashx/src/bashx.sh"
}
_bxrun() {
    local cur
    COMPREPLY=()
    cur=${COMP_WORDS[COMP_CWORD]}
    COMPREPLY=( $( compgen -W '$( _bxrun_lst )' -- $cur  ) )
}
_bxrun_zsh() {
    compadd `_bxrun_lst`
}
if type complete &gt;/dev/null 2&gt;/dev/null; then
    # bash
    complete -F _bxrun bxrun
else if type compdef &gt;/dev/null 2&gt;/dev/null; then
    # zsh
    compdef _bxrun_zsh bxrun
fi; fi
