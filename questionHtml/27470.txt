<p>Your kernel should export this information in <code>procfs</code>:</p>

<pre><code>cat /proc/sys/kernel/pid_max
</code></pre>

<p>This is the maximum number of unique process identifiers your system can support.</p>

<p>Since this is a file, <code>int fd = read("/proc/sys/kernel/pid_max");</code> is perfectly valid, from C.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Short answer to your question : Number of process possible in the linux system is <strong>UNLIMITED</strong>.</p>

<p>But there is a limit on number of process per user(except root who has no limit).</p>

<p>And you can check your user limits with below command (apposite to "max user processes").</p>

<pre><code>$ ulimit -a
core file size          (blocks, -c) 0
data seg size           (kbytes, -d) unlimited
scheduling priority             (-e) 0
file size               (blocks, -f) unlimited
pending signals                 (-i) 256447
max locked memory       (kbytes, -l) 64
max memory size         (kbytes, -m) unlimited
open files                      (-n) 128000
pipe size            (512 bytes, -p) 8
POSIX message queues     (bytes, -q) 819200
real-time priority              (-r) 0
stack size              (kbytes, -s) 10240
cpu time               (seconds, -t) unlimited
max user processes              (-u) 500000
virtual memory          (kbytes, -v) unlimited
file locks                      (-x) unlimited
</code></pre>

<p>If you want to increase the limit on number of process for a particular user(for eg: hadoop ) , you need to make below entry in  <strong>/etc/security/limits.conf</strong> </p>

<pre><code>hadoop - nproc 500000
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>kernel.pid_max is a limiting factor, but at least as important is kernel.threads-max .  It's worth noting that the default nproc ulimit for each user is kernel.threads-max divided by two, and that every thread counts toward a user's nproc limit.  Thus, "ps -u $USER" may make it appear that a user has not exhausted their nproc limit, but "ps -L -u $USER" could tell a very different story.</p>
