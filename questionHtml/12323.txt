<p>The reduce phase of the job fails with:</p>

<h1>of failed Reduce Tasks exceeded allowed limit.</h1>

<p>The reason why each task fails is:</p>

<p>Task attempt_201301251556_1637_r_000005_0 failed to report status for 600 seconds. Killing!</p>

<p>Problem in detail:</p>

<p>The Map phase takes in each record which is of the format: time, rid, data.</p>

<p>The data is of the format: data element, and its count.</p>

<p>eg: a,1 b,4 c,7 correseponds to the data of a record.</p>

<p>The mapper outputs for each data element the data for every record. eg:</p>

<p>key:(time, a,), val: (rid,data)
key:(time, b,), val: (rid,data)
key:(time, c,), val: (rid,data)</p>

<p>Every reduce receives all the data corresponding to same key from all the records. 
e.g: 
key:(time, a), val:(rid1, data) and 
key:(time, a), val:(rid2, data)
reach the same reduce instance.</p>

<p>It does some processing here and outputs similar rids.</p>

<p>My program runs without trouble for a small dataset such as 10MB. But fails when the data increases to say 1G, with the above mentioned reason. I don't know why this happens. Please help!</p>

<p><strong>Reduce code:</strong></p>

<p>There are two classes below:</p>

<ul>
<li><code>VCLReduce0Split</code></li>
<li><code>CoreSplit</code></li>
</ul>

     public void filters(MultiValueMap index, String ptoken, String RID, HashMap&lt;String, String&gt; hmap, float t, int size, float val, boolean j_l, HashMap&lt;String, Float&gt; alpha, HashMap&lt;String, CustomMapOverlap&gt; hmap_overlap, float j, float prefix_contri)
     {
            @SuppressWarnings("unchecked")

            ArrayList&lt;CustomMapPrefixTokens&gt; positions_list = (ArrayList&lt;CustomMapPrefixTokens&gt;) index.get(ptoken);

   public void positionFilter( float ubound,float alpha1, CustomMapPrefixTokens cmapt, float y_overlap, HashMap&lt;String, CustomMapOverlap&gt; hmap_overlap)
   {

     float y_overlap_total = 0;

            if(null!=hmap_overlap.get(cmapt.getRID()))

            }

   public ArrayList&lt;Float&gt; overlapCalc(float alpha1, int size, int y_size, CustomMapPrefixTokens cmapt, float j, HashMap&lt;String, Float&gt; alpha, boolean j_l, float RID_val, float val, float prefix_contri )
   {

            alpha.put(cmapt.getRID(), alpha1);
            float min1 = y_size-cmapt.getPosition();
            float min2 = size-j;
            float min = 0;

            float y_overlap = 0;

            if(min1&lt;min2)
            {
                min = min1;
            }
            else
            {
                min = min2;
            }
            if(j_l==true)
            {
                val = prefix_contri;    
            }                                       
            if(RID_val&lt;val)
            {
                y_overlap = RID_val;
            }
            else
            {
                y_overlap = val;
            }

            float ubound = y_overlap+min;

            ArrayList&lt;Float&gt; fl = new ArrayList&lt;Float&gt;();
            fl.add(ubound);
            fl.add(y_overlap);

            return fl;

   }


     public ArrayList&lt;CustomMapSimilarity&gt; calcSimilarity( String time, String RID, HashMap&lt;String,String&gt; hmap , HashMap&lt;String, Float&gt; alpha, HashMap&lt;String, CustomMapOverlap&gt; hmap_overlap)
     {

         float jaccard = 0;

         CustomMapSimilarity cms = new CustomMapSimilarity(null, null);   
         ArrayList&lt;CustomMapSimilarity&gt; alsim = new ArrayList&lt;CustomMapSimilarity&gt;();

        Iterator&lt;String&gt; iter = hmap_overlap.keySet().iterator();

        {
            String key = (String)iter.next();

            CustomMapOverlap val = (CustomMapOverlap)hmap_overlap.get(key);

             jaccard = unionIntersection(hmap, RIDpair);

                    alsim.add(cms);
                }

            }

        }

         return alsim;

     }


     public float unionIntersection( HashMap&lt;String,String&gt; hmap, String RIDpair)
     {


            StringTokenizer st = new StringTokenizer(RIDpair);

            String yRID = st.nextToken();

            String ydata = hmap.get(yRID);


            int total_union = 0;

            int xval = 0;
            int yval = 0;
            int part_union = 0;

        //  System.out.println("xdata:------*************&gt;"+xdata);

            ytokenizer.nextToken();

            HashMap&lt;String,Integer&gt; x = new HashMap&lt;String, Integer&gt;();
            HashMap&lt;String,Integer&gt; y = new HashMap&lt;String, Integer&gt;();


            String [] xparts;

                 xparts = datax.toString().split("\\|");


              String [] yparts;

                 yparts = datay.toString().split("\\|");


                 for(int i = 0; i&lt;xparts.length-1; i++)
                 {
                     int part_size = Integer.parseInt(xparts[i+1]);
                     x.put(xparts[i], part_size);

                     i++;
                 }

                 for(int i = 0; i&lt;yparts.length-1; i++)
                 {
                     int part_size = Integer.parseInt(yparts[i+1]);
                     y.put(xparts[i], part_size);

                     i++;
                 }


             Set&lt;String&gt; xset = x.keySet();
             Set&lt;String&gt; yset = y.keySet();

            for(String elm:xset )
            {

                yval = 0;

                xval = (Integer)x.get(elm);

                part_union = 0;
                int part_intersect = 0;
                if(yset.contains(elm)){

                    yval = (Integer) y.get(elm);

                if(xval&gt;yval)
                {
                    part_union = xval;
                    part_intersect = yval;
                }
                else
                {
                    part_union = yval;
                    part_intersect = xval;
                }
                total_intersect = total_intersect+part_intersect;
                }
                else

                    part_union = xval;

                part_union = 0;

                if(!xset.contains(elm))

                    part_union = (Integer) y.get(elm);
                    total_union = total_union+part_union;
                }

            }

            float jaccard = (float)total_intersect/total_union;

         return jaccard;

     }

    <div class="post-text" itemprop="text">
<p>The reason for the timeouts might be a long-running computation in your reducer without reporting the progress back to the Hadoop framework. This can be resolved using different approaches:</p>

<p>I. Increasing the timeout in <code>mapred-site.xml</code>:</p>

<pre><code>&lt;property&gt;
  &lt;name&gt;mapred.task.timeout&lt;/name&gt;
  &lt;value&gt;1200000&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>The default is <code>600000 ms = 600 seconds</code>.</p>

<p>II. Reporting progress every x records as in the <a href="http://hadoop.apache.org/docs/r1.1.1/api/org/apache/hadoop/mapred/Reducer.html">Reducer example in javadoc</a>:</p>

<pre><code>public void reduce(K key, Iterator&lt;V&gt; values,
                          OutputCollector&lt;K, V&gt; output, 
                          Reporter reporter) throws IOException {
   // report progress
   if ((noValues%10) == 0) {
     reporter.progress();
   }

   // ...
}
</code></pre>

<p>optionally you can increment a custom counter as in the <a href="http://hadoop.apache.org/docs/r1.1.1/api/org/apache/hadoop/mapred/Reducer.html">example</a>:</p>

<pre><code>reporter.incrCounter(NUM_RECORDS, 1);
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It's possible that you might have consumed all of Java's heap space or GC is happening too frequently giving no chance to the reducer to report status to master and is hence killed. </p>

<p>Another possibility is that one of the reducer is getting too skewed data, i.e. for a particular rid, a lot of records are there. </p>

<p>Try to increase your java heap by setting the following config:
<code>mapred.child.java.opts</code></p>

<p>to</p>

<p><code>-Xmx2048m</code></p>

<p>Also, try and reduce the number of parallel reducers by setting the following config to a lower value than what it currently has (default value is <code>2</code>):</p>

<p><code>mapred.tasktracker.reduce.tasks.maximum</code></p>
