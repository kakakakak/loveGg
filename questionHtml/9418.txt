<p>Basically I want to do following:</p>

<ol>
<li>Write a mapper and reducer class
with map and reduce function in a
java application external to Hadoop
environment.</li>
<li>Then send this job to Hadoop which will use mapper and
reducer function defined above.</li>
</ol>

<p>To try this I started with WordCount example that's described in the Hadoop examples. I used the same Map and Reduce function described in the example. Then I shipped this Job to Hadoop environment using <code>Job</code> class. While Map function is working I am not sure if Reduce function is getting executed. Also the job completes successfully but the output clearly shows that reduce function is not being applied. I am getting following output:</p>

<pre><code>A       1
Applications    1
Bad     1
By      1
For     1
For     1
HDFS    1
Hadoop  1
In      1
In      1
In      1
</code></pre>

<p>As you can see For / In words' count is not getting aggregated. Also I am getting a error.</p>

<pre><code>java.lang.NoClassDefFoundError: org/apache/commons/httpclient/HttpMethod
    at org.apache.hadoop.mapred.LocalJobRunner$Job.run(LocalJobRunner.java:242)
Caused by: java.lang.ClassNotFoundException: org.apache.commons.httpclient.HttpMethod
    at java.net.URLClassLoader$1.run(URLClassLoader.java:202)
    at java.security.AccessController.doPrivileged(Native Method)
    at java.net.URLClassLoader.findClass(URLClassLoader.java:190)
    at java.lang.ClassLoader.loadClass(ClassLoader.java:307)
    at sun.misc.Launcher$AppClassLoader.loadClass(Launcher.java:301)
    at java.lang.ClassLoader.loadClass(ClassLoader.java:248)
</code></pre>

<p>Can anyone suggest the solution to this problem?</p>
