<p>I am a Web developer. I have experience in Web technologies like JavaScript , Jquery , Php , HTML . I know basic concepts of C. Recently I had taken interest in learning more about <strong>mapreduce</strong> and <strong>hadoop</strong>. So I enrolled my self in <strong>parallel data processing in mapreduce</strong> course in my university. Since I dont have any prior programing knowledge in any object oriented languages like Java or C++ , how should I go about learning map reduce and hadoop. I have started to read <strong>Yahoo hadoop tutorials</strong> and also <strong>OReilly's Hadoop The Definitive Guide 2nd.Edition</strong>. </p>

<p>I would like you guys to suggest me ways I could go about learning mapreduce and hadoop.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can access Hadoop from many different languages and a number of resources set up Hadoop for you.  You could try Amazon's Elastic MapReduce (EMR), for instance, without having to go through the hassle of configuring the servers, workers, etc.  This is a good way to get your head around MapReduce processing while delaying a bit the issues of learning how to use HDFS well, how to manage your scheduler, etc.</p>

<p>It's not hard to search for your favorite language &amp; find Hadoop APIs for it or at least some tutorials on linking it with Hadoop.  For instance, here's a walkthrough on a PHP app run on Hadoop: <a href="http://www.lunchpauze.com/2007/10/writing-hadoop-mapreduce-program-in-php.html">http://www.lunchpauze.com/2007/10/writing-hadoop-mapreduce-program-in-php.html</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>Answer 1 :</strong></p>

<ul>
<li>It is very desirable to know Java. Hadoop is written in Java. Its popular Sequence File format is dependent on Java. </li>
<li>Even if you use Hive or Pig, you'll probably need to write your own UDF someday. Some people still try to write them in other languages, but I guess that Java has more robust and primary support for them.</li>
<li>Most Hadoop tools are not mature enough (like Sqoop, HCatalog and so on), so you'll see many Java error stack traces and probably you'll want to hack the source code someday</li>
</ul>

<p><strong>Answer 2</strong></p>

<ul>
<li>It is not required for you to know Java.</li>
<li>As the others said, it would be very helpful depending on how complex your processing may be. However, there is an incredible amount you can do with just Pig and say Hive. </li>
<li>I would agree that it is fairly likely you will eventually need to write a user defined function (UDF), however, I've written those in Python, and it is very easy to write UDFs in Python. </li>
<li>Granted, if you have very stringent performance requirements, then a Java based MapReduce program would be the way to go. However, great advancements in performance are being made all of the time in both Pig and Hive. </li>
<li>So, the short answer to your question is, "No", it is not required for you to know Java in order to perform Hadoop development.</li>
</ul>
