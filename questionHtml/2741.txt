<p>I need to compute the cosine similarity between strings in a list. For example, I have a list of over 10 million strings, each string has to determine similarity between itself and every other string in the list. What is the best algorithm I can use to efficiently and quickly do such task? Is the divide and conquer algorithm applicable? </p>

<p><strong>EDIT</strong></p>

<p>I want to determine which strings are most similar to a given string and be able to have a measure/score associated with the similarity. I think what I want to do falls in line with clustering where the number of clusters are not initially known. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Work with the transposed matrix. That is what Mahout does on Hadoop to do this kind of task fast (or just use Mahout).</p>

<p>Essentially, computing cosine similarity the naive way is bad. Because you end up computing a lot of 0 * something. Instead, you better work in <em>columns</em>, and <em>leave away all 0s there</em>.</p>

<p>You could try <a href="http://www.chokkan.org/software/simstring/" rel="nofollow">SimString</a>.</p>

<p>It is a C++ library (with Python or Ruby bindings) for approximate string matching.</p>

<p>It claims to find strings with high cosine similarity in under 1 millisecond for a database of 13 million strings.</p>

<p>The algorithm used is described <a href="http://www.aclweb.org/anthology-new/C/C10/C10-1096.pdf" rel="nofollow">here</a> based on pruning of inverted lists.</p>
