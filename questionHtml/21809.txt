<p>Had a very similar problem and found this question while googling for a solution.</p>

<p>Here is how I could resolve it (on Mac OS 10.10 with Hadoop 2.5.1). Not sure if the question is exactly the same problem: I checked the log files generated by the data-node (/usr/local/hadoop-2.2.0/logs/hadoop-mrp-datanode-mrp.local.out) and found the following entry:</p>

<pre><code>2014-11-09 17:44:35,238 FATAL org.apache.hadoop.hdfs.server.namenode.NameNode:
Exception in namenode join org.apache.hadoop.hdfs.server.common.InconsistentFSStateException: 
Directory /private/tmp/hadoop-kthul/dfs/name is in an inconsistent state: storage
directory does not exist or is not accessible.
</code></pre>

<p>Based on this, I concluded that something is wrong with the HDFS data on the datanode.</p>

<p>I deleted the directory with the HDFS data and reformatted HDFS:</p>

<pre><code>rm -rf /private/tmp/hadoop-kthul
hdfs namenode -format
</code></pre>

<p>Now, I am up and running again. Still wondering if /private/tmp is a good place to keep the HDSF data - looking options to change this.</p>

<p>So I've got Hadoop up and running. I had two problems (I think).  </p>

<ol>
<li>When starting up the NameNode and DataNode, I received the following error: <code>Unable to load realm info from SCDynamicStore.</code></li>
</ol>

<p>To fix this, I added the following two lines to my <code>hadoop-env.sh</code> file:</p>

<blockquote>
  <p>HADOOP_OPTS="${HADOOP_OPTS} -Djava.security.krb5.realm= -Djava.security.krb5.kdc="
  HADOOP_OPTS="${HADOOP_OPTS} -Djava.security.krb5.conf=/dev/null"</p>
</blockquote>

<p>I found those two lines in the solution to this post, <a href="http://stackoverflow.com/questions/7134723/hadoop-on-osx-unable-to-load-realm-info-from-scdynamicstore">Hadoop on OSX &quot;Unable to load realm info from SCDynamicStore&quot;</a>.  The Answer was posted by Matthew L Daniel.</p>

<ol start="2">
<li>I had formatted the NameNode folder more than once, which apparently screws things up? </li>
</ol>

<p>I can't verify this screws things up, because I don't have any errors in any of my log files, however once I followed <strong>Workaround 1</strong> (deleting &amp; recreating NameNode/DataNode folders, then reformatting) on this post, <a href="http://stackoverflow.com/questions/10097246/no-data-nodes-are-started">No data nodes are started</a>, I was able to load up the DataNode and get everything working. </p>

<p>Since native library isn't supported on Mac, if you want to suppress this warning:</p>

<pre><code>WARN util.NativeCodeLoader: Unable to load native-hadoop library for your platform... using builtin-java classes where applicable
</code></pre>

<p>Add this to the log4j.properties in ${HADOOP_HOME}/libexec/etc/hadoop:</p>

<pre><code># Turn of native library warning
