    <div class="post-text" itemprop="text">

<p>I want to output <code>gzip</code> and <code>lzo</code> formats at the same time in one job.</p>

<p>I used <code>MultipleOutputs</code>, and add two named outputs like this:</p>

<p><code>MultipleOutputs.addNamedOutput(job, "LzoOutput", GBKTextOutputFormat.class, Text.class, Text.class);</code></p>

<p><code>GBKTextOutputFormat.setOutputCompressorClass(job, LzoCodec.class);</code></p>

<p><code>MultipleOutputs.addNamedOutput(job, "GzOutput", TextOutputFormat.class, Text.class, Text.class);</code></p>

<p><code>TextOutputFormat.setOutputCompressorClass(job, GzipCodec.class);</code></p>

<p>(<code>GBKTextOutputFormat</code> here is written by myself which extends <code>FileOutputFormat</code>)</p>

<p>They are used in reducer like:</p>

<p><code>multipleOutputs.write("LzoOutput", NullWritable.get(), value, "/user/hadoop/lzo/"+key.toString());</code></p>

<p><code>multipleOutputs.write("GzOutput", NullWritable.get(), value, "/user/hadoop/gzip/"+key.toString());</code></p>

<p>The result is:</p>

<p>I can get outputs in the two path, but they are both in <code>gzip</code> format.</p>

<p>Someone can help me? Thanks!</p>

<p>==========================================================================</p>

<p>More:</p>

<p>I just looked at the source code of <code>setOutputCompressorClass</code> in <code>FileOutputFormat</code>, in which <code>conf.setClass("mapred.output.compression.codec", codecClass, CompressionCodec.class);</code></p>

<p>It seems that mapred.output.compression.codec in configuration will be reset when setOutputCompressorClass is called. </p>

<p>So the actual compression format is the one we set at last, and we cannot set two different compression formats in the same job ? Or there is something else ignored ?</p>
