<p>I have several  <code>(title , text )</code> ordered pairs obtained as an output from a MapReduce application in Hadoop using Java.</p>

<p>Now I would like to implement Word Count on the text field of these ordered pairs. </p>

<p>So my final output should look like :</p>

<pre><code>(title-a , word-a-1 , count-a-1 , word-a-2 , count-a-2 ....)

(title-b , word-b-1, count-b-1 , word-b-2 , count-b-2 ....)
.
.
.
.
(title-x , word-x-1, count-x-1 , word-x-2 , count-x-2 ....)
</code></pre>

<p>To summarize , I want to implement wordcount separately on the output records from first mapreduce. Can someone suggest me a good way to do it or how I can chain a second map reduce job to create the above output or format it better ?</p>

        private boolean readUntilMatch(byte[] match, boolean withinBlock)

            int i = 0;
            while (true) {
                int b = fsin.read();
                // end of file:
                if (b == -1)
                    return false;
                // save to buffer:
                if (withinBlock)
                    buffer.write(b);
                // check if we're matching:
        if (b == match[i]) {

                // see if we've passed the stop point:
                if (!withinBlock &amp;&amp; i == 0 &amp;&amp; fsin.getPos() &gt;= end)
                    return false;
            }
        }
    }

  }

  private Text outputKey = new Text();
  public void reduce(Text key, Iterable&lt;Text&gt; values,
                 Context context)
  throws IOException, InterruptedException {
for (Text value : values) {
      outputKey.set(constructPropertyXml(key, value));
      context.write(outputKey, null);
    }
 }

  public static String constructPropertyXml(Text name, Text value) {
    StringBuilder sb = new StringBuilder();
    sb.append("&lt;property&gt;&lt;name&gt;").append(name)
    .append("&lt;/name&gt;&lt;value&gt;").append(value)
    .append("&lt;/value&gt;&lt;/property&gt;");
    return sb.toString();
  }
}



    public static void main(String[] args) throws Exception
    {
            Configuration conf = new Configuration();

            conf.set("xmlinput.start", "&lt;page&gt;");
            conf.set("xmlinput.end", "&lt;/page&gt;");
            Job job = new Job(conf);
            job.setJarByClass(XmlParser11.class);
            job.setOutputKeyClass(Text.class);
            job.setOutputValueClass(Text.class);

            job.setMapperClass(XmlParser11.Map.class);
            job.setReducerClass(XmlParser11.Reduce.class);

            job.setInputFormatClass(XmlInputFormat1.class);
            job.setOutputFormatClass(TextOutputFormat.class);

            FileInputFormat.addInputPath(job, new Path(args[0]));
            FileOutputFormat.setOutputPath(job, new Path(args[1]));

            job.waitForCompletion(true);
    }
}
</code></pre>

<p>The wordcount code we find online does the word count of all files and gives the output . I want to do the wordcount for each of text fields separately. The above mapper is used to pull title and text from an XML document . Is there any way I can do the wordcount in the same mapper. If I do that , my next doubt is how do I pass it along with the already existing key value pairs (title,text) to the reducer. Sorry , I am not able to phrase my question properly but I guess the reader must have got some idea</p>

<p>I'm not sure if I have understood it properly. So I have many questions along with my answer.</p>

<p>First of all whoever has written this code is probably trying to show how to write a <strong>custom InputFormat</strong> to process xml data using MR. I don't know how it is related to your problem.</p>

<p><strong>To summarize , I want to implement wordcount separately on the output records from first mapreduce. Can someone suggest me a good way to do it</strong></p>

<p>Read the output file generated by first MR and do it.</p>

<p><strong>or how I can chain a second map reduce job to create the above output or format it better ?</strong></p>

<p>You can definitely chain jobs together in this fashion by writing multiple driver methods, one for each job. See <a href="http://developer.yahoo.com/hadoop/tutorial/module4.html#chaining" rel="nofollow">this</a> for more details and <a href="http://gandhigeet.blogspot.in/2012/12/as-discussed-in-previous-post-hadoop.html" rel="nofollow">this</a> for an example.</p>

<p><strong>I want to do the wordcount for each of text fields separately.</strong></p>

<p>What do you mean by <strong>separately</strong>?In the traditional wordcount program count of each word is calculated independently of the others.</p>

<p><strong>Is there any way I can do the wordcount in the same mapper.</strong></p>

<p>I hope you have understood the wordcount program properly. In the traditional wordcount program you read the input file, one line at a time, slit the line into words and then emit <strong>each word as the key</strong> with <strong>1 as the value</strong>. All this happens inside the Mapper, which is essentially the same Mapper. And then the total count for each word is determined in the Reducer part of your job. If you wish to emit the words with their total counts from the mapper itself you have to read the whole file in the Mapper itself and do the counting. For that you need to set <strong>isSplittable</strong> in your <strong>InputFormat</strong> to <strong>false</strong> so that your input file is read as a whole and goes to just one Mapper.</p>

<p>When you emit something from Mapper and if it is <strong>not a Map only job</strong>, the output of your Mapper automatically goes to the Reducer. Do you need something else?</p>
