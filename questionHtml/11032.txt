<p>Following is the client code on non hadoop system to perform actions on the secured remote HDFS.</p>

<pre><code>Configuration conf = new
        Configuration();
conf.set("hadoop.security.authentication", "kerberos");
conf.set("java.security.krb5.conf",krbPath);
conf.set("fs.defaultFS", "webhdfs://10.31.251.254:50070");
conf.set("fs.webhdfs.impl", org.apache.hadoop.hdfs.web.WebHdfsFileSystem.class.getName());
conf.set("com.sun.security.auth.module.Krb5LoginModule", "required");
conf.set("debug", "true");
conf.set("ticketCache", "DIR:/etc/");
System.out.print("Conf......");

UserGroupInformation.setConfiguration(conf);



UserGroupInformation.loginUserFromKeytab("Dummy@EXAMPLE.COM", keytab);
System.out.print("Obtained......");
URI uri = URI.create("webhdfs://Dummy:50070");
FileSystem fs = FileSystem.get(uri, conf);

if (fs.mkdirs(new Path("/testKerb2")))
    System.out.print("Directory created...");
</code></pre>

<p>I am able to perform the actions but the ticket configuration values are not read from krb5.conf. Ticket lifetime mentioned in conf file is 1m but the code generates a ticket with 1d lifetime. And also the tickets are not generated in configured ticketCache.</p>

<p>Please help in configuring so that code reads from krb5.conf file and generates tickets in configured path.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You have to set the location of the krb5.conf file in the System, not the Hadoop configurations, i.e. replace your line</p>

<pre><code>conf.set("java.security.krb5.conf",krbPath);
</code></pre>

<p>with </p>

<pre><code>System.setProperty( "java.security.krb5.conf", krbPath);
</code></pre>

<p>(but you probably figured this out yourself, given that the question is 5 months old)</p>
