<p>I have followed some tutorial online but they do not work with <code>Spark 1.5.1</code> on OS X El Capitan (10.11)</p>

<p>Basically I have run this commands download <code>apache-spark</code></p>

<pre><code>brew update
brew install scala
brew install apache-spark
</code></pre>

<p>updated the .bash_profile</p>

<pre><code># For a ipython notebook and pyspark integration
if which pyspark &gt; /dev/null; then
  export SPARK_HOME="/usr/local/Cellar/apache-spark/1.5.1/libexec/"
  export PYSPARK_SUBMIT_ARGS="--master local[2]"
fi
</code></pre>

<p>run </p>

<pre><code>ipython profile create pyspark
</code></pre>

<p>created a startup file <code>~/.ipython/profile_pyspark/startup/00-pyspark-setup.py</code> configured in this way </p>

<pre><code># Configure the necessary Spark environment
import os
import sys

# Spark home
spark_home = os.environ.get("SPARK_HOME")

# If Spark V1.4.x is detected, then add ' pyspark-shell' to
# the end of the 'PYSPARK_SUBMIT_ARGS' environment variable
spark_release_file = spark_home + "/RELEASE"
if os.path.exists(spark_release_file) and "Spark 1.4" in open(spark_release_file).read():
    pyspark_submit_args = os.environ.get("PYSPARK_SUBMIT_ARGS", "")
    if not "pyspark-shell" in pyspark_submit_args: pyspark_submit_args += " pyspark-shell"
    os.environ["PYSPARK_SUBMIT_ARGS"] = pyspark_submit_args

# Add the spark python sub-directory to the path
sys.path.insert(0, spark_home + "/python")

# Add the py4j to the path.
# You may need to change the version number to match your install
sys.path.insert(0, os.path.join(spark_home, "python/lib/py4j-0.8.2.1-src.zip"))

# Initialize PySpark to predefine the SparkContext variable 'sc'
execfile(os.path.join(spark_home, "python/pyspark/shell.py"))
</code></pre>

<p>I then run <code>ipython notebook --profile=pyspark</code> and the notebook works fine, but the <code>sc</code> (spark context) is not recognised.</p>

<p>Anyone managed to do this with <code>Spark 1.5.1</code>?</p>

<p><strong>EDIT:</strong> you can follow this guide to have it working </p>

<p><a href="https://gist.github.com/tommycarpi/f5a67c66a8f2170e263c" rel="nofollow">https://gist.github.com/tommycarpi/f5a67c66a8f2170e263c</a></p>

<p>I have Jupyter installed, and indeed It is simpler than you think:</p>

<ol>
<li>Install <a href="https://www.continuum.io/downloads#_macosx">anaconda</a> for OSX.</li>
<li><p>Install <a href="http://jupyter.org/">jupyter</a> typing the next line in your terminal <a href="http://jupyter.readthedocs.org/en/latest/install.html">Click me for more info</a>.</p>

<pre><code>ilovejobs@mymac:~$ conda install jupyter
</code></pre></li>
<li><p>Update jupyter just in case.</p>

<pre><code>ilovejobs@mymac:~$ conda update jupyter
</code></pre></li>
<li><p>Download <a href="https://spark.apache.org/downloads.html">Apache Spark</a> and compile it, or download and uncompress <a href="http://www.apache.org/dyn/closer.lua/spark/spark-1.5.1/spark-1.5.1-bin-hadoop2.6.tgz">Apache Spark 1.5.1 + Hadoop 2.6</a>.</p>

<pre><code>ilovejobs@mymac:~$ cd Downloads 
ilovejobs@mymac:~/Downloads$ wget http://www.apache.org/dyn/closer.lua/spark/spark-1.5.1/spark-1.5.1-bin-hadoop2.6.tgz
</code></pre></li>
<li><p>Create an <code>Apps</code> folder on your home (i.e):</p>

<pre><code>ilovejobs@mymac:~/Downloads$ mkdir ~/Apps
</code></pre></li>
<li><p>Move the uncompressed folder <code>spark-1.5.1</code> to the <code>~/Apps</code> directory. </p>

<pre><code>ilovejobs@mymac:~/Downloads$ mv spark-1.5.1/ ~/Apps
</code></pre></li>
<li><p>Move to the <code>~/Apps</code> directory and verify that spark is there.</p>

<pre><code>ilovejobs@mymac:~/Downloads$ cd ~/Apps
ilovejobs@mymac:~/Apps$ ls -l
drwxr-xr-x ?? ilovejobs ilovejobs 4096 ?? ?? ??:?? spark-1.5.1
</code></pre></li>
<li><p><em>Here is the first tricky part</em>. Add the spark binaries to your <code>$PATH</code>:</p>

<pre><code>ilovejobs@mymac:~/Apps$ cd
ilovejobs@mymac:~$ echo "export PATH=/home/ilovejobs/Apps/spark/bin" &gt;&gt; .profile
</code></pre></li>
<li><p><em>Here is the second tricky part</em>. Add this environment variables also:</p>

<pre><code>ilovejobs@mymac:~$ echo "export PYSPARK_DRIVER_PYTHON=ipython" &gt;&gt; .profile
ilovejobs@mymac:~$ echo "export PYSPARK_DRIVER_PYTHON_OPTS='notebook' pyspark" &gt;&gt; .profile
</code></pre></li>
<li><p>Source the profile to make these variables available for this <strong>terminal</strong></p>

<pre><code>ilovejobs@mymac:~$ source .profile
</code></pre></li>
<li><p>Create a <code>~/notebooks</code> directory.</p>

<pre><code>ilovejobs@mymac:~$ mkdir notebooks
</code></pre></li>
<li><p>Move to <code>~/notebooks</code> and run pyspark:</p>

<pre><code>ilovejobs@mymac:~$ cd notebooks
ilovejobs@mymac:~/notebooks$ pyspark
</code></pre></li>
</ol>

<p><strong>Now be happy, You should be able to run jupyter with a pyspark kernel (It will show it as a python 2 but it will use spark)</strong></p>

<p>Spark with IPython/Jupyter notebook is great and I'm pleased the Alberto was able to help you get it working.</p>

<p>For reference it's also worth considering 2 good alternatives that come prepackaged and can easily be integrated into a YARN cluster (if desired.)</p>

<p>Spark Notebook: <a href="https://github.com/andypetrella/spark-notebook" rel="nofollow">https://github.com/andypetrella/spark-notebook</a></p>

<p>Apache Zeppelin: <a href="https://zeppelin.incubator.apache.org/" rel="nofollow">https://zeppelin.incubator.apache.org/</a></p>

<p>At the time of writing Spark Notebook (v0.6.1) is more mature and you can prebuild an install against your Spark and Hadoop version here: <a href="http://spark-notebook.io/" rel="nofollow">http://spark-notebook.io/</a></p>

<p>Zeppelin (v0.5) looks very promising but doesn't offer as much functionality as Spark Notebook or IPython with Spark right now.</p>
