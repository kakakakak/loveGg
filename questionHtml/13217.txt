<p>I have 3 data nodes running, while running a job i am getting the following given below error ,</p>

<blockquote>
  <p>java.io.IOException: File /user/ashsshar/olhcache/loaderMap9b663bd9 could only be replicated to 0 nodes instead of minReplication (=1).  There are 3 datanode(s) running and 3 node(s) are excluded in this operation.
          at org.apache.hadoop.hdfs.server.blockmanagement.BlockManager.chooseTarget(BlockManager.java:1325)</p>
</blockquote>

<p>This error mainly comes when our DataNode instances have ran out of space or if DataNodes are not running.
I tried restarting the DataNodes but still getting the same error.</p>

<p>dfsadmin -reports  at my cluster nodes clearly shows a lots of space is available.</p>

<p>I am not sure why this is happending.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>1.Stop all Hadoop daemons </p>

<pre><code>for x in `cd /etc/init.d ; ls hadoop*` ; do sudo service $x stop ; done
</code></pre>

<p>2.Remove all files from <code>/var/lib/hadoop-hdfs/cache/hdfs/dfs/name</code> </p>

<pre><code>Eg: devan@Devan-PC:~$ sudo rm -r /var/lib/hadoop-hdfs/cache/
</code></pre>

<p>3.Format Namenode</p>

<pre><code>sudo -u hdfs hdfs namenode -format
</code></pre>

<p>4.Start all Hadoop daemons</p>

<pre><code>for x in `cd /etc/init.d ; ls hadoop*` ; do sudo service $x start ; done
</code></pre>

<p><a href="http://www.cloudera.com/content/cloudera/en/documentation/cdh4/latest/CDH4-Installation-Guide/cdh4ig_topic_27_3.html" rel="nofollow">Stop All Hadoop Service</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>What I usually do when this happens is that I go to <em>tmp/hadoop-username/dfs/</em> directory and manually delete the <em>data</em> and <em>name</em> folders (assuming you are running in a Linux environment).</p>

<p>Then format the dfs by calling <em>bin/hadoop namenode -format</em> (make sure that you answer with a capital <em>Y</em> when you are asked whether you want to format; if you are not asked, then re-run the command again).</p>

<p>You can then start hadoop again by calling <em>bin/start-all.sh</em></p>
