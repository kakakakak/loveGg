<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I don't believe I'm familiar with anything similar to Apache Mahout build on top of .NET, but I believe you could use the following approach to get pretty close (how close you can actually get depends on the specifics of what you're trying to do).</p>

<p>Mahout is in fact a collection of standard machine learning algorithms implemented on top of Apache Hadoop to allow them scale to large data sets, so to get the same effect in a .NET environment you'll need a distributed computation solution (and to keep with the spirit of Mahout, I'd use Map/Reduce implementation), and a machine learning library. Then you can pass the computation of the learning tasks to your Map/Reduce cluster.</p>

<p>For the Map/Reduce implementations you have the following options - there is <a href="http://code.google.com/p/qizmt/">Qizmt</a>, which is an open source implementation of Map/Reduce by MySpace; there is also <a href="http://code.google.com/p/hadoopdotnet/">Hadoop.NET</a> which is an attempt in class-for-class, api-for-api port of Hadoop from Java to C#; and last, but not least, you could look into using Hadoop itself by the way of <a href="http://wiki.apache.org/hadoop/HadoopStreaming">streaming</a>.</p>

<p>For machine learning you have the following .NET libraries - for plain Bayesian inference there is Microsoft's own <a href="http://research.microsoft.com/en-us/um/cambridge/projects/infernet/">Infer.NET</a>; another simple machine learning library is <a href="http://machine.codeplex.com/">Machine Learning for .NET</a>; and finally the most well rounded machine learning/numerics library for .NET that I'm aware of is <a href="http://accord-net.origo.ethz.ch/wiki/accord-net">Accord.NET</a>.</p>
