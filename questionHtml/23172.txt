<p>Hadoop is Consistent and partition tolerant, i.e. It falls under the CP category of the CAP theoram.</p>

<p>Hadoop is not available because all the nodes are dependent on the name node. If the name node falls the cluster goes down.</p>

<p>But considering the fact that the HDFS cluster has a secondary name node why cant we call hadoop as available. If the name node is down the secondary name node can be used for the writes.</p>

<p>What is the major difference between name node and secondary name node that makes hadoop unavailable.</p>

<p>Thanks in advance. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The namenode stores the HDFS filesystem information in a file named fsimage. Updates to the file system (add/remove blocks) are not updating the fsimage file, but instead are logged into a file, so the I/O is fast append only streaming as opposed to random file writes. When restaring, the namenode reads the fsimage and then applies all the changes from the log file to bring the filesystem state up to date in memory. This process takes time.</p>

<p>The secondarynamenode job is not to be a secondary to the name node, but only to periodically read the filesystem changes log and apply them into the fsimage file, thus bringing it up to date. This allows the namenode to start up faster next time. </p>

<p>Unfortunatley the secondarynamenode service is not a standby secondary namenode, despite its name. Specifically, it does not offer HA for the namenode. This is well illustrated <a href="http://www.youtube.com/watch?v=hEqQMLSXQlY">here</a>.</p>

<p>See <a href="http://hortonworks.com/blog/understanding-namenode-startup-operations-in-hdfs/">Understanding NameNode Startup Operations in HDFS</a>. </p>

<p>Note that more recent distributions (current Hadoop 2.6) introduces <a href="https://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HDFSHighAvailabilityWithNFS.html">namenode High Availability using NFS (shared storage)</a> and/or <a href="https://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HDFSHighAvailabilityWithQJM.html">namenode High Availability using Quorum Journal Manager</a>.</p>
