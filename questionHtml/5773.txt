<p>I was stuck at the same issue for last couple of hours but finally solved it.
I had the hadoop installation extracted by same user as one I am using to run hadoop. So user privilege is not issue.<br>
My cofiguration is like this:
Ubuntu linux machine on Google Cloud.</p>

<p>Hadoop installation /home/
Hadoop data directory /var/lib/hadoop
and the directory access bits are 777 so anybody can access.
I did ssh into the remote machine made changes to the config files and executed start-dfs.sh, then it gave me "Permission denied (Public key)"
So here is the solution:
In the same ssh terminal:</p>

<ol>
<li><strong>ssh-keygen</strong></li>
</ol>

<p>2.It will ask for folder location where it will copy the keys, I entered <strong>/home/hadoop/.ssh/id_rsa</strong></p>

<p>3.it will ask for pass phrase, keep it empty for simplicity.</p>

<p>4.<strong>cp /home/hadoop/.ssh/id_rsa.pub .ssh/authorized_keys</strong> 
(To copy the newly generated public key to auth file in your users home/.ssh directiry</p>

<ol start="5">
<li><p><strong>ssh localhost</strong></p></li>
<li><p><strong>start-dfs.sh</strong> 
(Now it should work!)</p></li>

<p>I think the problem is that root and user's ssh connection.
here is my copy <a href="http://askubuntu.com/questions/497895/permission-denied-for-rootlocalhost-for-ssh-connection">http://askubuntu.com/questions/497895/permission-denied-for-rootlocalhost-for-ssh-connection</a>
Solved my case.</p>

<p>By default, the SSH server denies password-based login for root. In /etc/ssh/sshd_config, </p>

<p>change:
PermitRootLogin without-password  to  PermitRootLogin yes</p>

<p>And restart SSH:      sudo service ssh restart</p>

<p>Or, you can use SSH keys. If you don't have one, create one using ssh-keygen (stick to the default for the key, and skip the password if you feel like it). Then do sudo -s (or whatever your preferred method of becoming root is), and add an SSH key to /root/.ssh/authorized_keys:</p>

<p>cat /home/user/.ssh/id_rsa.pub >> /root/.ssh/authorized_keys</p>
