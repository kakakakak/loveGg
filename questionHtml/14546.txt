<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you look at hadoop command documentation, you can see that <code>-libjars</code> is a generic option. For parsing generic option, you got to override the <code>ToolRunner.run()</code> method in your driver class as follows :</p>

<pre class="lang-java prettyprint-override"><code>public class TestDriver extends Configured implements Tool {
    @Override
    public int run(String[] args) throws Exception {
        Configuration conf = getConf();

        # Job configuration details            
        # Job submission  

        return 0;
    }        
}

public static void main(String[] args) throws Exception {
    int exitCode = ToolRunner.run(new TestDriver(), args);
    System.exit(exitCode);
}
</code></pre>

<hr>

<p>I'nk you are getting this exception from your driver code itself. Setting hcatalog-cor*.jar using -libjars option may not be available in client JVM(JVM in which driver code runs). Better you need to set this jar in HADOOP_CLASSPATH environment variable before executing the same using hadoop jar as follows</p>

<pre><code>export HADOOP_CLASSPATH=${HADOOP_CLASSPATH}:&lt;PATH-TO-HCAT-LIB&gt;/hcatalog-core-0.5.0-cdh4.4.0.jar;
hadoop jar MyJob.jar MyDriver -libjars hcatalog-core-0.5.0-cdh4.4.0.jar inputDir OutputDir
</code></pre>
