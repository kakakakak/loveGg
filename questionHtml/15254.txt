<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Think using s3 or any other file-system implementation as an replacement for HDFS/Namenode isn't possible so straightforward.
Tried the same thing with the Tachyon filesystem and failed, see <a href="https://groups.google.com/forum/#!topic/tachyon-users/u4OoBekGigA" rel="nofollow">https://groups.google.com/forum/#!topic/tachyon-users/u4OoBekGigA</a></p>

<p>Think other folks got it to work by:</p>

<ul>
<li>adding AbstractFileSystem implementation</li>
<li>patching Hadoop to not check permissions of staging artifacts</li>
</ul>

<p>So bottom-line, you can use s3 for reading the input and writing the output of a job but using it as an HDFS replacement for the execution-metadata layer isn't supported out of the box!</p>
