<p>I'am working with Cloudera CDH5.3 with 1 Namenode (ip:...169) and 3 slaves.</p>

<p>I have ElasticSearch 1.4.4 installed on my master machine (ip:...169).</p>

<p>I have downloaded the ES-Hadoop jar and added it to the path.</p>

<p>With that being said; I now want to load data from Hive to ES.</p>

<p>1) First of all, I created a table via a CSV file under table metastore (with HUE)</p>

<p>2) I defined an external table on top of ES in hive to write and load data in it later:  </p>

<pre><code>  ADD JAR 
  /usr/elasticsearch-hadoop-2.0.2/dist/elasticsearch-hadoop-hive-2.0.2.jar;

 CREATE EXTERNAL TABLE es_cdr(

 id bigint,

 calling int,

 called int,

 duration int,

 location string,

 date string)

 ROW FORMAT SERDE 'org.elasticsearch.hadoop.hive.EsSerDe'

 STORED BY 'org.elasticsearch.hadoop.hive.EsStorageHandler'

 TBLPROPERTIES(

 'es.nodes'='10.44.162.169',


 'es.resource' = 'indexOmar/typeOmar');
</code></pre>

<p>I've also added manually the serde snapshot jar via paramaters=> add file =>jar</p>
