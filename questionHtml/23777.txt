<p>My aim is to read all the files that starts with  "trans" in a directory  and convert them into a single file and load  that single file into HDFS location </p>

<p>my source directory is /user/cloudera/inputfiles/</p>

<p>Assume that inside that above directory , there are lot of file , but i need all the files that start with "trans"</p>

<p>my destination directory is /user/cloudera/transfiles/ </p>

<p>So i tried this command below </p>

<pre><code>hadoop dfs - getmerge /user/cloudera/inputfiles/trans* /user/cloudera/transfiles/records.txt
</code></pre>

<p>but the above command is not working .</p>

<p>If i try the below command then it works</p>

<pre><code>hadoop dfs - getmerge /user/cloudera/inputfiles   /user/cloudera/transfiles/records.txt
</code></pre>

<p>Any suggestion on how do i merge some files from a hdfs location and store that merged single file in another hdfs location  </p>

    <div class="post-text" itemprop="text">
<p>Below is the usage of getmerge command:</p>

<pre><code>Usage: hdfs dfs -getmerge &lt;src&gt; &lt;localdst&gt; [addnl]

Takes a source directory and a destination file as input and 
concatenates files in src into the destination local file. 
Optionally addnl can be set to enable adding a newline character at the
end of each file.
</code></pre>

<p>It expects directory as first parameter.</p>

<p>you can try cat command like this:</p>

<pre><code>hadoop dfs -cat /user/cloudera/inputfiles/trans* &gt; /&lt;local_fs_dir&gt;/records.txt
hadoop dfs -copyFromLocal /&lt;local_fs_dir&gt;/records.txt /user/cloudera/transfiles/records.txt
</code></pre>
