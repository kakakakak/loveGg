    <div class="post-text" itemprop="text">

<p>It made me happy when I heard about <code>parallelStream()</code> in Java 8, that processes on multiple cores and finally gives back the result within single JVM. No more lines of multithreading code. As far as I understand this is valid for single JVM only.</p>

<p>But what if I want to distribute the processing across different JVMs on a single host or even multiple hosts? Does Java 8 include any abstraction for simplifying it?</p>

<p>In a <a href="http://www.dreamsyssoft.com/java-8-lambda-tutorial/map-reduce-tutorial.php" rel="nofollow">tutorial at dreamsyssoft.com</a> a list of users</p>

<pre><code>private static List&lt;User&gt; users = Arrays.asList(
    new User(1, "Steve", "Vai", 40),
    new User(4, "Joe", "Smith", 32),
    new User(3, "Steve", "Johnson", 57),
    new User(9, "Mike", "Stevens", 18),
    new User(10, "George", "Armstrong", 24),
    new User(2, "Jim", "Smith", 40),
    new User(8, "Chuck", "Schneider", 34),
    new User(5, "Jorje", "Gonzales", 22),
    new User(6, "Jane", "Michaels", 47),
    new User(7, "Kim", "Berlie", 60)
);
</code></pre>

<p>is processed to get their average age like this:</p>

<pre><code>double average = users.parallelStream().map(u -&gt; u.age).average().getAsDouble();
</code></pre>

<p>In this case it is processed on single host.</p>

<p><strong>My question is</strong>: Can it be processed utilizing multiple hosts?</p>

<p>E.g. <code>Host1</code> processes the list below and returns <code>average1</code> for five users:</p>

<pre><code>new User(1, "Steve", "Vai", 40),
new User(4, "Joe", "Smith", 32),
new User(3, "Steve", "Johnson", 57),
new User(9, "Mike", "Stevens", 18),
new User(10, "George", "Armstrong", 24),
</code></pre>

<p>Similarly <code>Host2</code> processes the list below and returns <code>average2</code> for remaining five users:</p>

<pre><code>new User(2, "Jim", "Smith", 40),
new User(8, "Chuck", "Schneider", 34),
new User(5, "Jorje", "Gonzales", 22),
new User(6, "Jane", "Michaels", 47),
new User(7, "Kim", "Berlie", 60)
</code></pre>

<p>Finally <code>Host3</code> computes final result like:</p>

<pre><code>average = (average1 + average2)  / 2
</code></pre>

<p>Using distributed architecture it can be solved like remoting. Does Java 8 have some simpler way to solve the problem with some abstraction for it?</p>

<p>I know frameworks like Hadoop, Akka and Promises solve it. I am talking about pure Java 8. Can I get any docummentation and examples for <code>parallelStream()</code> for multiple hosts?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Here is the <a href="http://openjdk.java.net/projects/jdk8/features">list of features</a> scheduled for Java 8 as of September 2013.</p>

<p>As you can see, there is no feature dedicated to standardizing distributed computing over a cluster. The closest you have is <a href="http://openjdk.java.net/jeps/107">JEP 107</a>, which builds on the Fork/Join framework in JDK 7 to leverage multi-core CPU's. In Java 8, you will be able to use lambda expressions to perform bulk operations on collections in parallel by dividing the task among multiple processors. </p>

<p>Java 8 is also scheduled to feature <a href="http://openjdk.java.net/jeps/103">JEP 103</a>, which will also build on Java 7 Fork/Join to sort arrays in parallel. Meanwhile, since Fork/Join is clearly a big deal, it evolves further with <a href="http://openjdk.java.net/jeps/155">JEP 155</a>.</p>

<p>So there are no core Java 8 abstractions for distributed computing over a cluster--only over multiple cores. You will need to devise your own solution for real distributed computing using existing facilities.</p>

<p>As disappointing as that may be, I would point out that there are still wonderful open-source third party abstractions over Hadoop out there like <a href="http://cascalog.org/">Cascalog</a> and <a href="http://spark.incubator.apache.org/">Apache Spark</a>. Spark in particular lets you perform operations on your data in a distributed way through the <a href="http://spark.incubator.apache.org/docs/latest/scala-programming-guide.html">RDD</a> abstraction, which makes it feel like your data is just in a fancy array.</p>

<p>But you will have to wait for such things in core Java. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I'm not sure what will happen with Java 8 since it is too early to tell but there are a couple of open source projects that extend the map reduce capabilities of earlier functional programming languages that run in the JVM to distributed computing environments.</p>

<p>Recently, I took a traditional yet non-trivial Hadoop map reduce job (that takes raw performance data and prepares it for loading into an OLAP cube) and rewrote it in both Clojure running on Cascalog and Scala running on Spark. I documented my findings in a blog called <a href="http://glennengstrand.info/analytics/distributed/functional/programming" rel="nofollow">Distributed Computing and Functional Programming</a>. </p>

<p>These open source projects are mature and ready for prime time. They are supported by both Cloudera and Hortonworks.</p>
