    <div class="post-text" itemprop="text">
<blockquote>
  <p>I believe zookeeper is part of the Hadoop stack (right?)</p>
</blockquote>

<p><a href="http://zookeeper.apache.org/" rel="nofollow">ZooKeeper</a> (ZK) is not part of the Hadoop Stack. It's a Top Level Project (TLP) under Apache and is independent of Hadoop. So, first ZK has to be installed on EC2. Here are the <a href="http://pulasthisupun.blogspot.in/2012/08/setting-up-zookeeper-quorum-on-amazon.html" rel="nofollow">instructions</a> for the same.</p>

<blockquote>
  <p>how would I access it from an Elastic Mapreduce job in order to set and update a cluster-wide counter?</p>
</blockquote>

<p>Once installed ZK can be used to generate a cluster wide counter using the ZK API. Here (<a href="http://www.mail-archive.com/zookeeper-user@hadoop.apache.org/msg01976.html" rel="nofollow">1</a> and <a href="http://zookeeper-user.578899.n2.nabble.com/Sequence-Number-Generation-With-Zookeeper-td5378618.html" rel="nofollow">2</a>) discussions on the approach with the pros and cons. Here are some other <a href="http://srinathsview.blogspot.in/2012/04/generating-distributed-sequence-number.html" rel="nofollow">alternatives</a> for ZK for the same requirements.</p>

<p>You can, as Praveen Sripati answers.
But I wan't to clarify some points:</p>

<ol>
<li>Keep in mind, that zk has a limited write rate (~300 request per
second) </li>
<li>Clients can see stale data (zk don't guarantee read consistency across replicas).</li>
</ol>

<p>I suggest to use dedicated sequence generator server, which will generate sequences for you (and this service can use Zk or whatever it wants). One example of such service: <a href="https://github.com/kasabi/H1" rel="nofollow">https://github.com/kasabi/H1</a> </p>
