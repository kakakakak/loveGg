    <div class="post-text" itemprop="text">
<p><strong>Solution:</strong></p>

<p>Change line 1 of your code to:</p>

<pre><code>A= load '/user/cloudera' using PigStorage(':');
</code></pre>

<p>Having said that, are you sure your input data is in your home area? That seems unlikely. It's more likely to be in a folder within your home area, i.e. <code>/user/cloudera/input-data</code>.</p>

<p>Before running your job do:</p>

<pre><code>hdfs dfs -ls /user/cloudera
</code></pre>

<p>to confirm that input data is actually in that folder. If it's not, work out where it actually is, and make sure it is on the HDFS and not locally.</p>

<p><strong>Explanation:</strong></p>

<p>The relevant part of the logs is</p>

<pre><code>ERROR 2118: Input path does not exist: hdfs://quickstart.cloudera:8020/user/cloudera/user/cloudera
</code></pre>

<p>This suggests it's related to the input path. The part of your code that deals with the input path is</p>

<pre><code>A= load 'user/cloudera' using PigStorage(':');
</code></pre>

<p>By not adding a forward slash to <code>/user</code> then it assumes that everything is relative to your home area, so for example writing <code>load 'input'</code> would   lead the Pig job to read in <code>hdfs://quickstart.cloudera:8028/user/cloudera/input</code>. In your case then the missing slash means it adds it to your user area.</p>
