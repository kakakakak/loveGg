<p>I wrote a Map Reduce job in Hadoop Java API (1.0.3).
The job consists on summing all the values of a particular field (X) in my data and create a weighted distribution of other fields.</p>

<p>INPUT:</p>

<pre><code>1 field1_1 field2_1 field3_1 ... fieldX_1
2 field1_2 field2_2 field3_2 ... fieldX_2
3 field1_3 field2_3 field3_3 ... fieldX_3
</code></pre>

<p>Since one pair is emitted for any line in my data and that I need to use a single reducer in order to sum up all the values, I thought to set the same Reduce class as a Combiner.</p>

<p><strong>Total X summing:</strong></p>

<p>MAP OUTPUT:</p>

<pre><code>X fieldX_1
X fieldX_2 
X fieldX_3
X ... 
</code></pre>

<p>REDUCE OUTPUT:</p>

<pre><code>X fieldX_1+fieldX_2+fieldX_3+...
</code></pre>

<p>The weird thing that happens is that the combiner/reducer receives the same key multiple times:</p>

<pre><code>X [fieldX_1 fieldX_1 fieldX_1 ... fieldX_1]
X [fieldX_2 fieldX_2 fieldX_2 ...]
X [fieldX_3 fieldX_3 fieldX_3 ...]
X ...
</code></pre>

<p>I am sure about this because I am logging in the stderr all the  that are passed to each invocation of the reduce method for dubugging.</p>

<p>I want to add a more concrete example:</p>

<p>Data:</p>

<pre><code>1 field1_1 field2_1 field3_1 ... 10
2 field1_2 field2_2 field3_2 ... 20
3 field1_3 field2_3 field3_3 ... 30
4 field1_1 field2_1 field3_1 ... 10
5 field1_2 field2_2 field3_2 ... 40
6 field1_3 field2_3 field3_3 ... 20
...
</code></pre>

<p>Map output:</p>

<pre><code>X 10
X 20
X 30
X 10
X 40
X 20 
</code></pre>

<p>Reduce input (with combiner):</p>

<pre><code>X [10 10 10 10]
X [20 20 20]
X [30 30 30 30 30 30 30]
X [40 40]
</code></pre>

<p>Reduce output (with combiner):</p>

<pre><code>X 40
X 60
X 210
X 80
</code></pre>

<p>X is a constant label (field name).
To note that the reducer is called with the same key X and collection of identical values of X, e.g. [10 10 10 ...] or [30 30 30...].
Each sum will be output separately. I mean the algorithm works fine but at this stage needs an additional reduce step to sum up the duplicates.</p>

<p>Real log example:</p>

<pre><code>Nov 06, 2013 8:50:12 AM MYCLASS logInputError

</code></pre>

<p>If I remove the Combiner everything works fine.
I understand that the Combiner may be invoked 0, 1 or multiple times, but what about the Reducer? It should be invoked exactly once, isn't it?</p>

<p>But even more weird is that I am repeating a similar procedure for the fields distribution, and this happens only for X summing problem...</p>

<p><strong>Weighted Field Distribution</strong></p>

<p>MAP OUTPUT (example field1):</p>

<pre><code>field1_1 X_1
field1_2 X_2
field1_3 X_3
... 
</code></pre>

<p>REDUCE OUTPUT:</p>

<pre><code>field1(class1) fieldX(class1)+fieldX(class1)+fieldX(class1)+...
field1(class2) fieldX(class2)+fieldX(class2)+fieldX(class2)+...
field1(class3) fieldX(class3)+fieldX(class3)+fieldX(class3)+...
...
</code></pre>

<p>Essentially, for each value of field1 I am summing up all the associated values of fieldX and repeating the same procedure for several fields (field1, field2, field3...).</p>

<p>For those emitted pairs, the reducer receives the single key (field1(class1)) and the array of values ([fieldX(class1)...]) as should normally behave.</p>

<p><strong>Conclusion</strong></p>

<p>One consideration is that for the X summing problem the single key (X) maps a number of values equals to the size of the data (number of lines). Whilst, for the field weighted distribution the values are spread among the several class labels that field contains.</p>

<p>Is it a bug in my code or there is some procedural detail of Hadoop that I am not considering?</p>

<p>Standing to the M/R paradigm a Reducer class should receive all the values of a particular key at once, not divided in more partitions.</p>

<p>Hope to receive good feedback.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<blockquote>
  <p>The weird thing that happens is that the combiner receives the same key multiple times</p>
</blockquote>

<p>This is possible as the combiner may be called multiple times by the MapReduce framework, see <a href="http://hadoop.apache.org/docs/current/api/org/apache/hadoop/mapred/JobConf.html#setCombinerClass%28java.lang.Class%29" rel="nofollow">JobConf#setCombinerClass()</a>:</p>

<blockquote>
  <p>The framework may invoke the combiner <strong>0, 1, or multiple</strong> times, in both
  the mapper and reducer tasks. In general, the combiner is called as
  the sort/merge result is written to disk. The combiner must:</p>
  
  <ul>
  <li>be side-effect free</li>
  <li>have the same input and output key types and the same input and output value types</li>
  </ul>
</blockquote>
