  <p>Users create RDDs in two ways: by loading an external dataset, or by

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>An RDD is, essentially, the Spark representation of a set of data, spread across multiple machines, with APIs to let you act on it. An RDD could come from any datasource, e.g. text files, a database via JDBC, etc.</p>

<p>The formal definition is:</p>

<blockquote>
  <p>RDDs are fault-tolerant, parallel data structures that let users
  explicitly persist intermediate results in memory, control their
  partitioning to optimize data placement, and manipulate them using a
  rich set of operators.</p>
</blockquote>

<p>If you want the full details on what an RDD is, read one of the core Spark academic papers, <a href="https://www.cs.berkeley.edu/~matei/papers/2012/nsdi_spark.pdf">Resilient Distributed Datasets: A Fault-Tolerant Abstraction for In-Memory Cluster Computing</a></p>
    </div>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>RDD</strong> is a logical reference of a <code>dataset</code> which is partitioned across many server machines in the cluster. <strong>RDD</strong>s are Immutable and are self recovered in case of failure.</p>

<p><code>dataset</code> could be the data loaded externally by the user. It could be a json file, csv file or a text file with no specific data structure.</p>

<p><a href="http://i.stack.imgur.com/IEcsA.png"><img src="http://i.stack.imgur.com/IEcsA.png" alt="enter image description here"></a></p>

<p><strong>UPDATE</strong>: <a href="http://www.cs.berkeley.edu/~matei/papers/2012/nsdi_spark.pdf">Here</a> is an article describing RDD internals: </p>

<p>Hope this helps.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>To compare RDD with scala collection, below are few differences </p>

<ol>
<li>Same but runs on a cluster</li>
<li>Lazy in nature where scala collections are strict </li>
<li>RDD is always Immutable i.e., you can not change the state of the data in the collection </li>
<li>RDD are self recovered  i.e., fault-tolerant  </li>
</ol>
