<p>To create MapReduce jobs you can either use the old <code>org.apache.hadoop.mapred</code> package or the newer <code>org.apache.hadoop.mapreduce</code> package for Mappers and Reducers, Jobs ... The first one had been marked as deprecated but this got reverted meanwhile. Now I wonder whether it is better to use the old mapred package or the new mapreduce package to create a job and why. Or is it just dependent on whether you need stuff like the MultipleTextOutputFormat which is only available in the old mapred package?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Functionality wise there is not much difference between the old (<code>o.a.h.mapred</code>) and the new (<code>o.a.h.mapreduce</code>) API. The only significant difference is that records are pushed to the mapper/reducer in the old API. While the new API supports both pull/push mechanism. You can get more information about the pull mechanism <a href="http://stackoverflow.com/questions/7537797/how-to-pull-data-in-the-map-reduce-functions">here</a>.</p>

<p>Also, the old API has been <a href="https://issues.apache.org/jira/browse/MAPREDUCE-1735">un-deprecated</a> since 0.21. You can find more information about the new API <a href="http://www.cloudera.com/blog/2010/08/what%E2%80%99s-new-in-apache-hadoop-0-21/">here</a>.</p>

<p>As you mentioned some of the classes (like MultipleTextOutputFormat) have not been migrated to the new API, due to this and the above mentioned reason it's better to stick to the old API (although a translation is usually quite simple).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Both the old and new APIs are good. The new API is cleaner though. Use the new API wherever you can, and use the old one wherever you need specific classes that are not present in the new API (like <code>MultipleTextOutputFormat</code>)</p>

<p>But do take care not to use a mix of the old and new APIs in the same Mapreduce job. That leads to weird problems.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>Old API (mapred)</strong></p>

<ol>
<li><p>Exists in Package org.apache.hadoop.mapred</p></li>
<li><p>Provide A map/reduce job configuration.</p></li>
<li>Reduces values for a given key, based on the Iterator</li>
<li>Package <a href="http://hadoop.apache.org/docs/r2.2.0/api/org/apache/hadoop/mapred/package-summary.html" rel="nofollow">Summary</a></li>
</ol>

<p><strong>New API (mapreduce)</strong></p>

<ol start="5">
<li><p>Exists  in Package org.apache.hadoop.mapreduce</p></li>
<li><p>Job configuration is done by separate class, Called JobConf which is extension of Configuration<br>
     Class</p></li>
<li><p>Reduces values for a given key, based on the Iterable</p></li>
<li><p><a href="http://hadoop.apache.org/docs/r2.3.0/api/org/apache/hadoop/mapreduce/package-summary.html" rel="nofollow">Package Summary</a></p></li>
</ol>
