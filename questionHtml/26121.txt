<p>Hi i'm new to Hadoop and just started learning a couple days ago. I just followed the instructions from Digital Ocean to <a href="https://www.digitalocean.com/community/tutorials/how-to-install-hadoop-on-ubuntu-13-10" rel="nofollow">setup a Hadoop cluster</a>. Afterwards I just tried a simple example program called WordCount from the <a href="http://hadoop.apache.org/docs/current/hadoop-mapreduce-client/hadoop-mapreduce-client-core/MapReduceTutorial.html" rel="nofollow">Hadoop docs</a>. </p>

<p>My hadoop version is 2.5.1 which is the same version with what is used on the tutorial, and it's running on Ubuntu Precise. I'm ensuring that I've done the proper setup as the tutorial said. Here's the end of my <strong>~/.bashrc</strong> contents.</p>

<pre><code>        ...
#HADOOP VARIABLES START
export JAVA_HOME=/usr/lib/jvm/java-7-oracle
export HADOOP_INSTALL=/usr/local/hadoop
export PATH=$PATH:$HADOOP_INSTALL/bin
export PATH=$PATH:$HADOOP_INSTALL/sbin
export HADOOP_MAPRED_HOME=$HADOOP_INSTALL
export HADOOP_COMMON_HOME=$HADOOP_INSTALL
export HADOOP_HDFS_HOME=$HADOOP_INSTALL
export YARN_HOME=$HADOOP_INSTALL
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_INSTALL/lib/native
export HADOOP_OPTS="-Djava.library.path=$HADOOP_INSTALL/lib"
export HADOOP_PREFIX=/usr/local/hadoop
#HADOOP VARIABLES END
</code></pre>

<p>Also, I checked on my java home config and the result is like below</p>

<pre><code>sudo update-alternatives --config java
There are 3 choices for the alternative java (providing /usr/bin/java).

  Selection    Path                                            Priority   Status
------------------------------------------------------------
  0            /usr/lib/jvm/java-6-openjdk-amd64/jre/bin/java   1061      auto mode
  1            /usr/lib/jvm/java-6-openjdk-amd64/jre/bin/java   1061      manual mode
  2            /usr/lib/jvm/java-7-openjdk-amd64/jre/bin/java   1051      manual mode
* 3            /usr/lib/jvm/java-7-oracle/jre/bin/java          1         manual mode
</code></pre>

<p>So i changed all JAVA_HOME value both in bashrc and hadoop-env.sh files to <code>/usr/lib/jvm/java-7-oracle</code> . I'm ensuring as well that the Dfs and Yarn are both started.</p>

<p>However, when i <strong>compile</strong> the WordCount.java using this command</p>

<pre><code>hadoop com.sun.tools.javac.Main WordCount.java 
</code></pre>

<p>Nothing is going my way. I got this error. Note that i'm using Hadoop command instead bin/hadoop as the command is working properly since it was defined in <code>bashrc</code> file.</p>

<pre><code>Error: Could not find or load main class com.sun.tools.javac.Main
</code></pre>

<p>What is the possible cause of this error and how to get rid of this? It might be java classpath issue i think, but i'm still not be able to figure out the detail. Every workarounds regarding this problem i got are about executing <code>java</code> or <code>javac</code> command, not <code>hadoop</code> command. </p>

<p>I just want to get the sample program working first, before getting started to learn how it works. Any help would be appreciated. Thanks..</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><code>Assuming you are using Eclipse or any other IDE</code>,<br>
As i mentioned in <a href="http://stackoverflow.com/questions/27063886/map-reduce-client-jars-for-2-4-1-hadoop-in-eclipse/27086746#27086746"><strong>this</strong></a> post, create a <code>maven</code> based simple wordcount project with your class.<br>
So all dependencies will be handled. Next <code>Right click</code> on your project and select <code>Export</code> option and give a name <code>&lt;hadoop-example&gt;.jar</code> and then next, next generate a jar file for your wordcount project.   </p>

<blockquote>
  <p>You don't need to explicitly compile your programs, 
  Eclipse will do it for you once export is successful</p>
</blockquote>

<p>If you installed hadoop on the same machine, then <code>start all the daemons</code> and check using <code>jps</code> whether all daemons started or not.<br>
Else copy the jar file to Virtual Machine where hadoop is installed.
 Go to the jar location and run the following command.  </p>

<pre><code>hadoop jar &lt;hadoop-example&gt;.jar &lt;fully qualified main class name&gt; hdfsInputPath hdfsOutputPath  
</code></pre>

<p>This will run your main class (<code>WordCount</code> in your case)<br>
One should use the above command to run any hadoop program using command line by using jar file and then main class name(<code>WorkCount</code>) in your case.<br>
You can make the <code>WordCount</code> as application entry point Main class while exporting the jar file. So you don't need to give the fully qualifed name. it would be like this.   </p>

<pre><code>hadoop jar &lt;hadoop-example&gt;.jar  hdfsInputPath hdfsOutputPath 
</code></pre>

<p>Please try this and let us know if it helps you.<br>
<strong>Update:</strong> As mentioned in the comments, IDE is not using.  </p>

<blockquote>
  <p>The above paths you set up is used by <code>Map/Reduce</code> while running the
  program.</p>
</blockquote>

<p>But before you run you should make a jar which needs all the dependents to compile.<br>
So take one variable as <code>HADOOP_CLASSPATH_LIBS</code> and assign all the jars in the<br>
 <code>&lt;installed-hadoop&gt;/share/hadoop/&lt;subfolder&gt;/lib</code>.<br>
<code>For every jar, you should give the absolute path and export this variable</code>.<br>
Which will enable you to compile and then make a jar file. Once you are having jar, you can follow the above steps to run it.
If need more help, can assist.<br>
Hope it helps.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The <a href="http://hadoop.apache.org/docs/current/hadoop-mapreduce-client/hadoop-mapreduce-client-core/MapReduceTutorial.html" rel="nofollow">Apache Hadoop tutorial</a> assumes that the environmental variables are set as follows:  </p>

<pre><code>export JAVA_HOME=/usr/java/default
export PATH=$JAVA_HOME/bin:$PATH
export HADOOP_CLASSPATH=$JAVA_HOME/lib/tools.jar
</code></pre>

<p>Perhaps the <a href="https://www.digitalocean.com/community/tutorials/how-to-install-hadoop-on-ubuntu-13-10" rel="nofollow">Digital Ocean hadoop tutorial</a>, which I also followed, ought to recommend adding those two latter variables to the ~/.bashrc so that it ends up looking like this:</p>

<pre><code>#HADOOP VARIABLES START
export JAVA_HOME=/usr/lib/jvm/java-7-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
export HADOOP_CLASSPATH=$JAVA_HOME/lib/tools.jar
export HADOOP_INSTALL=/usr/local/hadoop
export PATH=$PATH:$HADOOP_INSTALL/bin
export PATH=$PATH:$HADOOP_INSTALL/sbin
export HADOOP_MAPRED_HOME=$HADOOP_INSTALL
export HADOOP_COMMON_HOME=$HADOOP_INSTALL
export HADOOP_HDFS_HOME=$HADOOP_INSTALL
export YARN_HOME=$HADOOP_INSTALL
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_INSTALL/lib/native
export HADOOP_OPTS="-Djava.library.path=$HADOOP_INSTALL/lib"
#HADOOP VARIABLES END
</code></pre>

<p>It worked for my installation.  See the new compiled class files listed in the output:</p>

<p>Before: </p>

<pre><code>ubuntu@mail:/usr/local/hadoop$ ls
bin  include  lib      LICENSE.txt  NOTICE.txt  README.txt  share              WordCount.java
etc  input    libexec  logs         output      sbin        WordCount_classes
</code></pre>

<p>After:</p>

<pre><code>ubuntu@mail:/usr/local/hadoop$ bin/hadoop com.sun.tools.javac.Main WordCount.java
ubuntu@mail:/usr/local/hadoop$ ls
bin      input    LICENSE.txt  output      share              WordCount$IntSumReducer.class
etc      lib      logs         README.txt  WordCount.class    WordCount.java
include  libexec  NOTICE.txt   sbin        WordCount_classes  WordCount$TokenizerMapper.class
</code></pre>

<p>Another helpful resource was this: </p>

<blockquote>
  <p><a href="http://ubuntuforums.org/archive/index.php/t-634996.html" rel="nofollow">http://ubuntuforums.org/archive/index.php/t-634996.html</a> Append the
  following lines to the opened .bashrc file, save it and close: export
  JAVA_HOME="/usr/lib/jvm/java-6-sun-1.6.0.22" export
  PATH=$PATH:$JAVA_HOME/bin issue the following command in the terminal:
  source $HOME/.bashrc</p>
  
  <p>please refer this blog post for more info
  (<a href="http://sureshatt.blogspot.com/2011/01/easiest-way-to-install-java-in-ubuntu.html" rel="nofollow">http://sureshatt.blogspot.com/2011/01/easiest-way-to-install-java-in-ubuntu.html</a>)</p>
</blockquote>
