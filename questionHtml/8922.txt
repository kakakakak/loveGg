    <div class="post-text" itemprop="text">

<p>I am looking for an S3 alternative which relies on a RESTful API, so that I can simply insert links such as <a href="http://datastore1.example.com/ID">http://datastore1.example.com/ID</a> and they are directly downloadable.
I have looked at RIAK and Bitcache. They both seem very nice: <a href="http://bitcache.org/api/rest">http://bitcache.org/api/rest</a> but they have one problem. I want to be the only one who can upload data. Else anyone could use our datastore by sending a PUT Request. </p>

<p>Is there a way to configure RIAK so that everyone can "GET" but not everyone can PUT or DELETE files except me? Are there other services which you can recommend?</p>

<p>Also adding Bounty :)</p>

<p>Requirements:</p>

<ul>
<li>RESTful API</li>
<li>Guests GET only</li>
<li>Runs on Debian</li>
</ul>

<p>Very nice to have:</p>

<ul>
<li>auto distributed</li>
</ul>

<p>EDIT: To clarify I don't want any connection to S3 I have great servers just lying around with harddrives and very good network connection (3Gbps) I don't need S3..</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The Riak authors <a href="https://wiki.basho.com/pages/viewpage.action?pageId=917629#Cassandra-HTTPBasedAPI" rel="nofollow">recommend</a> to put a HTTP proxy in front of Riak in order to provide access control. You can chose any proxy server you like (such as nginx or Apache), and any access control policy you like (such as authorization based on IP addresses, HTTP basic auth, or cookies, assuming your proxy server can handle it). For example, in nginx, you might specify <a href="http://wiki.nginx.org/HttpCoreModule#limit_except" rel="nofollow">limit_except</a> (likewise LimitExcept in Apache).</p>

<p>Alternatively, you could also add access control to Riak directly. It's based on Webmachine, so one approach would be to implement <a href="http://bitbucket.org/justin/webmachine/wiki/ExampleResources" rel="nofollow">is_authorized</a>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you are looking for a distributed file system, why don't you try hadoop hdfs?</p>

<p><a href="http://hadoop.apache.org/common/docs/r0.17.0/hdfs_design.html" rel="nofollow">http://hadoop.apache.org/common/docs/r0.17.0/hdfs_design.html</a></p>

<p>There is a Java API available:</p>

<p><a href="http://hadoop.apache.org/common/docs/r0.20.2/api/org/apache/hadoop/fs/FileSystem.html" rel="nofollow">http://hadoop.apache.org/common/docs/r0.20.2/api/org/apache/hadoop/fs/FileSystem.html</a></p>

<p>Currently, security is an issue - at least if you have access to a terminal:</p>

<p><a href="http://developer.yahoo.com/hadoop/tutorial/module2.html#perms" rel="nofollow">http://developer.yahoo.com/hadoop/tutorial/module2.html#perms</a></p>

<p>But you could deploy hdfs, put an application server (using the Java API) in front of it (GlassFish) and use Jersey to build the RESTful API:</p>

<p><a href="http://jersey.java.net/" rel="nofollow">http://jersey.java.net/</a></p>

<p>If you're interested in building such a thing, please let me know, for I may be building something similar quite soon.</p>

<p>You can use the Cloudera Hadoop Distribution to make life a bit more easy:</p>

<p><a href="http://www.cloudera.com/hadoop/" rel="nofollow">http://www.cloudera.com/hadoop/</a></p>

J.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can develop it yourself as a web app or a part of your existing application. It will consume HTTP requests, retrieve their URI component, convert it to S3 object name and use <a href="http://s3.amazonaws.com/doc/s3-developer-guide/SOAPGetObject.html" rel="nofollow">getObject()</a> to get its content (using one of available S3 SDKs, for example <a href="http://aws.amazon.com/sdkforjava/" rel="nofollow">AWS Java SDK</a>).</p>

<p>You can try a hosted solution - <a href="http://www.s3auth.com" rel="nofollow">s3auth.com</a> (I'm a developer). It's an open source project, and you can see how this mechanism is implemented internally at <a href="https://github.com/yegor256/s3auth/blob/master/s3auth-relay/src/main/java/com/s3auth/relay/SecuredHost.java" rel="nofollow">one of its core classes</a>. HTTP request is processed by the service and then re-translated to Amazon S3 internal authentication scheme.</p>
