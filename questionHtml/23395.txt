<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The instructions at <a href="http://wiki.apache.org/hadoop/HowToContribute">http://wiki.apache.org/hadoop/HowToContribute</a> explain some of the less obvious aspects of building hadoop - including this one. <strong>The main problem here is that Protocol Buffers are required by hadoop's YARN framework, and you most likely do not have them installed.</strong></p>

<p>It seems like the problem might be that your not interpretting the error correctly, so -- </p>

<p><strong>How to Isolate the Build error</strong></p>

<p>Any mojo which fails should be associated with a class.  </p>

<p>In this case, the mojo that is failing is here :</p>

<p><a href="https://svn.apache.org/repos/asf/hadoop/common/trunk/hadoop-maven-plugins/src/main/java/org/apache/hadoop/maven/plugin/protoc/ProtocMojo.java">https://svn.apache.org/repos/asf/hadoop/common/trunk/hadoop-maven-plugins/src/main/java/org/apache/hadoop/maven/plugin/protoc/ProtocMojo.java</a></p>

<p>Looking at the code, its clear that this Mojo requires the "protoc" program to run. </p>

<p><a href="http://code.google.com/p/protobuf/">http://code.google.com/p/protobuf/</a></p>

<p><strong>Solution</strong></p>

<p>To build hadoop, you need protocol buffers.  This can be installed on a *NIX machine from source, by running the following commands to grab the code from googlecode (taken from <a href="http://numbers.brighterplanet.com/2012/04/14/how-to-install-mosh-on-amazon-ec2/">http://numbers.brighterplanet.com/2012/04/14/how-to-install-mosh-on-amazon-ec2/</a>).</p>

<pre><code>wget http://protobuf.googlecode.com/files/protobuf-2.4.1.tar.gz
tar xzf protobuf-2.4.1.tar.gz
cd protobuf-2.4.1
./configure
make
sudo make install
sudo ldconfig
</code></pre>
