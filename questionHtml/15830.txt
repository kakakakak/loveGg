<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Cascading and Oozie are not in the same category.</p>

<p>Oozie is a workflow scheduler.</p>

<p>Cascading is an API for creating workflows. It is agnostic about schedulers, i.e., it should run with whatever scheduler system that you use.</p>

<p>There is perhaps some confusion because the Oozie docs mention a "DAG", and both run atop Hadoop.</p>

<p>Also, Cascading has a notion of "data availability" in the checkpoint support, which is supported in Oozie, albeit differently.</p>

<p>Personally i play around with both to some extend, what i found interesting with cascading is </p>

<p>1)concise and expressive in terms of simple keywords like flow,tap,pipe etc.,</p>

<p>2)amazing TDD based approach for local development and research</p>

<p>3)nice planner view(.dot file) and will be useful once the project is grown, so maintenance is ease.</p>

<p>4)DSL based approach using groovy,scala,cloujre. so no need to worry about learning any new language or rather hadoop.</p>

<p>5)simple cloud deployment(e.g. amazon support as raw jar deployment).</p>

<p>6)you can call anything like existing pig or hive or pure other MR jar as long as they expose java api.</p>

<p>7)amazing for ML and NLP related works.</p>
