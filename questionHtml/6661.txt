<td class="answercell">
    <div class="post-text" itemprop="text">
<p>This is the problem with spark communicating with Application Master. </p>

<p>The RM and NM talk to each other over RPC so the problem could be launch_container.cmd is not running correctly. Check that the NM has communicating with RM while submitting job</p>

<p>Try adding this to your yarn-site.xml:</p>

<pre><code>&lt;property&gt;
  &lt;name&gt;yarn.nodemanager.delete.debug-delay-sec&lt;/name&gt;
  &lt;value&gt;1200&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>This will ensure that the launch_container.cmd from the NM error seen does not get deleted ( will remain around for 20 mins - increase 1200 to a higher number if needed). Now, what you can do is try and run that launch_container.cmd script manually from the container dir and see where it bails out.</p>

<p>Hope this will help you.</p>
