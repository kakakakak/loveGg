<p>I am trying to create a PHP page to launch cluster and add jobs to the jobflow.</p>

<pre><code>$response = $emr-&gt;run_job_flow($nameOfRun, array(
        'Ec2KeyName' =&gt; 'hadoop',
        'HadoopVersion' =&gt; '0.20',
        'KeepJobFlowAliveWhenNoSteps' =&gt; $clusterAlive,
        'InstanceGroups' =&gt; array(
                array( // Group #1
                        'InstanceCount' =&gt; $numOfMaster,
                        'InstanceRole' =&gt; 'MASTER',
                        'InstanceType' =&gt; $masterInstanceType,
                        'Market' =&gt; 'ON_DEMAND',
                        'Name' =&gt; 'Master',
                ),
                array( // Group #2
                        'InstanceCount' =&gt; $numOfSlaves,
                        'InstanceRole' =&gt; 'CORE',
                        'InstanceType' =&gt; $slaveInstanceType,
                        'Market' =&gt; 'ON_DEMAND',
                        'Name' =&gt; 'Slaves',
                )
        ),
        'Placement' =&gt; array(
                'AvailabilityZone' =&gt; 'us-east-1d'
        )
));
</code></pre>

<p>The cluster is launched and I can see the job flow ID. But it starts to shut down immediately after launch. When I do a <code>elastic-mapreduce --list</code>, for that cluster the state is Failed.</p>

<p>Any known issues or anywhere I am doing wrong? Is AvailabilityZone may be causing the issue?</p>

<p>I checked from the AWS Console, it is giving this error - <code>The given SSH key name was invalid</code>
I checked, the access key and secret key I provided in config.inc.php are correct and working.</p>

Kartikeya Sinha</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Looks like there are four different gotchas that might cause the error, as outlined in this <a href="https://forums.aws.amazon.com/message.jspa?messageID=208470" rel="nofollow">AWS thread:</a></p>

<ol>
<li>Specifying the exact keypair name from the console.</li>
<li>Specifying a keypair that exists in a different region than your the region of your endpoint.</li>
<li>Using a keypair for a different AWS service, instead of one for EC2.</li>
<li>Using a renamed keypair pem file, when it should match the keypair in the console (really a special case of 1).</li>
</ol>

<p>If all that looks right, try deleting the keypair and making a new one.</p>

<hr>

<p>Based on a reading of the <a href="http://docs.amazonwebservices.com/AWSSDKforPHP/latest/#m=AmazonEMR/run_job_flow" rel="nofollow">PHP SDK</a> docs, I think the problem is that you're specifying <code>EC2KeyName</code> at all (your comments suggest you don't want to tie a particular pem file to the EC2 instances; you rather just want access with a secret key and access key).</p>

<blockquote>
  <p>Ec2KeyName - string - Optional - Specifies the name of the Amazon EC2 key pair that can be used to ssh to the master node as the user called “hadoop.” [Constraints: The value must be between 0 and 256 characters, and must match the following regular expression pattern: </p>
  
  <p>[\u0020-\uD7FF\uE000-\uFFFD\uD800\uDC00-\uDBFF\uDFFF\r\n\t]*]</p>
</blockquote>

<p>This parameter refers to the pem keypair files you can associate with EC2 instances. Your comments suggest you're speaking specifically about AccessKeys and SecretAccessKeys, which are different concepts. The pem files give you <code>ssh</code> access to EC2 instances. The latter authorize you against the AWS API.</p>

<p>You should be able to kill that optional parameter and have the job work, although you won't be able to <code>ssh</code> into its affiliated EC2 instances without a pem file.</p>
