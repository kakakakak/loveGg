<p>I'm trying to recreate Hadoop's <a href="http://hadoop.apache.org/docs/r1.1.1/mapred_tutorial.html">word count</a> map / reduce logic in a simple Scala program for learning</p>

<p>This is what I have so far</p>

<pre><code>val words1 = "Hello World Bye World"      
val words2 = "Hello Hadoop Goodbye Hadoop"

val input = List(words1,words2)           
val mapped = input.flatMap(line=&gt;line.split(" ").map(word=&gt;word-&gt;1))
    //&gt; mapped  : List[(String, Int)] = List((Hello,1), (World,1), (Bye,1), 
    //                                       (World,1), (Hello,1), (Hadoop,1), 
    //                                       (Goodbye,1), (Hadoop,1))

mapped.foldLeft(Map[String,Int]())((sofar,item)=&gt;{
    if(sofar.contains(item._1)){
        sofar.updated(item._1, item._2 + sofar(item._1))
    }else{
        sofar + item
    }
})                              
    //&gt;Map(Goodbye -&gt; 1, Hello -&gt; 2, Bye -&gt; 1, Hadoop -&gt; 2, World -&gt; 2)
</code></pre>

<p>This seems to work, but I'm sure there is a more idiomatic way to handle the reduce part (foldLeft) </p>

<p>I was thinking about perhaps a multimap,  but I have a feeling Scala has a way to do this easily </p>

<p>Is there? e.g. <strong>a way to add to a map, and if the key exists, instead of replacing it, adding the value to the existing value</strong>. I'm sure I've seen this quesion somewhere, but couldn't find it and neither the answer.</p>

<p>I know <code>groupBy</code> is the way to do it probably in the real world, but I'm trying to implement it as close as possible to the original map/reduce logic in the link above.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can use <a href="https://github.com/scalaz/scalaz">Scalaz's</a> <code>|+|</code> operator because <code>Maps</code> are part of the <a href="http://dnene.bitbucket.org/docs/scalaz-guide/ref-identity.html#id1">Semigroup</a> typeclass:</p>

<p>The <code>|+|</code> operator is the Monoid <code>mappend</code> function (a Monoid is any "thing" that can be "added" together. Many things can be added together like this: Strings, Ints, Maps, Lists, Options etc. An example:</p>

<pre><code>scala&gt; import scalaz._
import scalaz._

scala&gt; import Scalaz._
import Scalaz._

scala&gt; val map1 = Map(1 -&gt; 3 , 2 -&gt; 4)
map1: scala.collection.immutable.Map[Int,Int] = Map(1 -&gt; 3, 2 -&gt; 4)

scala&gt; val map2 = Map(1 -&gt; 1, 3 -&gt; 6)
map2: scala.collection.immutable.Map[Int,Int] = Map(1 -&gt; 1, 3 -&gt; 6)

scala&gt; map1 |+| map2
res2: scala.collection.immutable.Map[Int,Int] = Map(1 -&gt; 4, 3 -&gt; 6, 2 -&gt; 4)
</code></pre>

<p><strong>So in your case,</strong> rather then create a <code>List[(String,Int)]</code>, create a <code>List[Map[String,Int]]</code>, and then sum them:</p>

<pre><code>val mapped = input.flatMap(_.split(" ").map(word =&gt; Map(word -&gt; 1)))
mapped.suml
</code></pre>

<p>You could use a map that returns 0 as default value. Map offers withDefaultValue:</p>

<pre><code>def withDefaultValue[B1 &gt;: B](d: B1): Map[A, B1]
</code></pre>

<p>The same map with a given default value:</p>

<pre><code>val emptyMap = Map[String,Int]().withDefaultValue(0)
mapped.foldLeft(emptyMap)((sofar,item) =&gt; {
    sofar.updated(item._1, item._2 + sofar(item._1))
})  
</code></pre>
