    <div class="post-text" itemprop="text">

<p>I have the following rows in a HBase table called <code>test</code></p>

<p><code>ROW                                   COLUMN+CELL                                                                               row1                                 column=cf:a, timestamp=1429204170712, value=value1
 row2                                 column=cf:b, timestamp=1429204196225, value=value2
 row3                                 column=cf:c, timestamp=1429204213427, value=value3</code></p>

<p>I am trying to retrieve all the rows with rowkey matching prefix <code>row</code> using <strong>Suffix Globbing</strong>, as mentioned <a href="https://wiki.apache.org/hadoop/Hbase/Stargate" rel="nofollow">here</a></p>

<p>But why do I get <code>Bad request</code> when I try <code>http://localhost:8080/test/row*</code> where <code>localhost:8080</code> is where the HBase REST server Stargate is listening, <code>test</code> is the table and <code>row</code> is a partial rowkey. I executed it in a browser and in a REST client Poster (Firefox plugin). Executing the URL <code>http://localhost:8080/test/row*/cf</code> gives the response <code>value1</code> but I would like to retrieve the values in all the rows with rowkey matching prefix <code>row</code>.</p>

<p>I am running HBase 0.94.26, Stargate (came bundled with HBase), Hadoop 1.2.1, Ubuntu 12.04 virtual machine. </p>

<p>Is it possible to retrieve all the rows programmatically atleast?</p>

  <p>Multiple value queries of a row can optionally append a suffix glob on
  the row key. This is a restricted form of scanner which will return
  all values in all rows that have keys which contain the supplied key
