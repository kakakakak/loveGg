<p>I am new to MapReduce and would like your opinion on the best MapReduce approach for the following task.</p>

<p>I have a single large document in the format</p>

<pre><code>1 2 3
2
2 3 4 5
</code></pre>

<p>Each line has a list of numbers. I want to list each possible (pair) combination of numbers in any line. And I want the number of lines containing each given pair.</p>

<p>The result will be like</p>

<pre><code>element1 element2 occurrences
1        1        1
1        2        1
1        3        1
2        2        3
2        3        2
3        3        2
3        4        1
3        5        1
</code></pre>

<p>There are about 2M lines in the documents, and about 1.5M different numbers. And there will be about 2.5G different pairs of numbers to be counted. </p>

<p>The stright forward pseudo code is like:
Invoke map for each line in the document</p>

<pre><code>Map(int lineId, list&lt;int&gt; elements)
{
  for each pair of integers in elements
    emit(pair, 1)
}

Reduce((int, int) pair, list&lt;int&gt; counts)
{
  return sum(counts)
}
</code></pre>

<p>But this approach will require 2M mappers and 2.5G reducers. Is this a plausible way to go?
Planning on trying Hadoop on Azure.</p>

    <div class="post-text" itemprop="text">
<blockquote>
  <p>But this approach will require 2M mappers and 2.5G reducers. Is this a plausible way to go? Planning on trying Hadoop on Azure.</p>
</blockquote>

<p>This assumption is not correct.</p>

<p>The number of mappers for the <a href="http://hadoop.apache.org/docs/r0.20.0/api/org/apache/hadoop/mapred/FileInputFormat.html" rel="nofollow">FileInputFormat</a> is equal to the number of Input Splits. An Input Split can map to a block in HDFS, which is defaulted to 64MB. So, if the input file is 1024 MB, then 16 map tasks will be launched.</p>

<p>The number of reducers is configurable using the <a href="http://hadoop.apache.org/docs/r0.20.2/mapred-default.html" rel="nofollow">mapred.reduce.tasks</a> parameter which is defaulted to 1. Also, note that a <a href="http://hadoop.apache.org/docs/r0.20.0/api/org/apache/hadoop/mapred/FileInputFormat.html" rel="nofollow">combiner</a> can be used to make the job complete faster.</p>

<p>Would suggest to go through the <a href="http://shop.oreilly.com/product/0636920021773.do" rel="nofollow">Hadoop - The Definitive Guide</a> for better understanding of MapReduce and Hadoop.</p>
