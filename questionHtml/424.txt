<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Although it is possible to use <code>INSERT OVERWRITE</code> to get data out of Hive, it might not be the best method for your particular case. First let me explain what <code>INSERT OVERWRITE</code> does, then I'll describe the method I use to get tsv files from Hive tables.</p>

<p>According to <a href="https://cwiki.apache.org/confluence/display/Hive/LanguageManual+DML#LanguageManualDML-Writingdataintothefilesystemfromqueries">the manual</a>, your query will store the data in a directory in HDFS. The format will not be csv.</p>

<blockquote>
  <p>Data written to the filesystem is serialized as text with columns separated by ^A and rows separated by newlines. If any of the columns are not of primitive type, then those columns are serialized to JSON format.</p>
</blockquote>

<p>A slight modification (adding the <code>LOCAL</code> keyword) will store the data in a local directory.</p>

<pre><code>INSERT OVERWRITE LOCAL DIRECTORY '/home/lvermeer/temp' select books from table;
</code></pre>

<p>When I run a similar query, here's what the output looks like.</p>

<pre><code>[lvermeer@hadoop temp]$ ll
total 4
-rwxr-xr-x 1 lvermeer users 811 Aug  9 09:21 000000_0
[lvermeer@hadoop temp]$ head 000000_0 
"row1""col1"1234"col3"1234FALSE
"row2""col1"5678"col3"5678TRUE
</code></pre>

<p>Personally, I usually run my query directly through Hive on the command line for this kind of thing, and pipe it into the local file like so:</p>

<pre><code>hive -e 'select books from table' &gt; /home/lvermeer/temp.tsv
</code></pre>

<p>That gives me a tab-separated file that I can use. Hope that is useful for you as well.</p>

<p>Based on <a href="https://issues.apache.org/jira/browse/HIVE-3682">this patch-3682</a>, I suspect a better solution is available when using Hive 0.11, but I am unable to test this myself. The new syntax should allow the following.</p>

<pre><code>INSERT OVERWRITE LOCAL DIRECTORY '/home/lvermeer/temp' 
ROW FORMAT DELIMITED 
FIELDS TERMINATED BY ',' 
select books from table;
</code></pre>

<p>Hope that helps.</p>

    <div class="post-text" itemprop="text">
<p>I had a similar issue and this is how I was able to address it.</p>

<p>Step 1 - Loaded the data from hive table into another table as follows</p>

<blockquote>
  <p>DROP TABLE IF EXISTS TestHiveTableCSV;
  CREATE TABLE TestHiveTableCSV ROW FORMAT DELIMITED FIELDS TERMINATED BY ',' LINES TERMINATED BY '\n' AS
  SELECT Column List FROM TestHiveTable;</p>
</blockquote>

<p>Step 2 - Copied the blob from hive warehouse to the new location with appropriate extension</p>

<blockquote>
  <p>Start-AzureStorageBlobCopy <code>
  -DestContext $destContext</code>
  -SrcContainer "Source Container" <code>
  -SrcBlob "hive/warehouse/TestHiveTableCSV/000000_0"</code>
  -DestContainer "Destination Container" `
  -DestBlob "CSV/TestHiveTable.csv"</p>
</blockquote>

<p>Hope this helps!</p>

    <div class="post-text" itemprop="text">
<p>I was looking for a similar solution, but the ones mentioned here would not work. My data had all variations of whitespace (space, newline, tab) chars and commas. </p>

<p>To make the column data tsv safe, I replaced all \t chars in the column data with a space, and executed python code on the commandline to generate a csv file, as shown below:</p>

<pre><code>hive -e 'tab_replaced_hql_query' |  python -c 'exec("import sys;import csv;reader = csv.reader(sys.stdin, dialect=csv.excel_tab);writer = csv.writer(sys.stdout, dialect=csv.excel)\nfor row in reader: writer.writerow(row)")'
</code></pre>

<p>This created a perfectly valid csv. Hope this helps those who come looking for this solution.</p>
