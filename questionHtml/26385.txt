that execution id and ran the following command</p>

<pre><code>F:\HDP&gt;msbuild F:\HDP\hadoop-common-project\hadoop-common\src\main\winutils\winutils.sln /nologo/p:Configuration=Release;OutDir=bin
</code></pre>

<pre><code>C:\Program Files(x86)\MSBuild\Microsoft.Cpp\v4.0\V110\Microsoft.Cpp.Platform.targets(42,5): error MSB8020: The builds tools for Visual Studio 2010 (Platform Toolset ='v100') cannot be found.
To build using the v100 build tools, either click the Project menu or right-click the solution, and then select "Update VC++ Projects...". Install Visual Studio 2010 to build using the Visual Studio 2010 build tools.
[F:\HDP\hadoop-common-project\hadoop-common\src\main\winutils\libwinutils.vcxproj]
</code></pre>

<p>Since I do not have Visual Studio 2010 on my machine, the problem had to be

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I have followed following steps to install Hadoop 2.2.0</p>

<p>Steps to build Hadoop bin distribution for Windows</p>

<ol>
<li><p>Download and install Microsoft Windows SDK v7.1.</p></li>
<li><p>Download and install Unix command-line tool Cygwin.</p></li>
<li><p>Download and install Maven 3.1.1.</p></li>
<li><p>Download Protocol Buffers 2.5.0 and extract to a folder (say c:\protobuf).</p></li>
<li><p>Add Environment Variables JAVA_HOME, M2_HOME and Platform if not added already. 
Note : Variable name Platform is case sensitive. And value will be either x64 or Win32 for building on a 64-bit or 32-bit system. 
Edit Path Variable to add bin directory of Cygwin (say C:\cygwin64\bin), bin directory of Maven (say C:\maven\bin) and installation path of Protocol Buffers (say c:\protobuf).</p></li>
<li><p>Download hadoop-2.2.0-src.tar.gz and extract to a folder having short path (say c:\hdfs) to avoid runtime problem due to maximum path length limitation in Windows.</p></li>
<li><p>Select Start --> All Programs --> Microsoft Windows SDK v7.1 and open Windows SDK 7.1 Command Prompt. Change directory to Hadoop source code folder (c:\hdfs). Execute mvn package with options -Pdist,native-win -DskipTests -Dtar to create Windows binary tar distribution. </p></li>
<li><p>If everything goes well in the previous step, then native distribution hadoop-2.2.0.tar.gz will be created inside C:\hdfs\hadoop-dist\target\hadoop-2.2.0 directory.</p></li>
</ol>

<p><strong>Install Hadoop</strong></p>

<ol>
<li><p>Extract hadoop-2.2.0.tar.gz to a folder (say c:\hadoop).</p></li>
<li><p>Add Environment Variable HADOOP_HOME and edit Path Variable to add bin directory of HADOOP_HOME (say C:\hadoop\bin).</p></li>
</ol>

<p><strong>Configure Hadoop</strong></p>

<p><strong>C:\hadoop\etc\hadoop\core-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
                &lt;name&gt;fs.defaultFS&lt;/name&gt;
                &lt;value&gt;hdfs://localhost:9000&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>C:\hadoop\etc\hadoop\hdfs-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
                &lt;name&gt;dfs.replication&lt;/name&gt;
                &lt;value&gt;1&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
                &lt;name&gt;dfs.namenode.name.dir&lt;/name&gt;
                &lt;value&gt;file:/hadoop/data/dfs/namenode&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
                &lt;name&gt;dfs.datanode.data.dir&lt;/name&gt;
                &lt;value&gt;file:/hadoop/data/dfs/datanode&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>C:\hadoop\etc\hadoop\mapred-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
           &lt;name&gt;mapreduce.framework.name&lt;/name&gt;
           &lt;value&gt;yarn&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>C:\hadoop\etc\hadoop\ yarn-site.xml</strong></p>

<pre><code>&lt;configuration&gt;
        &lt;property&gt;
           &lt;name&gt;yarn.nodemanager.aux-services&lt;/name&gt;
           &lt;value&gt;mapreduce_shuffle&lt;/value&gt;
        &lt;/property&gt;
        &lt;property&gt;
           &lt;name&gt;yarn.nodemanager.aux-services.mapreduce_shuffle.class&lt;/name&gt;
           &lt;value&gt;org.apache.hadoop.mapred.ShuffleHandler&lt;/value&gt;
        &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p><strong>Format namenode</strong></p>

<p>For the first time only, namenode needs to be formatted.</p>

<pre><code>C:\Users\abhijitg&gt;cd c:\hadoop\bin 
c:\hadoop\bin&gt;hdfs namenode –format
</code></pre>

<p><strong>Start HDFS (Namenode and Datanode)</strong></p>

<pre><code>C:\Users\abhijitg&gt;cd c:\hadoop\sbin
c:\hadoop\sbin&gt;start-dfs
</code></pre>

<p><strong>Start MapReduce aka YARN (Resource Manager and Node Manager)</strong></p>

<pre><code>C:\Users\abhijitg&gt;cd c:\hadoop\sbin
c:\hadoop\sbin&gt;start-yarn
starting yarn daemons
</code></pre>

<p>Total four separate Command Prompt windows will be opened automatically to run <strong>Namenode, Datanode, Resource Manager, Node Manager</strong></p>

<p>Reference : <a href="http://www.srccodes.com/p/article/38/build-install-configure-run-apache-hadoop-2.2.0-microsoft-windows-os" rel="nofollow">Build, Install, Configure and Run Apache Hadoop 2.2.0 in Microsoft Windows OS</a></p>

<p><em>My BUILDING.txt says:</em><br></p>

<pre><code>Building on Windows
Requirements:

* Windows System
* JDK 1.6+
* Maven 3.0 or later
* Findbugs 1.3.9 (if running findbugs)
* ProtocolBuffer 2.5.0
* CMake 2.6 or newer
* Windows SDK or Visual Studio 2010 Professional
* Unix command-line tools from GnuWin32 or Cygwin: sh, mkdir, rm, cp, tar, gzip
* zlib headers (if building native code bindings for zlib)
* Internet connection for first build (to fetch all Maven and Hadoop dependencies)

If using Visual Studio, it must be Visual Studio 2010 Professional (not 2012).
Do not use Visual Studio Express.  It does not support compiling for 64-bit,
which is problematic if running a 64-bit system.  The Windows SDK is free to
download here:

http://www.microsoft.com/en-us/download/details.aspx?id=8279    
</code></pre>

    <div class="post-text" itemprop="text">
<p>I was able to fix this issue by below two steps.</p>

<p>1) Add the MsBuild path(C:\Windows\Microsoft.NET\Framework64\v4.0.30319) to Environment Variable Path</p>

<p>2) Installing  Microsoft-Visual-Studio-2010-Professional
<a href="http://download.cnet.com/Microsoft-Visual-Studio-2010-Professional/3000-2212_4-10618634.html" rel="nofollow">http://download.cnet.com/Microsoft-Visual-Studio-2010-Professional/3000-2212_4-10618634.html</a></p>

<p>Note : Before installing Visual-Studio-2010-Professional I tried Windows-SDK-7.1 which did not help me. So I uninstalled Windows-SDK-7.1 before installing Visual-Studio-2010-Professional. Also the link I provided for Visual-Studio-2010-Professional is a trial version. </p>
