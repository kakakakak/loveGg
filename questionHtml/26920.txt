    <div class="post-text" itemprop="text">
<p><strong>1) Common setup:</strong> Just download the Spark version on a local machine. Unzip it and follow <a href="http://spark.apache.org/docs/latest/spark-standalone.html#installing-spark-standalone-to-a-cluster" rel="nofollow">these</a> steps to set it up locally. </p>

<p><strong>2) Launching a cluster for production:</strong>  The Spark cluster mode overview available <a href="http://spark.apache.org/docs/1.1.0/cluster-overview.html" rel="nofollow">here</a> explains the key concepts when running a <em>Spark</em> cluster. <em>Spark</em> can be run both in a standalone way and on several existing cluster managers. Currently, several deployments options are available:</p>

<ul>
<li><p><em>Amazon EC2</em></p></li>
<li><p><em>Standalone mode</em> </p></li>
<li><p><em>Apache Mesos</em></p></li>
<li><p><em>Hadoop YARN</em>   </p></li>
</ul>

<p><em>EC2</em> scripts let you launch a cluster in about 5 minutes. In fact, if you are using <em>EC2</em>, the best way to go is using the script provided by spark. The standalone mode is the best for the deployment of <em>Spark</em> on a private cluster.</p>

<p>Normally, we use YARN as cluster manager when we have an existing <em>Hadoop</em> setup with <em>YARN</em>, and the same goes for <em>Mesos</em>. Instead, if you are creating a new cluster from the dust, I would recommend using the <em>Standalone mode</em>, considering you are not using <em>Amazon</em>'s <em>EC2</em> instances. <a href="http://spark.apache.org/docs/latest/spark-standalone.html" rel="nofollow">This</a> link shows some steps that help arranging a <em>Standalone Spark</em> cluster.</p>

    <div class="post-text" itemprop="text">
<p>I am using Sandbox from Hortonworks. It has hadoop, java and all the required environment to run the spark examples.</p>

<p>I would suggest you to write simple Java or Scala class in your IDE. Create SparkConf and SparkContext objects in your "SimpleApp.java".</p>

<pre><code>SparkConf conf = new SparkConf().setAppName(appName).setMaster("local[2]");
JavaSparkContext sc = new JavaSparkContext(conf);
</code></pre>

<p>Once you run <code>maven clean package or maven package</code> it will create jar file in your project's target folder. If it doesn't then create a JAR file using following command. You can find the SimpleApp.class file in <strong>"target/classes"</strong> folder. cd to this directory.</p>

<pre><code>jar cfve file.jar SimpleApp.class
</code></pre>

<p>Put this JAR file into your project in target directory. This JAR file contains the dependency of your SimpleApp class while submitting your job to Spark. I guess you have project structure like below.</p>

<pre><code>simpleapp
 - src/main/java
  - org.apache.spark.examples
    -SimpleApp.java
 - lib
  - dependent.jars (you can put all dependent jars inside lib directory)
 - target
  - simpleapp.jar (after compiling your source)
</code></pre>

<p>cd to your spark directory. I am using spark-1.4.0-bin-hadoop2.6. Your cmd looks like this.</p>

<pre><code>spark-1.4.0-bin-hadoop2.6&gt;
</code></pre>

<p>Start the master and worker using following commands.</p>

<pre><code>spark-1.4.0-bin-hadoop2.6&gt; ./sbin/start-all.sh
</code></pre>

<p>If this does not work then start master and slaves separately.</p>

<pre><code>spark-1.4.0-bin-hadoop2.6&gt; ./sbin/start-master.sh
spark-1.4.0-bin-hadoop2.6&gt; ./sbin/start-slaves.sh
</code></pre>

<p>Submit your spark program using Spark Submit. If you have structure like I explained then pass this argument in class.</p>

<pre><code>--class org.apache.spark.examples.SimpleApp
</code></pre>

<p>else</p>

<pre><code>--class SimpleApp
</code></pre>

<p>Finally submit your spark program through spark submit.</p>

<pre><code>spark-1.4.0-bin-hadoop2.6&gt;./bin/spark-submit --class SimpleApp --master local[2] /PATH-TO-YOUR-PROJECT-DIRECTORY/target/file.jar
</code></pre>

<p>Here I have used <strong>local[2] as a master</strong> so my program will run on two threads but you can pass your master URL in --master as a <code>--master spark://YOUR-HOSTNAME:7077</code></p>

<p>Port number 7077 is a default port number for Master URL. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>For development and testing, I recommend using sbt in local mode so that you can avail yourself of the test suites you will write for your code. The following advice is verbatim from Matei Zaharia regarding Spark development, and it's always worked for me:</p>

<p>"As a tip (and maybe this isn't documented well), I normally use SBT for
 development to avoid the slow build process, and use its interactive
 console to run only specific tests. The nice advantage is that SBT can keep
 the Scala compiler loaded and JITed across builds, making it faster to
 iterate. To use it, you can do the following:</p>

<ul>
<li>Start the SBT interactive console with sbt/sbt</li>
<li>Build your assembly by running the "assembly" target in the assembly
project: assembly/assembly</li>
<li>Run all the tests in one module: core/test</li>
<li>Run a specific suite: core/test-only org.apache.spark.rdd.RDDSuite (this
also supports tab completion)"</li>
</ul>

<p>Source:
<a href="http://mail-archives.apache.org/mod_mbox/spark-dev/201412.mbox/%3cCAAsvFP=V9FL=KvXNUeWVfFz8q04oj1exdwSaSNiab5Vc9hFkUg@mail.gmail.com%3e" rel="nofollow">http://mail-archives.apache.org/mod_mbox/spark-dev/201412.mbox/%3cCAAsvFP=V9FL=KvXNUeWVfFz8q04oj1exdwSaSNiab5Vc9hFkUg@mail.gmail.com%3e</a></p>
