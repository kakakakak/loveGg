<p>In Java, I encountered the following line:</p>

<pre><code>e.g.: (1 &amp; Integer.MAX_VALUE) % 4
e.g.: (2 &amp; Integer.MAX_VALUE) % 5
</code></pre>

<p>What does it do? I have tried the code, but I couldn't comprehend its purpose or functionality. What is the code trying to check?</p>

<p>Basically, it's <code>(int &amp; Integer.MAX_VALUE) % int</code>.</p>

<p>Actual code (from Hadoop training class):</p>

<pre><code>public int getPartition(StringPairWritable key, Text value, int numReduceTasks) {
    return (key.getLeft().hashCode() &amp; Integer.MAX_VALUE) % numReduceTasks;
}
</code></pre>

<p><code>i &amp; Integer.MAX_VALUE</code> does the same thing as this code:</p>

<pre><code>if(i &lt; 0) {
    i = (i + Integer.MAX_VALUE + 1);
}
</code></pre>

<p>The <code>%</code> is a regular remainder operation.</p>

<p>It's a quick way of ensuring that an integer is positive if you don't care about it's actual value (e.g. if you want to turn random numbers that can be both positive and negative into only positive values).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The <code>Integer.MAX_VALUE</code> is <code>0x7FFFFFFF</code>. Thus <code>num &amp; Integer.MAX_VALUE</code> clears the highest bit in <code>num</code>. The <code>% numReduceTasks</code> is normal remainder after division by <code>numReduceTasks</code>.</p>

<p>This is done to convert the signed number to the non-negative number and then get the evenly distributed value from <code>0</code> to <code>numReduceTasks-1</code>. Note that if you write <code>Math.abs(key.getLeft().hashCode()) % numReduceTasks</code> you may get negative number if <code>hashCode()</code> happens to be <code>Integer.MIN_VALUE</code> as <code>Math.abs(Integer.MIN_VALUE)</code> is still <code>Integer.MIN_VALUE</code>. So <code>&amp; Integer.MAX_VALUE</code> is a safer alternative.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are two parts here:</p>

<ul>
<li>What the function actually <em>does</em> from a Java perspective, and</li>
<li>What purpose the function actually <em>serves</em>, from a Hadoop perspective.</li>
</ul>

<p>Let's cover the Java side of it first.  It's fairly straightforward bitwise math, in that it clears the sign bit and turns the value into a positive integer.</p>

<p>That's easy enough to demonstrate here; let's assume that our key is -128876912, which is <code>0xF8517E90</code>.  The max value for an int is <code>0x7FFFFFFF</code>.</p>

<p>If we look at the actual math operation, the sign bit is cleared (along with quite a few other bits, in this case), and we get a positive integer value.</p>

<pre><code>1111 1000 0101 0001 0111 1110 1001 0000
0111 1111 1111 1111 1111 1111 1111 1111
---------------------------------------
0111 1000 0101 0001 0111 1110 1001 0000
</code></pre>

<p>If the value is positive, then the net result is that we get back the same value.</p>

<p>This is important, since a hash code can come back negative; I don't believe that you want a negative value for a reason that's important a bit later.</p>

<p>For the partitioning bit, this is a bit more Hadoop knowledge than I can truly claim, but after <a href="https://hadoop.apache.org/docs/stable/api/org/apache/hadoop/mapreduce/Partitioner.html#getPartition(KEY,%20VALUE,%20int)" rel="nofollow">reading the docs</a>, it informs you which partition the value falls under.  That is where the modulo comes in; you're guaranteed to get a value between <code>[0, partition)</code>, thus specifying which reducer the particular bit of data is processed by.</p>

<p>From my reading of it, this is one of the default supplied partitioners, and may not be entirely suitable for your uses (you may want to <a href="http://dailyhadoopsoup.blogspot.com/2014/01/partitioning-in-mapreduce.html" rel="nofollow">group your data in a different way</a>, for instance).</p>

<p>it computes the residue class of <code>i mod j</code>. Note that <code>i % j</code> is the remainder of a division, a symmetric (w.r.t. i == 0) function, whereas the residue class, which is a periodic function.</p>
