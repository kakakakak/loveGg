    <div class="post-text" itemprop="text">

<p>I have simple text table (delimited by ",") with the following format:</p>

<pre class="lang-sql prettyprint-override"><code>orderID INT, CustID INT, OrderTotal FLOAT, OrderNumItems INT, OrderDesc STRING
</code></pre>

<p>I want to insert this data into a Parquet table:
I've created the table using:</p>

<pre class="lang-sql prettyprint-override"><code>CREATE TABLE parquet_test (orderID INT, CustID INT, OrderTotal FLOAT, 
OrderNumItems INT, OrderDesc STRING) 
ROW FORMAT SERDE 'parquet.hive.serde.ParquetHiveSerDe' stored as 
INPUTFORMAT 'parquet.hive.DeprecatedParquetInputFormat' 
OUTPUTFORMAT 'parquet.hive.DeprecatedParquetOutputFormat';
</code></pre>

<p>but when I’m trying to insert the data using</p>

<pre class="lang-sql prettyprint-override"><code>insert overwrite table parquet_small_orders select * from small_orders;
</code></pre>

<p>it fails. any thoughts?</p>

<p>The answer by Matthieu Lieber describes how you can load data to a parquet table.</p>

<p>Couple of things before you post your next question.</p>

<ol>
<li><p>Please mention what version you are using. Parquet was not natively supported in Hive versions prior to 0.13 and you need to add some explicit jars to get the functionality.</p></li>
<li><p>Please add the logs/error messages. "It fails" is a vague description of the error and it makes things difficult to debug. The way you are doing the data load seems OK and it should work. However, a log would have made things clear as to what the issue is.</p></li>
</ol>

<p>If this is still an open issue, you may refer to <a href="http://www.cloudera.com/content/cloudera/en/documentation/core/latest/topics/cdh_ig_parquet.html" rel="nofollow">Cloudera's documentation</a> to see some basics as to how you can use Parquet with Hive.</p>
