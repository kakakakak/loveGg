    <div class="post-text" itemprop="text">

<p>I am working on a cluster where a dataset is kept in <code>hdfs</code> in distributed manner. Here is what I have: </p>

<pre><code>[hmi@bdadev-5 ~]$ hadoop fs -ls /bdatest/clm/data/
Found 1840 items
-rw-r--r--   3 bda supergroup          0 2015-08-11 00:32 /bdatest/clm/data/_SUCCESS
-rw-r--r--   3 bda supergroup   34404390 2015-08-11 00:32 /bdatest/clm/data/part-00000
-rw-r--r--   3 bda supergroup   34404062 2015-08-11 00:32 /bdatest/clm/data/part-00001
-rw-r--r--   3 bda supergroup   34404259 2015-08-11 00:32 /bdatest/clm/data/part-00002
....
....
</code></pre>

<p>The data is of the form: </p>

<pre><code>[hmi@bdadev-5 ~]$ hadoop fs -cat /bdatest/clm/data/part-00000|head
V|485715986|1|8ca217a3d75d8236|Y|Y|Y|Y/1X||Trimode|SAMSUNG|1x/Trimode|High|Phone|N|Y|Y|Y|N|Basic|Basic|Basic|Basic|N|N|N|N|Y|N|Basic-Communicator|Y|Basic|N|Y|1X|Basic|1X|||SAM|Other|SCH-A870|SCH-A870|N|N|M2MC|
</code></pre>

<p>So, what I want to do is to count the total number of lines in the original data file <code>data</code>. My understanding is that the distributed chunks like <code>part-00000</code>, <code>part-00001</code> etc have overlaps. So just counting the number of lines in <code>part-xxxx</code> files and summing them won't work. Also the original dataset <code>data</code> is of size <code>~70GB</code>. How can I efficiently find out the total number of lines?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>More effieciently -- you can use spark to count the no. of lines. The following code snippet helps to count the number of lines. </p>

<pre><code>text_file = spark.textFile("hdfs://...")
count = text_file.count();
count.dump();
</code></pre>

<p>This displays the count of no. of lines.</p>

<p><code>Note: The data in different part files will not overlap</code></p>

<p>Using <code>hdfs dfs -cat /bdatest/clm/data/part-* | wc -l</code> will also give you the output but this will dump all the data to the local machine and takes longer time.</p>

<p>Best solution is to use mapreduce or spark. Mapredue will take longer time to develop and execute. If the sprak is installed, this is the best choice. </p>

<pre><code>hadoop fs -cat /bdatest/clm/data/part-* | wc -l

See the accepted answer of <a href="http://stackoverflow.com/questions/9248846/writing-mapreduce-code-for-counting-number-of-records">Writing MApreduce code for counting number of records</a> for the solution.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>This is such a common task that I wish there is a subcommand in fs to do that (e.g. <code>hadoop fs -wc -l inputdir</code>) to avoid streaming all the content to one machine that performs the "wc -l" command.</p>

<p>To count lines efficiently, I often use hadoop streaming and unix commands as follows:</p>

<pre><code>hadoop jar ${HADOOP_HOME}/hadoop-streaming.jar \
  -Dmapred.reduce.tasks=1 \
  -input inputdir \
  -output outputdir \
  -mapper "bash -c 'paste &lt;(echo "count") &lt;(wc -l)'" \
  -reducer "bash -c 'cut -f2 | paste -sd+ | bc'" 
</code></pre>

<p>Every mapper will run "wc -l" on the parts it has and then a single reducer will sum up the counts from all the mappers.</p>
