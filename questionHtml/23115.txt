<p>I'm using Hadoop 2.5.1 with HBase 0.98.11 on Ubuntu 14.04</p>

<p>I could run it in Pseudo-distributed mode. Now that I want to run on distributed mode. I follow the instruction from sites and end up having an error in RUNTIME called "Error: org/apache/hadoop/hbase/HBaseConfiguration" (while there is no error when I compile the code).</p>

<p>After trying things, I found that if I comment the mapreduce.framework.name in mapred-site.xml and also stuffs in yarn-site, I could be able to run the hadoop successfully. </p>

<p>But I think it's the single-node running (I have no idea, just guessing by comparing the running time to what I ran in Pseudo and there is no MR in slave's node jps when running the job on master). </p>

    <div class="post-text" itemprop="text">
<p>I guess that you didn't set up your hadoop cluster correctly please follow these steps :</p>

<p><em>Hadoop Configuration:</em></p>

<p>step 1 : edit <code>hadoop-env.sh</code> as following:</p>

<pre><code># The java implementation to use.  Required.
export JAVA_HOME=/usr/lib/jvm/java-6-sun
</code></pre>

<p>step 2 : Now create a directory and set the required ownerships and permissions</p>

<pre><code>$ sudo mkdir -p /app/hadoop/tmp
$ sudo chown hduser:hadoop /app/hadoop/tmp
# ...and if you want to tighten up security, chmod from 755 to 750...
$ sudo chmod 750 /app/hadoop/tmp
</code></pre>

<p>step 3 : edit <code>core-site.xml</code></p>

<pre><code>&lt;property&gt;
  &lt;name&gt;hadoop.tmp.dir&lt;/name&gt;
  &lt;value&gt;/app/hadoop/tmp&lt;/value&gt;
&lt;/property&gt;

&lt;property&gt;
  &lt;name&gt;fs.default.name&lt;/name&gt;
  &lt;value&gt;hdfs://localhost:54310&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>step 5 : edit <code>mapred-site.xml</code></p>

<pre><code>  &lt;property&gt;
      &lt;name&gt;mapreduce.framework.name&lt;/name&gt;
      &lt;value&gt;yarn&lt;/value&gt;
    &lt;/property&gt;
</code></pre>

<p>step 6 : edit <code>hdfs-site.xml</code></p>

<pre><code>&lt;property&gt;
  &lt;name&gt;dfs.replication&lt;/name&gt;
          &lt;value&gt;1&lt;/value&gt;
&lt;/property&gt;

&lt;property&gt;
  &lt;name&gt;dfs.name.dir&lt;/name&gt;
  &lt;value&gt;file:///home/hduser/hadoopdata/hdfs/namenode&lt;/value&gt;
&lt;/property&gt;

&lt;property&gt;
  &lt;name&gt;dfs.data.dir&lt;/name&gt;
  &lt;value&gt;file:///home/hduser/hadoop/hadoopdata/hdfs/datanode&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>step 7 : edit <code>yarn-site.xml</code></p>

<pre><code>&lt;property&gt;
   &lt;name&gt;yarn.nodemanager.aux-services&lt;/name&gt;
   &lt;value&gt;mapreduce_shuffle&lt;/value&gt;
&lt;/property&gt;
</code></pre>

<p>finally format your hdfs (You need to do this the first time you set up a Hadoop cluster)</p>

<pre><code>$ /usr/local/hadoop/bin/hadoop namenode -format
</code></pre>

<p><em>Hbase Configuration:</em></p>

<p>edit you <code>hbase-site.xml</code>:</p>

<pre><code>&lt;property&gt;
  &lt;name&gt;hbase.rootdir&lt;/name&gt;
  &lt;value&gt;hdfs://localhost:54310/hbase&lt;/value&gt;
&lt;/property&gt; 

&lt;property&gt;
   &lt;name&gt;hbase.cluster.distributed&lt;/name&gt;
   &lt;value&gt;true&lt;/value&gt;
&lt;/property&gt;

&lt;property&gt;
   &lt;name&gt;hbase.zookeeper.quorum&lt;/name&gt;  
   &lt;value&gt;localhost&lt;/value&gt;
&lt;/property&gt;

&lt;property&gt;   
  &lt;name&gt;dfs.replication&lt;/name&gt;   
  &lt;value&gt;1&lt;/value&gt;    
&lt;/property&gt;

 &lt;property&gt;       
  &lt;name&gt;hbase.zookeeper.property.clientPort&lt;/name&gt;  
  &lt;value&gt;2181&lt;/value&gt;                                                                                 
 &lt;/property&gt;

 &lt;property&gt;
    &lt;name&gt;hbase.zookeeper.property.dataDir&lt;/name&gt;
    &lt;value&gt;/usr/local/hbase/zookeeper&lt;/value&gt;   
 &lt;/property&gt;                                                                                                                          
</code></pre>

<p>Hope this helps you                                                                                                                                                                                                                                                           </p>
