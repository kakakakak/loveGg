<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The query that worked for you does not launch a Map Reduce Job. It looks like there is a problem while launching Map Reduce Jobs. Can you check the hive logs (default location, assuming you are running hiveserver2 as user hive) – <code>/tmp/hive/hive.log</code> to see if it has the full error message.</p>

<p>One common error that you might find in the logs is the permission denied error. Mostly the ODBC driver logs on to as user hadoop which has no write access thus causing a failure in starting a Map Reduce Job. If you change this user to hdfs, your problem might be solved.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The difference between the <code>Select * from SomeTable;</code> and <code>Select colName from SomeTable;</code> queries you ran is that the later, <code>Select colName from SomeTable;</code> requires information from the <em>Hive Metastore</em> service. Since you are looking for a specific column, hive must first check the hive schema definition which is stored in the <em>Hive Metastore</em>.</p>

<p>Your issue is due to the fact that you cannot connect to the Hive Metastore, as indicated by the following message in your log file:</p>

<p><code>org.apache.thrift.transport.TTransportException: java.net.SocketTimeoutException: Read timed out</code></p>

<p>Try restarting your <em>Hive Metastore</em> service.</p>
