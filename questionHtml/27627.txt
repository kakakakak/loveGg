    <div class="post-text" itemprop="text">
<p>There are some pieces here and there if you search for them. For example <a href="http://code.google.com/p/octopy/" rel="nofollow">Octopy</a> and <a href="http://discoproject.org/" rel="nofollow">Disco</a> as well as <a href="https://github.com/bwhite/hadoopy" rel="nofollow">Hadoopy</a>.</p>

<p>However, I don't believe that any of them can compete Hadoop in terms of maturity, stability, scalability, performance, etc. For small cases they should suffice, but for something more "glorious", you have to stick to Hadoop.</p>

<p>Remember that you can still write map/reduce programs in Hadoop with python/jython.</p>

<p>EDIT : I've recently came across <a href="http://packages.python.org/mrjob/index.html" rel="nofollow">mrjob</a>. It seems great, as it eases the way to write map/reduce programs and then launch them on Hadoop or on Amazon's Elastic MapReduce platform. The article that brough the good news is <a href="http://musicmachinery.com/2011/09/04/how-to-process-a-million-songs-in-20-minutes/" rel="nofollow">here</a></p>

    <div class="post-text" itemprop="text">
<p>Another good option is <a href="https://github.com/klbostee/dumbo/wiki/Short-tutorial" rel="nofollow">Dumbo</a>.</p>

<p>Below is the code to run a map/reduce for word counting.</p>

<pre><code>def mapper(key,value):
  for word in value.split(): yield word,1
def reducer(key,values):
  yield key,sum(values)

if __name__ == "__main__":
  import dumbo
  dumbo.run(mapper,reducer)
</code></pre>

<p>To run it, just feed your text file <code>wc_input.txt</code> for counting, the output is saved as <code>wc_output</code>.</p>

<pre><code> python -m dumbo wordcount.py -hadoop /path/to/hadoop -input wc_input.txt -output wc_output
</code></pre>
