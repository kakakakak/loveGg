<td class="answercell">
    <div class="post-text" itemprop="text">
<p>HCatalog supports reading and writing files in any format for which a Hive SerDe (serializer-deserializer) can be written. By default, HCatalog supports RCFile, CSV, JSON, and SequenceFile formats. To use a custom format, you must provide the InputFormat, OutputFormat, and SerDe.</p>

<p>HCatalog is built on top of the Hive metastore and incorporates components from the Hive DDL. HCatalog provides read and write interfaces for Pig and MapReduce and uses Hive’s command line interface for issuing data definition and metadata exploration commands. It also presents a REST interface to allow external tools access to Hive DDL (Data Definition Language) operations, such as “create table” and “describe table”.</p>

<p>HCatalog presents a relational view of data. Data is stored in tables and these tables can be placed into databases. Tables can also be partitioned on one or more keys. For a given value of a key (or set of keys) there will be one partition that contains all rows with that value (or set of values).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In short, HCatalog opens up the hive metadata to other mapreduce tools. Every mapreduce tools has its own notion about HDFS data (example Pig sees the HDFS data as set of files, Hive sees it as tables). With having table based abstraction, HCatalog supported mapreduce tools do not need to care about where the data is stored, in which format and storage location (HBase or HDFS).</p>

<p>We do get the facility of WebHcat to submit jobs in an RESTful way if you configure webhcat along Hcatalog. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Here is a very basic example of how ho use HCATALOG.</p>

<p>I have a table in hive ,TABLE NAME is STUDENT which is stored in one of the HDFS location:</p>

<p><code>neethu  90
malini  90
sunitha 98
mrinal  56
ravi    90
joshua  8</code></p>

<p>Now suppose I want to load this table to pig for further transformation of data,
In this scenario I can use HCATALOG:</p>

<p>When using table information from the Hive metastore with Pig, add the -useHCatalog option when invoking pig:</p>

<p><code>pig -useHCatalog</code></p>

<p>(you may want to export HCAT_HOME 'HCAT_HOME=/usr/lib/hive-hcatalog/')</p>

<p>Now loading this table to pig:
<code>A = LOAD 'student' USING org.apache.hcatalog.pig.HCatLoader();</code></p>

<p>Now you have loaded the table to pig.To check the schema , just do a DESCRIBE on the relation.</p>

<p><code>DESCRIBE A</code></p>

<p>Thanks</p>
