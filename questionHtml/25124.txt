<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Modify datanode node/etc/hosts file. </p>

<p>Each line is divided into three parts. The first part is the network IP address, the second part is the host name or domain name, the third part is the host alias detailed steps are as follows: </p>

<ol>
<li><p>First check the host name:</p>

<p>cat / proc / sys / kernel / hostname</p>

<p>You will see a <code>HOSTNAME</code> attribute. Change the value of the IP behind on OK and then exit. </p></li>
<li><p>Use the command:</p>

<p><code>hostname ***. ***. ***. ***</code></p>

<p>Asterisk is replaced by the corresponding IP. </p></li>
<li><p>Modify the the hosts configuration similarly, as follows:</p>

<p>127.0.0.1 localhost.localdomain localhost
:: 1 localhost6.localdomain6 localhost6 
10.200.187.77 10.200.187.77 hadoop-datanode</p></li>
</ol>

<p>If the IP address is configured and successfully modified, or show host name there is a problem, continue to modify the hosts file.</p>

<p>I had the same problem: "Too many fetch failures" and very slow Hadoop performance (the simple wordcount example took more than 20 minutes to run on a 2-node cluster of powerful servers). I also got "WARN mapred.JobClient: Error reading task outputConnection refused" errors.</p>
