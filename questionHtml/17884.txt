    <div class="post-text" itemprop="text">
<p>I've just run into a similar problem, my code was trying to connect to <code>10.0.2.15</code> which was not exposed in my network. I'm using VirtualBox 4.3.20 on OS X 10.9.5.</p>

<p>I fixed this by following:</p>

<ol>
<li>Open Virtualbox->Preferences->Network->Host-Only Network tab</li>
<li>Add <code>vboxnet0</code> (or if there is one, just click the screwdriver)</li>
<li>Set IPv4 address to <code>192.168.56.1</code> (in my case), the mask <code>255.255.255.0</code></li>
<li>Enable DHCP server and set the ranges, i.e. server address <code>192.168.56.100</code>, bounds .101-.254</li>
<li>Confirm everything</li>
<li>Open you Sandbox VM settings</li>
<li>Go to network</li>
<li>Change the adapter settings to Host-Only Network and select <code>vboxnet0</code></li>
<li>Restart your Sandbox VM</li>
</ol>

<p>Just to verify everything is in order:</p>

<ol>
<li>In your Mac terminal, run ifconfig, you should see <code>vboxnet0</code> there</li>
<li>In your Sandbox VM cmd: cat /etc/hosts should have something like <code>192.168.56.101</code> instead of <code>10.0.2.15</code></li>
</ol>

<p>Now I can connect to the sandbox HDFS from my mac.</p>
