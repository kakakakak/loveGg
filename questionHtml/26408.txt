    <div class="post-text" itemprop="text">

<p>I have a <code>JavaRDD&lt;Model&gt;</code>, which i need to write it as more than one file with different layout [one or two fields in the RDD will be different between different layout].</p>

<p>When i use <code>saveAsTextFile()</code> its calling the <code>toString()</code> method of Model, it means same layout will be written as output.</p>

<p>Currently what i am doing is iterate the RDD using <code>map</code> transformation method and return the different model with other layout, so i can use saveAsTextFile() action to write as different output file.</p>

<p>Just because of one or two fields are different , i need to iterate the entire RDD again and create new RDD then save it as output file.</p>

<p><strong>For example:</strong></p>

<p><strong>Current RDD with fields:</strong></p>

<p>RoleIndicator, Name, Age, Address, Department</p>

<p><strong>Output File 1:</strong></p>

<p>Name, Age, Address</p>

<p><strong>Output File 2:</strong></p>

<p>RoleIndicator, Name, Age, Department</p>

<p>Is there any optimal solution for this?</p>

<p>Regards,
Shankar</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You want to use <code>foreach</code>, not <code>collect</code>.</p>

<p>You should define your function as an actual named class that extends <code>VoidFunction</code>. Create instance variables for both files, and add a <code>close()</code> method that closes the files. Your <code>call()</code> implementation will write whatever you need.</p>

<p>Remember to call <code>close()</code> on your function object after you're done.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It is possible with Pair RDD.
Pair RDD can be stored in multiple files in a single iteration by using Hadoop Custom output format.</p>

<pre><code>rdd.saveAsHadoopFile(path, key.class, value.class,CustomTextOutputFormat.class, jobConf);


public class FileGroupingTextOutputFormat extends MultipleTextOutputFormat&lt;Text, Text&gt; { 
  @Override
  protected Text generateActualKey(Text key, Text value) {
    return new Text();
  }

  @Override
  protected Text generateActualValue(Text key, Text value) {
    return value;
  }
// returns a dynamic file name based on each RDD element
  @Override
  protected String generateFileNameForKeyValue(Text key, Text value, String name) {
    return value.getSomeField() + "-" + name;
  }
}
</code></pre>
