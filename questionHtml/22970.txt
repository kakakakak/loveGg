    <div class="post-text" itemprop="text">

<p>First, I have read this post:<a href="http://stackoverflow.com/questions/21539694/is-there-an-equivalent-to-pwd-in-hdfs">Is there an equivalent to `pwd` in hdfs?</a>. It says there is no such 'pwd' in HDFS.</p>

<p>However, as I progressed with the instructions of <a href="http://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-common/SingleCluster.html" rel="nofollow">Hadoop: Setting up a Single Node Cluster</a>, I failed on this command:</p>

<pre><code>$ bin/hdfs dfs -put etc/hadoop input
put: 'input': No such file or directory
</code></pre>

<p>It's weird that I <strong>succeed on this command for the first time</strong> I went through the instructions, but <strong>failed for the second time</strong>. It's also weird that I <strong>succeed on this command on my friends computer</strong>, which has the same system (Ubuntu 14.04) and hadoop version (2.7.1) as mine.</p>

<p>Can anyone explain what happened here? Is there some 'pwd' in HDFS after all?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Firstly, You are trying to run the command <code>$ bin/hdfs dfs -put etc/hadoop input</code> with user that doesn't exist in the <strong>VM/HDFS</strong><br>
Let me clearly explain you with the following example in <strong>HDP VM</strong></p>

<pre><code>[root@sandbox hadoop-hdfs-client]# bin/hdfs dfs -put /etc/hadoop input
put: `input': No such file or directory 
</code></pre>

<p>Here I executed the command with root user and it didn't exist in the HDP VM. Check in the following command to list the users</p>

<pre><code>[root@sandbox hadoop-hdfs-client]# hadoop fs -ls /user
Found 8 items
drwxrwx---   - ambari-qa hdfs           0 2015-08-20 08:33 /user/ambari-qa
drwxr-xr-x   - guest     guest          0 2015-08-20 08:47 /user/guest
drwxr-xr-x   - hcat      hdfs           0 2015-08-20 08:36 /user/hcat
drwx------   - hive      hdfs           0 2015-09-04 09:52 /user/hive
drwxr-xr-x   - hue       hue            0 2015-08-20 09:05 /user/hue
drwxrwxr-x   - oozie     hdfs           0 2015-08-20 08:37 /user/oozie
drwxr-xr-x   - solr      hdfs           0 2015-08-20 08:41 /user/solr
drwxrwxr-x   - spark     hdfs           0 2015-08-20 08:34 /user/spark
</code></pre>

<p>In <code>HDFS</code>, If you want to <code>copy</code> a file and not mentioning the absolute path for destination argument, it will consider <code>home</code> of the logged user and place your file there. Here <code>root</code> user not found.</p>

<p>Now let's switch to <code>hive</code> user and test</p>

<pre><code>[root@sandbox hadoop-hdfs-client]# su hive
[hive@sandbox hadoop-hdfs-client]$ bin/hdfs dfs -put  /etc/hadoop input
[hive@sandbox hadoop-hdfs-client]$ hadoop fs -ls /user/hive
Found 1 items
drwxr-xr-x   - hive hdfs          0 2015-09-04 10:07 /user/hive/input
</code></pre>

<p>It means that we need to move input files to hdfs location.</p>

<p>Suppose you have input file named input.txt and we need to move to HDFS, then follow the below command.
Command: hdfs dfs -put /input_location /hdfs_location</p>

<p>In case no specific directory in HDFS
<strong>hdfs dfs -put /home/Desktop/input.txt /</strong></p>

<p>In case specific directory in HDFS (<strong>Note: We need to create a directory before proceeding</strong>)</p>

<p><strong>hdfs dfs -put /home/Desktop/input.txt /MR_input</strong></p>

<p>After that you can run the examples</p>

<p>bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.1.jar wordcount /input /output</p>

<p>Here Input and output are the paths which should be in HDFS.</p>
