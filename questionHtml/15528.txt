    <div class="post-text" itemprop="text">
<p>Unfortunately, the launch image of Dataproc was built without Snappy support. I've opened a bug to get this fixed for the next image. </p>

<p>A workaround:</p>

<ol>
<li><p>First create a small shell script that properly installs snappy and the native library support for it. For this we'll use the same native libraries bdutil uses. I called my script <code>setup-snappy.sh</code>:</p>

<pre><code>#!/bin/bash

pushd "$(mktemp -d)"
apt-get install -q -y libsnappy1
wget https://storage.googleapis.com/hadoop-native-dist/Hadoop_2.7.1-Linux-amd64-64.tar.gz
tar zxvf Hadoop_2.7.1-Linux-amd64-64.tar.gz -C /usr/lib/hadoop/
</code></pre></li>
<li><p>Copy the new shell script to a GCS bucket you own. For demonstration purposes, let's assume the bucket is <code>dataproc-actions</code>:</p>

<pre><code>gsutil cp ./setup-snappy.sh gs://dataproc-actions/setup-snappy.sh
</code></pre></li>
<li><p>When starting a cluster, specify initialization actions:</p>

<pre><code>gcloud beta dataproc clusters create --initialization-actions gs://dataproc-actions/setup-snappy.sh mycluster
</code></pre></li>
</ol>
