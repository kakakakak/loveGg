<p>The Contents of the ReadFromHadoopURL file are as follows:</p>

<pre><code>package com.felix.hadoop.training;

import java.io.InputStream;
import java.net.URL;

import org.apache.hadoop.fs.FsUrlStreamHandlerFactory;
import org.apache.hadoop.io.IOUtils;

//  URLCat
public class ReadFromHadoopURL {

  static {
        URL.setURLStreamHandlerFactory(new FsUrlStreamHandlerFactory());
  }

  public static void main(String[] args) throws Exception {
    InputStream in = null;
    try {
      in = new URL(args[0]).openStream();
      IOUtils.copyBytes(in, System.out, 4096, false);
    } finally {
      IOUtils.closeStream(in);
    }
      }
    }
</code></pre>

<p>Then I created a jar file ReadFromHadoopURL.jar .</p>

<p>Then I ran this command</p>

<pre><code>hadoop jar /home/training/Pradosh/ReadFromHadoopURL.jar ReadFromHadoopURL hdfs://localhost:54310/my/empfile

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>To run with the <code>hadoop jar</code> command, you have to implement a class containing <code>Map extends Mapper&lt;&gt;</code> class (and optionally <code>Reduce extends Reducer&lt;&gt;</code>) as illustrated in the canonical <a href="http://wiki.apache.org/hadoop/WordCount" rel="nofollow">WordCount Example</a>.</p>

<p>The above looks like you're trying to implement a simple Java console app to cat a file stored in HDFS.  If that's the case then you should run it with something like:</p>

<p><code>java -cp ./ReadFromHadoopURL.jar ReadFromHadoopURL hdfs://localhost:54310/my/empfile</code></p>

<p>Note: getting all of the Hadoop and Apache dependencies to work from the command line will likely require a much expanded cp/classpath.</p>
