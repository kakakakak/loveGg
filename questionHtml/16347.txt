<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Storing data in HDFS is a very different thing than analyzing it with the MapReduce paradigm.</p>

<p>When uploaded to HDFS, big data files are splited into blocks which are stored in the datanodes, and each block is replicated as many times as the configured replication factor (by default, 3). Data spliting is as simple as dividing the file by the configured block size.</p>

<p>MapReduce, as said, is a programming paradigm when analyzing big data files in order to obtain value added information. In a few words, each file block is assigned to a map task in order all the mappers perform the same operation on the chuncks; once finished, the output partial results are sent to the reducers in order to aggregate the data in some way.</p>
