<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Since your intention is just to copy the entire file from HDFS to your local FS, I would not suggest you to do it through a Hive query, because of the following reasons :</p>

<ul>
<li>It'll start a Mapreduce job which will take more time than a normal copy.</li>
<li>It'll create file(s) with different names(<code>000000_0, 000001_0 and so on</code>), which will require you to rename the file manually afterwards.</li>
<li>You <code>might</code> face problem in opening these files as they are without any extension. Your OS would be unable to choose an application to open these files on its own. In such a case you either have to rename the file or manually select an application to open it.</li>
</ul>

<p>To avoid these problems you could use <code>HDFS get command</code> :</p>

<pre><code>bin/hadoop fs -get /user/hive/warehouse/sample/sample.txt /opt/zxy/sample.txt
</code></pre>

<p>Simple n easy. But if you need to copy some selected data, then you have to use a Hive query.</p>

<p>HTH</p>
