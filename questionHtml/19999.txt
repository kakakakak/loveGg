    <div class="post-text" itemprop="text">

<p>I have a MySQL table storing some user generated content. For each piece of content, I have a title (VARCHAR 255) and a description (TEXT) column.</p>

<p>When a user is viewing a record, I want to find other records that are 'similar' to it, based on the title/description being similar.</p>

<p>What's the best way to go about doing this? I'm using PHP and MySQL.</p>

<p>My initial ideas are:</p>

<p>1) Either to strip out common words from the title and description to be left with 'unique' keywords, and then find other records which share those keywords.</p>

<p>E.g in the sentence: "Bob woke up at 5 am and went to school", the keywords would be: "Bob, woke, 5, went, school". Then if there's another record whose title talks about 'bob' and 'school', they would be considered 'similar'.</p>

<p>2) Or to use MySQL's full text search, though I don't know if this would be any good for something like this?</p>

<p>Which method would be better out of the two, or is there another method which is even better?</p>

    <div class="post-text" itemprop="text">
<p>I'll keep this short (it could be way too long)...</p>

<p>I would not select they keywords 'manually' or modify your original data.</p>

<p>MySQL supports full text search with MyISAM (not InnoDB) engine. A full description of the options available when querying the DB are available <a href="http://dev.mysql.com/doc/refman/5.0/en/fulltext-search.html" rel="nofollow">here</a>. The query can automatically get rid of common stop-words and words too common in the data set (more than 50% of the rows contains them) depending on the querying method. Query expansion is also available and the query type should be decided depending on your needs.</p>

<p>Consider also using a separate engine like Lucene. With Lucene you will probably have more functionalities and better indexing/searching. You can automatically get rid of common words (they get a low score and do not influence the search) and use things as stemming for instance. There is a little bit of a learning curve but I'll definitely look into it.</p>

<p><strong>EDIT:</strong></p>

<p>The MySQL '<a href="http://dev.mysql.com/doc/refman/5.0/en/fulltext-natural-language.html" rel="nofollow">full-text natural language search</a>' returns the most similar rows (and their relevance score) and is not a boolean matching search.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I have done something like this. I replace all of the spaces in the string with % then use LIKE in the where clause. Here, I will give you my code. It is from MSSQL but minor adjustments can be made to work it with MySQL. Hope it helps.</p>

<pre><code>CREATE FUNCTION [dbo].[fss_MakeTextSearchable] (@text NVARCHAR(MAX)) RETURNS NVARCHAR(MAX)
--replaces spaces with wildcard characters to return more matches in a LIKE condition
--              for example:
--              @text = 'my file' will return '%my%file%'
--              SELECT WHERE 'my project files' like @text would return true

AS
BEGIN
   DECLARE @searchableText NVARCHAR(MAX)

   SELECT @searchableText = '%' + replace(@text, ' ', '%') + '%' 

   RETURN @searchableText
END
</code></pre>

<p>Then use the function like this:</p>

<pre><code>SELECT @searchString = dbo.fss_MakeTextSearchable(@String)
</code></pre>

<p>Then in your query:</p>

<pre><code>Select * from Table where title LIKE @searchString 
</code></pre>
