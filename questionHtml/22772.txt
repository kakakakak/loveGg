<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Look into using the <a href="http://hipi.cs.virginia.edu/" rel="nofollow">HIPI library</a> - it stores a collection of images into an ImageBundle (which is more efficient that storing the individual image files in HDFS). They have a couple of examples too.</p>

<p>As for your code, you need to specify what input and output formats you plan to use. There is no current input format that hands the entire file over, but you can just extend FileInputFormat and create a RecordReader that emits <code>&lt;Text, BytesWritable&gt;</code> pairs, where the key is the filename, and the value is the bytes of the image file.</p>

<p>In fact <em><a href="http://books.google.com/books?id=Nff49D7vnJcC&amp;pg=PA206&amp;lpg=PA206&amp;dq=hadoop%20whole%20file%20input%20format&amp;source=bl&amp;ots=IieySt5zWv&amp;sig=RzWxqzrxDHXmhHsCnYOOieUVOCk&amp;hl=en&amp;sa=X&amp;ei=kBHGT9GsH_L46QGPkdmcBg&amp;ved=0CF4Q6AEwBQ#v=onepage&amp;q=hadoop%20whole%20file%20input%20format&amp;f=false" rel="nofollow">Hadoop - The Definitive Guide</a></em> has an example of this exact input format:</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you want to send all the images as input to MR task you just set the conf.setFileInputPath() to the directory of the input 
If You want to send selective images in a particular folder You can add multiple paths when you are setting conf.setFileInputPath();</p>

<p>One way is to create a Path[] one for each image. or just set it to comma separated string with all the paths.
Go through the following documentation </p>

<p><a href="http://hadoop.apache.org/docs/current/api/org/apache/hadoop/mapred/FileInputFormat.html" rel="nofollow">http://hadoop.apache.org/docs/current/api/org/apache/hadoop/mapred/FileInputFormat.html</a></p>

<p>And one more thing you have to set the Map input formats as Text, ByteArray
get the image features from that ByteArray input instead of creating new fileinputstream.</p>
