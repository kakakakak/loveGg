<p>No matter how much I tinker with the settings in <code>yarn-site.xml</code> i.e using all of the below options</p>

<pre><code>yarn.scheduler.minimum-allocation-vcores
yarn.nodemanager.resource.memory-mb
yarn.nodemanager.resource.cpu-vcores
yarn.scheduler.maximum-allocation-mb
yarn.scheduler.maximum-allocation-vcores
</code></pre>

<p>i just still cannot get my application i.e Spark to utilize all the cores on the cluster. The spark executors seem to be correctly taking up all the available memory, but each executor just keeps taking a single core and no more.</p>

<p>Here are the options configured in <code>spark-defaults.conf</code></p>

<pre><code>spark.executor.cores                    3
spark.executor.memory                   5100m
spark.yarn.executor.memoryOverhead      800
spark.driver.memory                     2g
spark.yarn.driver.memoryOverhead        400
spark.executor.instances                28
spark.reducer.maxMbInFlight             120
spark.shuffle.file.buffer.kb            200
</code></pre>

<p>Notice that <code>spark.executor.cores</code> is set to 3, but it doesn't work.
How do i fix this?</p>

<p>The problem lies not with <code>yarn-site.xml</code> or <code>spark-defaults.conf</code> but actually with the resource calculator that assigns the cores to the executors or in the case of MapReduce jobs, to the Mappers/Reducers.</p>

<p>The default resource calculator i.e <code>org.apache.hadoop.yarn.util.resource.DefaultResourceCalculator</code> uses only memory information for allocating containers and CPU scheduling is not enabled by default. To use both memory as well as the CPU, the resource calculator needs to be changed to <code>org.apache.hadoop.yarn.util.resource.DominantResourceCalculator</code> in the <code>capacity-scheduler.xml</code> file.</p>

<p>Here's what needs to change.</p>

<pre><code>&lt;property&gt;
    &lt;name&gt;yarn.scheduler.capacity.resource-calculator&lt;/name&gt;
    &lt;value&gt;org.apache.hadoop.yarn.util.resource.DominantResourceCalculator&lt;/value&gt;
&lt;/property&gt;
</code></pre>
