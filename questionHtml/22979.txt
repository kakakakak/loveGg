<td class="answercell">
    <div class="post-text" itemprop="text">
<blockquote>
  <p>1) Will this change affect the existing data in HDFS</p>
</blockquote>

<p>No, it will not. It will keep the old block size on the old files. In order for it to take the new block change, you need to rewrite the data. You can either do a <code>hadoop fs -cp</code> or a <code>distcp</code> on your data. The new copy will have the new block size and you can delete your old data.</p>

<p>2) Do I need to propogate this change to all he nodes in Hadoop cluster or only on the NameNode is sufficient?</p>

<p>I believe in this case you only need to change the NameNode. However, this is a very very bad idea. You need to keep all of your configuration files in sync for a number of good reasons. When you get more serious about your Hadoop deployment, you should probably start using something like Puppet or Chef to manage your configs.</p>

<p>Also, note that whenever you change a configuration, you need to restart the NameNode and DataNodes in order for them to change their behavior.</p>

<p>Interesting note: you can set the blocksize of individual files as you write them to overwrite the default block size. E.g., <code>hadoop fs -D fs.local.block.size=134217728 -put a b</code></p>
