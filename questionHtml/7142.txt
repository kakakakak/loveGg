<div>
    <div class="post-text" itemprop="text">

<p>Amazon's Elastic MapReduce tool seems to only have support for HBase v0.92.x and v0.94.x. </p>

<p>The <a href="http://docs.aws.amazon.com/ElasticMapReduce/latest/DeveloperGuide/emr-hbase-versions.html" rel="nofollow">documentation for the EMR AMIs and HBase</a> is seemingly out-of-date and there is no information about HBase on the newest release label emr-4.0.0.</p>

<p>Using this <a href="https://blogs.aws.amazon.com/bigdata/post/Tx3CFT0COINZ4N8/Getting-HBase-Running-on-Amazon-EMR-and-Connecting-it-to-Amazon-Kinesis" rel="nofollow">example from an AWS engineer</a>, I was able to concoct a way to install another version of HBase on the nodes, but it was ultimately unsuccessful.</p>

<p>After much trial and error with the Java SDK to provision EMR with better versions, I ask:</p>

<p>Is it possible to configure EMR to use more recent versions of HBase (e.g. 0.98.x and newer?)</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>After several days of trial, error and support tickets to AWS, I was able to implement HBase 0.98 on Amazon's ElasticMapReduce service. Here's how to do it using the Java SDK, some bash and ruby-based bootstrap actions.</p>

<p>Credit for these bash and ruby scripts goes to Amazon support. They are in-development scripts and not officially supported. It's a hack.</p>

<p><strong>Supported Version:</strong> HBase 0.98.2 for Hadoop 2</p>

<p>I also created mirrors in Google Drive for the supporting files incase Amazon pulls them down from S3.</p>

<p><strong>Java SDK example</strong></p>

<pre><code>RunJobFlowRequest jobFlowRequest = new RunJobFlowRequest()
    .withSteps(new ArrayList&lt;StepConfig&gt;())
    .withName("HBase 0.98 Test")
