    <div class="post-text" itemprop="text">

<p>I'm using Spark with MongoDB, and consequently rely on the <code>mongo-hadoop</code> drivers. I got things working thanks to input on my original question <a href="http://stackoverflow.com/questions/33391840/getting-spark-python-and-mongodb-to-work-together">here</a>.</p>

<p>My Spark job is running, however, I receive warnings that I don't understand. When I run this command</p>

<pre><code>$SPARK_HOME/bin/spark-submit --driver-class-path /usr/local/share/mongo-hadoop/build/libs/mongo-hadoop-1.5.0-SNAPSHOT.jar:/usr/local/share/mongo-hadoop/spark/build/libs/mongo-hadoop-spark-1.5.0-SNAPSHOT.jar --jars /usr/local/share/mongo-hadoop/build/libs/mongo-hadoop-1.5.0-SNAPSHOT.jar:/usr/local/share/mongo-hadoop/spark/build/libs/mongo-hadoop-spark-1.5.0-SNAPSHOT.jar my_application.py
</code></pre>

<p>it works, but gives me the following warning message</p>

<blockquote>
  <p>Warning: Local jar
  /usr/local/share/mongo-hadoop/build/libs/mongo-hadoop-1.5.0-SNAPSHOT.jar:/usr/local/share/mongo-hadoop/spark/build/libs/mongo-hadoop-spark-1.5.0-SNAPSHOT.jar
  does not exist, skipping.</p>
</blockquote>

<p>When I was trying to get this working, if I left out those paths when submitting the job it wouldn't run at all. Now, however, if I leave out those paths it does run</p>

<pre><code>$SPARK_HOME/bin/spark-submit  my_application.py
</code></pre>

<p>Can someone please explain what is going on here? I have looked through similar questions here referencing the same warning, and searched through the documentation.</p>

<p>By setting the options once are they stored as environment variables or something? I'm glad it works, but wary that I don't fully understand why sometimes and not others. </p>
