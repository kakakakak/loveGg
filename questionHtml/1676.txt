<p>I am trying to run an OOZIE workflow using spark-action using yarn-cluster as master. The app seems to run fine when I use spark-submit from command line like this:</p>

<pre><code>./bin/spark-submit --verbose --class com.demo.spark.hdfs.SparkHDFSDemo --master yarn-cluster hdfs://namenode:8020/user/root/word-count/spark-hdfs.jar
</code></pre>

<p>But when I try to run same application using spark-action it keeps running for about 20 minutes and then gets killed. Following is my job.properties:</p>

<pre><code>nameNode=hdfs://namenode:8020
jobTracker=job-tracker:8050
master=yarn-cluster
name=spark-hdfs-word-count
class=com.demo.spark.hdfs.SparkHDFSDemo
oozie.use.system.libpath=true
oozie.wf.application.path=${nameNode}/user/root/word-count/spark-hdfs-workflow.xml
jar=${nameNode}/user/root/word-count/spark-hdfs.jar
</code></pre>

<p>And below is my spark-hdfs-workflow.xml</p>

<pre><code>&lt;workflow-app name="word-count-001" xmlns="uri:oozie:workflow:0.1"&gt;
    &lt;start to="spark-hdfs" /&gt;
    &lt;action name="spark-hdfs"&gt;
        &lt;spark xmlns="uri:oozie:spark-action:0.1"&gt;
            &lt;job-tracker&gt;${jobTracker}&lt;/job-tracker&gt;
            &lt;name-node&gt;${nameNode}&lt;/name-node&gt;
            &lt;master&gt;${master}&lt;/master&gt;
            &lt;name&gt;${name}&lt;/name&gt;
            &lt;class&gt;${class}&lt;/class&gt;
            &lt;jar&gt;${jar}&lt;/jar&gt;
         &lt;/spark&gt;
        &lt;ok to="end" /&gt;
      &lt;error to="kill" /&gt;
   &lt;/action&gt;
   &lt;kill name="kill"&gt;
      &lt;message&gt;failed&lt;/message&gt;
   &lt;/kill&gt;
   &lt;end name="end" /&gt;
&lt;/workflow-app&gt;
</code></pre>
