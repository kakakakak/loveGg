<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong><a href="http://facebook.com">Facebook.com</a></strong></p>

<ul>
<li><strong><a href="http://hadoop.apache.org/hive/">Hive</a></strong> (Data warehouse for Hadoop, supports tables and a variant of SQL called hiveQL). Used for "simple summarization jobs, business intelligence and machine learning and many other applications"</li>
<li><strong><a href="http://code.google.com/p/the-cassandra-project/">Cassandra</a></strong> (Multi-dimensional, distributed key-value store). Currently used for Facebook's private messaging.</li>
</ul>

<p>Currently running 610 (soon to be 1000) Hadoop nodes in a single cluster with Hive datastore. Both Hive and Cassandra have been open-sourced by Facebook.</p>

<p>Facebook stats:</p>

<ul>
<li>More than 200 million active users</li>
<li>More than 100 million users log on to Facebook at least once each day</li>
<li>More than 30 million users update their statuses at least once each day</li>
<li>Average user has 120 friends on the site</li>
</ul>

<ul>

<p><strong><a href="http://yahoo.com">Yahoo.com</a></strong></p>

<ul>
<li><strong>PostgreSQL</strong> (modified) - A client can connect to any of the nodes in the cluster (or a policy restricted subset).  A query flows from the client to the server it chose to connect with. The SQL compiler on that node compiles and optimizes the query on that single node (no parallelism).</li>
</ul>

<p>Yahoo.com stats:</p>

<ul>
<li>24 billion events a day</li>
<li>2-petabyte, claims largest database (Mar 2008)</li>
</ul>
