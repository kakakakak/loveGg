<p>More specifically I want an interface to compare objects which can be only compared for equality [e.g. complex numbers] but don't have total order on them. 
It should have [Note that it only returns a boolean yes/no]</p>

<pre><code>boolean Equals(T object1, T object2);
</code></pre>

<p>and a hashcode function. So that when I distribute them into buckets using objects say to collect "equal" objects, 2 "equal" objects don't end up in two different buckets. </p>

<pre><code>int getHashCode(T object);
</code></pre>

<p>Does Java have one? I searched and couldn't find it.</p>

<p>I am trying to use this in Hadoop Map Reduce to distribute "equal" objects to same reduce job so that I can operate on all "equal" objects. I only care about whether objects are equal or not and don't need total order. But if two objects are equal they should have same hash code. Otherwise they will end up in two different reduce jobs.</p>

<p>Please note that I know about equals and hashcode of object. But I want an external comparator which say depends on only part of an object. So object's notion of equality differs from mine.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There's no built-in type that is used for this in Java. It's a "hole" in the collections design, IMO. There's the string-specific <a href="http://download.oracle.com/javase/6/docs/api/java/text/Collator.html">Collator</a> class which is about as close as it gets, I'm afraid.</p>

<p>There's no way of customizing the built-in maps to use a specific kind of equality comparison, worse luck. It's <em>entirely reasonable</em> to want this functionality, and a real pain that it's not already present.</p>

<p>You could create your own such interface of course, and write your own map variants which use it... but having to do so sucks :(</p>

<p>The type you want is Guava's <a href="http://docs.guava-libraries.googlecode.com/git/javadoc/com/google/common/base/Equivalence.html"><code>Equivalence</code></a>.  However, you may be disappointed, as in Java <code>Collection</code> and <code>Map</code> are fairly rigidly specified in terms of <code>Object.equals</code>, and you will not find implementations of those in Guava that use an alternate equivalence. You can, however, simulate that behavior a bit using <code>myEquivalence.wrap(myObject)</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In the end I opted to write what I do in similar cases. 
If I need special equality/hash - like for example keeping weak references. You can wrap the key like that. Overall it's not very different from the interface approach but it creates dumb instances (like HashMap/Hashtable  for bucket entries). You might need extra unwrapping for keySet() and so on...</p>

<pre><code>package t1;

public abstract class KeyX&lt;Key&gt; implements java.io.Serializable {
    private static final long serialVersionUID = 0l;

    final Key key;
    final int hash;
    protected KeyX(Key key){
        this.key = key;
        this.hash = hashCode(key);
    }

    protected abstract int hashCode(Key key);

    //Key, Key will be way too strict and it'd required, key.getClass().isInstance(y) prior calling
    protected abstract boolean equals(Key x, Object y);

    @Override
    public final boolean equals(Object obj) {
        if (obj==this)
            return true;
        if (!(obj instanceof KeyX)){
            return false;
        }
        final KeyX&lt;?&gt; other = (KeyX&lt;?&gt;) obj;
        return this.key==other.key ||  (hash==other.hash &amp;&amp; other.key!=null  &amp;&amp; equals(this.key, other.key)); 

    }

    @Override
    public final int hashCode() {
        return hash;
    }

    public final Key unwrap(){
        return key;
    }
}
</code></pre>
