    <div class="post-text" itemprop="text">

<p>I have a class that must have some static methods. Inside these static methods I need to call the method getClass() to make the following call:</p>

<pre><code>public static void startMusic() {
  URL songPath = getClass().getClassLoader().getResource("background.midi");
}
</code></pre>

<p>However Eclipse tells me:</p>

<pre><code>Cannot make a static reference to the non-static method getClass() 
from the type Object
</code></pre>

<p>What is the appropriate way to fix this compile time error?</p>

The problem is that the JRE will be using the bootstrap class-loader at that stage, which will not have application resources on the class-path (of the bootstrap loader).</span>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As for the code example in the question, the standard solution is to reference the class explicitly by its name, and it is even possible to do without <code>getClassLoader()</code> call:</p>

<pre><code>class MyClass {
  public static void startMusic() {
    URL songPath = MyClass.class.getResource("background.midi");
  }
}
</code></pre>

<p>This approach still has a back side that it is not very safe against copy/paste errors in case you need to replicate this code to a number of similar classes.</p>

<p>And as for the exact question in the headline, there is a trick posted in the <a href="http://stackoverflow.com/a/17397143/972463">adjacent thread</a>:</p>

<pre><code>Class currentClass = new Object() { }.getClass().getEnclosingClass();
</code></pre>

<p>It uses a nested anonymous <code>Object</code> subclass to get hold of the execution context. This trick has a benefit of being copy/paste safe...</p>

    <div class="post-text" itemprop="text">
<p>I wrestled with this myself.  A nice trick is to use use the current thread to get a ClassLoader when in a static context.  This will work in a Hadoop MapReduce as well.  Other methods work when running locally, but return a null InputStream when used in a MapReduce.</p>

<pre><code>public static InputStream getResource(String resource) throws Exception {
   ClassLoader cl = Thread.currentThread().getContextClassLoader();
   InputStream is = cl.getResourceAsStream(resource);
   return is;
}
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><code>getClass()</code> method is defined in Object class with the following signature:</p>

<blockquote>
  <p>public final Class getClass()</p>
</blockquote>

<p>Since it is not defined as <code>static</code>, you can not call it within a static code block. See these answers for more information: <a href="http://stackoverflow.com/questions/5152428/non-static-to-static-problem">Q1</a>, <a href="http://stackoverflow.com/questions/2042813/calling-non-static-method-in-static-method-in-java">Q2</a>, <a href="http://stackoverflow.com/questions/6270690/non-static-method-cannot-be-referenced-in-static-context">Q3</a>.</p>

<p>If you're in a static context, then you have to use the class literal expression to get the Class, so you basically have to do like:</p>

<blockquote>
  <p>Foo.class</p>
</blockquote>

<p>This type of expression is called <strong>Class Literals</strong> and they are explained in <a href="http://java.sun.com/docs/books/jls/second_edition/html/expressions.doc.html#251530">Java Language Specification Book</a> as follows:</p>

<blockquote>
  <p>A class literal is an expression consisting of the name of a class, interface, array, or primitive type followed by a `.' and the token class. The type of a class literal is Class. It evaluates to the Class object for the named type (or for void) as defined by the defining class loader of the class of the current instance.</p>
</blockquote>

<p>You can also find information about this subject on <a href="http://docs.oracle.com/javase/6/docs/api/java/lang/Class.html">API documentation</a> for Class.</p>
