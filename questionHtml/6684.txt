<p>Message from terminal with command:</p>

<pre><code>hduser@ubuntu:/usr/local/hadoop$ hadoop fs -put /usr/local/input-data/ /input
put: /usr/local/input-data (Permission denied)

hduser@ubuntu:/usr/local/hadoop$ 
</code></pre>

<p>After using sudo and adding hduser to sudouser:</p>

<pre><code>hduser@ubuntu:/usr/local/hadoop$ sudo bin/hadoop fs -put /usr/local/input-data/ /inwe
put: org.apache.hadoop.security.AccessControlException: Permission denied: user=root, access=WRITE, inode="":hduser:supergroup:rwxr-xr-x

hduser@ubuntu:/usr/local/hadoop$ 
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I had similar situation and here is my approach which is somewhat different:</p>

<pre><code> HADOOP_USER_NAME=hdfs hdfs dfs -put /root/MyHadoop/file1.txt /
</code></pre>

<p>What you actually do is you read local file in accordance to your local permissions but when placing file on HDFS you are authenticated like user <code>hdfs</code>. You can do this with other ID (beware of real auth schemes configuration but this is usually not a case).</p>

<p>Advantages:</p>

<ol>
<li>Permissions are kept on HDFS.</li>
<li>You don't need <code>sudo</code>.</li>
<li>You don't need actually appropriate local user 'hdfs' at all.</li>
<li>You don't need to copy anything or change permissions because of previous points.</li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You are experiencing two separate problems here:</p>

<hr>

<pre><code>hduser@ubuntu:/usr/local/hadoop$ hadoop fs -put /usr/local/input-data/ /input put: /usr/local/input-data (Permission denied)
</code></pre>

<p>Here, the user <code>hduser</code> does not have access to the local directory <code>/usr/local/input-data</code>. That is, your local permissions are too restrictive. You should change it.</p>

<hr>

<pre><code>hduser@ubuntu:/usr/local/hadoop$ sudo bin/hadoop fs -put /usr/local/input-data/ /inwe put: org.apache.hadoop.security.AccessControlException: Permission denied: user=root, access=WRITE, inode="":hduser:supergroup:rwxr-xr-x
</code></pre>

<p>Here, the user <code>root</code> (since you are using sudo) does not have access to the HDFS directory <code>/input</code>. As you can see: <code>hduser:supergroup:rwxr-xr-x</code> says only <code>hduser</code> has write access. Hadoop doesn't really respect <code>root</code> as a special user.</p>

<hr>

<p>To fix this, I suggest you change the permissions on the local data:</p>

<pre><code>sudo chmod -R og+rx /usr/local/input-data/
</code></pre>

<p>Then, try the <code>put</code> command again <strong>as hduser</strong>.</p>
