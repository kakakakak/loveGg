    <div class="post-text" itemprop="text">

<p>I have installed and configured Hadoop 2.5.2 for a 10 node cluster. 1 is acting as masternode and other nodes as slavenodes.</p>

<p>I have problem in executing hadoop fs commands. hadoop fs -ls command is working fine with HDFS URI. It gives message "ls: `.': No such file or directory" when used without HDFS URI</p>

<pre><code>ubuntu@101-master:~$ hadoop fs -ls
15/01/30 17:03:49 WARN util.NativeCodeLoader: Unable to load native-hadoop 
ibrary for your platform... using builtin-java classes where applicable
ls: `.': No such file or directory
ubuntu@101-master:~$ 
</code></pre>

<p>Whereas, executing the same command with HDFS URI </p>

<pre><code>ubuntu@101-master:~$ hadoop fs -ls hdfs://101-master:50000/
15/01/30 17:14:31 WARN util.NativeCodeLoader: Unable to load native-hadoop      
library for your platform... using builtin-java classes where applicable
Found 3 items
drwxr-xr-x   - ubuntu supergroup          0 2015-01-28 12:07 hdfs://101-master:50000/hvision-data
-rw-r--r--   2 ubuntu supergroup   15512587 2015-01-28 11:50 hdfs://101-master:50000/testimage.seq
 drwxr-xr-x   - ubuntu supergroup          0 2015-01-30 17:03 hdfs://101-master:50000/wrodcount-in
 ubuntu@101-master:~$ 
</code></pre>

<p>I am getting exception in MapReduce program due to this behavior. jarlib is referring to the HDFS file location, whereas, I want jarlib to refer to the jar files stored at the local file system on the Hadoop nodes.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The behaviour that you are seeing is expected, let me explain what's going on when you are working with <code>hadoop fs</code> commands.</p>

<p>The command's syntax is this: <code>hadoop fs -ls [path]</code></p>

<p>By default, when you don't specify <code>[path]</code> for the above command, hadoop expands the path to <code>/home/[username]</code> in hdfs; where <code>[username]</code> gets replaced with linux username who is executing the command.</p>

<p>So, when you execute this command:</p>

<pre><code>ubuntu@xad101-master:~$ hadoop fs -ls
</code></pre>

<p>the reason you are seeing the error is <code>ls: '.': No such file or directory</code> because hadoop is looking for this path <code>/home/ubuntu</code>, it seems like this path doesn't exist in hdfs.</p>

<p>The reason why this command:</p>

<pre><code>ubuntu@101-master:~$ hadoop fs -ls hdfs://101-master:50000/
</code></pre>

<p>is working because, you have explicitly specified <code>[path]</code> and is the root of the hdfs. You can also do the same using this:</p>

<pre><code>ubuntu@101-master:~$ hadoop fs -ls /
</code></pre>

<p>which automatically gets evaluated to the root of hdfs.</p>

<p>Hope, this clears the behaviour you are seeing while executing <code>hadoop fs -ls</code> command.</p>

<p>Hence, if you want to specify local file system path use <code>file:///</code> url scheme.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are a couple things at work here; based on "jarlib is referring to the HDFS file location", it sounds like you indeed have an HDFS path set as your <code>fs.default.name</code>, which is indeed the typical setup. So, when you type <code>hadoop fs -ls</code>, this is indeed trying to look inside HDFS, except it's looking in your <a href="http://stackoverflow.com/questions/27684746/hadoop-fs-ls-listing-files-in-the-present-working-directory">current working directory</a>, which should be something like <code>hdfs://101-master:50000/user/ubuntu</code>. The error message is unfortunately somewhat confusing since it doesn't tell you that <code>.</code> was interpreted to be that full path. If you <code>hadoop fs -mkdir /user/ubuntu</code> then <code>hadoop fs -ls</code> should start working.</p>

<p>This problem is unrelated to your "jarlib" problem; whenever you want to refer files explicitly stored in the local filesystem, but where the path goes through Hadoop's <code>Path</code> resolution, you simply need to add <code>file:///</code> to force Hadoop to refer to the local filesystem. For example:</p>

<pre><code>hadoop fs -ls file:///tmp
</code></pre>

<p>Try passing your jar file paths as fille <code>file:///path/to/your/jarfile</code> and it should work.</p>
