<p>I do the following from a hive table myTable.</p>

<p>INSERT OVERWRITE LOCAL DIRECTORY '/myDir/out' SELECT concat_ws('',NAME,PRODUCT,PRC,field1,field2,field3,field4,field5) FROM myTable;</p>

<p>So, this command generates 2 files 000000_0 and 000001_0 inside the folder out/.</p>

<p>But, I need the contents as a single file. What should I do? </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are multiple files in the directory because every reducer is writing one file. If you really need the contents as a single file, run your map reduce job with only <strong>1 reducer</strong> which will write to a single file.</p>

<p>However depending on your data size, this might not be a good approach to run a single reducer. </p>

<p>Edit: Instead of forcing hive to run 1 reduce task and output a single reduce file, it would be better to use hadoop fs operations to merge outputs to a single file.
For example</p>

<pre><code>hadoop fs -text /myDir/out/* | hadoop fs -put - /myDir/out.txt
</code></pre>
