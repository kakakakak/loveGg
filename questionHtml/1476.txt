<p>I have a jar file on all my Hadoop nodes at <code>/home/ubuntu/libs/javacv-0.9.jar</code> , with some other jar files.</p>

<p>When my <em>MapReduce</em> application is executing on <em>Hadoop</em> nodes, I am getting this exception</p>

<pre><code>java.io.FileNotFoundException: File does not exist hdfs://192.168.0.18:50000/home/ubuntu/libs/javacv-0.9.jar
</code></pre>

<p>How can I resolve this exception? How can my jar running in Hadoop access 3rd party libraries from the local file system of the Hadoop node?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You need to copy your file to HDFS and not to the local filesystem.</p>

<p>To copy files to HDFS you need to use:</p>

<pre><code>hadop fs -put localfile hdfsPath
</code></pre>

<p>Other option is to change the file path to:</p>

<pre><code>file:///home/ubuntu/libs/javacv-0.9.jar
</code></pre>

<p>To add jar files to the classpath, take a look at <a href="https://hadoop.apache.org/docs/current/api/org/apache/hadoop/filecache/DistributedCache.html" rel="nofollow">DistributedCache</a>:</p>

<pre><code>DistributedCache.addFileToClassPath(new Path("file:///home/ubuntu/libs/javacv-0.9.jar"), job);
</code></pre>

<p>You may need to iterate over all jar files in that directory.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Another option would be to use <a href="https://hadoop.apache.org/docs/current/api/org/apache/hadoop/filecache/DistributedCache.html" rel="nofollow">distributed cache</a>'s <code>addFileToClassPath(new Path("/myapp/mylib.jar"), job);</code> to submit the Jar files that should be added to the classpath of your mapper and reducer tasks.</p>

<blockquote>
  <p>Note: Make sure you copy the jar file to HDFS first.</p>
</blockquote>

<p>You could even add jar files to class path by using hadoop command line argument <code>-libjars &lt;jar_file&gt;</code>.</p>

<blockquote>
  <p>Note: Make sure your MapReduce application implements <a href="https://hadoop.apache.org/docs/current/api/org/apache/hadoop/util/ToolRunner.html" rel="nofollow">ToolRunner</a> to allow <code>-libjars</code> option from command line.</p>
</blockquote>
