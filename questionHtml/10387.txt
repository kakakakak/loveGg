<p>I am having trouble setting up the <code>Php-Java Bridge</code> setup properly.</p>

<p>I will explain what I have done.</p>

<ul>
<li>My site is in pure php</li>
<li>For our payment transaction process we need to set up a php-java bridge</li>
<li>I followed this link to setup the bridge <a href="http://php-java-bridge.sourceforge.net/doc/installation.php" rel="nofollow">PHP-JAVA BRIDGE INSTALATION.</a></li>
<li>Here I learned that I need to have a private jvm to install the bridge.</li>
<li>So 1st i installed apache-tomcat-6.0.14 in Private JVM using my c-panel. After instalation it asked me to Map a domain to private JVM. So I mapped my domain example.com (which is the only option available) to it.</li>
<li>Then it asked to enable a traffic redirection from Apache web server to my Java application server (there was a check box and i clicked it)</li>
<li>Finally it asked me to deploy the WAR File (JavaBridge.WAR was my file) and everthing seems fine</li>
<li>Now when i go to <a href="http://example.com/JavaBridge/" rel="nofollow">http://example.com/JavaBridge/</a> I could see the javabridge examples and it works fine.</li>
</ul>

<p><strong>SO FAR SO GOOD</strong></p>

<p>Now my problem starts here when I try to access a java class file from php. A sample <code>test.php</code> is what I create and put the following code into it.</p>

<pre><code>  &lt;?php
        require_once("http://example.com:portnumber/JavaBridge/java/Java.inc");
        $System = java("java.lang.System");
        echo $System-&gt;getProperties(); //This Part echo's correctly and shows the data so it means i can access Java.inc Correctly

        $path_e24class = getcwd(). '/e24PaymentPipe.class'; //This part fails both test.php and java class file e24PaymentPipe.class are in the same directory in publich_html folder
        java_require($path_e24class);
        $pipe = new Java("e24PaymentPipe");
        $pipe-&gt;setAction("1");
?&gt;
</code></pre>

<p>My site contents reside in the public_html folder and the WAR file are deployed in private jvm.</p>

<p>These are the error message am getting.</p>

<pre><code>  1) Warning: java_require() not supported anymore. Please use tomcat or jee hot deployment instead
