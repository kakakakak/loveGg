<td class="answercell">
    <div class="post-text" itemprop="text">
<p>COBOL is a programming language, not a file format.  If what you need is to export files produced by COBOL programs, you can use the same technique as if those files were produced by C, C++, Java, Perl, PL/I, Rexx, etc.</p>

<p>In general, you will have three different data sources: flat files, VSAM files, and a DBMS such as DB2 or IMS.</p>

<p>DMBSs have export utilities to copy the data into flat files.  Keep in mind that data in DB2 will likely be normalized and thus you likely need the contents of related tables in order to make sense of the data.</p>

<p>VSAM files can be exported to flat files via the <a href="http://publib.boulder.ibm.com/infocenter/zos/v1r13/index.jsp?topic=/com.ibm.zos.r13.idai200/toc.htm">IDCAMS</a> utility.</p>

<p>I would strongly suggest you get the files into a text format before transferring them to another box with a different code page.  Trying to deal with mixed text (which must have its code page translated) and binary (which must not have its code page translated but which likely must be converted from big endian to little endian) is harder than doing the conversion up front.</p>

<p>The conversion can likely be done via the SORT utility on the mainframe.  Mainframe SORT utilities tend to have extensive data manipulation functions.  There are other mechanisms you could use (other utilities, custom code written in the language of your choice, purchased packages) but this is what we tend to do in these circumstances.</p>

<p>Once you have your flat files converted such that all data is text, you can transfer them to your Hadoop boxes via FTP or SFTP or FTPS.</p>

<p>This isn't an exhaustive coverage of the topic, but it will get you started.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Not pull, but push: use the <a href="https://dovetail.com/products/coz.html" rel="nofollow">Co:Z Launcher</a> from Dovetailed Technologies.</p>

<p>For example (JCL excerpt):</p>

<pre><code>//FORWARD  EXEC PGM=COZLNCH
//STDIN    DD *
hadoop fs -put &lt;(fromfile /u/me/data.csv) /data/data.csv
# Create a catalog table
hive -f &lt;(fromfile /u/me/data.hcatalog)
/*
</code></pre>

<p>where /u/me/data.csv (the mainframe-based data that you want in Hadoop) and /u/me/data.hcatalog (corresponding HCatalog file) are z/OS UNIX file paths.</p>

<p>For a more detailed example, where the data happens to be log records, see <a href="http://www-01.ibm.com/support/knowledgecenter/SSKKZM_1.3.0_ims/fuwutsk_big_data_hadoop.dita" rel="nofollow">Extracting logs to Hadoop</a>.</p>
