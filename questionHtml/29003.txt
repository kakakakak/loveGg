<p>I have found following question on some site , but I am not sure about correct answer .</p>

<blockquote>
  <p><em>Which Hadoop service in a MapReduce program spawns a new Mapper?</em></p>
</blockquote>

<ol>
<li>The <code>JobTracker</code> spawns a new <code>Mapper</code> to process all the records in a single file.  </li>
<li>The <code>TaskTracker</code> spawns a new <code>Mapper</code> to process each key-value pair.  </li>
<li>The <code>TaskTracker</code> spawns a new <code>Mapper</code> to process all the records in a single input split.  </li>
<li>The <code>JobTracker</code> calls the <code>TaskTracker</code>'s <code>configure ()</code> method, then its <code>map ()</code> method and finally its <code>close ()</code> method.  </li>
</ol>

<p>Site says Answer is option 2 , but I am confuse with Answer 3.</p>

<p>As in java doc of <code>mapper</code> I have found following thing :</p>

<blockquote>
  <p>The Hadoop Map-Reduce framework spawns one map task for each <code>InputSplit</code> generated by the <code>InputFormat</code> for the job. <code>Mapper</code> implementations can access the Configuration for the job via <code>theJobContext.getConfiguration()</code>.<br>
  The framework first calls <code>setup(org.apache.hadoop.mapreduce.Mapper.Context)</code>, followed by <code>map(Object, Object, Context)</code> for each key/value pair in the <code>InputSplit</code>. Finally <code>cleanup(Context)</code> is called.
  All intermediate values associated with a given output key are subsequently grouped by the framework, and passed to a <code>Reducer</code> to determine the final output. Users can control the sorting and grouping by specifying two key <code>RawComparator</code> classes.</p>
</blockquote>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It is 2</p>

<p>Map Task is JVM</p>

<p>Mapper is just a java class or an object </p>

<p>The job scheduler creates one Map Task for each split: (MapReduce1 page 191 Hadoop The Definitive Guide 3rd). </p>

<p>The Application Master creates a map task object for each split:(MapReduce2 page 199 of the same book).</p>

<p>Either case, it is not taskTrack spawns the Map task.</p>

<p>TaskTrack spawns Mapper class to handle each key/value pair.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There are two terms to be considered here. </p>

<ul>
<li><strong>Mapper</strong> as in the Map task spawned by the Tasktracker in a separate
JVM to process an input split. ( all of it ). For TextInputFormat , this would be a specific number of lines from your input file. </li>
<li><strong>Map method</strong> that is called for every record(key-value pair) in the split. Mapper.map(...) . In case of TextInputFormat, each map method (invocation)will process a line in your input split. </li>
</ul>

<p>Answer 3 seems to be the most accurate answer in this context. </p>

<p>Reference :- The Definitive Guide Edition 3</p>

<p>Correct answer is option-3</p>

<p>2.The TaskTracker spawns a new Mapper  to process each key-value pair.</p>

<p>3.The TaskTracker spawns a new Mapper to process all the records in a single input split. </p>

<p>For me, it does not make sense to spawn a new Mapper for each key-value pair - Think of 64MB block size contains 1000 records(key-value pairs). I do not think framework creates 1000 mapper here.</p>

<p>A Mapper task process single input split( not a file or block or some of records)</p>
