    <div class="post-text" itemprop="text">

<p>I have a cluster. On that cluster I have nodes. On those nodes I have a collection of different processes running (or not) that I would like to see a quick overview of. I wrote this bash script:</p>

<pre class="lang-sh prettyprint-override"><code>#!/usr/bin/env bash
set -o nounset

# Print out the Java process list and store it in a variable
readonly jps=$(jps -v)

# Declare two associative arrays
declare -A up
declare -A pid

# If the process in the in the saved list, set to 1, otherwise 0
up[accumulo_master]=$(echo ${jps} | grep -c 'Dapp=master')
up[accumulo_proxy]=$(echo ${jps} | grep -c 'Dapp=proxy')

# Store the PID of the process
pid[accumulo_master]=$(echo ${jps} | grep 'Dapp=master' | awk '{print $1}')
pid[accumulo_proxy]=$(echo ${jps} | grep 'Dapp=proxy' | awk '{print $1}')

echo Cluster Node Status: $(${wht})${me}$(${off})
echo -----------------------------------------
printf "%-28s %-5s %-5s\n" Component Up? PID
echo -----------------------------------------

for i in "${!up[@]}"
do
    printf "%-28s %-5s %-5s\n" $i ${up[$i]} ${pid[$i]};
done |
sort
</code></pre>

<p>The real script is the same, but with more elements in both the up and pid associative arrays. This prints out something like the following:</p>

<pre class="lang-txt prettyprint-override"><code>Cluster Node Status: box1
-----------------------------------------
Component                    Up?   PID  
-----------------------------------------
accumulo_master              1    10493
accumulo_proxy               1    10493
</code></pre>

<p>The problem is, the PID is always the same - it always prints the first PID that it comes across, repeated for each row - and I don't understand why. If I change the script so that lines like this:</p>

<pre><code>pid[accumulo_master]=$(echo ${jps} | grep 'Dapp=master' | awk '{print $1}')
</code></pre>

<p>look like this:</p>

<pre><code>pid[accumulo_master]=$(jps -v | grep 'Dapp=master' | awk '{print $1}')
</code></pre>

<p>i.e. - if I run <code>jps -v</code> every time, it works as expected - it just takes <em>much much longer</em> to run.</p>

<p>Any ideas?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Okay, so I figured this out just after I posted. It was, as it always is with bash, a quoting problem.</p>

<p>If you do this:</p>

<pre><code>readonly jps=$(jps -v)
echo $jps
</code></pre>

<p>It will print the whole multi-line output from <code>jps -v</code> on one single line. This means that <code>awk {print $1}</code> will <em>always</em> print out the first value - because it's line based and there's only one line.</p>

<p>If instead you do this:</p>

<pre><code>readonly jps=$(jps -v)
echo "$jps"
</code></pre>

<p>It prints out with multi-lines intact, which was what I was expecting and makes everything work.</p>

<p>I just needed to change the script so lines like this:</p>

<pre><code>pid[accumulo_master]=$(echo ${jps} | grep 'Dapp=master' | awk '{print $1}')
</code></pre>

<p>look like hits:</p>

<pre><code>pid[accumulo_master]=$(echo "${jps}" | grep 'Dapp=master' | awk '{print $1}')
</code></pre>
