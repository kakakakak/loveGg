    <div class="post-text" itemprop="text">

<p>What's the best practice for integrating Cassandra and Hive? </p>

<p>An old question on Stackoverflow (<a href="http://stackoverflow.com/questions/5935787/cassandra-wih-hive">Cassandra wih Hive</a>) points to Brisk, which has now become a subscription-only Datastax Enterprise product.</p>

<p>A google search only points to two open jira issues,</p>

<ul>
<li><a href="https://issues.apache.org/jira/browse/CASSANDRA-4131" rel="nofollow">https://issues.apache.org/jira/browse/CASSANDRA-4131</a></li>
<li><a href="https://issues.apache.org/jira/browse/HIVE-1434" rel="nofollow">https://issues.apache.org/jira/browse/HIVE-1434</a></li>
</ul>

<p>but none of them has resulted in any code committed in one of the two projects.</p>

<p>Is the only way to integrate Cassandra and Hive patching the Cassandra/Hive source code? Which solution are you using in your stack?</p>

    <div class="post-text" itemprop="text">
<p>I did the same research a month ago, to reach to the same conclusion. 
Brisk is no longer available as a community download, and besides patching the Cassandra/Hive code, the only way to throw map/reduce jobs at your Cassandra database is to use DSE -- Datastax Enterprise, which I believe is free for any use but production clusters. </p>

<p>You might have a look at <a href="http://hbase.apache.org/" rel="nofollow">HBase</a> which is based on <a href="http://hadoop.apache.org/hdfs/" rel="nofollow">HDFS</a>. </p>
