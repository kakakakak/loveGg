    <div class="post-text" itemprop="text">

<p>I was reading Hadoop documentation when I came across two function declarations returning a reference to an abstract class:</p>

<pre><code>public FSDataInputStream open(Path f) throws IOException
public abstract FSDataInputStream open(Path f, int bufferSize) throws IOException
</code></pre>

<p>Apart from differences in the parameters, why do these two functions have different return type, one explicitly declared <code>abstract</code> while other not?</p>

<p>Thanks.</p>

    <div class="post-text" itemprop="text">
<p>To answer the original question too, <code>abstract</code> is not part of the return type, it's a modifier of the method itself, just like <code>public</code>. The return types of the two methods are identical.</p>

    <div class="post-text" itemprop="text">
<p>Accoding <a href="http://docs.oracle.com/javase/tutorial/java/IandI/abstract.html" rel="nofollow">documentation</a>:</p>

<blockquote>
  <p>An abstract method is a method that is declared without an
  implementation (without braces, and followed by a semicolon), like
  this:</p>

<pre><code>abstract void moveTo(double deltaX, double deltaY);
 If a class includes abstract methods, the class itself must be declared abstract, as in:

public abstract class GraphicObject {
   // declare fields
   // declare non-abstract methods
   abstract void draw();
}
</code></pre>
  
  <p>When an abstract class is subclassed, the subclass usually provides
  implementations for all of the abstract methods in its parent class.
  However, if it does not, the subclass must also be declared abstract.</p>
</blockquote>

<p><strong>1.</strong> <code>public abstract FSDataInputStream open(Path f, int bufferSize) throws IOException</code>, Means this <strong>method is an abstract method</strong>, so its just an Declaration not an Implementation. So this method can be either a method in an Interface or in an Abstract Class. And so <strong>this method must be implemented by the very first Concrete subclass</strong> that implements the Interface or extends the class which contains this method.</p>

<p><strong>2.</strong> <code>public FSDataInputStream open(Path f) throws IOException</code>, this method if declared inside the interface, then its an <strong>abstract method</strong>, <strong>else a Non-Abstract method</strong> within an Abstract or Non-Abstract class.</p>
