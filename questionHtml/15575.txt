<p>Trying to port some existing PIG scripts that run perfectly locally and on Hortonworks to HDInsights and I am getting a class not found error when trying to read Avro files via Pig's piggybank (piggybank is included in the HDInsight distribution). </p>

<p>Wonder if anyone else has got Avro to work in HDInsight to work or has a work around (.eg. copy which jar where?)</p>

<p>In more detail ... Tracing this down I ran PIG directly via remote terminal on the Azure HDInsight instance. </p>

<p>The error I see is: java.lang.ClassNotFoundException: org.json.simple.parser.ParseException</p>

<p>This is I think, the json-simple library at <a href="https://code.google.com/p/json-simple/" rel="nofollow">https://code.google.com/p/json-simple/</a></p>

<p>I have tried adding this to a few places in the HDInsight VM (and explicitly registering the library in PIG) and still get the error.</p>

<p>The error is simple to reproduce by RDP into the Hadoop command line prompt on the desktop of an HDinsight Azure instance:</p>

<p>Start PIG with details ...</p>

<pre><code>c:\apps\dist\pig-0.11.0.1.3.1.0-06\bin&gt;pig -verbose -warning
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Setting environment variables such as PIG_CLASSPATH is not recommended for now in HDInsight, since you don't have admin access to the head node to do so. </p>

<p>Instead, there are <strong>two dimensions</strong> of a workaround we can use with Pig+Avro in HDInsight</p>

<p>A. Put the Avro and JSON jars in <strong>Windows Azure Storage Blobs</strong>, and register them from there. </p>

<p>B. Additionally for Avro, make a <strong>dummy empty class with a different name than Avro</strong> and put into the Piggybank. </p>

<p>The class loader wasn’t finding the necessary dependencies, because Avro was already in the Piggybank it doesn't search the right places for the JSON dependency.</p>

<p>The details of this snag... I believe it is related to how Java loads classes using custom class loaders. The way Pig implements this REGISTER keyword is that it uses a custom class loader that adds search locations to include these custom jars, but in this case the piggybank jar that contains the AvroStorage class is already in the classpath so it didn’t use the custom class loader. The problem then is that when Pig tries to instantiate it and it needs more custom jars, it doesn’t use the custom class loader so it can’t find the needed classes.</p>

<p><strong>The workaround:</strong> To work around this problem I created an empty class, MyAvroStroage, that just vacuously inherits from AvroStorage, and put it into its own little jar (it’s just 650 bytes). Then I put this with all its dependencies on WASB, register them and use MyAvroStorage instead of AvroStorage then everything works fine.</p>

<p>Possible Workaround:</p>

<p><strong>1. Copy to WASB</strong> 
For Part A, Copy jars to WASB as "PigExtras" folder for example. In another it was these commands (your source locations will vary, and your destination locations will be your container and account). For Part B, point to the jar of the dummy code MyAvroStorage.jar to upload it as well…</p>

<pre><code>hadoop fs -copyFromLocal C:\files\MyAvroStorage.jar wasb://container@account.blob.core.windows.net/PigExtras/MyAvroStorage.jar
hadoop fs -copyFromLocal c:\apps\dist\pig-0.11.0.1.3.1.0-06\piggybank.jar wasb://container@account.blob.core.windows.net/PigExtras/piggybank.jar
hadoop fs -copyFromLocal c:\apps\dist\oozie-3.3.2.1.3.1.0-06\oozie-win-distro\share\lib\pig\avro-1.5.3.jar wasb://container@account.blob.core.windows.net/PigExtras/avro-1.5.3.jar
hadoop fs -copyFromLocal C:\apps\dist\oozie-3.3.2.1.3.1.0-06\oozie-win-distro\share\lib\oozie\json-simple-1.1.jar wasb://container@account.blob.core.windows.net/PigExtras/json-simple-1.1.jar
</code></pre>

<p><strong>2. Create a simple repro script,</strong> myscript.pig, and put that on Windows Azure Storage Blob as well. You can register a whole folder I believe with *.jar wildcard syntax.</p>

<pre><code>REGISTER wasb:///PigExtras/*.jar;
myset = load 'wasb:///example/data/test.avro' using MyAvroStorage();
dump myset
</code></pre>

<p>Then upload the .pig to WASB as well</p>

<pre><code>hadoop fs -copyFromLocal C:\files\myscript.pig wasb://container@account.blob.core.windows.net/PigExtras/myscript.pig
</code></pre>

<p><strong>3. Submit the job</strong> to the cluster from a client:</p>

<pre><code>$job = New-AzureHDInsightPigJobDefinition -File    wasb://container@account.blob.core.windows.net/PigExtras/myscript.pig
Start-AzureHDInsightJob -Cluster $clust.Name -Subscription $sub.SubscriptionId -JobDefinition $job -Certificate $sub.Certificate
</code></pre>
