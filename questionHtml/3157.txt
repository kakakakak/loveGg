<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you use the default Log4j.properties file the logging settings get overridden by environment variables from the startup script. If you want to use the default log4j and just simply want to change the logging level, use <code>$HADOOP_CONF_DIR/hadoop-env.sh</code></p>

<p>For example, to change your logger to DEBUG log level and DRFA logger, use</p>

<pre><code>export HADOOP_ROOT_LOGGER="DEBUG,DRFA"
</code></pre>

    <div class="post-text" itemprop="text">
<ol>
<li>You could remove the <code>log4j.properties</code> from your <code>hadoop jar</code></li>
<li>OR make sure that your <code>jar</code>/<code>log4j.properties</code> is first in the classpath (log4j picks the first <code>log4j.properties</code> from the classpath that it finds)</li>
<li>OR specify the system variable: <code>-Dlog4j.configuration=PATH_TO_FILE</code></li>
</ol>

<p>See the <a href="https://logging.apache.org/log4j/1.2/manual.html" rel="nofollow">documentation</a> to learn how log4j finds the configuration.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>Maven Packaging:</strong> </p>

<p>Once I realized I needed to add my custom <code>debug-log.properties</code> file to src/main/java/resources, Maven added it to the application.jar root directory and then it was just a matter of referring to it or not in <code>-Dlog4j.configuration=debug-log.properties</code> from the command line. </p>

<p><strong>Oozie <code>&lt;java&gt;</code> Action:</strong> </p>

<p>In regard to Oozie, use <code>&lt;java-opts&gt;-Dlog4j.configuration=${log4jConfig}&lt;/java-opts&gt;</code> in the workflow.xml actions and define the following in a job.properties file. </p>

<pre><code>#one of the following log4j.config parameters must be defined 
#log4jConfig=log4j.properties
log4jConfig=debug-log.properties
</code></pre>

<p><strong>Oozie <code>&lt;map-reduce&gt;</code> Action:</strong> </p>

<pre><code> &lt;property&gt;
      &lt;name&gt;mapred.child.java.opts&lt;/name&gt;
      &lt;value&gt;-Dlog4j.configuration=${log4jConfig}&lt;/value&gt;
 &lt;/property&gt; 
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Modify the <code>log4j</code> file inside <code>HADOOP_CONF_DIR</code>. Note that hadoop job wont consider the log4j file of your application. It will consider the one inside <code>HADOOP_CONF_DIR</code>. </p>

<p>If you want to force hadoop to use some other <code>log4j</code> file, try one of these:</p>

<ol>
<li><p>You can try what @Patrice said. ie. </p>

<p>-Dlog4j.configuration=file:/path/to/user_specific/log4j.xml</p></li>
<li><p>Customize HADOOP_CONF_DIR/log4j.xml and set the logger level for "your" classes as per your wish. Other user(s) wont be affected due to this unless both are having classes with same package structure. This wont work for core hadoop classes as all users will get afftected.</p></li>
<li><p>Create your customized log4j file. Replicate the directory HADOOP_CONF_DIR and put your log4j file inside it. export HADOOP_CONF_DIR to your conf directory. Other users will point to the default one. </p></li>
</ol>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Put log4j.configuration option <em>in</em> the child java options.</p>

<p>I.e.</p>

<pre><code>hadoop jar ... -Dmapred.child.java.opts=-Dlog4j.configuration=file:/...../log4j_debug.properties
</code></pre>

<p>You must put log4j_debug.properties file on all slave servers in a same directory path like /home/yourname/log4j_debug.properties or /tmp/log4j_debug.properties</p>

<p>This setting overwrites mapred.child.java.opts settings.
If you want to use with another options like -Xmx32m, which means 32MB heap size,  then do like following:</p>

<pre><code>hadoop jar ... -Dmapred.child.java.opts='-Xmx32m -Dlog4j.configuration=file:/...../log4j_debug.properties'
</code></pre>
