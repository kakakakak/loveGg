<p>i have algorithm that will go through a large data set read some text files and search for specific terms in those lines. I have it implemented in Java, but I didnt want to post code so that it doesnt look i am searching for someone to implement it for me, but it is true i really need a lot of help!!! This was not planned for my project, but data set turned out to be huge, so teacher told me I have to do it like this.  </p>

<p><strong>EDIT</strong>(i did not clarified i previos version)The data set I have is on a Hadoop cluster, and I should make its MapReduce implementation</p>

<p>I was reading about MapReduce and thaught that i first do the standard implementation and then it will be more/less easier to do it with mapreduce. But didnt happen, since algorithm is quite stupid and nothing special, and map reduce...i cant wrap my mind around it.</p>

<p>So here is shortly pseudo code of my algorithm</p>

<pre><code>LIST termList   (there is method that creates this list from lucene index)
FOLDER topFolder

INPUT topFolder
IF it is folder and not empty
    list files (there are 30 sub folders inside)
    FOR EACH sub folder
        GET file "CheckedFile.txt"
        analyze(CheckedFile)
    ENDFOR
END IF


Method ANALYZE(CheckedFile)

read CheckedFile
WHILE CheckedFile has next line
    GET line
    FOR(loops through termList)
            GET third word from line
          IF third word = term from list
        append whole line to string buffer
    ENDIF
ENDFOR
END WHILE
OUTPUT string buffer to file
</code></pre>

<p>Also, as you can see, each time when "analyze" is called, new file has to be created, i understood that map reduce is difficult to write to many outputs??? </p>

<p>I understand mapreduce intuition, and my example seems perfectly suited for mapreduce, but when it comes to do this, obviously I do not know enough and i am STUCK!</p>

<p>Please please help. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Map Reduce is easily implemented using some nice Java 6 concurrency features, especially Future, Callable and ExecutorService.</p>

<p>I created a Callable that will analyse a file in the way you specified</p>

<pre><code>public class FileAnalyser implements Callable&lt;String&gt; {

  private Scanner scanner;
  private List&lt;String&gt; termList;

  public FileAnalyser(String filename, List&lt;String&gt; termList) throws FileNotFoundException {
    this.termList = termList;
    scanner = new Scanner(new File(filename));
  }

  @Override
  public String call() throws Exception {
    StringBuilder buffer = new StringBuilder();
    while (scanner.hasNextLine()) {
      String line = scanner.nextLine();
      String[] tokens = line.split(" ");
      if ((tokens.length &gt;= 3) &amp;&amp; (inTermList(tokens[2])))
        buffer.append(line);
    }
    return buffer.toString();
  }

  private boolean inTermList(String term) {
    return termList.contains(term);
  }
}
</code></pre>

<p>We need to create a new callable for each file found and submit this to the executor service. The result of the submission is a Future which we can use later to obtain the result of the file parse.</p>

<pre><code>public class Analayser {

  private static final int THREAD_COUNT = 10;

  public static void main(String[] args) {

    //All callables will be submitted to this executor service
    //Play around with THREAD_COUNT for optimum performance
    ExecutorService executor = Executors.newFixedThreadPool(THREAD_COUNT);

    //Store all futures in this list so we can refer to them easily
    List&lt;Future&lt;String&gt;&gt; futureList = new ArrayList&lt;Future&lt;String&gt;&gt;();

    //Some random term list, I don't know what you're using.
    List&lt;String&gt; termList = new ArrayList&lt;String&gt;();
    termList.add("terma");
    termList.add("termb");

    //For each file you find, create a new FileAnalyser callable and submit
    //this to the executor service. Add the future to the list
    //so we can check back on the result later
    for each filename in all files {
      try {
        Callable&lt;String&gt; worker = new FileAnalyser(filename, termList);
        Future&lt;String&gt; future = executor.submit(worker);
        futureList.add(future);
      }
      catch (FileNotFoundException fnfe) {
        //If the file doesn't exist at this point we can probably ignore,
        //but I'll leave that for you to decide.
        System.err.println("Unable to create future for " + filename);
        fnfe.printStackTrace(System.err);
      }
    }

    //You may want to wait at this point, until all threads have finished
    //You could maybe loop through each future until allDone() holds true
    //for each of them.

    //Loop over all finished futures and do something with the result
    //from each
    for (Future&lt;String&gt; current : futureList) {
      String result = current.get();
      //Do something with the result from this future
    }
  }
}
</code></pre>

<p>My example here is far from complete, and far from efficient. I haven't considered the sample size, if it's really huge you could keep looping over the futureList, removing elements that have finished, something similar to:</p>

<pre><code>while (futureList.size() &gt; 0) {
      for (Future&lt;String&gt; current : futureList) {
        if (current.isDone()) {
          String result = current.get();
          //Do something with result
          futureList.remove(current);
          break; //We have modified the list during iteration, best break out of for-loop
        }
      }
}
</code></pre>

<p>Alternatively you could implement a producer-consumer type setup where the producer submits callables to the executor service and produces a future and the consumer takes the result of the future and discards then future.</p>

<p>This would maybe require the produce and consumer be threads themselves, and a synchronized list for adding/removing futures.</p>

<p>Any questions please ask.</p>
