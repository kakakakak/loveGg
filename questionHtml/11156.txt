    <div class="post-text" itemprop="text">

<p>There is some hype with languages like Python (that I like very much) and Ruby but I was writing a medium-size system in Python and feel the lack of some tools that I would have if I was using Java: Eclipse features, JUnit integration and some language-features like catching some errors in compile time.</p>

<p>Some people says that Java is dead as a language. But large important software are written in Java: Hadoop, Lucene and many others.</p>

<p>Is Java a better language than Python or Ruby for medium/large infrastructure software like these?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You make a valid point in saying that Java is more suited to enterprise software. That's really where Java shines: it works very well for enterprise programming.</p>

<p>However, your gripes about the tools in Python and Ruby are unfounded. PyDev for Eclipse takes full advantage of the same features that you would enjoy if you were developing in Java. Ruby has a similar plugin for Eclipse. Unit testing frameworks like pyunit, nose, and others exist for both languages (and I believe PyDev has support for unit tests). Also, just like a Java compiler would catch an error at compile time, a Python script will give syntax errors before you run it, along with an (IMO) much more understandable error system than a Java compiler.</p>

<p>Since Python and Ruby are scripting languages, their area of expertise lies more in small tasks, rapid development, and using frameworks like Django or Rails. So yes, more heavy-duty languages like Java are great for enterprise, but there's something to be said for the elegance and ease of use of Python and Ruby.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>All much of a muchness really. There are features in Python/Ruby that your aware of that make them a more appealing offering than say Java. Static typing can be your best friend most of the time, and a pain for the other.</p>

<p>Lack of tooling is related to adoption. Java/c# has a larger adoption.</p>

<p>Corporates/Enterprises and medium to large organizations, like to procure solutions with contracts as opposed to technical merits.</p>

<p>From experience Python does have some speed issues, I can't comment on Ruby, but I imagine that if your applications require dedicated speed I would not use python.</p>

<p>Also threading and multi-core processing is not as good as it could be in python, although 2.7 seems to improve a lot of the old problems, I think the GIL problem still remains for threading.</p>

<p>My only comment on Java is that it now has Oracle behind it, and depending where your sitting and how you like Oracle, might change your attitude to Java. especially if your designing for Open Source solutions. </p>
