<td class="answercell">
    <div class="post-text" itemprop="text">
<p>HDFS/Hadoop is not compatible on windows, there is a cygwin approach for windows but its not complete. You will save tons of your time and effort running hadoop on Linux.</p>

<p>Check the examples for Hbase MR on Hbase site:
<a href="http://hbase.apache.org/book/mapreduce.example.html" rel="nofollow">http://hbase.apache.org/book/mapreduce.example.html</a>
I use CDH libraries, below link guides you how to setup the maven dependencies:
<a href="http://www.cloudera.com/content/cloudera-content/cloudera-docs/CDH4/latest/CDH-Version-and-Packaging-Information/cdhvd_topic_8.html" rel="nofollow">http://www.cloudera.com/content/cloudera-content/cloudera-docs/CDH4/latest/CDH-Version-and-Packaging-Information/cdhvd_topic_8.html</a></p>

<p>Basically you will need 3 libraries: hadoop-client (hadoop-core/hadoop-common) zookeeper and hbase (hbase-common) to compile your Hbase map reduce program.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Have you tried Hortonworks <a href="http://hortonworks.com/products/hdp-windows/" rel="nofollow">HDP for Windows</a>? They claim it is open source and it looks like they provide at least minimally needed set of documentation for developers. So I think it is moreless doable and more, advice you at least to try their approach.</p>

<p>My own approach for 'trail' development is to use Cloudera <a href="http://www.cloudera.com/content/cloudera/en/products-and-services/cdh.html" rel="nofollow">Hadoop distribution</a> over any Ubuntu/CentOS VM on Windows host (lot of options for guest VM <a href="http://www.cloudera.com/content/cloudera-content/cloudera-docs/DemoVMs/Cloudera-QuickStart-VM/cloudera_quickstart_vm.html" rel="nofollow">including their own free bundle</a>). <a href="http://www.cloudera.com/content/cloudera/en/products-and-services/cloudera-manager.html" rel="nofollow">Cloudera Manager</a> (which has free 'standard' edition) simplifies initial setup. Actually they look like 'standard de facto' if you need good enough solution with reasonable level of documentation and it should be free while it is small.</p>

<p>Having Hadoop installed on VM you can access it based on any of large set of client libraries. I don't recommend actually to have Hadoop server on Windows. At least for some time from now (end of 2013).</p>

<p>OK, with 'pure' Apache solution everything looks much more complex.</p>
