<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The fundamental difference between Big Data platform (Hadoop) and traditional database is that Big Data platform applies schema on read - while we define a schema upfront in traditional databases. So, in traditional databases, we can only inset those data that fits into the known schema.  Hadoop at it core is a file system called HDFS and distributed compute on the file system using the map-reduce algorithm. Normally we don't ask, what kind of preparation we need to copy a file onto a your computer. You copy it first - as is – because we are copying into a file system like NTFS, FAT, HFSX, ZFS, etc. Similarly HDFS is a file system. Tools like Hive gives you an SQL interface to Hadoop. But, under the hood, Hive is taking the raw file and applying a schema at the time of read. Since the processing is fast and distributed, this is not an issue. That is why - we typically don’t use Hadoop for real time transaction processing systems. </p>

<p>While you can do most transformations you asked except delete, typically you shouldn’t be doing a lot of transformations in a Hadoop based system. You provide raw data and use tools like Hive or write your own map-reduce code to access it. They run at the time of access.  Hence, the level of pre-processing you do should be relatively small. </p>

<p>You could use several visualization tools on Hadoop to draw the kind of plots you are looking for.</p>

<p>For more analysis algorithms – see apache mahout. 
<a href="http://mahout.apache.org/users/basics/algorithms.html" rel="nofollow">http://mahout.apache.org/users/basics/algorithms.html</a></p>
