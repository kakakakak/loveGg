<p>I'm following the Hadoop2OnWindows Wiki <a href="https://wiki.apache.org/hadoop/Hadoop2OnWindows" rel="nofollow">here</a>, and I've read the <code>BUILDING.txt</code> in the source tree many times. <p>I have installed Windows SDK 7.1</p>

<p>The following things are in my PATH:</p>

<ul>
<li><code>Cygwin/bin</code></li>
<li>MSBuild (<code>C:\Windows\Microsoft.NET\Framework64\v4.0.30319</code>)</li>
<li>Protocol Buffer 2.5.0</li>
<li>Zlib and Zlib/include (so that it has access to zlib.h)</li>
<li>Findbugs</li>
<li>Maven</li>
</ul>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>From the log, it is obviously that you miss zlib.h
Did you set the env var ZLIB_HOME to the directory containing zlib.h?
Note that put zlib.h directory in PATH is wrong and not necessary, only zlib bin directory is needed, as said in Building guide <a href="https://svn.apache.org/viewvc/hadoop/common/branches/branch-2/BUILDING.txt?view=markup" rel="nofollow">https://svn.apache.org/viewvc/hadoop/common/branches/branch-2/BUILDING.txt?view=markup</a></p>

<p>Also, zlib.h requires some header not present in Windows, so you will need to download those headers and put in the same folder with zlib.h. 
For more details, see <a href="https://stackoverflow.com/questions/341817/is-there-a-replacement-for-unistd-h-for-windows-visual-c">Is there a replacement for unistd.h for Windows (Visual C)?</a> to get unistd.h,
and <a href="https://gist.github.com/ashelly/7776712" rel="nofollow">https://gist.github.com/ashelly/7776712</a> to get getopt.h</p>

<p>For those folks on Win32 (like me), you may have faced much more troubles. The key solution should be editing .sln and .vcxprj file of winutils and native package so that they are compatible with Win32 platform.</p>
