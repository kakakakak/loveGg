<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I've never used Camus. But I believe this is a Kafka related error and it has to do with how you're encoding/decoding the message. I believe the important lines in your stack trace are </p>

<pre><code>Caused by: com.linkedin.camus.coders.MessageDecoderException: java.lang.NullPointerException
  at com.linkedin.camus.etl.kafka.coders.KafkaAvroMessageDecoder.init(KafkaAvroMessageDecoder.java:40)
  at com.linkedin.camus.etl.kafka.coders.MessageDecoderFactory.createMessageDecoder(MessageDecoderFactory.java:24)
</code></pre>

<p>How are you telling Kafka to use your Avro encoding? You've commented out the following line in your config,</p>

<pre><code>#kafka.message.coder.schema.registry.class=com.linkedin.camus.example.schemaregistry.DummySchemaRegistry
</code></pre>

<p>So are you setting that somewhere else in code? If you're not, I would suggest uncommenting out that config value and setting it to whatever avro class you're trying to decode/encode in.</p>

<p>It might take you some debugging to use the right classpath and such, but I believe this is an easily solvable problem.</p>

<p><strong>EDIT</strong>
In responding to your comments, I have a couple comments of my own.</p>

<ol>
<li><strong>I have never used Camus.</strong> So debugging the errors you get from Camus is not something that I'll be able to do very well or at all. So you'll have to spend some time (<em>maybe several hours</em>) researching and trying different things to get it to work.</li>
<li>I doubt <a href="https://github.com/linkedin/camus/blob/master/camus-example/src/main/java/com/linkedin/camus/example/schemaregistry/DummySchemaRegistry.java" rel="nofollow">DummySchemaRegistry</a> is the correct configuration value that you need. Anything starting with Dummy is probably not a valid configuration option.</li>
<li>Doing a simple google search about camus and schema registry revealed some interesting links, <a href="https://github.com/linkedin/camus/tree/master/camus-schema-registry/src/main/java/com/linkedin/camus/schemaregistry" rel="nofollow">SchemaRegistry Classes</a>, <a href="https://github.com/linkedin/camus/blob/master/camus-kafka-coders/src/main/java/com/linkedin/camus/etl/kafka/coders/KafkaAvroMessageEncoder.java" rel="nofollow">KafkaAvroMessageEncoder</a>. Those are more likely to be the correct config values that you need. Just my guess, because again, I've never used Camus.</li>
<li><a href="https://groups.google.com/forum/#!topic/camus_etl/RzSHsDzOdow" rel="nofollow">This</a> could also be of some use to you. I don't know if you've seen it. But if you haven't, I'm pretty sure googling the specific error you get is probably something you should before coming to Stack overflow.</li>
</ol>
