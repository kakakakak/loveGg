<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In order to forcefully let the namenode leave safemode, following command should be executed:</p>

<pre><code> bin/hadoop dfsadmin -safemode leave
</code></pre>

<p>You are getting <code>Unknown command</code> error for your command as <code>-safemode</code> isn't a sub-command for <code>hadoop fs</code>, but it is of <code>hadoop dfsadmin</code>.</p>

<p>Also after the above command, I would suggest you to once run <code>hadoop fsck</code> so that any inconsistencies crept in the hdfs might be sorted out.</p>

<p><strong>Update:</strong></p>

<p>Use <code>hdfs</code> command instead of <code>hadoop</code> command for newer distributions:</p>

<pre><code>hdfs dfsadmin -safemode leave
</code></pre>

<p><code>hadoop dfsadmin</code> has been deprecated and so is <code>hadoop fs</code> command, all hdfs related tasks are being moved to a separate command <code>hdfs</code>. </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Namenode enters into safemode when there is shortage of memory. As a result the HDFS becomes readable only. That means one can not create any additional directory or file in the HDFS. To come out of the safemode, the following command is used: </p>

<pre><code>hadoop dfsadmin -safemode leave
</code></pre>

<p>If you are using cloudera manager:</p>

<pre><code>go to &gt;&gt;Actions&gt;&gt;Leave Safemode
</code></pre>

<p>But it doesn't always solve the problem. The complete solution lies in making some space in the memory. Use the following command to check your memory usage.</p>

<pre><code>free -m
</code></pre>

<p>If you are using cloudera, you can also check if the HDFS is showing some signs of bad health. It probably must be showing some memory issue related to the namenode. Allot more memory by following the options available. I am not sure what commands to use for the same if you are not using cloudera manager but there must be a way. Hope it helps! :)</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you use Hadoop version 2.6.1 above, while the command works, it complains that its depreciated. I actually could not use the <code>hadoop dfsadmin -safemode leave</code> because I was running Hadoop in a Docker container and that command magically fails when run in the container, so what I did was this. I checked doc and found <code>dfs.safemode.threshold.pct</code> in documentation that says </p>

<blockquote>
  <p>Specifies the percentage of blocks that should satisfy the minimal
  replication requirement defined by dfs.replication.min. Values less
  than or equal to 0 mean not to wait for any particular percentage of
  blocks before exiting safemode. Values greater than 1 will make safe
  mode permanent.</p>
</blockquote>

<p>so I changed the <code>hdfs-site.xml</code> into the following (In older Hadoop versions, apparently you need to do it in <code>hdfs-default.xml</code>:</p>

<pre><code>&lt;configuration&gt;
    &lt;property&gt;
        &lt;name&gt;dfs.safemode.threshold.pct&lt;/name&gt;
        &lt;value&gt;0&lt;/value&gt;
    &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>
