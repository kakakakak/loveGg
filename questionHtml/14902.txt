<p>When using the <code>--incremental append</code> flag in the <code>sqoop import</code>, the job will fail.</p>

<p><code>ERROR tool.ImportTool: Imported Failed: Wrong FS: s3n://&lt;api_key&gt;:&lt;api_secret&gt;@bucket/folder/</code></p>

<p>Here is the full command:</p>

<p><code>
sqoop import --connect jdbc:postgresql://someplace.net:12345/db --warehouse-dir s3n://&lt;key&gt;:&lt;private&gt;@bucket/folder/ --table my_table --hive-drop-import-delims --escaped-by "\\" --username root --password safepass -m 2 --split-by id  --incremental append --check-column id
</code></p>

<p>The exact same command without the <code>--incremental' and</code>--check-column` work without any error and upload to the correct s3 directory as expected. I want to be able to run an incremental import and have it uploaded to s3.</p>

<p>I'm using Sqoop 1.4.1-cdh4.1.3</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The reason this is happening is that Sqoop writes files to a temporary directory when in "incremental" mode. This is to avoid partial writes to the final destination directory. After writing everything to the temp directory, it will issue a "rename" for each file to get them into the final destination directory, which obviously won't work across filesystems (HDFS -> S3). </p>

<p>The way to make this work is to make Sqoop use S3 for the temporary directory as well. Before issuing the job, set the system property that Sqoop relies on as the base for the temporary files (see <a href="https://github.com/apache/sqoop/blob/trunk/src/java/org/apache/sqoop/util/AppendUtils.java" rel="nofollow">AppendUtils</a>):</p>

<pre><code>System.setProperty("sqoop.test.import.rootDir", &lt;AUTHENTICATED_S3N_URL&gt;);
</code></pre>

<p>Also, when renaming the files, Sqoop gets a handle on the FS by getting the default FS from the Hadoop config, so you'll also need to set this to S3N:</p>

<pre><code>FileSystem.setDefaultUri(configuration, &lt;AUTHENTICATED_S3N_URL&gt;);
</code></pre>

<p>It's not pretty, but it works.</p>
