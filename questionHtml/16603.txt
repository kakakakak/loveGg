<p>My hadoop-1.2.1 configurations are blow:</p>

<pre><code>***core-site.xml***

&lt;configuration&gt;
    &lt;property&gt;
        &lt;name&gt;fs.default.name&lt;/name&gt;
        &lt;value&gt;hdfs://localhost:9000&lt;/value&gt;
    &lt;/property&gt;
    &lt;property&gt;
        &lt;name&gt;hadoop.tmp.dir&lt;/name&gt;
        &lt;value&gt;/home/dsk/hadoop/tmp-storage&lt;/value&gt;
    &lt;/property&gt;
&lt;/configuration&gt;


***hdfs-site.xml***


&lt;configuration&gt;
    &lt;property&gt;
        &lt;name&gt;dfs.replication&lt;/name&gt;
        &lt;value&gt;1&lt;/value&gt;
    &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>I want to use Hadoop Filesystem API to make new direction in HDFS. So I wrote in my code:</p>

<pre><code>    @Test
    public void testDirectory() throws Exception {

        Configuration configuration = new Configuration();
        FileSystem hdfs = HDFSUtils.getFileSystem(configuration);
        Path path = new Path("/test_dir");
        boolean isSuccess = hdfs.mkdirs(path);
        System.out.println("mkdir   test_dir" + (isSuccess ? " success" : " failure"));
    }
</code></pre>

<p>My <code>class HDFSUtils</code> is :</p>

<pre><code>    public static FileSystem getFileSystem(Configuration conf) {

        FileSystem hdfs = null;
        try {

            hdfs = FileSystem.get(conf);
            return hdfs;

        } catch (Exception e) {
            e.printStackTrace();
        }
        return hdfs;
    }
</code></pre>

<p>I want my hdfs  hierarchical structure like below:
<img src="http://i.stack.imgur.com/ReTHj.png" alt="enter image description here">
But I got <code>mkdir test_dir failure</code> when I tested with JUnit.</p>

<p>My working IDE is Eclipse, in <code>conf</code> folder two <code>xml</code> files having been included<img src="http://i.stack.imgur.com/394G3.png" alt="enter image description here"></p>

<p>I still have some problems on how to manipulate files on HDFS using Filesystem API. 
At first I used <code>URL</code> method and it worked.</p>

<pre><code>String fileUrl = "hdfs://localhost:9000/firstSQL.sql";
</code></pre>

<p>But when I changed it to HDFS FileSystem API, I can't find my files:</p>

<pre><code>    Configuration conf = new Configuration();
    FileSystem hdfs = HDFSUtils.getFileSystem(conf);
    Path path = new Path("/firstSQL.sql");
</code></pre>

<p>The Failure Traces is <code>java.io.FileNotFoundException: File /firstSQL.sql does not exist.</code></p>

<p>But I really have <code>firstSQL.sql</code> in HDFS.
<img src="http://i.stack.imgur.com/ba51L.png" alt="enter image description here"></p>

<p>I am confused of the <strong>real path of my file</strong> on HDFS.</p>
