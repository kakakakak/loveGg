    <div class="post-text" itemprop="text">

<p>I have just installed <code>Google Cloud platform</code> for a free trial. In order to run <code>MapReduce</code> tasks with <code>DataStore</code>, the <a href="https://cloud.google.com/hadoop/running-with-datastore-connector" rel="nofollow">docs</a> says to run </p>

<pre><code>./bdutil --upload_files "samples/*" run_command ./test-mr-datastore.sh
</code></pre>

<p>But I couldn't get this file on my local and there's a good reason for that, this way to run <code>MapReduce</code> jobs seem to be deprecated see this on <a href="https://cloud.google.com/hadoop/running-with-datastore-connector" rel="nofollow">github</a>. Is that true, is there an alternative way to create MapReduce tasks from local command lines without requiring <code>BigQuery</code> ?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><strong>NOTE</strong>: Google team <a href="https://github.com/GoogleCloudPlatform/bdutil/blob/master/CHANGES.txt" rel="nofollow">removed DataStore connector</a> from <code>bdutil</code> v1.3.0 (2015-05-27) going forward so you might need to use older version or use GCS or BigQuery as proxy to accessing your data in DataStore.</p>

<p>I try to cover as much as I can, but <code>bdutil</code> is require lots more detail which is hard to document it in this answer, but I hope this can give you enough to start:</p>

<ul>
<li><p>Setup Google Cloud SDK - <a href="https://cloud.google.com/sdk/" rel="nofollow">detail</a></p>

<pre><code># Download SDK
curl https://sdk.cloud.google.com | bash
# Restart your shell
exec -l $SHELL
# Authenticate to GCP
gcloud auth login
# Select Project
gcloud config set project PROJECT_NAME
</code></pre></li>
<li><p>Download and extract <a href="https://cloud.google.com/hadoop/bdutil" rel="nofollow">bdutil</a> source code that contains DataStore connector.  </p>

<pre><code># Download source which contains DataStore connector
wget https://github.com/GoogleCloudPlatform/bdutil/archive/1.2.1.tar.gz -O bdutil.tar.gz
# Extract source
tar -xvzf bdutil.tar.gz
cd bdutil-*/
</code></pre></li>
<li><p>Create bdutil custom environment variable file. Please refer to <a href="https://cloud.google.com/hadoop/setting-up-a-hadoop-cluster#configuringacluster" rel="nofollow">bdutil configuration documentation</a> about creating correct configuration file, since you need to specify project, number of servers, GCS bucket, machine type, etc... </p></li>
<li><p>Deploy your Hadoop instances (<a href="https://cloud.google.com/hadoop/setting-up-a-hadoop-cluster#deployinstances" rel="nofollow">Full documentation</a>) using <code>datastore_env.sh</code></p>

<pre><code>./bdutil deploy -e YOUR_ENV_FILE.sh,datastore_env.sh
</code></pre></li>
<li><p>Connect to Hadoop Master node</p>

<pre><code>./bdutil shell
</code></pre></li>
<li><p>Now in Master node you can run your MapReduce Job which will have access to DataStore as well.</p></li>
<li><p>Turn down your Hadoop Cluster</p>

<pre><code> ./bdutil delete
</code></pre></li>
</ul>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The Datastore connector connector is indeed deprecated.</p>

<p>To your question "is there an alternative way to create MapReduce tasks from local command line", one option is to use <a href="https://cloud.google.com/dataflow/what-is-google-cloud-dataflow" rel="nofollow">Google Cloud Dataflow</a>. It's not MapReduce per se, but it's the programming model for parallel data processing which has replaced MapReduce at Google. The Dataflow SDK includes <a href="https://cloud.google.com/dataflow/java-sdk/JavaDoc/com/google/cloud/dataflow/sdk/io/DatastoreIO" rel="nofollow">support for Datastore access</a>.</p>

<p>Unlike Hadoop, you don't have to set up a cluster. You just write your code (using the Dataflow SDK) and submit your job from the CLI. The Datastore service will create the needed workers on the fly to process your job and then terminate them.</p>
