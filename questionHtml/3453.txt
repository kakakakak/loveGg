<p>I am using <code>Hadoop2.2</code>. I see that my jobs are completed with success. I can browse the filesystem to find the output. However, when I browse <code>http://NNode:8088/cluster/apps</code>, I am unable to see any applications that have been completed so far ( I ran 3 wordcount jobs, but none of it is seen here).</p>

<p>Are there any configurations that need to be taken into account?</p>

<p>Here is the <code>yarn-site.xml</code></p>

<pre><code>&lt;property&gt;
        &lt;name&gt;yarn.resourcemanager.hostname&lt;/name&gt;

    &lt;/property&gt;

        &lt;name&gt;yarn.nodemanager.aux-services&lt;/name&gt;
        &lt;value&gt;mapreduce_shuffle&lt;/value&gt;
    &lt;/property&gt;
&lt;!--
  &lt;property&gt;
    &lt;name&gt;yarn.nodemanager.aux-services.mapreduce.shuffle.class&lt;/name&gt;

  &lt;/property&gt;
--&gt;
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>After applications are completed, their responsibility might be moved to <code>Job History Server</code>. So check <code>Job History Server</code> URL. It normally listen on port <code>19888</code>. E.g.</p>

<p><code>http://&lt;job_history_server_address&gt;:19888/jobhistory</code></p>

<p>Log directories and log retain durations are configurable in <code>yarn-site.xml</code>. With <code>YARN</code>, even one can aggregate logs to a single (configurable) location.</p>

<p>Sometimes, even though application is listed, logs are not available (I am not sure if its due to some bug in <code>YARN</code>). However, almost each time I was able to get the logs using command line:</p>

<blockquote>
  <p>yarn logs -applicationId the_application_id</p>
</blockquote>

<p>Athough there are multiple options. Use help for details:</p>

<blockquote>
  <p>yarn logs --help</p>
</blockquote>
