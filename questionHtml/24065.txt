<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can easily do this using Hive external tables and partitioning your table by day. For example, create your table as such:</p>

<pre><code>create external table mytable(...) 
partitioned by (day string) 
location '/user/hive/warehouse/mytable';
</code></pre>

<p>This will essentially create an empty table in the metastore and make it point to <code>/user/hive/warehouse/mytable</code>.</p>

<p>Then you can load your data in this directory with the format <strong>key=value</strong> where <strong>key</strong> is your partition name (here "day") and value is the value of your partition. For example:</p>

<pre><code>hadoop fs -put /log/20130115 /user/hive/warehouse/mytable/day=20130115
</code></pre>

<p>Once your data is loaded there, it is in the HDFS directory, but the Hive metastore doesn't know yet that it belongs to the table, so you can add it this way:</p>

<pre><code>alter table mytable add partition(day='20130115');
</code></pre>

<p>And you should be good to go, the metastore will be updated with your new partition, and you can now query your table on this partition.</p>

<p>This should be trivial to script, you can create a cron job running once a day that will do these command in order and find the partition to load with the <code>date</code> command, for example continuously doing this command:</p>

<pre><code>hadoop fs -test /log/`date +%Y%m%d`
</code></pre>

<p>and checking if <code>$?</code> is equal to 0 will tell you if the file is here and if it is, you can transfer it and add the partition as described above.</p>

<p>You can make use of LOAD DATA command provided by Hive. It exactly matches your use case. Specify a directory in your local file system and make Hive tables from it.</p>

<p>Example usage - 
LOAD DATA LOCAL INPATH '/home/user/some-directory'
OVERWRITE INTO TABLE table</p>
