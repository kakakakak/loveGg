<p>I am trying my hands on Hadoop streaming using Python. I have written simple map and reduce scripts by taking help from <a href="http://pages.cs.brandeis.edu/~cs147a/lab/hadoop-example/" rel="nofollow">here</a></p>

<p><code>map</code> script is as follows :</p>

<pre><code>#!/usr/bin/env python

import sys, urllib, re

title_re = re.compile("&lt;title&gt;(.*?)&lt;/title&gt;", re.MULTILINE | re.DOTALL | re.IGNORECASE)

for line in sys.stdin:
    url = line.strip()
    match = title_re.search(urllib.urlopen(url).read())
    if match :
        print url, "\t", match.group(1).strip()
</code></pre>

<p>and <code>reduce</code> script is as follows : </p>

<pre><code>#!/usr/bin/env python

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>this error is just a generic error, that too many Map tasks failed to complete:</p>

<blockquote>
  <h1>of failed Map Tasks exceeded allowed limit</h1>
</blockquote>

<p>you can use the EMR Console to navigate to the logs for the individual Map / Reduce tasks.
Then you should be able to see what the issue is.</p>

<p>In my case - I got this error when I made small mistakes, like setting the path to the Map script incorrectly.</p>

<p>steps to view the logs of the Tasks:</p>

<p><a href="http://antipatterns.blogspot.nl/2013/03/amazon-emr-map-reduce-error-of-failed.html" rel="nofollow">http://antipatterns.blogspot.nl/2013/03/amazon-emr-map-reduce-error-of-failed.html</a></p>
