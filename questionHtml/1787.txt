    <div class="post-text" itemprop="text">

<p>I would like to easily load data in my HBase table. I thought using the <code>ImportTsv</code> tool would be ideal. With something like this :</p>

<pre><code>hbase org.apache.hadoop.hbase.mapreduce.ImportTsv '-Dimporttsv.separator=;' -Dimporttsv.columns=HBASE_ROW_KEY,f:i tab import.tsv
</code></pre>

<p>I want values in the column "f:i" to be stored as <strong>bytes (hex) NOT as strings</strong>. Because direct consequence is that I am unable to query that column with filters needing to make integers comparisons. </p>

<p><em>1 - If I use put in the shell :</em></p>

<pre><code>p = Put.new(Bytes.toBytes('r1'))
p.add(bytes('f'), Bytes.toBytes('i'), Bytes.toBytes(10));
tab.put(p)
</code></pre>

<p><em>I get :</em></p>

<pre><code>r1  column=f:i, timestamp=1398519413393, value=\x00\x00\x00\x00\x00\x00\x00\x0A
</code></pre>

<p><em>2 - If I use the ImportTsv tool I get :</em></p>

<pre><code>r1  column=f:i, timestamp=1398519413393, value=10
</code></pre>

<p>But in this case my scans with the following filter (as an example) won't work anymore :</p>

<pre><code>f = SingleColumnValueFilter.new(
  Bytes.toBytes('f'),
  Bytes.toBytes('i'),
  CompareFilter::CompareOp::LESS_OR_EQUAL,
  BinaryComparator.new(Bytes.toBytes(70))
)
</code></pre>

<p>So basically, is there a simple way to fine tune ImportTsv tool so that it stores the numbers like in the first case ? </p>

<p>Thanks a lot for your help !</p>

    <div class="post-text" itemprop="text">
<p>Tony, no luck.
Importtsv is wrong tool for binary data. Actually it is not good tool at all.</p>

<p>Looks like you need solution similar to what I do:</p>

<ul>
<li>MapReduce job which imports your data and outputs <code>HFile</code> image.</li>
<li><code>completebulkload</code> tool to bulk load prepared <code>HFile</code>.</li>
</ul>

<p>Reference: <a href="https://hbase.apache.org/book/arch.bulk.load.html" rel="nofollow">https://hbase.apache.org/book/arch.bulk.load.html</a></p>

<p>More details:</p>

<ul>
<li>For importing MapReduce job you actually need only mapper. This mapper shall produce sequence of <code>Put</code> objects. Look for importtsv itself.</li>
<li>Rest of importing job is just configured with things like <code>HFileOutputFormat2.configureIncrementalLoad(Job, HTable)</code>.</li>
<li>I recommend to use HFileV2 due to number of reasons starting from luck of HFile V1 support in modern <code>HBase</code> clusters.</li>
<li><code>completebulkload</code> is just ready-to-use tool. I personally have my custom MapReduce job for this stage because I have native things like Snappy in my tables and don't want to install any native things on client. So I just start single mapper which takes HFile image from HDFS and merge with specified table.</li>
</ul>

<p>Looks somewhat complex but indeed it worth to do yourself. Benefit is MUCH more efficient ETL operations.</p>

    <div class="post-text" itemprop="text">
<p>I had the same issue, and ended up writing a small bash script to encode the hexadecimal characters of the tsv files.</p>

<p><em>encode.sh</em></p>

<pre><code>#!/bin/bash
# Transforms hexadecimal characters, e.g. \xFF to binary
# http://stackoverflow.com/questions/10929453/bash-scripting-read-file-line-by-line
while IFS='' read -r line || [[ -n "$line" ]]; do
    echo -e "$line"
done &lt; "$1"
</code></pre>

<p><code>./encode.sh $TABLE.tsv|hadoop fs -put - $HDFS_PATH/$TABLE.tsv</code></p>
