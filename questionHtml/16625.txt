    <div class="post-text" itemprop="text">

<p>By default, when a <a href="http://storm.incubator.apache.org/" rel="nofollow">Storm</a> spout or bolt encounters an exception, it restarts the spout or bolt and tries again.  Is there any configuration option to make it stop the topology, perhaps after N repeated attempts?  (For example, Hadoop tries 4 times before giving up.)</p>

<p>I had a Storm topology run for 77 days with one bolt raising an exception on every tuple.  In situations like that, I'd rather it fail so that I notice that something's wrong.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There is no option for halting the topology (currently). And honestly, killing the whole topology just because an exception is brute force IMHO.  </p>

<p>In your scenario, those exceptions should be handled in the application layer.</p>

<blockquote>
  <p>Is there any configuration option to make it stop the topology, perhaps after N repeated attempts?     </p>
</blockquote>

<p>There is no ready solution for that but you can do that and keep track the retried tuples in the Spout. If a threshold is met, then log the tuple or send it to a messaging queue.</p>

<blockquote>
  <p>I had a Storm topology run for 77 days with one bolt raising an exception on every tuple.</p>
</blockquote>

<p>Then maybe there is a bug in your bolt's code?</p>

<p>One strategy is to send failed tuples to a massage queue or an event bus (such as HornetQ, Apache Kafka, Redis) and having a listener so you will be notified immediately about a poisonous tuple.</p>
