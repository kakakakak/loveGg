<p>For <strong>hadoop-2.6.0</strong> you can download binaries from <a href="http://www.barik.net/archive/2015/01/19/172716/">Titus Barik blog >></a>.</p>

<p>I have not only needed to point <code>HADOOP_HOME</code> to extracted directory <code>[path]</code>, but also provide system property <code>-Djava.library.path=[path]\bin</code> to load native libs (dll).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If we directly take the binary distribution of Apache Hadoop 2.2.0 release and try to run it on Microsoft Windows, then we'll encounter ERROR util.Shell: Failed to locate the winutils binary in the hadoop binary path.</p>

<p>The binary distribution of Apache Hadoop 2.2.0 release does not contain some windows native components (like winutils.exe, hadoop.dll etc). These are required (not optional) to run Hadoop on Windows.</p>

<p>So you need to build windows native binary distribution of hadoop from source codes following "BUILD.txt" file located inside the source distribution of hadoop. You can follow the following posts as well for step by step guide with screen shot</p>

<p><a href="http://www.srccodes.com/p/article/38/build-install-configure-run-apache-hadoop-2.2.0-microsoft-windows-os">Build, Install, Configure and Run Apache Hadoop 2.2.0 in Microsoft Windows OS </a></p>

<p><a href="http://www.srccodes.com/p/article/39/error-util-shell-failed-locate-winutils-binary-hadoop-binary-path">ERROR util.Shell: Failed to locate the winutils binary in the hadoop binary path</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The statement
java.io.IOException: Could not locate executable null\bin\winutils.exe</p>

<p>explains that the null is received when expanding or replacing an Environment Variable. If you see the Source in Shell.Java in Common Package you will find that HADOOP_HOME variable is not getting set and you are receiving null in place of that and hence the error.</p>

<p>So, HADOOP_HOME needs to be set for this properly or the variable hadoop.home.dir property.</p>

<p>Hope this helps.</p>

Kamleshwar.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you face this problem when running a self-contained local application with Spark (i.e., after adding spark-assembly-x.x.x-hadoopx.x.x.jar or the Maven dependency to the project), a simpler solution would be to put winutils.exe (download from <a href="https://github.com/srccodes/hadoop-common-2.2.0-bin/archive/master.zip" rel="nofollow">here</a>) in "C:\winutil\bin". Then you can add winutils.exe to the hadoop home directory by adding the following line to the code: </p>

<p>System.setProperty("hadoop.home.dir", "c:\\winutil\\")</p>

<p>Source: Click <a href="http://nanxiao.me/en/build-apache-spark-application-in-intellij-idea-14-1/" rel="nofollow">here</a></p>
