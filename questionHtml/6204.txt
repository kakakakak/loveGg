<p>Disclaimer: I have never solved a problem like this before.</p>

<p>I can think of a couple of ways to think of your problem:</p>

<ul>
<li>you are trying to cluster each line to a set of clusters
<ul>
<li>check into datamining algorithms</li>
</ul></li>
<li>the diff between each line in a cluster will be small, between lines from two different clusters will be rather bigger</li>
<li>you could quickly gather similar lines with, by comparing the set intersection of two lines: <code>set(line1.split) &amp; set(line2.split)</code> - the element count in the resulting set is an indicator of how close these two lines are.</li>
</ul>

<p>A bit of python code could look like this:</p>

<pre><code>import fileinput

CLUSTER_COUNT = 5
MAX_DISTANCE = 5

def main():
    clusters = [Cluster() for i in range(CLUSTER_COUNT)]
    MAXDISTANCE = 3
    for line in  fileinput.input():
        words = set(line.split())
        cluster = sorted(clusters, key=lambda c: c.distanceTo(words))[0]
        cluster.addLine(words, line)

    # print out results (FIXME: write clusters to separate files)
    for cluster in clusters:
        print "CLUSTER:", cluster.intersection
        for line in cluster.lines:
            print line
        print "-" * 80
        print

class Cluster(object):
    def __init__(self):
        self.intersection = set()
        self.lines = []
    def distanceTo(self, words):
        if len(self.intersection) == 0:
            return MAX_DISTANCE 
        return len(words) - len(self.intersection &amp; words)
    def addLine(self, words, line):
        self.lines.append(line) 
        if len(self.intersection) == 0:
            self.intersection = words
        else:
            self.intersection = self.intersection &amp; words

if __name__ == '__main__':
    main()
</code></pre>

<p>If you run it on your main data, you should end up with a couple of clusters. Note: alter the code to write the clusters to separate files. I think you will want to run the clusters through the code again, recursively, until you find the subsets you're interested in.</p>

    <div class="post-text" itemprop="text">
<p>What I would do here is write a <em>parameter recogniser</em>. This recognises the following sub-strings (when surrounded by whitespace) as parameters:</p>

<ul>
<li>decimal number</li>
<li><em>a.b.c.d</em>, where <em>a,b,c,d</em> are decimal numbers</li>
<li><em>filename</em>.php</li>
</ul>

<p>No doubt there will be more, but the list shouldn't grow too big. Then you replace each  parameter by a place-holder: %d, %url, %phpfile. Now you can just sort the strings.  </p>

<p>You can find unrecognised parameter types by looking through the output strings that occur rarely. For instance, if there is a parameter type <em>h:m:s</em> for a time of day, strings containing this unsubstituted parameter will be unique, or nearly so, and you can find this new parameter type by simply eyeballing a list of the 100 or so 'most nearly unique' strings. Then add <em>h:m:s</em> to your list, replace all such occurrences with %time, and run it again.</p>

<p>Interesting question... Just a spontaneous idea that can be coded quite quickly:</p>

<ul>
<li>Decide for each word whether it's <em>data</em> (11.22.33.55, 3829, somepage.php, etc.) or <em>description</em> (Load, Connection, page, etc.) by counting the frequency of each word in a sample of your data set. <em>Description</em> words presumably occur significantly more often than a particular <em>data</em> word. You'll have to tweak the threshold to find one that partitions the words into the two categories. If this doesn't work for all cases, e.g., because there's an IP address that occurs very often, a manual black list should fix this.</li>
<li>Process each line by computing a signature from the set of its <em>description</em> words only (a string hash of the concatenated <em>description</em> words should work). Count the occurrence of each signature.</li>
</ul>

<p>Performance (very rough estimate): in the 1. phase it suffices to sample your data, in the 2. phase you have to sequentially process each line, so you are within O(n) where n is the number of lines in your data set (use a hash map for O(1) insert in the 1. phase and O(1) test in the 2. phase). Memory consumption depends on the number of distinct words (1. phase) and distinct sentences (2. phase). If you have a large variation of words, the dictionary for counting occurrences in the first phase could become problematic. </p>

<p>In Python, as data structures I would try with the special (performant) dict data type <a href="http://docs.python.org/library/collections.html#collections.Counter" rel="nofollow">Counter</a>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>It is very hard to answer without any assumptions on the input. </p>

<p>My approach would be:</p>

<ol>
<li>Take a look at your input and make rules about the grouping, and implement these rules (possibly with regexes). </li>
<li>Then create a "misc" group for those that didn't match any.</li>
<li>Collect data, and see if there are patterns in that group that you can write regexes for. </li>
<li>Repeat until you have a satisfyingly small "misc" group.</li>
</ol>

<p><strong>Only you can tell (without assumptions)</strong> if you want to group together a particular set of "similar" strings or not. There must be a finite amount of error message types, so this should yield a good enough result after a few iterations.</p>

<p>Another approach might be, with the assumption that the similarity is based on the numbers and IP addresses in the strings, is to search for those, and use these numbers to classify the items. This again, involves being more concrete about your rules, but might spare you the iterated rule-making (if the assumption is actually valid).</p>

<p>One could think of measuring the levenshtein-distance between strings (or using a similar algorithm), but for classification that would require way too many steps.</p>

<p>If it were up to me, I would use python and regular expressions.  Disclaimer: I've never used python on the scale that you're talking about.</p>

<p>Here are the first two items from your sample output, rewritten in regular expression notation:</p>

<ul>
<li>r"Load of page \w+.php by client [.\d]+ failed due to error \d+"</li>
<li>r"Connection to [.\d]+ port \d+ timed out client [.\d]+ source port \d+"</li>
</ul>

<p>Not so bad, right?  You can use these as the "keys" for your top-level buckets.</p>

<p>Then your code for matching a given string (s) to a bucket becomes incredibly simple:</p>

<pre><code>for bucket in buckets:
   if re.match(bucket.regex, s):
      bucket.matchingStrings.append(s)
      break
# else if no buckets match, generate a new bucket/regex for s
</code></pre>

<p>Generating  these regular expressions will be the tricky part.  You need to have rules for choosing parts of the string that need to be generalized out.  In your case, the general parts seem to be numbers, IP addresses and filenames.  You'll have to come up with the right expressions for each scenario, but here's a simple substitution that replaces numbers in the string with the regex pattern representing a number.</p>

<pre><code>pattern = r"\d+"
re.sub(pattern, pattern, "Failed to count from 0 to 600")
# returns r"Failed to count from \d+ to \d+"
</code></pre>

<p>I bet you would get pretty far with the \d+ substitution and nothing more.  </p>

<p>I would do something like this.</p>

<pre><code>map&lt;string, int&gt; mStringToInt;

struct structOneRecord
{   
    vector&lt;int&gt; vWords; 
    vector&lt;vector&lt;int&gt;&gt; vLines; // All the Lines under this record

    vector&lt;int&gt; vIndexOfMutableWords;

    bool operator &lt; (const structOneRecord &amp;n) const
    {
        if(vWords.size() != n.vWords.size())
            return vWords.size() &lt; n.vWords.size();
        else
        {           
            // Count diferences     
            vector&lt;int&gt; vCurrentIndexs;         
            for(int i=0; i&lt;vWords.size(); i++)
            {
                if(vWords[i] != n.vWords[i])                                
                    vCurrentIndexs.push_back(i);                                    
            }

            if(vCurrentIndexs.size() == 0)
                return false;

            int iHalf = vWords.size() / 2; // The diferences can't be bigger than hald the phrase
            if(vCurrentIndexs.size() &lt; iHalf)
            {
                if(vIndexOfMutableWords.size() == 0)
                    return false;
                else
                {
                    if(vIndexOfMutableWords.size() == vCurrentIndexs.size())
                    {
                        for(int i=0; i&lt;vIndexOfMutableWords.size(); i++)
                        {
                            if(vIndexOfMutableWords[i] != vCurrentIndexs[i])
                                vWords[vCurrentIndexs[0]] &lt; n.vWords[vCurrentIndexs[0]]; // Not equal
                        }
                    }
                }
            }


            return  vWords[vCurrentIndexs[0]] &lt; n.vWords[vCurrentIndexs[0]];

        }
    }
};


vector&lt;string&gt; SplitString(const string &amp;strInput, char cDelimiter, bool bSkipEmpty)
{
    vector&lt;string&gt; vRetValue;

    stringstream ss(strInput);  
    string strItem;
    while(std::getline(ss, strItem, cDelimiter))    
    {   
        // Skip Empty
        if(bSkipEmpty &amp;&amp; strItem.size()==0)     
            continue;

        vRetValue.push_back(strItem);
    }

    return vRetValue;
}



void main()
{

// To Test
    vector&lt;string&gt; vInput;
    vInput.push_back("Connection to 11.22.33.44 port 3940 timed out client 1.2.3.4 source port 3940");
    vInput.push_back("Error loading page somepage.php by client 2.3.4.5");

    set&lt;structOneRecord&gt; sRecords;

    for(int i=0; i&lt;vInput.size(); i++)
    {
        vector&lt;string&gt; vWords = CMkDevStringUtilities::SplitString(vInput[i], ' ', true);

        structOneRecord stRecord;
        stRecord.vWords.resize(vWords.size());

        for(int j=0; j&lt;vWords.size(); j++)
        {
            map&lt;string, int&gt;::iterator mIte = mStringToInt.find(vWords[j]);
            if(mIte == mStringToInt.end())

        if(sIte != sRecords.end())

            sIte-&gt;vLines.push_back(stRecord.vWords);
            if(sIte-&gt;vIndexOfMutableWords.size() == 0)

                // Count diferences     
                vector&lt;int&gt; vCurrentIndexs;         
                for(int i=0; i&lt;stRecord.vWords.size(); i++)
                {
                    if(sIte-&gt;vWords[i] != stRecord.vWords[i])                               
                        vCurrentIndexs.push_back(i);                                    
                }

                sIte-&gt;vIndexOfMutableWords = vCurrentIndexs;
            }
        }
        else    
        {
            stRecord.vLines.push_back(stRecord.vWords);
            sRecords.insert(stRecord);          
        }
    }
}
