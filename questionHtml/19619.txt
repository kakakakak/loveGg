<p>I am trying to save thousands of models produced by ML Pipeline. As indicated in the answer <a href="http://stackoverflow.com/questions/32121046/run-3000-random-forest-models-by-group-using-spark-mllib-scala-api">here</a>, the models can be saved using the below code.</p>

<pre><code>import java.io._

def saveModel(name: String, model: PipelineModel) = {
  val oos = new ObjectOutputStream(new FileOutputStream(s"/some/path/$name"))
  oos.writeObject(model)
  oos.close
}

schools.zip(bySchoolArrayModels).foreach{
  case (name, model) =&gt; saveModel(name, Model)
}
</code></pre>

<p>I have tried using "s3://some/path/$name" and "/user/hadoop/some/path/$name". I would like the models to be saved on amazon s3 eventually but they both fails with messages indicating the path cannot be found.</p>

<p>Any advice would be helpful. Thanks!</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>One way to save a model to HDFS is as following:</p>

<pre><code>// persist model to HDFS
sc.parallelize(Seq(model), 1).saveAsObjectFile("hdfs:///user/root/linReg.model")
</code></pre>

<p>Saved model can then be loaded as:</p>

<pre><code>val linRegModel = sc.objectFile[LinearRegressionModel]("linReg.model").first()
</code></pre>

<p>For more details see (<a href="https://phdata.io/exploring-spark-mllib-part-4-exporting-the-model-for-use-outside-of-spark/" rel="nofollow">ref</a>)</p>
