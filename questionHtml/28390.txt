<p>EDIT: For Kepler you need SR2 (4.3.2) and the Java 8 <a href="https://wiki.eclipse.org/JDT/Eclipse_Java_8_Support_For_Kepler">feature patch</a>. Starting with I20140318-0830 all <a href="http://download.eclipse.org/eclipse/downloads/">Luna (4.4) builds</a> contain support for Java 8 "out-of-the-box". The final version of Eclipse Luna (4.4) will be released at the end of June 2014.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p><a href="http://waynebeaton.wordpress.com/about/" rel="nofollow">Wayne Beaton</a> (Director of Open Source Projects at the <a href="http://www.eclipse.org/" rel="nofollow">Eclipse Foundation</a>) just published (February 2014) an article on "<a href="http://waynebeaton.wordpress.com/2014/02/04/building-and-running-java-8-support/" rel="nofollow">Building and Running Java 8 Support</a>" (with Eclipse Luna 4.4Mx).</p>

<p>It includes:</p>

<ul>
<li>Getting the <a href="https://jdk8.java.net/" rel="nofollow">sources of JDK8</a></li>
<li>Downloading the <a href="http://download.eclipse.org/eclipse/downloads/" rel="nofollow">latest milestone of the Eclipse SDK</a> (4.4Mx)</li>
<li>Configuring the “<code>vm</code>” option in the <code>eclipse.ini</code> file to use the Java 8 JRE included with the downloaded JDK. </li>
<li>Adding <a href="http://www.eclipse.org/egit/" rel="nofollow">Egit</a></li>
<li>Clone the <code>BETA_JAVA8</code> branch of the Git repo <a href="https://git.eclipse.org/gitroot/jdt/eclipse.jdt.core.git" rel="nofollow">JDT Core</a>, <a href="https://git.eclipse.org/gitroot/jdt/eclipse.jdt.ui.git" rel="nofollow">JDT UI</a> and <a href="https://git.eclipse.org/gitroot/jdt/eclipse.jdt.debug.git" rel="nofollow">JDT Debug</a> repos.</li>
<li>Using the handy “Import all projects” feature to automatically pull all the code into my workspace.</li>
<li>Setting various JRE <a href="http://www.oracle.com/technetwork/java/javasebusiness/downloads/java-archive-downloads-javase14-419411.html" rel="nofollow">1.4</a>, <a href="http://www.oracle.com/technetwork/java/javasebusiness/downloads/java-archive-downloads-javase5-419410.html" rel="nofollow">5</a> and <a href="http://www.oracle.com/technetwork/java/javasebusiness/downloads/java-archive-downloads-javase6-419409.htmlc" rel="nofollow">6</a>.</li>
<li>Putting the right mapping for those JRE in the <a href="https://wiki.eclipse.org/Execution_Environments" rel="nofollow">execution environment</a>.</li>
</ul>

<hr>

<p>For Eclipse Kepler 4.3+, <a href="http://stackoverflow.com/users/3166303/leeor">leeor</a> mentions <a href="http://stackoverflow.com/questions/17619638/eclipse-kepler-java-8/21624103#comment33727560_21624103">in the comments</a>:</p>

<blockquote>
  <p>As of Feb 21st, you can install the beta 8 JDT via "<strong><code>install new software...</code></strong>" in Eclipse (I use Kepler) here: <a href="http://wiki.eclipse.org/JDT/Eclipse_Java_8_Support_%28BETA%29" rel="nofollow">JDT/Eclipse Java 8 Support (BETA)</a></p>
</blockquote>

<pre><code>http://build.eclipse.org/eclipse/builds/4P/siteDir/updates/4.3-P-builds

Follow the steps.
Go to Help->Eclipse market place
In the search box type "java 8 kepler"
The first link that appears is Java 8 support for Kepler SR2
select it and install it.
It will take a few minutes.
You may need to restart Eclipse after completion.</p>
