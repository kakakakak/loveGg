<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You need to do two things:</p>

<ol>
<li><p>Delete the main hadoop storage directory from every node. This directory is defined by the hadoop.tmp.dir property in your hdfs-site.xml.</p></li>
<li><p>Reformat the namenode:</p></li>
</ol>

<pre class="lang-bsh prettyprint-override"><code>hadoop namenode -format
</code></pre>

<p>If you only do (2), it will only remove the metadata stored by the namenode, but won't get rid of all the temporary storage and datanode blocks.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>So this is what I have had to do in the past.</p>

<p>1. Navigate to your hadoop directory on your NameNode, then stop all the hadoop processes. By running the default stop-all script. This will also stop DFS. e.g.</p>

<pre><code>cd myhadoopdirectory
bin/stop-all.sh
</code></pre>

<p>2. Now On every machine in your cluster (Namenodes, JobTrackers, datanodes etc.) delete all files in your main hadoop storage mine is set to the temp folder in the root folder. Yours can be found in the conf hdfs-site.xml file under hadoop.tmp.dir property e.g.</p>

<pre><code>cd /temp/
rm -r *
</code></pre>

<p>3. Finally go back to your name node, and format it by going to your hadoop directory and running 'bin/hadoop namenode -format' e.g. </p>

<pre><code>cd myhadoopdirectory
bin/hadoop namenode -format
</code></pre>

<p>4. Start up your cluster again by running the following command. It will also startup DFS again.</p>

<pre><code>bin/start-all.sh
</code></pre>

<p>5. And it should work.</p>
