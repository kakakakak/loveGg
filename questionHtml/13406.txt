<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The <code>java.lang.UnsupportedClassVersionError</code> definitely thrown when <code>pio-start-all</code> or other scripts it runs still use <strong>earlier version</strong> of Java than required by <code>predictionIO</code>. You must thoroughly check what Java version is used.</p>

<p>One of the way is to try under <code>root</code>:</p>

<pre><code>alternatives --config java
</code></pre>

<p>You will have something like this</p>

<pre><code>There are 2 choices for the alternative java (providing /usr/bin/java).

Selection    Path                                            Priority   Status
------------------------------------------------------------
* 0            /usr/lib/jvm/java-7-oracle/jre/bin/java          1062      auto mode
  1            /usr/lib/jvm/java-6-openjdk-amd64/jre/bin/java   1061      manual mode
  2            /usr/lib/jvm/java-7-oracle/jre/bin/java          1062      manual mode

Press enter to keep the current choice[*], or type selection number:
</code></pre>

<p>Choose the latest Java version.</p>

<p>You can configure <code>JAVA_HOME</code> as well. Please, <a href="https://access.redhat.com/documentation/en-US/JBoss_Communications_Platform/5.0/html/Platform_Installation_Guide/sect-Configuring_Java.html" rel="nofollow">check the instructions for RedHat</a>.</p>
