<p>I'm new at using Flume and Hadoop so I'm trying to setup the simplest (but somewhat helpful/realistic) example I can. I'm using the <a href="http://hortonworks.com/products/hortonworks-sandbox/" rel="nofollow">HortonWorks Sandbox</a> in a VM client. After following one tutorial 12 (which involves setting up and using Flume) everything seems to be working correctly. </p>

<p>So I setup my own flume.conf that should </p>

<ul>
<li>Read from an apache access log</li>
<li>Use a memory channel</li>
<li>Write to the HDFS</li>
</ul>

<p>Simple enough right? Here's my conf file</p>

<pre><code>agent.sources=exec-source
agent.sinks=hdfs-sink
agent.channels=ch1

agent.sources.exec-source.type=exec
agent.sources.exec-source.command=tail -F /var/log/httpd/access_log

agent.sinks.hdfs-sink.type=hdfs
agent.sinks.hdfs-sink.hdfs.path=/flume/events
agent.sinks.hdfs-sink.hdfs.filePrefix=apacheaccess
agent.sinks.hdfs-sink.hdfs.rollInterval=10
agent.sinks.hdfs-sink.hdfs.rollSize=0

agent.channels.ch1.type=memory
agent.channels.ch1.capacity=1000

agent.sources.exec-source.channels=ch1
agent.sinks.hdfs-sink.channel=ch1
</code></pre>

<p>I've seen several people have problems writing to HDFS, and in most cases it was that there weren't enough logs to fill the HDFS block. However, rollInterval=10 should generate a new file every 10 seconds, as long as at least 1 line is written to it. I can run "tail -F /var/log/httpd/access_log" in another window and see lines being written to the log fairly consistantly, so I don't think it's that.</p>

<pre><code>agent.sources=exec-source
agent.sinks=hdfs-sink
agent.channels=ch1
</code></pre>

<p>so the agent name is 'agent' flume expects that while running the flume-agent you should use the same name as specified in the config file so the command should be</p>

<pre><code>/usr/lib/flume/bin/flume-ng agent -n agent
</code></pre>
