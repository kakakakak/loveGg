<p><p>
hi,I have code written in Hadoop and now I try to migrate to Spark. The mappers and reducers are fairly complex. So I tried to reuse Mapper and Reducer classes of already existing <strong>Hadoop</strong> code inside <strong>spark</strong> program. Can somebody tell me how do I achieve this?</p>

<p><strong>EDIT:</strong>
<br>
So far, I have been able to reuse <strong>mapper</strong> class of standard hadoop word-count example in spark, implemented as below<br><br>
<strong>wordcount.java</strong></p>

<pre><code>import scala.Tuple2; 

import org.apache.spark.SparkConf;
import org.apache.spark.api.java.*;
import org.apache.spark.api.java.function.*;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.conf.*;
import org.apache.hadoop.io.*;
import org.apache.hadoop.mapred.*;
import org.apache.hadoop.util.*;

import java.io.*;
import java.util.Arrays;
import java.util.List;
import java.util.regex.Pattern;

public final class wordcount  extends Configured  implements Serializable {
  public static int main(String[] args) throws Exception{

        SparkConf sparkConf = new SparkConf().setMaster("spark://IMPETUS-I0203:7077").setAppName("wordcount");
        JavaSparkContext ctx = new JavaSparkContext(sparkConf); //created Spark context
        JavaRDD&lt;String&gt; rec = ctx.textFile("hdfs://localhost:54310/input/words.txt"); //Record Reader

  //creating a  Pair RDD whose key=some arbitrary number, value = a Record
        JavaPairRDD&lt;LongWritable,Text&gt; lines =rec.mapToPair(s-&gt;new Tuple2&lt;LongWritable,Text&gt;(new LongWritable(s.length()),new Text(s)));

  //transforming 'lines' RDD to another such that it returns for example ('word',1) tuple.
      JavaPairRDD&lt;Text,IntWritable&gt; ones = lines.flatMapToPair(it -&gt; {

          NotSerializableException notSerializable = new NotSerializableException();
          JobConf conf = new JobConf(new Configuration(), wordcount.class);
          conf.setJobName("WordCount");
          conf.setOutputKeyClass(Text.class);
          conf.setOutputValueClass(IntWritable.class);
          Path inp = new Path("hdfs://localhost:54310/input/darcy.txt");
          FileInputFormat.addInputPath(conf, inp);
          FileOutputFormat.setOutputPath(conf, out);

          WordCountMapper mapper = new WordCountMapper();
        mapper.configure(conf);
        OutputCollector&lt;Text,IntWritable&gt; output = new outputcollector&lt;Text,IntWritable&gt;() ;

            mapper.map(it._1, it._2 , output, Reporter.NULL);

          return  ((outputcollector)output).getList();
        });

        ones.saveAsTextFile("hdfs://localhost:54310/output/41");
        return 0;
  }
}
</code></pre>

<p><br>
<strong>WordCountMapper.java</strong></p>

<pre><code>import java.io.*;
import java.util.StringTokenizer;
import org.apache.hadoop.io.*;
import org.apache.hadoop.mapred.*;
import java.io.Serializable;

public class WordCountMapper extends MapReduceBase implements Mapper&lt;LongWritable, Text, Text, IntWritable&gt;,Serializable
{
      private final static IntWritable one = new IntWritable(1);
      private Text word = new Text();

      public void map(LongWritable key, Text value, OutputCollector&lt;Text, IntWritable&gt; output, Reporter reporter) throws IOException
      {
            NotSerializableException notSerializable = new NotSerializableException();
            String line = value.toString();
            StringTokenizer tokenizer = new StringTokenizer(line);
            while (tokenizer.hasMoreTokens())
            {  
               word = new Text();
               word.set(tokenizer.nextToken());
               output.collect(word, one);
            }
       }
}
</code></pre>

<p><br>
<strong>outputcollector.java</strong></p>

<pre><code>import java.util.ArrayList;
import java.util.List;
import org.apache.hadoop.mapred.*;
import scala.Tuple2;

public class outputcollector&lt;K extends Object, V extends Object&gt; implements OutputCollector&lt;K, V&gt;{
    private List&lt;Tuple2&lt;K,V&gt;&gt; writer = new ArrayList&lt;Tuple2&lt;K,V&gt;&gt;();
    @Override
    public void collect(K key, V value) {
        try{
            writer.add(new Tuple2&lt;K,V&gt;(key,value));
        }catch(Exception e){
            System.out.println(e+ "\n\n****output collector error\n\n");
        }
    }
    public List&lt;Tuple2&lt;K,V&gt;&gt; getList(){
        return writer;
    }
}
</code></pre>

<p><br>
This code works perfectly fine and I can <strong>successfully</strong> submit this spark job. It is somehow highly <strong>inefficient</strong> compared pure spark program. It takes about 50 times longer than a simple spark word-count example. Input file is <strong>1 GB</strong>. Input file exists on <strong>HDFS</strong>. Running in <strong>standalone</strong> mode. <p>
I am unable to find the reason why this code is as <strong>slow</strong> as a sloth. Here, I am using WordCountMapper.java in order to simply collect the <strong>pair(word,1)</strong>. That is also working in memory. So I don't see why my code has to be so slower than standard spark word-count example. <p>
So, can anyone suggest a <strong>better approach</strong> of reusing WordCountMapper.java (hadoop mapper) in spark? or explain the reason why it is so slow? Or anything that helps achieve my ultimate goal? (mentioned in my question at the top).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The basic way of converting a mapreduce to spark is:</p>

<pre><code>rdd.mapPartitions(partition -&gt; 
    setup() //map setup
    partition.map( item =&gt; 
        val output = process(item) 
        if (!partition.hasNext) {
           // Some cleanup code here
        }
    )
).groupByKey().mapPartitions( //similarly for reduce code).saveAsHadoopFile(//params) //To save on HDFS
</code></pre>

<p>Following link points to a set of two articles on cloudera. Not everything is discussed but if you go through it you get the gist of how to convert some parts of hadoop jobs into spark. For example how to do setup and cleanup.</p>

<p><a href="http://blog.cloudera.com/blog/2014/09/how-to-translate-from-mapreduce-to-apache-spark/" rel="nofollow">http://blog.cloudera.com/blog/2014/09/how-to-translate-from-mapreduce-to-apache-spark/</a></p>

<p><strong>Note</strong>: I have tried converting mapreduce to spark but it has resulted in a slower application. Maybe it is my own inefficiency in using scala or maybe spark is not suited for batch jobs. So be aware of this as well.</p>
