<p>I have set <code>JAVA_HOME</code> correctly and when I <code>echo</code> it, I see the correct value:</p>

<pre><code>$ echo $JAVA_HOME
/usr/lib/jvm/java-6-openjdk-amd64/
</code></pre>

<p>I have also put the <code>JAVA_HOME</code> value to the <strong>hadoop-env.sh</strong>.</p>

<pre><code>$ grep JAVA_HOME conf/hadoop-env.sh 
# The only required environment variable is JAVA_HOME.  All others are
# set JAVA_HOME in this file, so that it is correctly defined on
export JAVA_HOME="/usr/lib/jvm/java-6-openjdk-amd64/"
</code></pre>

<p>Despite all of this, hadoop still shows the <code>JAVA_HOME</code> variable totally different. I dont know where it gets this value. Any suggestions? </p>

<pre><code>$ hadoop noname -format
Warning: $HADOOP_HOME is deprecated.

/usr/bin/hadoop: line 320: /usr/lib/jvm/java-6-sun/bin/java: No such file or directory
/usr/bin/hadoop: line 390: /usr/lib/jvm/java-6-sun/bin/java: No such file or directory
</code></pre>

<p>I am pretty sure this is not the ideal solution. but this is what I did:</p>

<p>I put echo statements all throughout <code>/usr/local/hadoop/bin/hadoop</code> to see when <code>JAVA_HOME</code> gets set to a statically (seems static, anyhow) defined path of <code>/usr/lib/jvm/java-6-sun</code>.</p>

<p>I can tell this happens pretty much right away. And I had no idea why this was happening. I installed java in <code>/usr/local/jdk1.7.0_17</code> and correctly set up JAVA_HOME in my path and everything. funny thing, is I dont even have java 6 installed!</p>

<p>so here is the hacky part. I went to where it expected to find java-6-sun, and set a symlink to my java install </p>

<pre><code>cd /usr/lib/jvm
sudo ln -s /usr/local/jdk1.7.0_17/ java-6-sun
</code></pre>

<p>then everything worked. extremely hacky I know, but maybe this post could help someone else.</p>
