<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I had the same problem as yours, and I found out you need to include all jars in your hadoop installation's share/hadoop folder. An example is that you have included <code>share/hadoop/mapreduce/*.jar</code>, but missed out <code>share/hadoop/mapreduce/lib/*.jar</code> one level deeper. Here is my <code>common.loader</code> setting that works (just swap out the hadoop location prefix):</p>

<pre><code>common.loader=${catalina.base}/lib,${catalina.base}/lib/*.jar,${catalina.home}/lib,${catalina.home}/lib/*.jar,${catalina.home}/../lib/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/common/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/common/lib/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/hdfs/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/hdfs/lib/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/mapreduce/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/mapreduce/lib/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/tools/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/tools/lib/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/yarn/*.jar,/localdisk/hadoop-2.4.0/share/hadoop/yarn/lib/*.jar
</code></pre>

<td class="answercell">

<p>I am on HAdoop 2.6.2 and Sqoop 1.99.6, did the below </p>

<pre><code> edit $SQOOP_HOME/server/conf/catalina.properties
 modified common.loader and set the paths to my hadoop and hive dir's  
</code></pre>

<p>(or) a lazy fix (as sudo) -</p>

<pre><code>ln -s $HADOOP_HOME/share/hadoop/common /usr/lib/hadoop 
ln -s $HADOOP_HOME/share/hadoop/hdfs /usr/lib/hadoop-hdfs 
ln -s $HADOOP_HOME/share/hadoop/mapreduce /usr/lib/hadoop-mapreduce 
ln -s $HADOOP_HOME/share/hadoop/yarn /usr/lib/hadoop-yarn 
mkdir -p /usr/lib/hive
chmod 775 /usr/lib/hive
ln -s $HIVE_HOME/lib /usr/lib/hive/lib 
</code></pre>

<p>Also, modified org.apache.sqoop.submission.engine.mapreduce.configuration.directory property in SQOOP_HOME/server/conf/sqoop.properties and set that to a proper hadoop conf dir.</p>

<p>Hope this helps.</p>
