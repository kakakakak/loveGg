<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I believe you can do this with hadoop <code>streaming</code>.</p>

<p>As per <code>tom white</code>, author of <code>hadoop definitive guide, 3rd edition</code>. Page <code># 622</code>, Appendix C.</p>

<p>He used <code>hadoop</code> to execute a bash <code>shell</code> script as a <code>mapper</code>. </p>

<p>In your case you need to use <code>perl script</code> instead of that <code>bash shell script</code>.</p>

<p><strong>Use Case:</strong> He has a <code>lot of small files</code>(one big tar file input), his shell script converts them into <code>few big files</code>(one big tar file output).</p>

<p>He used hadoop to process them in parallel by giving <code>bash shell script</code> as mapper. Therefore this mapper works with input files parallely and produce results.</p>

<p><strong>example hadoop command:</strong>(copy pasted)</p>

<pre><code>hadoop jar $HADOOP_INSTALL/contrib/streaming/hadoop-*-streaming.jar \
-D mapred.reduce.tasks=0 \
-D mapred.map.tasks.speculative.execution=false \
-D mapred.task.timeout=12000000 \
-input ncdc_files.txt \
-inputformat org.apache.hadoop.mapred.lib.NLineInputFormat \
-output output \
-mapper load_ncdc_map.sh \
-file load_ncdc_map.sh
</code></pre>

<p>Replace <code>load_ncdc_map.sh</code> with your <code>xyz.perl</code> in both places(last 2 lines in command).</p>

<p>Replace <code>ncdc_files.txt</code> with another text file which contains the list of your input files to be processed.(5th line from bottom)</p>

<p><strong>Assumptions Taken:</strong> You have a fully functional hadoop cluster running and your perl script is error free.</p>
