<p>What do you think of the answer for Question 4 mentioned in this <a href="http://www.scribd.com/doc/234685897/CCD-410-FREE-Demo-Questions-1" rel="nofollow">site</a> will be ?</p>

<p>Is the answer right or wrong</p>

<p><strong>QUESTION: 4</strong></p>

<pre><code>In the standard word count MapReduce algorithm, why might using a combiner reduce theoverall Job running time?

A. Because combiners perform local aggregation of word counts, thereby allowing the mappers to process input data faster.
B. Because combinersperform local aggregation of word counts, thereby reducing the number of mappers that need to run.
C. Because combiners perform local aggregation of word counts, and then transfer that data toreducers without writing the intermediate data to disk.
D. Because combiners perform local aggregation of word counts, thereby reducing the number of key-value pairs that need to be snuff let across the network to the reducers.

Answer:A
</code></pre>

<p>and </p>

<p><strong>QUESTION: 3</strong></p>

<pre><code>What happens in a MapReduce job when you set the number of reducers to one?

A. A single reducer gathers and processes all the output from all the mappers. The output iswritten in as many separate files as there are mappers.
B. A single reducer gathers andprocesses all the output from all the mappers. The output iswritten to a single file in HDFS.
C. Setting the number of reducers to one creates a processing bottleneck, and since the number of reducers as specified by the programmer is used as a reference value only, the MapReduceruntime provides a default setting for the number of reducers.
D. Setting the number of reducers to one is invalid, and an exception is thrown.
Answer:A
</code></pre>

<p>From my understanding answers to the above questions </p>

<pre><code>Question 4: D
Question 3: B
</code></pre>

<p><strong>UPDATE</strong></p>

<pre><code>You have user profile records in your OLTP database,that you want to join with weblogs you have already ingested into HDFS.How will you obtain these user records?
Options
A. HDFS commands
B. Pig load
C. Sqoop import
D. Hive
Answer:B
</code></pre>

<p>and for updated question my answer I am doubtfull with <strong>B</strong> and <strong>C</strong></p>

<p><strong>EDIT</strong></p>

<p>Right Answer: <strong>Sqoop</strong>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>As far my understanding both the answers are wrong. </p>

<p>I haven't worked much with the <code>Combiner</code> but everywhere I found it to be working on outputs of <code>Mapper</code>. The answer to <strong>Question No 4</strong> should <strong>D</strong>. </p>

<p>Again from practical experience I've found that the number of output files is always equal to the number of <code>Reducer</code>s. So the answer to the <strong>Question No 3</strong> should be <strong>B</strong>. This may not be the case when using <code>MultipleOutputs</code> but that's not common.</p>

<p>Finally I think Apache won't lie about MapReduce ( exceptions do occur :). The answer to the both the questions are available in their <a href="http://wiki.apache.org/hadoop/HadoopMapReduce#Combine" rel="nofollow"><strong>wiki page</strong></a>. have a look.        </p>

<p>By the way, I liked the <em>"100% Pass-Guaranteed or Your Money Back!!!"</em>  quote on the link you provided ;-) </p>

<p><strong>EDIT</strong><br>
Not sure about the question in the update section since I've little knowledge on Pig &amp; Sqoop. But certainly  the same can be achieved using Hive by creating external tables on the HDFS data &amp; then joining.   </p>

<p><strong>UPDATE</strong><br>
After comments from user <strong><em>milk3422</em></strong> &amp; the owner, I did some searching and find out that my assumption of Hive being the answer to the last question is wrong since another OLTP database is involved. The proper answer should be <strong>C</strong> as Sqoop is designed to transfer data between HDFS and relational databases.   </p>
