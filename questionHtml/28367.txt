<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You need to find the logs from your mappers and reducers, since this is the place where the job is failing (as indicated by <code>java.lang.RuntimeException: PipeMapRed.waitOutputThreads(): subprocess failed with code 1</code>). This says that your R script crashed.</p>

<p>If you are using the Hortonworks Hadoop distribuion, the easiest way is to open your jobhistory. It should be at <code>http://127.0.0.1:19888/jobhistory</code> . It should be possible to find the log in the filesystem using the command line as well, but I haven't yet found where.</p>

<ol>
<li>Open <code>http://127.0.0.1:19888/jobhistory</code> in your web browser</li>
<li>Click on the <em>Job ID</em> of the failed job</li>
<li>Click the <em>number</em> indicating the failed job count</li>
<li>Click an <em>attempt</em> link</li>
<li>Click the <em>logs</em> link</li>
</ol>

<p>You should see a page which looks something like</p>

<pre><code>Log Type: stderr
Log Length: 418
Traceback (most recent call last):
  File "/hadoop/yarn/local/usercache/root/appcache/application_1404203309115_0003/container_1404203309115_0003_01_000002/./mapper.py", line 45, in &lt;module&gt;
    mapper()
  File "/hadoop/yarn/local/usercache/root/appcache/application_1404203309115_0003/container_1404203309115_0003_01_000002/./mapper.py", line 37, in mapper
    for record in reader:
_csv.Error: newline inside string
</code></pre>

<p>This is an error from my Python script, the errors from R look a bit different.</p>

<p>source: <a href="http://hortonworks.com/community/forums/topic/map-reduce-job-log-files/" rel="nofollow">http://hortonworks.com/community/forums/topic/map-reduce-job-log-files/</a></p>
