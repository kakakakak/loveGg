<p>I am creating a school management system that handles file uploads and downloads from teachers and students. Furthermore, there will be a large amount of database writes and reads (e.g. which classes a student is taking, what the student's grades are etc.)</p>

<p>I want dynamic web pages updated with python using an AJAX model on the front end.</p>

<p>On the back end I want to use python for file handling, database reads to show user their content, database writes when a user updates his/her content on the web app and <strong>memcache</strong>.</p>

<p>The stack I am using is...</p>

<p>CentOS + Hadoop + Hypertable + Python</p>

<p>I am currently going through the <a href="http://docs.pylonsproject.org/projects/pyramid/1.1/index.html" rel="nofollow">Pyramid</a> docs and after going past a few chapters, I don't see how pyramid helps me accomplish what I can do in PHP, and if it does do this, why so complicated?</p>

<p>Is pyramid the right tool, or should I be using <a href="http://web2py.com/" rel="nofollow">web2py</a>?</p>

<p>If pyramid is the best tool, where can I see some sample code of a complex application built in pyramid. I tend to understand best when reading other's code.</p>

<p><em>Django cannot be used due to inflexibility</em></p>

<blockquote>
  <p>I don't see how pyramid helps me accomplish what I can do in PHP, and if it does do this, why so complicated?</p>
</blockquote>

<ol>
<li><p>A web framework like Pyramid is not a web framework like PHP.  They're different.</p></li>
<li><p>Complexity is a matter of "experience".  If you're experienced with PHP, Python seems complex.  If your experienced with RoR, PHP seems complex.  Everything that's new seems complex.</p></li>
<li><p>Python has a dozen or so web frameworks of varying capabilities.  None of them will look like PHP.  Zero.  They'll all be different (and appear complex).</p></li>
<li><p>Consequently, if you don't like one.  Move on.  There are a lot of choices.  Keep trying different ones.  </p></li>
<li><p>Ask <strong>specific</strong> questions.  "I don't see how pyramid helps me accomplish what I can do in PHP" is too vague to discuss further.  If there's a specific thing you want to know about, search for that <strong>specific</strong> question (it's probably already been asked).   If you don't find anything, ask the <strong>specific</strong> question.  Code samples help.</p></li>
</ol>
