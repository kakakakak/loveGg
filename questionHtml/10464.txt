<p>I don't know, if this type of question is already asked or not. Actually I don't know what to search for.  Am asking at the right place?</p>

<p><strong>Just as an example</strong>, I always wonder how the social media giants like Facebook manages their user settings module... What would be the database design and how they manage to hide the user updates on his friends' timeline if he has chosen not to show his updates on that particular friends timeline. I mean if I had been programming there then I would have loaded all the settings value in an array and there would be many conditional statements to check each and every user setting and accordingly printed data.</p>

<p>But I think this would make that code unmanageable because there would be so many conditions which could lead to undesired results.</p>

<p>So my question is, is there any better approach to do this?</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Facebook's data is maintained in document repository (Nosql) and efficient indexing is used to quickly find the tags and searches. This approach of search and data storage is markedly different from relational database based data storage and search. </p>

<p>Google also uses similar scheme to map the entire web and promptly give you back the result.</p>

<p>So in simpler terms you data is stored and indexed the way Google indexes messages, only difference is, the data is also lying with Facebook.</p>

<p>The related technologies are <a href="http://en.wikipedia.org/wiki/Big_data" rel="nofollow">Bigdata</a>, <a href="http://en.wikipedia.org/wiki/MongoDB" rel="nofollow">Mongodb</a>, <a href="http://en.wikipedia.org/wiki/Apache_Hadoop" rel="nofollow">Apache Hadoop</a>. And one of the leading index management and search algorithm is <a href="http://en.wikipedia.org/wiki/Lucene" rel="nofollow">Lucene</a>. <a href="http://en.wikipedia.org/wiki/Elasticsearch" rel="nofollow">Apache Elasticsearch</a> is an user friendly package around Lucene. </p>

<p>So facebook treats these security critaria like tag (in simple language) and does google like search and presents you in a pleasing frontend, not sounding like a search engine.</p>

<p><strong>While setting up your system, you can use elasticsearch to have faster search. Elasticsearch is makes implementation of lucene easier. It definitely will have some learning curve. Elasticsearch can also be used along with <a href="http://en.wikipedia.org/wiki/Relational_database_management_system" rel="nofollow">rdbms</a>, in this case your data is saved in database but indexes also maintained to faster search. Definitely the cost would be disk-space. It makes it possible to have many criteria but still being able to get result quicker.</strong></p>

<p>A quick <a href="http://www.elasticsearchtutorial.com/elasticsearch-in-5-minutes.html" rel="nofollow">tutorial</a> on elasticsearch.</p>

<p>Regarding facebook's db schema's and how it works and why its a good design, here are some articles that would explain to you why:</p>

<p><a href="https://www.facebook.com/notes/facebook-engineering/tao-the-power-of-the-graph/10151525983993920" rel="nofollow">The power of the graph</a></p>

<p>This is posted by facebook and it explains how they are managing data. They use <a href="https://www.usenix.org/conference/atc13/technical-sessions/presentation/bronson" rel="nofollow">TAO data model</a> and through the application of <a href="http://en.wikipedia.org/wiki/Graph_theory" rel="nofollow">graph theory</a> and other complicated algorithms and advanced memoray caching and data handling, they can efficiently manage lots of user data..</p>

<p>but regarding to your question: <br/><em>What would be the database design and how they manage to hide the user updates on his friends' timeline if he has chosen not to show his updates on that particular friends timeline?</em></p>

<ol>
<li><p>I think this post would give you some insights on what kind of db structure facebook has and what would be the functionality of it for every user: <a href="http://www.codedodle.com/2014/12/social-network-friends-database.html" rel="nofollow">Social Network Friends Relationship Database Design</a></p></li>
<li><p>Usually, the hiding of user updates on your friends' timeline if you have not shown your update to that particular friend is managed by storing values in the database.. you can create a <code>view_type</code> table in db and that would determine what kind of view the user can see, then issue a <code>where</code> condition in your sqls based on the view the user has selected.. there are still many ways to handle this and a <strong>good database structure is needed</strong> for this and of course planning for a good and efficient database is a very important and strict procedure..</p></li>
</ol>
