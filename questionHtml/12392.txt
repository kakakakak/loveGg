<p>How can I fixed this?</p>

<pre><code>/tmp/hadoop-yarn/staging/ubuntu/.staging/job_1450038005671_0025/job.jar: Under replicated BP-938294433-10.0.1.190-1450037861153:blk_1073744219_3398. Target Replicas is 10 but found 3 replica(s).
</code></pre>

<p>I get this when I run <code>hadoop fsck /</code> in my master node.
I assume I should change a <code>.xml</code> file in conf or something similar I just don't know which file to change.</p>

<p>Note that <code>dfs.replication</code> in <code>hdfs-site.xml</code> is already set to 3. I don't have <code>dfs.replication.max</code> in my <code>hdfs-site.xml</code> file.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The replication count for files submitted as part of your job (jars, etc) is controlled by the parameter <code>mapreduce.client.submit.file.replication</code> (or <code>mapred.submit.replication</code> in pre 2.4 clusters) in mapred-site.xml. You can adjust this down for clusters that are smaller than 10 nodes, or just ignore the message from fsck.</p>

<p>FWIW, there is a <a href="https://issues.apache.org/jira/browse/MAPREDUCE-2845" rel="nofollow">JIRA</a> for this but I doubt it will ever get worked.</p>

<p>HDFS configuration file <code>hdfs-site.xml</code> should contain <code>dfs.replication</code> property which describes block replication factor:</p>

<pre><code>&lt;configuration&gt;
  &lt;property&gt;
    &lt;name&gt;dfs.replication&lt;/name&gt;
    &lt;value&gt;3&lt;/value&gt;
  &lt;/property&gt;
&lt;/configuration&gt;
</code></pre>

<p>Default <code>hdfs-site.xml</code> location is <code>/etc/hadoop/hdfs-site.xml</code> </p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>You can ignore. /tmp/hadoop-yarn/staging/ubuntu/.staging/job_1450038005671_0025/job.jar, it is a job resource. dfs.replication does not have impact on job resources.</p>

<ol>
<li>Job resources such as jar files, files passed using -files (distributed cache) will be copied to HDFS using 10 as replication factor</li>
<li>When the job is running, these job resources (code) will be copied to the container/task to process the data</li>
<li>Once the job is completed based up on thresholds these resources will be automatically recycled.</li>
</ol>

<p>This feature helps in implementing data locality (where code goes to data) while processing the data.</p>
