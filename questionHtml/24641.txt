<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Yes it looks like you need to use a different version of Hadoop (or build the latest Mahout from source) if you want this to work. You got a <em>NoSuchMethodError</em> so the first thing to do is check if the <em>ProgramDriver</em> is in the distribution of hadoop you're using.</p>

<p>Looking at the API docs for the various version you can see that its in v0.0.20.x but had been removed from the newer versions. </p>

<ul>
<li><a href="http://hadoop.apache.org/common/docs/r0.20.205.0/api/index.html" rel="nofollow">http://hadoop.apache.org/common/docs/r0.20.205.0/api/index.html</a></li>
<li><a href="http://hadoop.apache.org/common/docs/r2.0.0-alpha/api/index.html" rel="nofollow">http://hadoop.apache.org/common/docs/r2.0.0-alpha/api/index.html</a> (your version)</li>
</ul>

<p>Looking at the JIRA for Mahout you can see a bug was submitted for a similar problem to this on July 11th and has been fixed in version 0.8.</p>

<ul>
<li>[MAHOUT-1044] <a href="https://issues.apache.org/jira/browse/MAHOUT-1044" rel="nofollow">https://issues.apache.org/jira/browse/MAHOUT-1044</a></li>
</ul>

<p><strong>Update:</strong></p>

<p>Shouldn't your command have <strong>jar</strong> after the hadoop command? Something like:</p>

<blockquote>
  <p>$ hadoop jar /home/hadoop/mahout/core/target/mahout-core-0.7-job.jar
  etc</p>
</blockquote>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>@BinaryNerd is right. There is a bug in Mahout as detailed in:</p>

<ul>
<li>[MAHOUT-1044] <a href="https://issues.apache.org/jira/browse/MAHOUT-1044" rel="nofollow">https://issues.apache.org/jira/browse/MAHOUT-1044</a></li>
</ul>

<p>Mahout 0.7 command line gives the NoClassDef ProgramDriver error as detailed in the first part of your question. This will be fixed in 0.8 or you can edit your bin/mahout as detailed in the bug report to fix the brace placement.</p>

<p>I ran into the same issue using mahout under Cloudera CDH 4.1.2. Changing the brace in my bin/mahout fixed it.</p>
