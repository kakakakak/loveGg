<p>I have  Hadoop installed in this location</p>

<blockquote>
  <p>/usr/local/hadoop$</p>
</blockquote>

<blockquote>
  <p>hduser@ubuntu:/usr/local/hadoop$ bin/hadoop dfs -ls</p>
</blockquote>

<p>This gave me the files in the dfs</p>

<pre><code>Found 3 items
drwxr-xr-x   - hduser supergroup          0 2014-03-20 03:53 /user/hduser/gutenberg
drwxr-xr-x   - hduser supergroup          0 2014-03-24 22:34 /user/hduser/mytext-output
-rw-r--r--   1 hduser supergroup        126 2014-03-24 22:30 /user/hduser/text.txt

<p>Next time, I tried the same command in a different manner </p>

<blockquote>
  <p>hduser@ubuntu:/usr/local/hadoop$ hadoop dfs -ls</p>
</blockquote>

<p>It also gave me the same result.</p>

<p>Could some one please explain why both are working despite of executing the ls command from different folders. I hope you guys understood my question.Just explain me  difference between these two :</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>In unix an executable file can be executed in two ways, either by giving the absolute/relative path or commands in system executables path(path should be specified in PATH variable)</p>

<p>When you execute <code>bin/hadoop dfs -ls</code> should be inside the directory /usr/local/hadoop. Or <code>/usr/local/hadoop/bin/hadoop dfs -ls</code> will also work</p>

<p>There is one environment variable PATH in unix which keeps in the list of executable location by default it keeps the following path <code>/usr/local/bin:/bin:/usr/bin:/usr/local/sbin:/usr/sbin:/sbin:</code> . Whenever we execute any command like ls, mkdir etc it is taking from the one location in  PATH variable. When you give the command hadoop(it will be taken from the path /usr/local/hadoop/bin/). Since you have specified the path /usr/local/hadoop/bin/ in PATH variable. Use the following command to check the value of your PATH variable</p>

<pre><code>echo $PATH
</code></pre>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There is one simple concept in LINUX Operating System. You can say <strong>bashrc</strong> file is linked to the terminal. So when you open the terminal then it loads all the variable from the <strong>bashrc</strong> file. Any executable appended in the path variable defined in the bashrc file can be loaded from any directory you presently are. </p>

<p>Now to answer your question. Since <code>hadoop fs -ls /</code> was working fine for you so that means the executable is in your <strong>PATH</strong> variable. and for the latter case <code>bin/hadoop fs -ls /</code> you are manullay going to the folder where the executable is present. This is similar to running any executable in LINUX.</p>
