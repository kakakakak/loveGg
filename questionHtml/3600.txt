<p>You can assign more memory by editing the conf/mapred-site.xml file and adding the property:</p>

<pre><code>  &lt;property&gt;
    &lt;name&gt;mapred.child.java.opts&lt;/name&gt;
    &lt;value&gt;-Xmx1024m&lt;/value&gt;
  &lt;/property&gt;
</code></pre>

<p>This will start the hadoop JVMs with more heap space.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>For anyone using RPM or DEB packages, the documentation and common advice is misleading. These packages install hadoop configuration files into /etc/hadoop. These will take priority over other settings.</p>

<p>The /etc/hadoop/hadoop-env.sh sets the maximum java heap memory for Hadoop, by Default it is:<br></p>

<pre>   export HADOOP_CLIENT_OPTS="-Xmx128m $HADOOP_CLIENT_OPTS"</pre>

<p>This Xmx setting is too low, simply change it to this and rerun<br></p>

<pre>   export HADOOP_CLIENT_OPTS="-<b>Xmx2048m</b> $HADOOP_CLIENT_OPTS"</pre>

<p>I installed hadoop 1.0.4 from the binary tar and had the out of memory problem. I tried Tudor's, Zach Garner's, Nishant Nagwani's and Andris Birkmanis's solutions but none of them worked for me.  </p>

<p>Editing the bin/hadoop to ignore $HADOOP_CLIENT_OPTS worked for me:</p>

<pre><code>...
elif [ "$COMMAND" = "jar" ] ; then
     CLASS=org.apache.hadoop.util.RunJar
    #Line changed this line to avoid out of memory error:
    #HADOOP_OPTS="$HADOOP_OPTS $HADOOP_CLIENT_OPTS"
    # changed to:
     HADOOP_OPTS="$HADOOP_OPTS "
...
</code></pre>

<p>I'm assuming that there is a better way to do this but I could not find it.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>On Ubuntu using DEB install (at least for Hadoop 1.2.1) there is a <code>/etc/profile.d/hadoop-env.sh</code> symlink created to <code>/etc/hadoop/hadoop-env.sh</code> which causes it to load every time you log in.  In my experience this is not necessary as the <code>/usr/bin/hadoop</code> wrapper itself will eventually call it (through <code>/usr/libexec/hadoop-config.sh</code>).  On my system I've removed the symlink and I no longer get weird issues when changing the value for <code>-Xmx</code> in <code>HADOOP_CLIENT_OPTIONS</code> (because every time that <code>hadoop-env.sh</code> script is run, the client options environment variable is updated, though keeping the old value)</p>
