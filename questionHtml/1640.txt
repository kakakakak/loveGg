<td class="answercell">
    <div class="post-text" itemprop="text">
<p>After some days struggling with it, I finally came up with a solution. I had to add the corresponding <code>hadoop-client</code> dependency to avoid this strange exception.</p>

<p>After that, some other already <a href="http://stackoverflow.com/questions/22129321/connection-refused-when-running-sparkpi-locally?rq=1">reported errors</a> appeared. The solution to the connection refused problem was:</p>

<ol>
<li>Change <code>sbin/start-master.sh</code> and/or <code>sbin/start-slaves.sh</code> and set <code>$SPARK_MASTER_IP</code> to <code>hostname -f</code> instead of <code>hostname</code>. It seems Akka only works with fully qualified names and not with hostnames or ip addresses.</li>
<li>Set also <code>$SPARK_MASTER_IP</code> in <code>conf/spark-env.sh</code> to <code>hostname -f</code> so that cluster workers can reach the master.</li>
<li>Be sure that <code>conf/slaves</code> also uses fully qualified domain names and not hostnames/ip addresses.</li>
</ol>

<p>After these changes, everything just worked.</p>

<p>Hope it helps someone else!</p>
