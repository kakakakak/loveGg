<p>I'm trying to practice some data mining algorithms using hadoop. </p>

<p><strong>Use Apache mahout which runs on top of Hadoop.</strong>
[<a href="http://mahout.apache.org/][1]" rel="nofollow">http://mahout.apache.org/][1]</a></p>

<p>Can I do this with HDFS alone, or do I need to use the sub-projects like hive/hbase/pig?</p>

<p><strong>HDFS is a file system of hadoop. It stands for Hadoop distributed file system. No matter what tool you are going to use in Hadoop stack, they should process the data which is in the distributed environment. So, you can't do anything just with HDFS. You need any of the computation technicques/tools like Map Reduce, Pig, Hive and etc.</strong></p>

<p>Hope this helps!</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Hadoop is a tool for Distributed/parallel data processing. Mahout is a data mining/ machine learning framework that can work standalone mode as well as in Hadoop distribution environment. The decision to use it as standalone or with Hadoop boils down to the size of the historical data that needs to be mined. If the data size is of the order of Terabytes and Petabytes, you typically use Mahout with Hadoop.</p>

<p>Mahout supports 3 machine Learning algorithms, recommendation, clustering and classification. <strong>Mahout in action</strong> book by manning does a very good job of explaining this.   Weka is another similar open source projects. All these come under a category called machine learning frameworks. </p>

<p>Refer to the blog which talks about a use case about how <a href="http://wp.me/p2oZRH-41" rel="nofollow">Mahout and Hadoop distributed file system works</a>? As a precursor to this, there is also a blog on <a href="http://wp.me/p2oZRH-3P" rel="nofollow">Component architecture</a> of how each of these tools fit together for a data mining problem in Hadoop /Mahout ecosystem.</p>

<p>I hope it help.</p>

    <div class="post-text" itemprop="text">
<p>You have to use different tools in Hadoop ecosystem depending on their strengths. </p>

<p><strong><em>HIVE/HBASE</em></strong> are good to handle structured data</p>

<p><strong><em>SQOOP</em></strong> is used to import structured data from traditional RDBMS database Oracle, SQL Server etc.</p>

<p><strong><em>FLUME</em></strong> is used for processing Un-structured data. </p>

<p>You can use Content Management System to process Un-structured data &amp; Semi-Structured data - Tera Or Peta bytes of data. If you are storing un-structured data, I prefer to store the data in CMS and use meta data information in <strong><em>NoSQL</em></strong> database like <strong><em>HBASE</em></strong> (e.g. <code>Image Id , MD5SUM of image</code>)</p>

<p>To process Big data streaming, you can use <strong>PIG</strong>.</p>

<p><strong><em>Spark™:</em></strong> A fast and general compute engine for Hadoop data. Spark provides a simple and expressive programming model that supports a wide range of applications, including ETL, machine learning, stream processing, and graph computation</p>

<p>Have a look at <a href="http://www.dummies.com/how-to/content/structured-data-storage-and-processing-in-hadoop.html" rel="nofollow">Structured Data</a> and <a href="http://www.dummies.com/how-to/content/unstructured-data-in-a-big-data-environment.html" rel="nofollow">Un-Structured data</a> handling in Hadoop</p>

<p>Have a look at complete hadoop <a href="https://hadoopecosystemtable.github.io/" rel="nofollow">ecosystem</a> and this <a href="http://stackoverflow.com/questions/13911501/when-to-use-hadoop-hbase-hive-and-pig/33433532#33433532">SE question</a></p>
