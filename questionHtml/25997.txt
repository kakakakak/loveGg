    <div class="post-text" itemprop="text">
<p>Hadoop is a distributed computing framework. It is a <strong>De facto</strong> standard for <strong>Data Management</strong> (Distributed Storage + Distributed Processing). So Hadoop is a technology for all who involves in Data Management Life Cycle(Capturing, Storage, Processing, and Reporting). Hadoop is used by the following roles:</p>

<ul>
<li>Admin</li>
<li>Developer</li>
<li>Data Analyst</li>
<li>Data Scientist</li>
<li>Business Analyst</li>
<li>Functional Consultant</li>
<li>etc...</li>
</ul>

<p>Though Hadoop and most of it's Eco System is written in Java, but it is used by all kinds of People in the Enterprise. So we need several interfaces to target all the audience and to increase the adaptability.</p>

<p><strong>Hadoop Project Management Committee</strong> initiated several projects to support non java programmers, non programmers, SQL programmers etc...</p>

<p>The following are utilities and projects to support all varieties of Audience:</p>

<p><strong>Hadoop Streaming:</strong> It is an utility offered by Hadoop to allow non Java programers to write MapReduce programs using other languages like Perl, PHP, Python, Shell, R, C, C++, Scala, Groovy, Ruby etc..</p>

<p>Hadoop Streaming = Hadoop + Console (STDOUT/STDIN) + External Programs.</p>

<p>Hadoop streaming is a bit slow compared to Native Java MapReduce, but is useful to integrate the legacy code which is written in non java and it is also good for integrating Data Science tool kits like R and Python with Hadoop.</p>

<p>There are several projects which are developed based on Hadoop Streaming:</p>

<ul>
<li>RHadoop : R and Hadoop</li>
<li>Dumbo   : Python + Hadoop</li>
<li>mrjob   : Python + Hadoop</li>
</ul>

<p><strong>Hadoop Pipes:</strong> It is an utility offered by Hadoop to allow non Java programers to write MapReduce programs using C++.</p>

<p><strong>Pydoop:</strong> It is a Python Module to write MapReduce programs using Python. It internally uses Hadoop Pipes. So It is Python wrapper over Hadoop Pipes.</p>

<p><strong>Pig:</strong> It offers a scripting like language called Pig Latin to analyse your data by performing a series of transformations and aggregations. Pig Latin is easy to learn and it is a data flow language. It is a right tool for people who do not have any programming background.</p>

<p><strong>Hive/Impala/Drill/Tajo/Presto/MRQL:</strong> All these are Distributed SQL Engines over Hadoop. These offer a SQL like Query language to run ad hoc queries, data summarisation. It is good choice for SQL programmers, Database analysts, Data Warehouse programmers.</p>

<p>All the above projects and utilities allow non java programmers to write their data analysis using their choice of language. Hadoop with Java has great advantage and full control over data in terms of Key Value pairs.</p>

<p>The conclusion here is, we can do data analysis with Hadoop without Java programming. </p>

<p><strong>Is Java necessary for Hadoop?</strong></p>

<p>Hadoop is built in Java but to work on Hadoop you didn't require Java.
It is preferred if you know Java, then you can code on mapreduce. If you are not familiar with Java. You can focus your skills on Pig and Hive to perform the same functionality. They are similar tools like SQL with different way to writing syntax. If you are coding Background like python, C++ etc. You can convert there codes into Java using libraries like Jython for Python. </p>

<p><strong>How much Java is necessary for Hadoop?</strong></p>

<p>If you want to learn Java only for Hadoop then here are list of topic you must learn:
1. Basic of Core Java
- Variables, Classes, Functions, Inheritance, Package, Handling Error, Flow Controls, Arrays, API.
- Some useful string function   //Useful for data filter
- Collections and Generic - ArrayList, Hashtable, etc. </p>

<p>To get more detail about what are topics you need to learn in Java, check this article: 
<a href="https://www.dezyre.com/article/-how-much-java-is-required-to-learn-hadoop/103" rel="nofollow">https://www.dezyre.com/article/-how-much-java-is-required-to-learn-hadoop/103</a></p>

<blockquote>
  <p><strong><em>What is exact use of hadoop?</em></strong></p>
</blockquote>

<p>The Apache Hadoop software library is a framework that allows for the distributed storage and processing of large data sets across clusters of computers using simple programming models. Refer to documentation at <a href="https://hadoop.apache.org/" rel="nofollow">Apache-Hadoop</a></p>

<p>Hadoop provides highly scalable, cost-effective,fast,flexible and resilient data storage &amp; Analytics platform. </p>

<blockquote>
  <p><strong><em>Is it necessary to know java for learning Hadoop?</em></strong></p>
</blockquote>

<p>Not required. But you are looking at optimization of your Map Reduce jobs, java provides that flexibility.</p>

<p>Alternatives if you are not interested in java</p>

<p><strong>PIG</strong>: A high-level data-flow language and execution framework for parallel computation. It is recommended for people, who are experts in scripting languages like Python.</p>

<p><strong>HIVE</strong>: A data warehouse infrastructure that provides data summarization and ad- hoc querying. It is recommended for people, who are experts in SQL programming as syntax is similar to SQL</p>

<p><code>PIG</code> and <code>HIVE</code> hides complexity of Map Reduce Jobs for developers. They provide higher level of abstraction to provide solution to business problems.</p>

<p><code>PIG</code> and <code>HIVE</code>, both translates scripts/queries to a series of MAP reduce jobs. On the performance front, they are not efficient compared to traditional MAP reduce jobs implemented in java. </p>

<p>Refer to this article for <a href="http://www.tomsitpro.com/articles/mapreduce-hadoop-Pig_Latin_programming-streaming_mapreduce-cascading_project,2-361.html" rel="nofollow">Java Alternatives</a></p>

<p><strong><em>On job front</em></strong>, it depends your expertise and your selection of eco-system in Hadoop framework. It is difficult to answer.</p>

<p><strong><em>Before Hadoop</em></strong>, we do not have best framework/platform, which can provide same advantages of Hadoop with BigData.  To implement Word count type of programs, you have to write scripts, execute and consolidate the results from datanodes. </p>

<p>You can see comparison between GFS and HDFS at <a href="http://www.warse.org/pdfs/2014/icetetssp106.pdf" rel="nofollow"><strong><em>GFS Vs HDFS</em></strong></a>. Get good insight into HDFS at <a href="http://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HdfsDesign.html" rel="nofollow"><strong><em>HDFS design</em></strong></a></p>
