<p>Environment: ubuntu 14.04, hadoop 2.6</p>

<p>After I type the <code>start-all.sh</code> and <code>jps</code>, <code>DataNode</code> doesn't list on the terminal </p>

<pre><code>&gt;jps
9529 ResourceManager
9652 NodeManager
9060 NameNode
10108 Jps
9384 SecondaryNameNode
</code></pre>

<p>according to this answer : <a href="http://stackoverflow.com/questions/11889261/datanode-process-not-running-in-hadoop">Datanode process not running in Hadoop</a></p>

<p>I try its best solution</p>

<ul>
<li><code>bin/stop-all.sh (or stop-dfs.sh and stop-yarn.sh in the 2.x serie)</code></li>
<li><code>rm -Rf /app/tmp/hadoop-your-username/*</code></li>
<li><code>bin/hadoop namenode -format (or hdfs in the 2.x series)</code></li>
</ul>

<p>However, now I get this:</p>

<pre><code>&gt;jps
20369 ResourceManager
26032 Jps
20204 SecondaryNameNode
20710 NodeManager
</code></pre>

<p>As you can see, even the <code>NameNode</code> is missing, please help me.</p>

    <div class="post-text" itemprop="text">
<blockquote>
  <p>FATAL org.apache.hadoop.hdfs.server.datanode.DataNode: Exception in secureMain
  java.io.IOException: All directories in dfs.datanode.data.dir are invalid: <strong>"/usr/local/hadoop_store/hdfs/datanode/"</strong></p>
</blockquote>

<p>This error may be due to wrong permissions for <code>/usr/local/hadoop_store/hdfs/datanode/</code> folder.</p>

<blockquote>
  <p>FATAL org.apache.hadoop.hdfs.server.namenode.NameNode: Failed to start namenode.
  org.apache.hadoop.hdfs.server.common.InconsistentFSStateException: Directory <strong>/usr/local/hadoop_store/hdfs/namenode</strong> is in an inconsistent state: storage directory does not exist or is not accessible.</p>
</blockquote>

<p>This error may be due to wrong permissions for <code>/usr/local/hadoop_store/hdfs/namenode</code> folder or it does not exist. To rectify this problem follow these options:</p>

<p><strong>OPTION I:</strong></p>

<p>If you don't have the folder <code>/usr/local/hadoop_store/hdfs</code>, then create and give permission to the folder as follows:</p>

<pre><code>sudo mkdir /usr/local/hadoop_store/hdfs
sudo chown -R hadoopuser:hadoopgroup /usr/local/hadoop_store/hdfs
sudo chmod -R 755 /usr/local/hadoop_store/hdfs
</code></pre>

<p>Change <code>hadoopuser</code> and <code>hadoopgroup</code> to your hadoop username and hadoop groupname respectively. Now, try to start the hadoop processes. If the problem still persists, try option 2.</p>

<p><strong>OPTION II:</strong></p>

<p>Remove the contents of <code>/usr/local/hadoop_store/hdfs</code> folder:</p>

<pre><code>sudo rm -r /usr/local/hadoop_store/hdfs/*
</code></pre>

<p>Change folder permission:</p>

<pre><code>sudo chmod -R 755 /usr/local/hadoop_store/hdfs
</code></pre>

<p>Now, start the hadoop processes. It should work.</p>

<blockquote>
  <p><strong>NOTE: Post the new logs if error persists.</strong></p>
</blockquote>

<p><strong>UPDATE:</strong></p>

<p>In case you haven't created the hadoop user and group, do it as follows:</p>

<pre><code>sudo addgroup hadoop
sudo adduser --ingroup hadoop hadoop
</code></pre>

<p>Now, change ownership of <code>/usr/local/hadoop</code> and <code>/usr/local/hadoop_store</code>:</p>

<pre><code>sudo chown -R hadoop:hadoop /usr/local/hadoop
sudo chown -R hadoop:hadoop /usr/local/hadoop_store
</code></pre>

<p>Change your user to hadoop:</p>

<pre><code>su - hadoop
</code></pre>

<p>Enter your hadoop user password. Now your terminal should be like:</p>

<p><code>hadoop@ubuntu:$</code></p>

<p>Now, type:</p>

<p><code>$HADOOP_HOME/bin/start-all.sh</code></p>

<p>or</p>

<p><code>sh /usr/local/hadoop/bin/start-all.sh</code></p>
