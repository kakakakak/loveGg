<pre><code>ADD JAR wasb:///HdiSamples/StorageAnalytics/hive-serde-microsoft-wa-0.13.0.jar;

ALTER TABLE AggregateStorageLogs3 ADD IF NOT EXISTS PARTITION(StorageAccount = 'mystorageacc2')
LOCATION 'wasb://$logs@mystorageacc2.blob.core.windows.net/blob/';

ALTER TABLE AggregateStorageLogs3 ADD IF NOT EXISTS PARTITION(StorageAccount = 'mystorageacc3')
LOCATION 'wasb://$logs@mystorageacc3.blob.core.windows.net/blob/';
</code></pre>

<p>Then I tried to count the records in the external table to find the total number of log entries across all storage accounts like this:</p>

<pre><code>SET hive.mapred.supports.subdirectories= true; 
SET mapred.input.dir.recursive=true; 
SET hive.exec.dynamic.partition = true;
SET hive.exec.dynamic.partition.mode=nonstrict;
ADD JAR wasb:///HdiSamples/StorageAnalytics/hive-serde-microsoft-wa-0.13.0.jar;

SELECT COUNT(*)
FROM AggregateStorageLogs3  
</code></pre>

<p>However, the query fails with a large stack trace after several hours. I am very much a newbie with Hadoop so the stack trace means nothing to me. It contains a Log4J error, but I'm not sure if that is the cause of the overall job failure and in any case, I can't tell what is causing that error. I'm guessing I did something wrong with the data load, because the same query works when I do it on a single storage account without the partitions.</p>

<p>Please can anybody tell me what I'm doing wrong?</p>
