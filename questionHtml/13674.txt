    <div class="post-text" itemprop="text">

<p>I am currently working in a few projects with <a href="http://www.mongodb.org/">MongoDB</a> and <a href="http://cassandra.apache.org/">Apache Cassandra</a> respectively. I am also using Solr a lot and I am handling "lots" of data with them (approx. 1-2TB). I've heard of <a href="http://www.greenplum.com/">Greenplum</a> and <a href="http://www.vertica.com/">Vertica</a> the first time in the last week and I am not really sure, where to put them in my brain. They seem to me like Dataware House (DWH) solutions and I haven't really worked DWH. And they seem to cost lots of money (e.g. $60k for 1TB storage in Greenplum). I am currently not handling Petabyte of data and won't do so I think, but products like cassandra seem also to be able to handle this</p>

<blockquote>
  <p>Cassandra is the acknowledged NoSQL leader when it comes to
  comfortably scaling to terabytes or petabytes of data.</p>
  
  <p>via <a href="http://www.datastax.com/why-cassandra">http://www.datastax.com/why-cassandra</a></p>
</blockquote>

<p>So my question: Why should people use Greenplum &amp; Co? Is there a huge advantage in comparison to these other products? </p>

<p>Thanks.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Cassandra, Greenplum and Vertica all handle huge amounts of data but in very different ways. </p>

<p>Some made up usecases where each database has its strengths:</p>

<p>Use cassandra for:</p>

<pre><code>tweets.insert(key:user, data:blob);
tweets.get(key:user)
</code></pre>

<p>Use greenplum for:</p>

<pre><code>begin;
update account set balance = balance - 10 where account_id = 1;
update account set balance = balance + 10 where account_id = 2;
commit;
</code></pre>

<p>Use Vertica for:</p>

<pre><code>select sum(balance)
over (partition by region order by account rows unbounded preceding)
from transactions;
</code></pre>

<p>I'm a Vertica DBA and prior to that was a developer with Vertica. Michael Stonebreaker (the guy behind Ingres, Vertica, and other databases) has some critiques of NoSQL that are worth listening to.</p>

<p>Basically, here are the advantages of Vertica as I see them:</p>

<ul>
<li>it's rather fast on large amounts of data</li>
<li>it's performance is similar (so I can gather) to other data warehousing solutions but it's advantage is clustering and commodity hardware. So you can scale by adding more commodity hardware. It looks cheap in terms of overall cost per TB. (Going from memory not an exact quote.)</li>
<li>Again, it's for data warehousing.</li>
<li>You get to use traditional SQL and tables. It's under the hood that's different.</li>
</ul>

<p>I can't speak to the other products, but I'm sure a lot of them are fine too.</p>

<p><strong>Edit</strong>: Here's a talk from Stonebreaker: <a href="http://www.slideshare.net/Dataversity/newsql-vs-nosql-for-new-oltp-michael-stonebraker-voltdb" rel="nofollow">http://www.slideshare.net/Dataversity/newsql-vs-nosql-for-new-oltp-michael-stonebraker-voltdb</a></p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>We've been working in Hadoop for 4 years, and Vertica for 2. We had massive loading and indexing problems with our tables in MySQL. We were running on fumes with our home-grown sharding solution. We could have invested heavily in developing a more sophisticated sharding solution, which would have been quite painful, imo. We could have thought harder about what data we absolutely needed to keep in a SQL database.</p>

<p>But at the end of the day, switching from MySQL to Vertica was what we chose. Vertica performance patterns are quite different from MySQL's, which comes with its own headaches. But it can load a lot of data very quickly, and it is good at heavy duty queries that would make MySQL's head spin.</p>

<p>The way I see it, Vertica is a solution when you are already invested in SQL and need a heavier duty SQL database. I'm not an expert, so I couldn't tell you what a transition to Oracle or DB2 would have been like compared to Vertica, neither in terms of integration effort or monetary cost.</p>

<p>Vertica offers a lot of features we've barely looked into. Those might be very attractive to others with use cases different to ours.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Pivotal, formerly Greenplum, is the well-funded spinoff from EMC, VMware and GE.  Pivotal's market are enterprises (and Homeland Cybersecurity agencies) with multi-Petabyte size databases needing complex analytics and high speed ETL.  Greenplum’s origin is a PostgreSQL DB redesigned for Map Reduced MPP, with later additions for columnar-support and HDFS.  It marries the best of SQL + NoSQL making NewSQL.</p>

<p>Features:</p>

<ul>
<li>In 2015H1 most of their code, including Greenplum DB &amp; HAWQ, will go
Open Source.  Some advanced management &amp; performance features at the
top of the stack will remain proprietary.</li>
<li>MPP (Massively Parallel Processing) share-nothing RDBMS database designed for multi-terrabyte to multi-petabyte environments.</li>
<li>Full SQL Compliance - supporting all versions of SQL:  ‘92, ‘99, 2003 OLAP, etc. 100% compatible with PostgreSQL 8.2.
•Only SQL over HADOOP capable of handling all 99 queries used by the TPC-DS benchmark standard without rewriting. The competition cannot do many of them and are significantly slower. SIGMON whitepaper.</li>
<li>ACID compliance.</li>
<li>Supports data stored in HDFS, Hive, HBase, Avro, ProtoBuf, Delimited Text and Sequence Files.</li>
<li>Solr/Lucene integration for multi-lingual full-text search embedded in the SQL.</li>
<li>Incorporates Open Source Software: Spring, Cloud Foundry, Redis.io, RabbitMQ, Grails, Groovy, Open Chorus, Pig, ZooKeeper, Mahout, MADlib, MapR. Some of these are used at EBSCO.</li>
<li>Native connectivity to HBase, which is a popular column-store-like technology for Hadoop.</li>
<li>VMware's participation in $150m investment in MongoDB will likely lead to integration of petabyte-scale XML files.</li>
<li>Table-by-table specification of distribution keys allow you to design your table schemas to take advantage of node-local joins and group bys, but will perform will even without this.</li>
<li>Row and/or Column-oriented data storage.  It is the only database where a table can be polymorphic with both columnar and row-based partitions as defined by the DBA.</li>
<li>A column-store table can have a different compression algorithm per column because different datatypes have different compression characteristics to optimize their storage.</li>
<li>Advanced Map-Reduce-like CBO Query Optimizer – queries can be run on hundreds of thousands of nodes.</li>
<li>It is the only database with a dynamic distributed pipeline execution model for query processing. While older databases rely on materialized execution Greenplum doesn't have to write data to disk with every intermediate query step. It streams data to the next stage of a query plan in memory, and never has to materialize the data to disk, so it's much faster than what anybody has demonstrated on Hadoop.</li>
<li>Complex queries on large data sets are solved in seconds or even sub-seconds.</li>
<li>Data management – provides table statistics, table security.</li>
<li>Deep analytics – including data mining or machine learning algorithms using MADlib. Deep Semantic Textual Analytics using GPText.</li>
<li>Graphical Analysis - billion edge distributed in-memory graph database and algorithms using GraphLab.</li>
<li>Integration of SQL, Solr indexes, GPText, MADlib and GraphLab in a single query for massive syntactical parsing and graph/matrix affinity analysis for deep search analytics.</li>
<li>Fully ODBC/JDBC compliant.</li>
<li>Distributed ETL rate of 16 TB/hr!!  Integration with Talend available.</li>
<li>Cloud support: Pivotal plans to package its Cloud Foundry software so that it can be used to host Pivotal atop other clouds as well, including Amazon Web Services' EC2. Pivotal data management will be available for use in a variety of cloud settings and will not be dependent on a proprietary VMware system.  Will target OpenStack, vSphere, vCloud Director, or private brands.  IBM announced it has standardized on Cloud Foundry for its PaaS. Confluence page.</li>
<li>Two hardware "appliance" offerings: Isilon NAS &amp; Greenplum DCA.</li>
</ul>
