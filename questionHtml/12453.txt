<p>I'm inexperienced with building Java projects, and I'm trying to build a tokenizer for Solr (found here: <a href="https://github.com/LucidWorks/auto-phrase-tokenfilter" rel="nofollow">https://github.com/LucidWorks/auto-phrase-tokenfilter</a>). I'm getting an error about an unresolvable dependency during compilation. The missing dependencies are org.restlet and org.restlet.servlet. The repository for these is here: <a href="http://maven.restlet.com/org/restlet/jee/org.restlet/2.1.1/" rel="nofollow">http://maven.restlet.com/org/restlet/jee/org.restlet/2.1.1/</a>. Ant/Ivy/Maven appears to be looking on repo1.maven.org to find restlet dependencies instead. I'm unsure how to get ivy to look in the correct place to find these.</p>

<p>I tried downloading restlet manually, but I'm unsure where to put the jar files. It looks like somewhere in /usr/share/maven-repository/, but I've had no luck with putting them in there, and I'm not sure if ant/ivy/maven would look in there if I just drop them in the right place there anyway.</p>

<p>Any help or direction would be appreciated.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Here's what I ended up doing to resolve the dependency issue. I'm not sure which one of the module lines I configured is the one picking up the dependency. The chain is what did it though - it will process repositories in the order they're specified.</p>

<pre><code>&lt;ivysettings&gt;
    &lt;settings defaultResolver="my-chain"/&gt;
    &lt;resolvers&gt;
        &lt;chain name="my-chain"&gt;
                &lt;ibiblio name="restlet" m2compatible="true" root="http://maven.restlet.com"/&gt;
                &lt;ibiblio name="central" m2compatible="true"/&gt;
        &lt;/chain&gt;
    &lt;/resolvers&gt;
    &lt;modules&gt;
        &lt;module organization="org.restlet.jee" resolver="restlet"/&gt;
        &lt;module organization="restlet" resolver="restlet"/&gt;
        &lt;module organization="org.restlet" resolver="restlet"/&gt;
    &lt;/modules&gt;
&lt;/ivysettings&gt;
</code></pre>
