<p>I am running MR program on a quad-core machine with hyper-threading, and Hadoop 1.2.1 is run in pseudo-distributed mode on that machine.</p>

<h1>What I did</h1>

<p>I have a 1GB file, and ran the program multiple times with different settings for map tasks and slots (no reduce), and here's the result I've got:</p>

<pre><code>  split     mapred.tasktracker.    No. of
  size      map.tasks.maximum      process   time   
--------------------------------------------------------
  iGB       1                      1         04m 13s
  1GB       2                      1         04m 10s
  1GB       4                      1         04m 12s

  500MB     1                      2         02m 14s
  500MB     2                      2         02m 13s
  500MB     4                      2         02m 13s

  250MB     1                      2         02m 17s
  250MB     2                      2         02m 16s
  250MB     4                      2         02m 18s
  250MB     8                      2         02m 16s

  125MB     1                      2         02m 25s
  125MB     2                      2         02m 23s
  125MB     4                      2         02m 24s
  125MB     8                      2         02m 24s
</code></pre>

<h1>Question</h1>

<p>The result made sense to me when the split size is set to 1GB and 500MB. But, I expected to get up to 4 java processes in the TOP screen when the split size = 250MB and the slot size is greater or equal to 4. Or, when split size = 125MB and the slot size is greater or equal to 4.
(I see a few more java processes but they are sleeping and using 0.1 %CPU)</p>

<p>The MR programs are written in Python, so I'm using Hadoop Streaming, and I do see 1~2 Python processes at each run. Does it mean that 2 python processes and 2 java processes occupied all available cores? But the machine is quad-core with hyper threading, so doesn't that mean that I could have up to 8 processes running at the same time?
Is this the memory issue?? CPU usage is low. Or am I missing something here?</p>

<p>I've tried adding this, and still can't see more than 2 processes running at the same time..</p>

<pre><code>  &lt;property&gt;
    &lt;name&gt;mapred.child.java.opts&lt;/name&gt;
    &lt;value&gt;-Xmx512m&lt;/value&gt;
  &lt;/property&gt; 
</code></pre>

<h1>Edit</h1>

<p>When I use <code>mapreduce</code> instead of <code>mapred</code> in the property, I get more info while running but it slows down because of printing the log. 
The INFO shows something like this:</p>

<pre><code>mapred.MapTask: numReduceTasks: 1
mapred.MapTask: io.sort.mb = 100
streaming.PipeMapRed: PipeMapRed exec [/Users/hadoop/./map.py]
...
streaming.StreamJob: map 0% reduce 0%
mapred.MapTask: Spilling map output: record full = true
mapred.MapTask: Finished spill 0

etc...
</code></pre>

<h1>Edit</h1>

<p>Another thing is that when I add more reduce tasks, I do see more java processes in the TOP screen. It's just not adding more MAP processes...</p>
