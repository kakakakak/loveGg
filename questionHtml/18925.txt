<td class="answercell">
    <div class="post-text" itemprop="text">
<p>If you are referring to the file in the tests directory of the package, it's not really meant for data this wide, nor it's clear to me you should use kmeans when you have as many columns as rows, roughly. If you have k centers, D dimensions and P points, you are fitting kD parameters to P points. If D and P are about the same size, I don't think this is a statistically sound procedure. Even if I am wrong and it is, the data is partitioned by row. There is no scalability on the number of columns. You need to look into a different algorithm. It's not clear what your target data size is. 300MB is not really mapreduce sizes. This memory problem  usually happens because each container allocates all of its memory to a java process and none or little is left for the R process. Please see  help("hadoop.settings").</p>
