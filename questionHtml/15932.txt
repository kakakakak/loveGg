<p>I need to modify LD_LIBRARY_PATH JAVA_LIBRARY_PATH and CLASSPATH before running hadoop  job at cluster. In LD_LIBRARY_PATH and JAVA_LIBRARY_PATH i need to add location of some jars which are required while running the job, As these jars are available at my cluster, similar with CLASSPATH.</p>

<p>I have a 3 NODE cluster, I need to modify this LD_LIBRARY_PATH and CLASSPATH for all the 3 data nodes in such a way that jars available at my cluster node at added to classpath, so that the following jar are available while running the job as i am avoiding jar distribution while running the job to use all ready available jar on cluster nodes. I have tried the given below options</p>

<p>1.I have tried modifying hadoop-env.sh to modify CLASSPATH</p>

<pre><code>export HADOOP_TASKTRACKER_OPTS="-classpath:/opt/oracle/oraloader-2.0.0-2/jlib/
</code></pre>

<p>but the above thing modify HADOOP_CLASSPATH not the CLASSPATH</p>

<ol>
<li><p>For LD_LIBRARY_PATH and JAVA_LIBRARY_PATH i have tired adding given below property in mapred-site.xml as suggested a my place but that didn't work.</p>

<p>&lt; property ></p>

<p>&lt; name >mapred.child.env&lt; /name ></p>

<p>&lt; value >JAVA_LIBRARY_PATH=/opt/oracle/oraloader-2.0.0-2/lib/&lt; /value ></p>

<p>&lt; value >LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/opt/oracle/oraloader-2.0.0-2/lib/&lt; /value ></p>

<p>&lt; description>User added environment variables for the task tracker child processes. Example : 1) A=foo This will set the env variable A to foo 2) B=$B:c This is inherit tasktracker's B env variable.
&lt; /description></p>

<p>&lt; /property></p></li>
</ol>

<p>I have also restarted my all 3 data nodes,all tasktrakers and 2 NAMENOdes. Still these variables are not set and my hadoop job is not able to find all those jar files required  for running the test.</p>

<p>ERROR LOG::</p>

<blockquote>
  <p>Error: java.lang.ClassNotFoundException:
  oracle.i18n.text.OraDateFormat    at
  java.net.URLClassLoader$1.run(URLClassLoader.java:202)</p>
</blockquote>

<p>When i do echo HADOOP_CLASSPATH at my cluster nodes, all the required for running hadoop job are coming. But i think the following jars need to be added in JAVA_LIBRARY_PATH but it not coming.</p>

<p>Do not reinvent the wheel. </p>

<p>If your implementation uses <code>ToolRunner</code> (which you really should use if you implement map-reduce in Java) then you can use <code>-libjars jar1,jar2</code> to ship your jars to the cluster. </p>

<p>Check out "Side Data Distribution" section in "Hadoop: The definitive guide" by Tom White.</p>
