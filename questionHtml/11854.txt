    <div class="post-text" itemprop="text">

<p>I have been looking for "MapReduce implementation of Shortest path search algorithms". </p>

<p>However, all the instances I could find "computed the shortest distance form node x to y", and none actually output the "<strong>actual shortest path like x-a-b-c-y</strong>".</p>

<p>As for what am I trying to achieve is that I have graphs with hundreds of 1000s of nodes and I need to perform frequent pattern analysis on shortest paths among the various nodes. This is for a research project I am working on.</p>

<p>It would be a great help if some one could point me to some implementation (if it exists) or give some pointers as to <code>how to hack the existing SSSP implementations to generate the paths along with the distances</code>.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Basically these implementations work with some kind of messaging. So messages are send to HDFS between map and reduce stage.</p>

<p>In the reducer they are grouped and filtered by distance, the lowest distance wins. When the distance is updated in this case, you have to set the vertex (well, some ID probably) where the message came from. </p>

<p>So you have additional space requirement per vertex, but you can reconstruct every possible shortest path in the graph. </p>

<p>Based on your comment:</p>

<p>yes probably </p>

<blockquote>
  <p>I will need to write another class of the vertex object to hold this
  additional information. Thanks for the tip, though it would be very
  helpful if you could point out where and when I can capture this
  information of where the minimum weight came from, anything from your blog maybe :-)</p>
</blockquote>

<p>Yea, could be a quite cool theme, also for Apache Hama. Most of the implementations are just considering the costs not the real path. In your case (from the blog you've linked above) you will have to extract a vertex class which actually holds the adjacent vertices as <code>LongWritable</code> (maybe a list instead of this split on the text object) and simply add a parent or source id as field (of course also <code>LongWritable</code>). 
You will set this when propagating in the mapper, that is the for loop that is looping over the adjacent vertices of the current key node.</p>

<p>In the reducer you will update the lowest somewhere while iterating over the grouped values, there you will have to set the source vertex in the key vertex by the vertex that updated to the minimum.</p>

<p>You can actually get some of the vertices classes here from my blog:
<a href="http://codingwiththomas.blogspot.de/2011/04/graph-exploration-with-hadoop-mapreduce.html" rel="nofollow">Source</a>
or directly from the repository:
<a href="http://code.google.com/p/hama-shortest-paths/source/browse/#svn/trunk/hama-gsoc/src/de/jungblut/mapreduce/graphSource" rel="nofollow">Source</a></p>

<p>Maybe it helps you, it is quite unmaintained so please come back to me if you have a specific question. </p>

<p>Here is the same algorithm in BSP with Apache Hama:</p>

<p><a href="https://github.com/thomasjungblut/tjungblut-graph/blob/master/src/de/jungblut/graph/bsp/SSSP.java" rel="nofollow">https://github.com/thomasjungblut/tjungblut-graph/blob/master/src/de/jungblut/graph/bsp/SSSP.java</a></p>
