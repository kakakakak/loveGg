<p>I'm trying to run an <code>Apache Hive</code> web interface, so I downloaded the lastest version of ant (<code>Ant 1.9.4</code>), and when I execute the command to produce the <code>hive.war</code> file</p>

<pre><code> ANT_LIB=/opt/ant/lib bin/hive --service hwi
</code></pre>

<p>I get the following error:</p>

<pre><code> cp=/tmp/Jetty_0_0_0_0_9999_hive.hwi.0.10.0.war__hwi__ae9cmk/jsp
    cp=null
    work dir=/tmp/Jetty_0_0_0_0_9999_hive.hwi.0.10.0.war__hwi__ae9cmk/jsp
    extension dir=/usr/lib/jvm/jdk1.7.0_65/jre/lib/ext:/usr/java/packages/lib/ext

   compilerSourceVM=1.5

15/02/23 09:56:59 ERROR compiler.Compiler: Error compiling file: /tmp/Jetty_0_0_0_0_9999_hive.hwi.0.10.0.war__hwi__ae9cmk/jsp//org/apache/jsp/index_jsp.java     [javac] warning: 'includeantruntime' was not set, defaulting to build.sysclasspath=last; set to false for repeatable builds

<p>I'm executing that code as root. My <code>JAVA_HOME</code> is correctly set at the <code>.bashrc</code>:</p>

<pre><code>export JAVA_HOME=/usr/lib/jvm/jdk1.7.0_65
export HADOOP_LIBEXEC_DIR=/usr/lib/hadoop/libexec
export HADOOP_CONF_DIR=/etc/hadoop/conf

export PATH=$PATH:$JAVA_HOME/bin
</code></pre>

<p>and <code>.bash_profile</code> files:</p>

<h1>.bash_profile</h1>

<pre><code># Get the aliases and functions
if [ -f ~/.bashrc ]; then
        . ~/.bashrc
fi

# User specific environment and startup programs
export JAVA_HOME=/usr/lib/jvm/jdk1.7.0_65
PATH=$PATH:$HOME/bin:$JAVA_HOME/bin

export PATH
</code></pre>

<p>I want to know why is <code>Ant</code> referencing the <code>javac</code> at <code>"/usr/lib/jvm/jdk1.7.0_65/jre"</code>. Moreover, here is my <code>JDK</code> structure:</p>

<pre><code>[root@ip-10-32-205-19 ~]# cd $JAVA_HOME

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>The error message says:</p>

<blockquote>
  <p>Perhaps JAVA_HOME does not point to the JDK. It is currently set to
  "/usr/lib/jvm/jdk1.7.0_65/jre"</p>
</blockquote>

<p>However, your .bashrc has:</p>

<blockquote>
  <p>export JAVA_HOME=/usr/lib/jvm/jdk1.7.0_65</p>
</blockquote>

<p>You likely need to run <code>source ~/.bashrc</code> in your Bash shell to reset JAVA_HOME to the root of your JDK installation. Then run Ant again.</p>
