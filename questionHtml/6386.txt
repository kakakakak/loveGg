<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Just faced the same issue (Win 8.1 + Hadoop 2.7.0 [build from sources]). </p>

<p>The problem turned out to be (ol' good) space in path name where java is located (under C:\Program Files\ dir). What i did was this:</p>

<p>1) Copy JDK dir to C:\Java\jdk1.8.0_40</p>

<p>2) edit \etc\hadoop\hadoop-env.cmd and change: set JAVA_HOME=c:\Java\jdk1.8.0_40</p>

<p>3) run cmd and execute hadoop-env.cmd</p>

<p>4) now check 'hadoop version' whether it's still complaining (my wasn't)</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Check your <code>JAVA_HOME</code>.</p>

<p>If it is C:\Program Files\Java\jdk1.7.0_65. Then you will encounter such issues.
Made it to C:\MyDrive\Java\jdk1.7.0_65 and it worked out. The space in "Program Files" creates issues.</p>

<p>PATH is <code>PATH;%JAVA_HOME%\bin</code></p>

<p>If you are using windows, then surely You are going to face issues like (due to X64 and X86 issues) : </p>

<h1>1 .</h1>

<p>FATAL datanode.DataNode: Exception in secureMain java.lang.NullPointerException
and  </p>

<h1>2.</h1>

<p>FATAL namenode.NameNode: Failed to start namenode. java.lang.UnsatisfiedLinkError: org.apache.hadoop.io.nativeio.NativeIO$Windows.access0(Ljava/lang/String;I)</p>

<h1>Solution :</h1>

<p>Copy these files  hadoop.dll, hadoop.exp, hadoop.lib, hadoop.pdb, libwinutils.lib, winutils.exe, winutils.pdb from the <a href="https://github.com/srccodes/hadoop-common-2.2.0-bin/archive/master.zip" rel="nofollow">link</a>
to your bin folder of hadoop installed loaction which looks like this.  ..\HadoopInstalled\hadoop\hadoop-2.6.0\hadoop-2.6.0\bin</p>
