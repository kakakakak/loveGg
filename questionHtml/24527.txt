    <div class="post-text" itemprop="text">

<p>Consider I have a single File which is <code>300MB</code>. The <code>block</code> size is <code>128MB</code>.
So the input file is divided into the following chunks and placed in HDFS.</p>

<pre><code>Block1: 128MB
Block2: 128MB
Block3: 64MB.
</code></pre>

<p>Now Does each block's data has <code>byte offset</code> information contained in it.
That is, do the blocks have the following <code>offset</code> information?</p>

<pre><code>Block1: 0-128MB of File
Block2  129-256MB of File
Block3: 257MB-64MB of file
</code></pre>

<p>If so, how can I get the byte-offset information for Block2 (That is it starts at 129MB) in Hadoop.
This is for understanding purposes only. Any hadoop command-line tools to get this kind of meta data about the blocks?</p>

<p><strong>EDIT</strong></p>

<p>If the byte-offset info is not present, a mapper performing its map job on a block will start consuming lines from the beginning. If the offset information is present, the mapper will skip till it finds the next EOL and then starts processing the records.
So I guess byte offset information is present inside the blocks.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Disclaimer: I might be wrong on this one I have not read that much of the HDFS source code.</p>

<p>Basically, datanodes manage blocks which are just large blobs to them. They know the block id but that its. The namenode knows everything, especially the mapping between a file path and all the block ids of this file and where each block is stored. Each block id can be stored in one or more locations depending of its replication settings.</p>

<p>I don't think you will find public API to get the information you want from a block id because HDFS does not need to do the mapping this way. On the opposite you can easily know the blocks and their locations of a file. You can try explore the <a href="https://github.com/apache/hadoop-common/tree/trunk/hadoop-hdfs-project/hadoop-hdfs/src/main/java/org/apache/hadoop/hdfs/server" rel="nofollow">source code</a>, especially the blockmanager package. </p>

<p>If you want to learn more, <a href="http://www.aosabook.org/en/hdfs.html" rel="nofollow">this article about the HDFS architecture</a> could be a good start.</p>

    <div class="post-text" itemprop="text">
<p>You can run <code>hdfs  fsck /path/to/file -files -blocks</code> to get the list of blocks.</p>

<p>A <a href="https://github.com/apache/hadoop-common/blob/990db99601b28e2a292a0c1322ffe4f334e90c47/hadoop-hdfs-project/hadoop-hdfs/src/main/java/org/apache/hadoop/hdfs/protocol/Block.java" rel="nofollow"><code>Block</code></a> does not contain offset info, only length. But you can use <a href="https://github.com/apache/hadoop-common/blob/990db99601b28e2a292a0c1322ffe4f334e90c47/hadoop-hdfs-project/hadoop-hdfs/src/main/java/org/apache/hadoop/hdfs/protocol/LocatedBlocks.java" rel="nofollow"><code>LocatedBlocks</code></a> to get all blocks of a file and from this you can easily reconstruct each block what offset it starts at. </p>
