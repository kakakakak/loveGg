<p>Using Java which is the best way to merge two arrays of class based on some value of the class?</p>

<p>Example, We have these two Classes:</p>

<pre><code>public class C1{
  public String id="";
  public String value="";
  public String tot="";
}

public Class C2{
 public String id="";
 public String tot="";
}
</code></pre>

<p>And in some point of our code we have two Arrays like:</p>

<pre><code>            //id -value - tot
C1 a [] = { {"1","value#1",""}, 
            {"2","value#2",""}, 
            {"3","value#3",""}, 
            {"4","value#4",""}
    };
                //id - tot
    C2 b [] = { {"1","2"}, 
                {"2","11"}, 
                {"4","15"}
};
</code></pre>

<p>The final array should be like:</p>

<pre><code>C1 f [] = { {"1","value#1","2"}, 
            {"2","value#2","11"}, 
            {"3","value#3",""}, 
            {"4","value#4","15"}
};
</code></pre>

<p>I'm trying to figure out the best  way to achieve this result without reading one or another array from start to end, because here the two arrays have only few rows, but in reality they both can have a length of 100k+...</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>This is something database engines are doing for living. Lets see how they doing it:
a) to join very small dataset with other one - nested loops are used. It is least efficient way. <br>
b) If one set is small enough to fit in memory - you can put its as key-value pairs in the hashmap and then do one path over smaller data set to find all matches. <br>
c) If arrays are sorted by key - you can do merger-sort join. You simply get over both dataset and looking for the matches. If one array is already sorted - it might make sense to sort the second one and then use this algorithm. <br>
d) If both dataset are too big to fit a memory - you can use grace hash join. You divide both datasets into X buckets by hashing their join key. If you use the same hash function then you can be sure that you only have to join corresponding buckets. Then you use any of the above techniques to join buckets.<br>
e) Hadoop and Map/Reduce is good option, if you have really big data sets. Internally it will sort both datasets and then let you to do the join.<br>
f) You can take Hadoop with Hive, simply put data into its table and let it to do the join. I see this option as most practical for big data volumes.<br></p>
