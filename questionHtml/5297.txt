<p>What is the best approach to get the Map Output keys to a reducer in reverse order? By default the reducer receives all keys in ascending order of keys. Any help or comments widely appreciated. </p>

<p>In simple words, in the normal scenario, if a map emits keys <em>1,4,3,5,2</em> the reducer receives the same as <em>1,2,3,4,5</em>. I would like the reducer to receive <em>5,4,3,2,1</em> instead. </p>

<p>In Hadoop 1.X, you can specify a custom comparator class for your outputs using <a href="http://hadoop.apache.org/common/docs/r1.0.3/api/org/apache/hadoop/mapred/JobConf.html#setOutputKeyComparatorClass%28java.lang.Class%29" rel="nofollow"><code>JobConf.setOutputKeyComparatorClass</code></a>.</p>

<p>Your comparator must implement the <a href="http://hadoop.apache.org/common/docs/r1.0.3/api/org/apache/hadoop/io/RawComparator.html" rel="nofollow"><code>RawComparator</code> interface</a>.</p>

<p>With Hadoop 2.X, this is done by using <a href="http://hadoop.apache.org/common/docs/current/api/org/apache/hadoop/mapreduce/Job.html#setSortComparatorClass%28java.lang.Class%29" rel="nofollow"><code>Job.setSortComparatorClass</code></a>, still with an implementation of <code>RawComparator</code>.</p>

    <div class="post-text" itemprop="text">
<p>Sample, simple  code </p>

<pre><code>class MyKeyComparator extends WritableComparator {
    protected DescendingKeyComparator() {
        super(Text.class, true);
    }

    @SuppressWarnings("rawtypes")
    @Override
    public int compare(WritableComparable w1, WritableComparable w2) {
        Text key1 = (Text) w1;
        Text key2 = (Text) w2;          
        return -1 * key1.compareTo(key2);
    }
}
</code></pre>

<p>Then add it it to the job</p>

<pre><code>job.setSortComparatorClass(MyKeyComparator.class);
</code></pre>

<p>you can change the below text type as per ur use.</p>

<pre><code>Text key1 = (Text) w1;
Text key2 = (Text) w2; 
</code></pre>
