<td class="answercell">
    <div class="post-text" itemprop="text">
<p><code>One simple way available.</code> Which Doesn't Need Explicit Sorting.</p>

<p>Assuming you have <code>one reducer</code> running. You can Override a <code>cleanup()</code> method in reducer class.</p>

<p>A cleanup() method is used in reducer to do house keeping activities at the end of the reduce task.</p>

<p>But you can make use of it. As cleanup() method is going to be executed only once after the reduce task. </p>

<p><code>By the end of your reduce task you will be holding only last key-value pair. Now, instead of emiting that output from reduce() method emit it from cleanup() method.</code></p>

<p>You can keep your context.write() only inside the cleanup().</p>

<pre><code>@Override
protected void cleanup(Context context){

    context.write(//keep your key-values here);
}
</code></pre>

<p>I believe this does your job effortlessly and you will get the desired result instantly by using the above 3 lines of code.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>Set number of reducers to 1. And in map side override the default sort method to sort in descending order and set the comparartor class in your driver code <code>job.setSortComparatorClass.</code> And get only the first Key,value from your reduce call.</p>

<pre><code>public class MysortComparator extends WritableComparator
{
    protected MysortComparator()
    {
        super(Text.class,true);
    }
    @SuppressWarnings("rawtypes")
    public int compare(WritableComparable w,WritableComparable w1)
    {
        Text s=(Text)w;
        Text s1=(Text)w1;
        return -1 * s.compareTo(s1);
}
</code></pre>

<p>Also u can overwrite reducer's run method to read only the first record and pass it to the reduce call and ignore other records. This will avoid the overhead if your single reducer is going to take large key/value pairs.</p>

<pre><code>public void run(Context context) throws IOException, InterruptedException {
  setup(context);
  int rec_cnt = 0;
  while (context.nextKey() &amp;&amp; rec_cnt++ &lt; 1) {
    reduce(context.getCurrentKey(), context.getValues(), context);
  }
  cleanup(context);
}
</code></pre>
