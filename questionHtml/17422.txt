<p>The Mesos master Docker image is built with the following Dockerfile</p>

<pre><code>FROM ubuntu:14.04

ENV MESOS_V 0.24.0

# update
RUN apt-get update
RUN apt-get upgrade -y

# dependencies
RUN apt-get install -y wget openjdk-7-jdk build-essential python-dev python-boto libcurl4-nss-dev libsasl2-dev maven libapr1-dev libsvn-dev

# mesos
RUN wget http://www.apache.org/dist/mesos/${MESOS_V}/mesos-${MESOS_V}.tar.gz
RUN tar -zxf mesos-*.tar.gz
RUN rm mesos-*.tar.gz

WORKDIR mesos
RUN mkdir build
RUN ./configure
RUN make
RUN make install

RUN ldconfig

EXPOSE 5050

</code></pre>

</code></pre>

<pre><code>LIBPROCESS_IP=172.17.0.72 mesos-slave --master=zk://172.17.0.75:2181/mesos
</code></pre>

<p>The pyspark script is:</p>

<pre><code>import os
import pyspark

src = 'file:///{}/README.md'.format(os.environ['SPARK_HOME'])

conf.set('spark.executor.uri', 'http://d3kbcqa49mib13.cloudfront.net/spark-1.5.0-bin-hadoop2.6.tgz')
conf.setAppName('my_test_app')

sc = pyspark.SparkContext(conf=conf)

lines = sc.textFile(src)
words = lines.flatMap(lambda x: x.split(' '))
word_count = (words.map(lambda x: (x, 1)).reduceByKey(lambda x, y: x+y))
print(word_count.collect())
</code></pre>

    <div class="post-text" itemprop="text">
<p>After a lot more experimentation, it looks like it was an issue with the IP address of the host machine (using its local network address, 192.168.xx.xx) when it should have been using its Docker host IP (172.17.xx.xx).</p>

<p>I managed to get things running with:</p>

<pre><code>LIBPROCESS_IP=172.17.xx.xx python test_spark.py
</code></pre>

<p>I'm now hitting a different error, but it seems unrelated, so I think this command solves my problem.</p>

<p>I'm not familiar enough with Mesos/Spark yet to understand <em>why</em> this fixes things, so if someone wants to add an explanation, that would be very helpful.</p>
