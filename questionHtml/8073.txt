    <div class="post-text" itemprop="text">

<p>I am using the <code>hdp:tool-tasklet</code> to execute a data transformation job on hadoop.
Reading from the documentation, it seems one can specify a property to be available when the actual implementation is invoked - at least for the tool-runner:</p>

<pre><code>&lt;hdp:tool-runner id="someTool" tool-class="org.foo.SomeTool" run-at-startup="true"&gt;
   &lt;hdp:arg value="data/in.txt"/&gt;
   &lt;hdp:arg value="data/out.txt"/&gt;

   **property=value**
&lt;/hdp:tool-runner&gt;
</code></pre>

<p>The documentation also mentions that:</p>

<blockquote>
  <p>The (tool) tasklet element supports the same configuration options as
   #hadoop:tool-runner[tool-runner] except for run-at-startup (which does not apply for a workflow)</p>
</blockquote>

<p>My tasklet is defined like this:</p>

<pre><code> &lt;hdp:tool-tasklet id="transformTask"
    jar="classpath:hadoop-artifacts/transform-job.jar"
    scope="step"&gt;
    &lt;hdp:arg value="#{jobParameters['inputFile']}" /&gt;
    &lt;hdp:arg value="${spring.hadoop.fsUri}/${transform.output.path}" /&gt;
    ingest.schemaXml=#{jobExecutionContext['schemaXml']}
&lt;/hdp:tool-tasklet&gt;
</code></pre>

<p>I am trying to use it later in a bean definition (within the transform-job.jar) :</p>

<pre><code>&lt;bean id="schemaDecoder" class="com.foo.bar.transform.beans.SchemaDecoder"&gt;
    &lt;constructor-arg value="${ingest.schemaXml}"&gt;&lt;/constructor-arg&gt;
&lt;/bean&gt;
</code></pre>

<p>and I get an exception:</p>

<blockquote>
  <p>Could not resolve placeholder 'ingest.schemaXml' in string value
  "${ingest.schemaXml}"</p>
</blockquote>

<p>However, if I use the <code>hdp:jar-tasklet</code> instead to define my job, the property gets propagated. I get into other problems with classloaders though...</p>

<p>So my question is, are user defined properties supported by the tool-tasklet? And if not, what are my options?
The value of the property is defined dynamically at run time, so property files would not work..</p>

<p>thank you in advance,</p>

<p>mihai </p>
